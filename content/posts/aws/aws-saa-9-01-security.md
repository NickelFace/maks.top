## KMS — Key Management Service

KMS is the central encryption service in AWS. All other AWS services that offer "encryption at rest" delegate to KMS under the hood.

### Key types

| Key type | Who manages rotation | Cost | Cross-account |
|---|---|---|---|
| **AWS-managed CMK** (`aws/s3`, `aws/ebs` …) | AWS (automatic, annual) | Free | No |
| **Customer-managed CMK** | You (optional auto-rotation) | $1/month + $0.03/10k API calls | Yes (via key policy) |
| **Data key** (generated by `GenerateDataKey`) | You store it | Counted in API calls | N/A |

### Envelope encryption

1. Call `GenerateDataKey` → KMS returns a **plaintext data key** + an **encrypted data key**.
2. Use the plaintext key to encrypt your data locally (e.g., AES-256).
3. Discard the plaintext key; store the encrypted data key alongside the ciphertext.
4. To decrypt: call `Decrypt` with the encrypted data key → KMS returns the plaintext key → decrypt locally.

> **📌 Tip:** Envelope encryption keeps large data out of KMS (max payload for `Encrypt`/`Decrypt` API = 4 KB). `GenerateDataKey` is the pattern for S3, EBS, RDS encryption.

### Key rotation

- **AWS-managed keys**: rotated automatically every year, cannot disable.
- **Customer-managed keys**: enable via console or `enable-key-rotation`. New backing key material generated; old material retained to decrypt data encrypted before rotation.
- **Imported key material**: rotation NOT supported automatically — must rotate manually by re-importing and updating the key alias.

### Multi-region keys

Replicated keys share the same key material and key ID prefix. Used for:
- Active-active multi-region applications (encrypt in `us-east-1`, decrypt in `eu-west-1`).
- DynamoDB Global Tables with client-side encryption.

> **📌 Tip:** Multi-region keys are NOT global. Each replica is a separate resource in its region; you must explicitly create replicas.

### Cross-account encryption

Grant another account access by modifying the **key policy** (not just IAM policies). The external account's IAM policy must also allow the KMS action.

## Secrets Manager

Secrets Manager stores credentials, API keys, tokens with built-in **automatic rotation**.

### Key features

| Feature | Detail |
|---|---|
| **Automatic rotation** | Lambda function rotates the secret on a schedule; AWS provides managed Lambda functions for RDS, Redshift, DocumentDB |
| **Versioning** | `AWSCURRENT` / `AWSPENDING` / `AWSPREVIOUS` labels; apps always request `AWSCURRENT` |
| **Cross-account** | Secret resource policy grants access to another account |
| **Encryption** | Always encrypted with KMS (default: `aws/secretsmanager` key) |
| **Cost** | $0.40/secret/month + $0.05 per 10k API calls |

### Secrets Manager vs SSM Parameter Store

| Dimension | Secrets Manager | Parameter Store Standard | Parameter Store Advanced |
|---|---|---|---|
| **Cost** | $0.40/secret/month | Free | $0.05/param/month |
| **Max size** | 65 KB | 4 KB | 8 KB |
| **Automatic rotation** | Yes (Lambda) | No | No (but can trigger EventBridge) |
| **Cross-account** | Yes (resource policy) | No | No |
| **KMS integration** | Always (mandatory) | Optional (`SecureString`) | Optional (`SecureString`) |
| **Versioning / labels** | Yes | Limited (latest only) | Limited |
| **Best for** | DB passwords, API keys needing rotation | Config values, feature flags, non-sensitive params | Larger configs, policies |

> **📌 Tip:** If the question mentions "automatic rotation of database credentials" → Secrets Manager. If it mentions "free, simple config storage" → Parameter Store Standard.

## ACM — Certificate Manager

ACM provides **free TLS/SSL certificates** for use within AWS.

| Feature | Detail |
|---|---|
| **Public certs** | Free; auto-renewed before expiry |
| **Private CA** | Paid ($400/month for CA + $0.75/cert) |
| **Private key** | Cannot be exported (ever, for public certs) |
| **Integrations** | ALB, NLB, CloudFront, API Gateway, Elastic Beanstalk |
| **Region** | Cert must be in **same region as the resource**, EXCEPT CloudFront which requires `us-east-1` |

> **📌 Tip:** ACM certificate for CloudFront must be in `us-east-1` regardless of where the distribution serves traffic. This is a classic exam trap.

## WAF — Web Application Firewall

WAF operates at Layer 7 and inspects HTTP/HTTPS requests.

### Web ACL components

| Component | Description |
|---|---|
| **Web ACL** | Container for rules; attached to one or more resources |
| **Rule group** | Reusable set of rules (managed or custom) |
| **Managed rules** | Pre-built by AWS or Marketplace (e.g., `AWSManagedRulesCommonRuleSet`, OWASP Top 10) |
| **Custom rules** | IP set match, geo match, regex match, rate-based |
| **Rate-based rule** | Counts requests per IP per 5-min window; auto-blocks excess |

### WAF integrations

- ALB (regional)
- API Gateway (regional)
- CloudFront (global — deployed to edge locations)
- AppSync
- Cognito User Pools

> **📌 Tip:** WAF on CloudFront protects globally at edge (before traffic hits your origin). WAF on ALB protects only at the regional level. For global DDoS + WAF protection, attach WAF to CloudFront.

## Shield

| Tier | Cost | Protection | Extras |
|---|---|---|---|
| **Standard** | Free (always on) | Layer 3/4 — SYN floods, UDP reflection, volumetric attacks | Automatic |
| **Advanced** | $3,000/month + data transfer fees | Layer 3/4/7 (with WAF); protects EC2, ELB, CloudFront, Route 53, Global Accelerator | DDoS Response Team (DRT), attack forensics, **financial protection** (bill credits for scaling costs during DDoS) |

> **📌 Tip:** The exam loves "financial protection against DDoS" — that is **Shield Advanced only**. Shield Standard protects against common attacks automatically, but no bill credits, no DRT.

## GuardDuty

GuardDuty is a **threat detection** service. It analyses log data — no agents required.

### Data sources

| Source | Threats detected |
|---|---|
| **VPC Flow Logs** | Port scanning, unusual traffic patterns, C2 communication |
| **DNS Logs** | DNS exfiltration, crypto-mining domains |
| **CloudTrail Management Events** | Unusual API calls, privilege escalation, credential abuse |
| **CloudTrail S3 Data Events** | S3 bucket data exfiltration |
| **EKS Audit Logs** | Kubernetes anomalies |
| **RDS Login Events** | Brute-force, unusual login patterns |

### Auto-remediation pattern

```
GuardDuty Finding → EventBridge Rule → Lambda (block IP in WAF / isolate EC2 / revoke IAM key)
```

> **📌 Tip:** GuardDuty **detects** threats; it does NOT block them. Blocking requires EventBridge + Lambda + WAF/Security Group. GuardDuty ≠ WAF.

## Inspector

Inspector performs **automated vulnerability assessments**.

| Target | What it scans |
|---|---|
| **EC2 instances** | OS CVEs, network reachability (via SSM agent) |
| **Lambda functions** | Software package vulnerabilities in function code |
| **ECR container images** | CVE scan on push or continuously |

- Findings include a risk score and CVE ID.
- Integrates with Security Hub for aggregated view.

> **📌 Tip:** Inspector is about **CVEs and OS vulnerabilities**, not about data classification. If the question is about detecting PII in S3 → that's Macie.

---

## Macie

Macie uses **machine learning to discover sensitive data** in S3.

- Analyses S3 objects for PII (names, SSNs, credit card numbers, etc.)
- Generates findings for policy violations (e.g., public bucket containing sensitive data)
- Integrates with Security Hub and EventBridge for automated response

> **📌 Tip:** Macie is **S3-only**. It does not scan databases, EBS volumes, or other storage types. Any question about "sensitive data discovery in S3" → Macie.

---

## Security Hub

Security Hub **aggregates security findings** from multiple AWS services into a single pane.

### Integrated sources

- GuardDuty
- Inspector
- Macie
- IAM Access Analyzer
- Firewall Manager
- AWS Config (security-related rules)
- Third-party tools (via ASFF format)

### Features

- **Security standards**: CIS AWS Foundations Benchmark, PCI DSS, NIST 800-53 — automated compliance checks
- **Findings**: normalized using ASFF (Amazon Security Finding Format)
- **Aggregation**: multi-account, multi-region via delegated administrator
- **Actions**: send findings to EventBridge for automated remediation

## Exam Traps Summary

| Trap | Correct answer |
|---|---|
| "Financial protection against DDoS bill" | Shield **Advanced** (not Standard) |
| "Global WAF + DDoS protection" | WAF attached to **CloudFront** |
| "GuardDuty blocks malicious IPs" | False — GuardDuty only detects; use EventBridge + Lambda to block |
| "Macie scans RDS / EBS" | False — Macie is **S3-only** |
| "Free automatic credential rotation" | Secrets Manager is **not free** ($0.40/secret/month); Parameter Store has no rotation |
| "Encrypt data > 4KB with KMS directly" | Not possible — use **envelope encryption** (`GenerateDataKey`) |
| "ACM cert for CloudFront in any region" | Must be in **us-east-1** |
| "Auto-rotate imported KMS key material" | Not supported — manual rotation only |
| "Cross-account secret sharing" | Secrets Manager (resource policy); Parameter Store cannot do cross-account |
