## CloudWatch Metrics

CloudWatch собирает и отслеживает метрики из AWS-сервисов и пользовательских источников.

### Основные концепции

| Концепция | Описание |
|---|---|
| **Namespace** | Контейнер для метрик (например, `AWS/EC2`, `AWS/RDS`). Кастомные метрики в собственном namespace. |
| **Dimension** | Пара имя/значение для идентификации метрики (например, `InstanceId=i-1234`). До 30 на метрику. |
| **Standard monitoring** | По умолчанию для большинства сервисов — точки данных каждые **5 минут** |
| **Detailed monitoring** | Дополнительная стоимость — точки данных каждую **1 минуту** (включается на ресурс) |
| **High-resolution metrics** | Кастомные метрики с гранулярностью 1 секунда через `--storage-resolution 1` |
| **Retention** | `< 60с`: 3 часа; `60с`: 15 дней; `300с`: 63 дня; `1 час`: 15 месяцев |

### Кастомные метрики

```bash
# Публикация кастомной метрики
aws cloudwatch put-metric-data \
  --namespace "MyApp/Performance" \
  --metric-name "ActiveUsers" \
  --value 42 \
  --dimensions Environment=prod
```

> **📌 Совет:** Использование RAM и дисковое пространство по умолчанию НЕ собираются CloudWatch на EC2. Установите **CloudWatch Agent** для отправки этих данных как кастомных метрик.

## CloudWatch Logs

### Структура

| Компонент | Описание |
|---|---|
| **Log group** | Контейнер для log streams (например, `/aws/lambda/my-function`) |
| **Log stream** | Последовательность log-событий из одного источника |
| **Retention** | Не истекает (по умолчанию) → настраивается от 1 дня до 10 лет |
| **Metric filter** | Сканирует log-события по паттернам и увеличивает CloudWatch-метрику |
| **Subscription filter** | Стримит лог-данные почти в реальном времени в Kinesis, Firehose или Lambda |

### CloudWatch Logs Insights

SQL-подобный язык запросов для интерактивного анализа:

```sql
fields @timestamp, @message
| filter @message like /ERROR/
| stats count(*) as errorCount by bin(5m)
| sort errorCount desc
| limit 20
```

- Поддерживает кросс-аккаунтные и кросс-региональные запросы (требует шаринга log group данных).
- Результаты запросов можно сохранять и визуализировать в CloudWatch дашбордах.

## CloudWatch Alarms

### Состояния алармов

| Состояние | Значение |
|---|---|
| `OK` | Метрика в пределах порогового значения |
| `ALARM` | Метрика превысила пороговое значение |
| `INSUFFICIENT_DATA` | Недостаточно точек данных (часто на новых ресурсах) |

### Действия аларма

- **SNS-уведомление** — email, HTTP-endpoint, SQS, Lambda и т.д.
- **EC2-действие** — остановить, завершить, перезагрузить, восстановить инстанс
- **Auto Scaling** — масштабировать группу вверх или вниз
- **Systems Manager OpsCenter** — создать OpsItem

### Составные аларма (Composite alarms)

Объединяют несколько алармов через логику AND/OR. Полезны для снижения шума:

```
Composite ALARM = (CPU_Alarm AND Memory_Alarm) OR DiskIO_Alarm
```

> **📌 Совет:** Составные аларма снижают шум ("alarm storms"), требуя одновременного выполнения нескольких условий.

## Amazon EventBridge (ранее CloudWatch Events)

EventBridge — современная шина событий для AWS.

| Функция | Детали |
|---|---|
| **Default bus** | Принимает события от всех AWS-сервисов |
| **Custom bus** | Для событий собственных приложений |
| **Partner bus** | Интеграция с SaaS (Zendesk, Datadog и др.) |
| **Rules** | Совпадение событий по паттерну; маршрутизация к одному или нескольким таргетам |
| **Scheduled rules** | Cron или rate-выражения (замена CloudWatch Scheduled Events) |
| **Targets** | Lambda, Step Functions, SQS, SNS, Kinesis, ECS Task, CodePipeline и 20+ других |

| Функция | Мониторит |

## CloudTrail

CloudTrail записывает **каждый API-вызов** в вашем AWS-аккаунте.

### Типы событий

| Тип | Примеры | По умолчанию? | Стоимость |
|---|---|---|---|
| **Management events** | CreateBucket, RunInstances, PutRolePolicy | Да (бесплатно, 90 дней Event History) | Бесплатно для первой копии на регион |
| **Data events** | S3 GetObject/PutObject, Lambda Invoke | Нет (высокообъёмные, opt-in) | Дополнительная стоимость |
| **Insights events** | Нетипичный объём API-вызовов / частота ошибок | Нет (opt-in) | Дополнительная стоимость |

### Конфигурация trail

- **Однорегиональный trail** — логирует события только в одном регионе
- **Всерегиональный trail** — рекомендуется; логирует все регионы в один S3-bucket
- **Доставка**: S3 (всегда) + опционально CloudWatch Logs + SNS-уведомление
- **Целостность лог-файлов**: SHA-256 дайджест-файлы; обнаруживает подделку

### CloudTrail Lake

Неизменяемое, полностью управляемое хранилище данных событий с SQL-запросами. Хранение до 7 лет.

> **📌 Совет:** CloudTrail Event History = 90 дней бесплатно в консоли. Для более долгого хранения или кастомных запросов → создайте trail в S3 или используйте CloudTrail Lake.

## AWS Config

Config записывает **состояние конфигурации** AWS-ресурсов со временем и оценивает соответствие.

### Основные концепции

| Концепция | Описание |
|---|---|
| **Configuration item** | Снимок конфигурации ресурса в определённый момент |
| **Configuration history** | Временная шкала всех изменений конфигурации |
| **Config rule** | Оценивает, соответствует ли конфигурация ресурса требованиям |
| **Managed rules** | Готовые (например, `encrypted-volumes`, `restricted-ssh`, `s3-bucket-public-read-prohibited`) |
| **Custom rules** | Lambda-функции для кастомной логики соответствия |
| **Remediation** | Автоматическая (SSM Automation) или ручная; исправление несоответствующих ресурсов |
| **Aggregator** | Сбор данных Config по аккаунтам и регионам в единый вид |

### Config vs CloudTrail

| Вопрос | Используйте |
|---|---|
| "Какая текущая/прошлая конфигурация у этого ресурса?" | **AWS Config** |
| "Кто сделал этот API-вызов и когда?" | **CloudTrail** |
| "Изменилась ли эта security group и как она выглядела раньше?" | **AWS Config** (показывает diff) |
| "Какой IAM-пользователь удалил этот S3-bucket?" | **CloudTrail** |

> **📌 Совет:** AWS Config **не бесплатный**. Взимается плата за каждый записанный configuration item (~$0.003) и за каждую оценку Config rule (~$0.001).

> **📌 Совет:** Config Rules оценивают соответствие и могут запускать ремедиацию, но **не могут запретить** создание ресурса в реальном времени. Для превентивных мер → SCP или IAM.

## Trusted Advisor

Trusted Advisor предоставляет **рекомендации лучших практик** по пяти категориям.

### Категории

| Категория | Примеры проверок |
|---|---|
| **Cost Optimization** | Простаивающие EC2-инстансы, недоиспользуемые RDS, несвязанные EIP |
| **Performance** | Использование EC2 provisioned IOPS, настройки CloudFront TTL |
| **Security** | Разрешения S3-bucket, MFA на root, широко открытые порты security group |
| **Fault Tolerance** | Multi-AZ RDS, снимки EBS старше 7 дней, проверки работоспособности Route 53 |
| **Service Limits** | VPC на регион, лимиты EC2 On-Demand, тома EBS |

### Требования к плану поддержки

| Уровень проверок | Требуемый план |
|---|---|
| **7 основных проверок** (базовая безопасность + лимиты сервисов) | Бесплатно (все планы) |
| **Все проверки** + автообновление + CloudWatch-аларма для лимитов | Поддержка **Business** или **Enterprise** |

> **📌 Совет:** Полный доступ к Trusted Advisor требует Business или Enterprise поддержки — не Developer или free-tier.

## AWS Health Dashboard

| Вид | Назначение |
|---|---|
| **Service Health** (публичный) | Глобальные события AWS-инфраструктуры (публичная страница статуса) |
| **Personal Health Dashboard** | События, влияющие на **ваши конкретные ресурсы** (например, запланированное обслуживание EC2 в вашем аккаунте) |

Personal Health интегрируется с EventBridge: запускайте Lambda при наступлении события обслуживания для ваших инстансов.

## X-Ray — распределённая трассировка

X-Ray трассирует запросы при их прохождении через распределённые приложения (микросервисы, Lambda, API Gateway и т.д.).

### Концепции

| Концепция | Описание |
|---|---|
| **Trace** | Сквозная запись одного запроса; состоит из сегментов |
| **Segment** | Работа, выполненная одним сервисом |
| **Subsegment** | Детализация внутри сегмента (вызов БД, HTTP-запрос) |
| **Service map** | Визуальный граф всех сервисов и их связей |
| **Sampling** | Не каждый запрос трассируется (настраиваемая ставка, по умолчанию 5% + 1 запрос/с) |
| **Annotation** | Пара ключ-значение, индексированная для фильтрации |
| **Group** | Выражение фильтра, ограничивающее трассы для дашбордов/алертов |

### Интеграция

- **Lambda**: включить активную трассировку в конфигурации функции
- **EC2/ECS**: установить X-Ray daemon; SDK в коде приложения
- **API Gateway**: включить X-Ray трассировку на стейдже
- **Elastic Beanstalk**: X-Ray daemon устанавливается автоматически

> **📌 Совет:** X-Ray для **трассировки микросервисов / распределённых систем**. Не для анализа логов (CloudWatch Logs) или аудита (CloudTrail). Вопрос "как найти, какой downstream-сервис вызывает задержку" → X-Ray.

## Ловушки экзамена

| Ловушка | Правильный ответ |
|---|---|
| "CloudTrail хранит логи 1 год по умолчанию" | Ложь — Event History **90 дней**; создайте trail для более долгого хранения |
| "AWS Config бесплатный" | Ложь — плата за config item + за оценку rule |
| "Config Rules могут блокировать создание несоответствующих ресурсов" | Ложь — Config только **обнаруживает**, не предотвращает |
| "Полный Trusted Advisor на free tier" | Ложь — требует **Business или Enterprise** поддержки |
| "RAM/диск метрики в CloudWatch по умолчанию на EC2" | Ложь — нужен **CloudWatch Agent** |
| "X-Ray для аудита API-вызовов" | Ложь — X-Ray для **распределённой трассировки**; CloudTrail для аудита API |
| "CloudWatch Events vs EventBridge" | Одно и то же — EventBridge **новое название** |
| "Detailed monitoring по умолчанию" | Ложь — по умолчанию стандартный мониторинг **5 минут** |
