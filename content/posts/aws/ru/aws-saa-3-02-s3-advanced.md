## S3 Продвинутые возможности

В этой статье разобраны функции, часто встречающиеся в ситуационных вопросах SAA-C03: движение данных, производительность при масштабировании и защита данных.

---

## Lifecycle-правила

Lifecycle-правила автоматизируют переходы между классами хранения и удаление объектов.

### Действия

| Тип правила | Что делает |
|---|---|
| Transition | Перемещает объект в более дешёвый класс через N дней |
| Expiration | Удаляет объекты (или версии) через N дней |
| Abort incomplete multipart | Очищает незавершённые загрузки через N дней |

**Ограничения по времени (минимальные дни до перехода):**

| Откуда | Куда | Мин. дней |
|---|---|---|
| Standard | Standard-IA или One Zone-IA | **30 дней** |
| Standard | Glacier Instant Retrieval | **90 дней** |
| Standard-IA | Glacier | 0 (без доп. минимума) |
| Любой класс | Glacier Deep Archive | 0 (но минимальный срок класса применяется) |

> **📌 Tip:** 30-дневный минимум для Standard → Standard-IA — частая ловушка на экзамене. Нельзя перейти в IA раньше 30 дней, независимо от настроек правила.

**Lifecycle-правила применяются к:**
- Текущим версиям объектов
- Неактуальным версиям (при включённом версионировании)
- Незавершённым multipart-загрузкам

**Практический паттерн — поэтапная архивация:**
```
День 0:   Загрузка → Standard
День 30:  Переход → Standard-IA
День 90:  Переход → Glacier Instant Retrieval
День 365: Переход → Glacier Deep Archive
День 730: Истечение (удаление)
```

## Репликация

### CRR vs SRR

| Функция | CRR (межрегиональная) | SRR (в том же регионе) |
|---|---|---|
| Регион назначения | Другой регион | Тот же регион |
| Применение | Compliance, снижение задержки, DR | Агрегация логов, живая копия между аккаунтами |
| Требование к версионированию | **Оба** бакета должны иметь версионирование | То же требование |
| Задержка | Выше (межрегионально) | Ниже |

**Ключевые особенности:**
- Реплицируются только **новые объекты** после настройки — существующие объекты **не реплицируются ретроактивно** (для этого — S3 Batch Operations)
- Маркеры удаления реплицируются опционально (не по умолчанию)
- Удаление конкретной версии **не реплицируется** (защита от каскадного удаления)
- Репликация **не транзитивна**: A → B, B → C ≠ A → C

### Replication Time Control (RTC)

- SLA: 99.99% объектов реплицируются **за 15 минут**
- Стоит дополнительно (за ГБ и за объект)
- Включает метрики в CloudWatch

> **📌 Tip:** Стандартная репликация не гарантирует время. RTC — ответ, когда в сценарии требуется конкретная временная гарантия репликации.

## Производительность S3

### Базовая пропускная способность

S3 автоматически масштабируется. Базовые показатели на префикс:

| Операция | Запросов/сек на префикс |
|---|---|
| PUT/COPY/POST/DELETE | **3 500** |
| GET/HEAD | **5 500** |

**Параллелизм через префиксы:** распределите объекты по нескольким префиксам, чтобы умножить пропускную способность. Четыре префикса = 22 000 GET/с.

```
bucket/2024-01/images/...
bucket/2024-02/images/...
bucket/2024-03/images/...
```

### Multipart Upload

| Порог | Рекомендация |
|---|---|
| > 100 МБ | Рекомендуется |
| > 5 ГБ | **Обязательно** |

Преимущества: параллельная загрузка, повтор отдельных частей, возобновление прерванных загрузок.

```bash
# Инициировать
aws s3api create-multipart-upload --bucket my-bucket --key large-file.zip

# Загрузить части (каждая ≥ 5 МБ, кроме последней)
aws s3api upload-part --bucket my-bucket --key large-file.zip \
  --part-number 1 --upload-id <id> --body part1.bin

# Завершить
aws s3api complete-multipart-upload --bucket my-bucket --key large-file.zip \
  --upload-id <id> --multipart-upload file://parts.json
```

### Transfer Acceleration

- Маршрутизирует загрузки через **edge-узлы CloudFront** → магистраль AWS → бакет S3
- Быстрее для загрузок от географически удалённых клиентов
- Использует отдельный endpoint: `<bucket>.s3-accelerate.amazonaws.com`
- Дополнительная плата за ГБ

> **📌 Tip:** Transfer Acceleration полезен для **загрузки** (PUT) — не обязательно для скачивания. Экзамен часто проверяет именно это. Также бесполезен для локальных пользователей.

### Byte-Range Fetches

Запросить только часть объекта через HTTP-заголовок `Range`. Позволяет:
- Параллельные загрузки (разделить файл на части)
- Получить только заголовок файла (например, первые 256 байт)

## Уведомления о событиях

S3 может инициировать обработку при создании, удалении или восстановлении объектов.

| Назначение | Применение |
|---|---|
| **SNS** | Fan-out на несколько подписчиков |
| **SQS** | Очередь для независимой обработки |
| **Lambda** | Serverless-обработка в реальном времени |
| **EventBridge** | Сложная маршрутизация, несколько получателей, фильтрация |

**Преимущества EventBridge перед нативными событиями S3:**
- Фильтрация по метаданным, размеру объекта, префиксу/суффиксу
- Маршрутизация на 18+ типов получателей
- Воспроизведение и архивирование событий
- Кросс-аккаунтная доставка

> **📌 Tip:** Для сложной маршрутизации или нескольких потребителей одного события — EventBridge. Для простой однополучательной обработки — нативные события S3 (Lambda/SQS/SNS).

## S3 Select и Glacier Select

Запрос данных внутри объектов с помощью SQL-выражений — без скачивания всего объекта.

- Поддерживает CSV, JSON, Parquet
- Фильтрация на стороне сервера — возвращаются только совпадающие строки
- До 400% быстрее и на 80% дешевле, чем скачивание + фильтрация на клиенте
- Glacier Select: та же концепция для объектов в Glacier Flexible Retrieval

```sql
SELECT s.name, s.age FROM S3Object s WHERE s.age > 30
```

## Access Points

Упрощают доступ к общим бакетам S3 в больших организациях.

- У каждого Access Point есть своё **DNS-имя** и **политика**
- Политики можно ограничить конкретным VPC
- Упрощают управление политиками бакета — вместо одной огромной политики у каждой команды/приложения свой точка доступа

```
Политика бакета (базовая)
  └─ Access Point: data-science-ap → VPC vpc-abc, IAM role data-science-role
  └─ Access Point: finance-ap → IAM role finance-role
```

## Object Lambda

Трансформирует объекты S3 «на лету» при их получении — без хранения изменённых копий.

Примеры использования:
- Скрыть персональные данные в документах перед выдачей
- Конвертировать формат изображений по запросу
- Добавить водяной знак
- Обогатить данные из DynamoDB

Архитектура: `Клиент → S3 Object Lambda Access Point → Lambda → S3 бакет`

## S3 Object Lock

Защищает объекты от удаления или перезаписи на заданный период (WORM — Write Once Read Many).

| Режим | Кто может обойти | Применение |
|---|---|---|
| **Governance** | Пользователи с `s3:BypassGovernanceRetention` | Внутренний compliance с возможностью обхода |
| **Compliance** | **Никто** — даже root | Регуляторные требования (SEC 17a-4, FINRA) |

**Период хранения (Retention Period):** фиксированный срок. Объекты нельзя удалить до его истечения.

**Legal Hold:** независим от периода хранения. Включается/выключается пользователями с `s3:PutObjectLegalHold`. Блокирует удаление вне зависимости от Retention.

> **📌 Tip:** Compliance mode = никто не может удалить, даже root. Governance mode = привилегированные пользователи могут обойти. Legal Hold — дополнительный замок, который можно добавить/снять отдельно.

## Pre-signed URLs

Предоставляют временный доступ к приватному объекту S3 без его публикации.

```bash
# Сгенерировать pre-signed URL на 1 час
aws s3 presign s3://my-bucket/private-file.pdf --expires-in 3600
```

- URL содержит учётные данные подписанта — доступ проверяется по правам создателя в момент запроса
- Работает для GET (скачивание) и PUT (загрузка)
- Срок действия: до 12 часов (консоль/CLI), до 7 дней (SDK с SigV4)

**Сценарии:** защищённое скачивание из веб-приложения, прямая загрузка в S3 минуя сервер, временный доступ к файлу.

## Сводка ловушек экзамена

| Ловушка | Правильный ответ |
|---|---|
| Transfer Acceleration ускоряет скачивание | В первую очередь помогает **загрузке**; сценарий — межрегиональные/дальние загрузки |
| Replication Time Control (RTC) бесплатен | RTC **стоит дополнительно** — стандартная репликация без гарантии времени |
| Репликация ретроактивна | **Не ретроактивна** — только новые объекты; для существующих — S3 Batch Operations |
| Standard → Standard-IA в любое время | Нужно дождаться **минимум 30 дней** |
| Object Lock Compliance может обойти администратор | Никто не может обойти Compliance mode — даже root |
| S3 Select скачивает весь объект | Фильтрует на стороне сервера — возвращает только совпадающие данные |
