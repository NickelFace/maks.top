## KMS — Key Management Service

KMS — центральный сервис шифрования в AWS. Все другие AWS-сервисы с "шифрованием в покое" используют KMS под капотом.

### Типы ключей

| Тип ключа | Кто управляет ротацией | Стоимость | Кросс-аккаунт |
|---|---|---|---|
| **AWS-managed CMK** (`aws/s3`, `aws/ebs` …) | AWS (автоматически, ежегодно) | Бесплатно | Нет |
| **Customer-managed CMK** | Вы (опциональная авторотация) | $1/мес + $0.03/10k API-вызовов | Да (через key policy) |
| **Data key** (через `GenerateDataKey`) | Вы храните его | Считается в API-вызовах | N/A |

### Конвертная (envelope) шифровка

1. Вызвать `GenerateDataKey` → KMS возвращает **plaintext data key** + **зашифрованный data key**.
2. Зашифровать данные локально plaintext-ключом (например, AES-256).
3. Уничтожить plaintext-ключ; хранить зашифрованный data key рядом с шифротекстом.
4. Для расшифровки: вызвать `Decrypt` с зашифрованным data key → KMS вернёт plaintext-ключ → расшифровать локально.

> **📌 Совет:** Конвертное шифрование нужно потому, что максимальный размер данных для API `Encrypt`/`Decrypt` = 4 КБ. `GenerateDataKey` — это паттерн для S3, EBS, RDS.

### Ротация ключей

- **AWS-managed ключи**: ротируются автоматически раз в год, нельзя отключить.
- **Customer-managed ключи**: включается через консоль или `enable-key-rotation`. Старый материал ключа сохраняется для расшифровки ранее зашифрованных данных.
- **Импортированный материал ключа**: автоматическая ротация НЕ поддерживается — только вручную.

### Мульти-регионные ключи

Реплицированные ключи имеют один и тот же материал ключа. Использование:
- Активно-активные мульти-регионные приложения (шифровать в `us-east-1`, расшифровывать в `eu-west-1`).
- DynamoDB Global Tables с клиентским шифрованием.

> **📌 Совет:** Мульти-регионные ключи НЕ глобальные. Каждая реплика — отдельный ресурс в своём регионе; реплики нужно создавать явно.

### Кросс-аккаунтное шифрование

Доступ другому аккаунту предоставляется изменением **key policy** (недостаточно только IAM-политик). IAM-политика внешнего аккаунта также должна разрешать KMS-действие.

## Secrets Manager

Secrets Manager хранит учётные данные, API-ключи, токены с встроенной **автоматической ротацией**.

### Ключевые особенности

| Особенность | Детали |
|---|---|
| **Автоматическая ротация** | Lambda-функция ротирует секрет по расписанию; AWS предоставляет managed Lambda для RDS, Redshift, DocumentDB |
| **Версионирование** | Метки `AWSCURRENT` / `AWSPENDING` / `AWSPREVIOUS`; приложения всегда запрашивают `AWSCURRENT` |
| **Кросс-аккаунт** | Resource policy секрета даёт доступ другому аккаунту |
| **Шифрование** | Всегда через KMS (по умолчанию: ключ `aws/secretsmanager`) |
| **Стоимость** | $0.40/секрет/мес + $0.05 за 10k API-вызовов |

### Secrets Manager vs SSM Parameter Store

| Параметр | Secrets Manager | Parameter Store Standard | Parameter Store Advanced |
|---|---|---|---|
| **Стоимость** | $0.40/секрет/мес | Бесплатно | $0.05/параметр/мес |
| **Макс. размер** | 65 КБ | 4 КБ | 8 КБ |
| **Авторотация** | Да (Lambda) | Нет | Нет (можно через EventBridge) |
| **Кросс-аккаунт** | Да (resource policy) | Нет | Нет |
| **KMS** | Обязательно | Опционально (`SecureString`) | Опционально (`SecureString`) |
| **Лучше для** | Пароли БД, API-ключи с ротацией | Конфиги, флаги, несекретные данные | Большие конфиги, политики |

> **📌 Совет:** Вопрос про "автоматическую ротацию учётных данных базы" → Secrets Manager. Вопрос про "бесплатное простое хранилище конфигов" → Parameter Store Standard.

## ACM — Certificate Manager

ACM предоставляет **бесплатные TLS/SSL-сертификаты** для использования внутри AWS.

| Особенность | Детали |
|---|---|
| **Публичные сертификаты** | Бесплатно; авторенев до истечения срока |
| **Private CA** | Платно ($400/мес за CA + $0.75/сертификат) |
| **Приватный ключ** | Нельзя экспортировать (никогда, для публичных сертификатов) |
| **Интеграции** | ALB, NLB, CloudFront, API Gateway, Elastic Beanstalk |
| **Регион** | Сертификат должен быть в **том же регионе**, что и ресурс — КРОМЕ CloudFront, которому нужен `us-east-1` |

> **📌 Совет:** ACM-сертификат для CloudFront должен быть в `us-east-1` независимо от того, где дистрибуция обслуживает трафик. Классическая ловушка экзамена.

## WAF — Web Application Firewall

WAF работает на уровне Layer 7 и инспектирует HTTP/HTTPS-запросы.

### Компоненты Web ACL

| Компонент | Описание |
|---|---|
| **Web ACL** | Контейнер для правил; привязывается к одному или нескольким ресурсам |
| **Rule group** | Переиспользуемый набор правил (managed или custom) |
| **Managed rules** | Готовые от AWS или Marketplace (например, `AWSManagedRulesCommonRuleSet`, OWASP Top 10) |
| **Custom rules** | IP set match, geo match, regex match, rate-based |
| **Rate-based rule** | Считает запросы по IP за 5 минут; автоблокировка превышения |

### Интеграции WAF

- ALB (региональный)
- API Gateway (региональный)
- CloudFront (глобальный — развёртывается в edge locations)
- AppSync
- Cognito User Pools

> **📌 Совет:** WAF на CloudFront защищает глобально на edge (до того, как трафик достигает origin). WAF на ALB — только на региональном уровне. Для глобальной DDoS + WAF защиты — WAF на CloudFront.

## Shield

| Уровень | Стоимость | Защита | Дополнительно |
|---|---|---|---|
| **Standard** | Бесплатно (всегда включён) | Layer 3/4 — SYN floods, UDP reflection, объёмные атаки | Автоматически |
| **Advanced** | $3,000/мес + трафик | Layer 3/4/7 (с WAF); защищает EC2, ELB, CloudFront, Route 53, Global Accelerator | DDoS Response Team (DRT), криминалистика атак, **финансовая защита** (кредиты на счёт за масштабирование во время DDoS) |

> **📌 Совет:** Экзамен любит "финансовую защиту от DDoS" — это **только Shield Advanced**. Shield Standard защищает от типичных атак автоматически, но без кредитов и DRT.

## GuardDuty

GuardDuty — сервис **обнаружения угроз**. Анализирует логи — агенты не нужны.

### Источники данных

| Источник | Обнаруживаемые угрозы |
|---|---|
| **VPC Flow Logs** | Сканирование портов, нетипичные паттерны трафика, C2-связь |
| **DNS Logs** | DNS-эксфильтрация, домены криптомайнинга |
| **CloudTrail Management Events** | Нетипичные API-вызовы, эскалация привилегий, злоупотребление учётными данными |
| **CloudTrail S3 Data Events** | Эксфильтрация данных из S3 |
| **EKS Audit Logs** | Аномалии Kubernetes |
| **RDS Login Events** | Брутфорс, нетипичные паттерны входа |

### Паттерн авторемедиации

```
GuardDuty Finding → EventBridge Rule → Lambda (блокировать IP в WAF / изолировать EC2 / отозвать IAM-ключ)
```

> **📌 Совет:** GuardDuty **обнаруживает** угрозы; он их НЕ блокирует. Блокировка требует EventBridge + Lambda + WAF/Security Group. GuardDuty ≠ WAF.

## Inspector

Inspector выполняет **автоматизированные оценки уязвимостей**.

| Цель | Что сканируется |
|---|---|
| **EC2 инстансы** | CVE операционной системы, сетевая доступность (через SSM agent) |
| **Lambda функции** | Уязвимости пакетов в коде функции |
| **ECR образы контейнеров** | CVE-сканирование при push или непрерывно |

- Находки включают оценку риска и CVE ID.
- Интегрируется с Security Hub для сводного просмотра.

> **📌 Совет:** Inspector — про **CVE и уязвимости ОС**, не про классификацию данных. Если вопрос про обнаружение PII в S3 → это Macie.

## Macie

Macie использует **машинное обучение для обнаружения чувствительных данных** в S3.

- Анализирует объекты S3 на PII (имена, SSN, номера кредитных карт и т.д.)
- Генерирует находки по нарушениям политики (например, публичный bucket с чувствительными данными)
- Интегрируется с Security Hub и EventBridge для автоматического реагирования

> **📌 Совет:** Macie работает **только с S3**. Не сканирует базы данных, EBS-тома или другие хранилища. Вопрос о "поиске чувствительных данных в S3" → Macie.

## Security Hub

Security Hub **агрегирует выводы по безопасности** из нескольких AWS-сервисов в единый интерфейс.

### Интегрированные источники

- GuardDuty
- Inspector
- Macie
- IAM Access Analyzer
- Firewall Manager
- AWS Config (правила, связанные с безопасностью)
- Сторонние инструменты (через формат ASFF)

### Функции

- **Стандарты безопасности**: CIS AWS Foundations Benchmark, PCI DSS, NIST 800-53 — автоматизированные проверки соответствия
- **Находки**: нормализованы в формате ASFF (Amazon Security Finding Format)
- **Агрегация**: мультиаккаунт, мульти-регион через делегированного администратора
- **Действия**: отправка находок в EventBridge для автоматической ремедиации

## Ловушки экзамена

| Ловушка | Правильный ответ |
|---|---|
| "Финансовая защита от DDoS-счёта" | Shield **Advanced** (не Standard) |
| "Глобальная WAF + DDoS защита" | WAF привязанный к **CloudFront** |
| "GuardDuty блокирует вредоносные IP" | Ложь — GuardDuty только обнаруживает; для блокировки — EventBridge + Lambda |
| "Macie сканирует RDS / EBS" | Ложь — Macie работает **только с S3** |
| "Бесплатная автоматическая ротация учётных данных" | Secrets Manager **не бесплатный** ($0.40/секрет/мес); Parameter Store не имеет ротации |
| "Зашифровать данные > 4КБ через KMS напрямую" | Невозможно — используйте **конвертное шифрование** (`GenerateDataKey`) |
| "ACM-сертификат для CloudFront в любом регионе" | Должен быть в **us-east-1** |
| "Авторотация импортированного KMS-ключа" | Не поддерживается — только ручная ротация |
| "Шаринг секрета между аккаунтами" | Secrets Manager (resource policy); Parameter Store не поддерживает кросс-аккаунт |
