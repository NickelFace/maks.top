## Elastic Load Balancing и Auto Scaling

ELB распределяет входящий трафик между целевыми объектами. ASG автоматически регулирует количество инстансов EC2 в зависимости от нагрузки. Вместе они составляют основу горизонтального масштабирования в AWS.

---

## Типы балансировщиков нагрузки

| Тип LB | Уровень OSI | Протоколы | Ключевая особенность | Цели |
|---|---|---|---|---|
| **ALB** (Application) | L7 | HTTP, HTTPS, gRPC | Маршрутизация по пути/хосту/заголовку, WebSockets | EC2, Lambda, IP, контейнеры |
| **NLB** (Network) | L4 | TCP, UDP, TLS | Сверхнизкая задержка, статический IP на AZ | EC2, IP, ALB |
| **GWLB** (Gateway) | L3/L4 | IP (все) | Прозрачная встроенная инспекция | Виртуальные устройства (firewall, IDS) |
| **CLB** (Classic) | L4/L7 | TCP, HTTP, HTTPS | **Устаревший** — не использовать в новых проектах | EC2-Classic (deprecated) |

### Краткое руководство по выбору

| Сценарий | Ответ |
|---|---|
| Маршрутизировать `/api/` на один сервис, `/web/` на другой | **ALB** |
| Нужен фиксированный/Elastic IP для вайтлистинга клиентов | **NLB** |
| Встроенная инспекция трафика Firewall/IDS/DLP | **GWLB** |
| Публикация сервиса через PrivateLink | **NLB** (за PrivateLink) |
| Миллионы запросов в секунду, сверхнизкая задержка | **NLB** |
| Аутентификация пользователей перед бэкендом | **ALB** + Cognito/OIDC |
| gRPC-микросервисы | **ALB** |

## ALB — Application Load Balancer

### Правила слушателя

Правила оцениваются в порядке приоритета. Каждое правило содержит условия и действие.

**Условия:** паттерн пути, хост-заголовок, HTTP-заголовок, строка запроса, IP источника

**Действия:** forward (в target group), redirect, фиксированный ответ, authenticate (Cognito / OIDC)

```
Правило 1: ЕСЛИ путь = /api/*       ТО forward → api-target-group
Правило 2: ЕСЛИ хост = admin.*      ТО authenticate-cognito + forward → admin-tg
Правило 3: ЕСЛИ заголовок X-Beta = true ТО forward → beta-target-group
По умолчанию: forward → main-target-group
```

### Целевые группы (Target Groups)

| Тип цели | Применение |
|---|---|
| **Instance** | EC2 инстансы по ID |
| **IP** | Любой IP (включая локальные через DX/VPN) |
| **Lambda** | Serverless-цели (HTTP → JSON-событие) |
| **ALB** | Цепочка NLB → ALB |

### Sticky Sessions

Направляет одного клиента к одной и той же цели через cookie.

| Тип cookie | Имя | Срок действия |
|---|---|---|
| Cookie приложения | Пользовательское (не AWSALB*) | Задаётся приложением |
| Основанный на времени | `AWSALB` | Настраивается на target group (1с – 7 дней) |

### Дополнительные возможности ALB

- **Интеграция WAF:** прикрепите AWS WAF Web ACL для фильтрации вредоносных запросов
- **Аутентификация:** встроенная поддержка OIDC и Amazon Cognito в правилах слушателя
- **gRPC:** поддерживается (в отличие от NLB и CLB)
- **WebSocket:** поддерживается

## NLB — Network Load Balancer

### Ключевые характеристики

- Работает на L4 — нет осведомлённости об HTTP-заголовках
- **Один статический Elastic IP на AZ** — клиенты могут вайтлистить конкретные IP
- **Сверхнизкая задержка** (~100 мкс против ~400 мс у ALB)
- Поддерживает TLS-терминацию
- Может регистрировать **ALB как цель** (цепочка NLB → ALB)

### Балансировка между AZ (Cross-Zone Load Balancing)

| Тип LB | По умолчанию | Стоимость |
|---|---|---|
| ALB | **Включена** | Бесплатно |
| NLB | **Отключена** | Платная при включении |
| GWLB | **Отключена** | Платная при включении |
| CLB | Отключена | Бесплатно при включении |

> **📌 Tip:** Для ALB cross-zone включена по умолчанию и бесплатна. Для NLB/GWLB отключена по умолчанию — включение стоит дополнительно за межAZ трафик. Это различие проверяется на экзамене.

### Зональный DNS

NLB предоставляет зональное DNS-имя на каждую AZ (`<az>.<name>.<region>.elb.amazonaws.com`) для клиентов, которым нужна AZ-локальная маршрутизация (снижает межAZ расходы).

## GWLB — Gateway Load Balancer

Используется для развёртывания, масштабирования и управления сторонними виртуальными сетевыми устройствами (firewall, IDS/IPS, DLP, инспекция пакетов).

**Принцип работы:**
```
Клиент → GWLB (точка входа)
           ↓ (туннель GENEVE, порт 6081)
        Устройство Firewall (инспектирует + возвращает трафик)
           ↓
         GWLB → маршрутизирует к назначению
```

- Использует инкапсуляцию **GENEVE** (не HTTP)
- Трафик прозрачен для источника/назначения — переадресация IP не нужна
- Целевая группа: виртуальные устройства (EC2-инстансы)

> **📌 Tip:** Если на экзамене упоминается «встроенная инспекция трафика», «bump-in-the-wire» или «сторонний firewall/IDS в AWS» — ответ **GWLB**.

## Auto Scaling Group (ASG)

Поддерживает заданное количество EC2-инстансов, заменяя нездоровые и масштабируя в зависимости от нагрузки.

### Launch Template vs Launch Configuration

| Параметр | Launch Template | Launch Configuration |
|---|---|---|
| Статус | **Рекомендуется** | Устаревший |
| Версионирование | **Да** (несколько версий) | Нет |
| Несколько типов инстансов | **Да** (смешанные инстансы) | Нет |
| Смешивание Spot + On-Demand | **Да** | Ограниченно |
| Требуется для | Новейших функций EC2 | — |

Всегда используйте **Launch Templates** для новых ASG.

### Проверки состояния (Health Checks)

| Тип | Что проверяется | По умолчанию |
|---|---|---|
| **EC2** | Статус инстанса (системный + инстанса) | Да |
| **ELB** | Health check целевой группы (HTTP-ответ) | Опционально |

Включите ELB health checks для замены инстансов, которые проходят EC2-проверку, но не проходят проверку приложения.

### Cooldown, Warm-up и масштабирование

| Параметр | Назначение | По умолчанию |
|---|---|---|
| **Cooldown** | Ожидание после scale-out/in перед следующим действием | 300 с |
| **Instance Warm-up** | Время на готовность нового инстанса (метрики не учитываются) | Настраивается |

## Политики масштабирования

| Тип политики | Принцип работы | Применение |
|---|---|---|
| **Target Tracking** | Поддерживает целевое значение метрики; ASG масштабируется автоматически | CpuUtilization 60%, ALBRequestCountPerTarget 1000 |
| **Step Scaling** | Разные размеры корректировки в зависимости от диапазона нарушения | CPU 70–80% → добавить 2; 80–90% → добавить 4 |
| **Simple Scaling** | Одна корректировка при срабатывании alarm; ждёт cooldown | Проще, но медленнее Step |
| **Scheduled** | Масштабирование по расписанию | Увеличить в 8:00 пн–пт, уменьшить в 18:00 |
| **Predictive** | ML-прогноз; преждевременное масштабирование | Повторяющиеся паттерны нагрузки |

**Рекомендация:** Target Tracking для большинства нагрузок. Predictive для предсказуемого циклического трафика.

### Политика завершения ASG

Порядок завершения по умолчанию:
1. AZ с наибольшим числом инстансов
2. Самый старый Launch Template / Launch Configuration
3. Инстанс, ближайший к следующему часу выставления счёта

---

## Lifecycle Hooks

Приостанавливают запуск/завершение инстанса для выполнения пользовательских действий.

| Состояние | Точка хука | Применение |
|---|---|---|
| `EC2_INSTANCE_LAUNCHING` | До ввода инстанса в эксплуатацию | Установка ПО, получение конфига из S3, тесты |
| `EC2_INSTANCE_TERMINATING` | До завершения инстанса | Drain соединений, архивирование логов, отмена регистрации |

```
Запуск:     Pending → Pending:Wait (хук) → Pending:Proceed → InService
Завершение: Terminating → Terminating:Wait (хук) → Terminating:Proceed → Terminated
```

Таймаут хука: 1 час по умолчанию (максимум 48 ч). Завершить через `complete-lifecycle-action`.

## Instance Refresh

Механизм rolling-обновления для замены инстансов ASG после изменения launch template.

- Задайте **минимальный процент здоровых инстансов** (например, 90%) — ASG поддерживает его во время обновления
- Задайте **время прогрева** — время до считывания нового инстанса здоровым
- Поддерживает автоматический откат при сбоях health check

```bash
aws autoscaling start-instance-refresh \
  --auto-scaling-group-name my-asg \
  --preferences '{"MinHealthyPercentage": 90, "InstanceWarmup": 300}'
```

## Сводка ловушек экзамена

| Ловушка | Правильный ответ |
|---|---|
| NLB поддерживает маршрутизацию по HTTP-пути | **ALB** маршрутизирует по пути; NLB только L4 |
| ALB имеет статический IP | **NLB** имеет статический IP на AZ; у ALB динамический IP |
| ALB поддерживает UDP | ALB **не поддерживает** UDP; NLB поддерживает |
| GWLB для CDN/кэширования | GWLB для **встроенной инспекции трафика** (firewall, IDS) |
| Cross-zone бесплатен для NLB | Cross-zone для NLB **платный** при включении |
| ASG заменяет инстанс при сбое EC2 health check | Замена при сбое на уровне приложения только если **включён ELB health check** |
| Launch Configuration — текущий best practice | Используйте **Launch Templates** — LC устаревший |
