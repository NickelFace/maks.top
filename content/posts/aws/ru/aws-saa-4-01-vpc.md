## Основы VPC

Virtual Private Cloud (VPC) — логически изолированная сеть внутри AWS. В каждом регионе для каждого аккаунта автоматически создаётся VPC по умолчанию (CIDR `172.31.0.0/16`).

---

## Компоненты VPC

| Компонент | Описание |
|---|---|
| **CIDR-блок** | IP-диапазон VPC (например, `10.0.0.0/16`). До 5 CIDR на VPC (1 основной + 4 вторичных) |
| **Подсеть** | Часть CIDR VPC, привязанная к одной AZ; бывает публичной или приватной |
| **Таблица маршрутизации** | Набор правил, определяющих, куда направляется трафик |
| **Internet Gateway (IGW)** | Обеспечивает доступ в интернет для публичных подсетей; горизонтально масштабируемый, HA |
| **NAT Gateway / NAT Instance** | Позволяет ресурсам в приватных подсетях инициировать исходящие интернет-соединения |
| **Security Group** | Stateful-фаервол на уровне инстанса (ENI) |
| **NACL** | Stateless-фаервол на уровне подсети |
| **VPC Endpoints** | Приватное подключение к сервисам AWS без IGW (см. статью 4.02) |

### Планирование CIDR

AWS резервирует 5 IP-адресов в каждой подсети:
- `.0` — адрес сети
- `.1` — роутер VPC
- `.2` — DNS-сервер
- `.3` — зарезервировано
- `.255` — широковещательный (не используется, но зарезервирован)

Для подсети `/24` (256 адресов): **251 используемый IP**.

> **📌 Tip:** На экзамене могут спросить, сколько IP доступно в /28 (16 всего − 5 = **11**). Всегда вычитайте 5.

## Подсети: публичные и приватные

| Тип | Маршрут к IGW | Получают публичный IP | Применение |
|---|---|---|---|
| **Публичная** | Да (`0.0.0.0/0 → igw-xxx`) | Да (при включённом auto-assign) | ALB, bastion hosts, NAT Gateway |
| **Приватная** | Нет | Нет | Серверы приложений, БД, Lambda в VPC |

Подсеть становится публичной, когда в её таблицу маршрутизации **добавляется маршрут к Internet Gateway**. Сам IGW должен быть прикреплён к VPC.

---

## Таблицы маршрутизации

Каждая подсеть связана ровно с одной таблицей маршрутизации. Приоритет: **самый специфичный маршрут побеждает**.

**Основная (main) таблица маршрутизации** — создаётся автоматически вместе с VPC; используется подсетями без явной привязки.

**Пользовательские таблицы** — создаются для гибкого управления по типам подсетей.

| Получатель | Цель | Значение |
|---|---|---|
| `10.0.0.0/16` | `local` | Внутренний трафик VPC; всегда присутствует, нельзя удалить |
| `0.0.0.0/0` | `igw-xxx` | Выход в интернет (публичная подсеть) |
| `0.0.0.0/0` | `nat-xxx` | Исходящий интернет через NAT (приватная подсеть) |
| `10.1.0.0/16` | `pcx-xxx` | Маршрут через VPC Peering |

## Internet Gateway vs Egress-Only IGW

| Параметр | Internet Gateway (IGW) | Egress-Only IGW |
|---|---|---|
| Версия IP | IPv4 + IPv6 | **Только IPv6** |
| Направление | Входящий + исходящий | **Только исходящий** (как NAT для IPv6) |
| Применение | Публичные ресурсы | IPv6-инстансы в приватных подсетях, которым нужен интернет |
| Прикреплён к | VPC | VPC |

> **📌 Tip:** Egress-Only IGW — аналог NAT Gateway для IPv6: позволяет инстансам с IPv6 инициировать исходящие соединения, оставаясь недоступными из интернета.

## NAT Gateway vs NAT Instance

| Параметр | NAT Gateway | NAT Instance |
|---|---|---|
| Управление | **Полностью управляемый** AWS | Управляемый EC2-инстанс |
| Высокая доступность | HA **в одной AZ** (разворачивать в каждой AZ для полной HA) | Настраивается вручную |
| Пропускная способность | До **100 Гбит/с** (масштабируется автоматически) | Ограничена типом инстанса |
| Стоимость | Почасово + за ГБ (дороже) | Стоимость EC2 (дешевле при малом объёме) |
| Security Groups | **Нельзя** прикрепить SG | **Можно** прикрепить SG |
| Port Forwarding | Не поддерживается | Поддерживается |
| Bastion Host | Нет | Может совмещать |
| Размещение | Должен быть в **публичной подсети** | Должен быть в **публичной подсети** |
| Source/Dest Check | Н/Д | Нужно **отключить** Source/Dest Check |

**Архитектура NAT Gateway с HA по AZ:**

```
AZ-A приватная подсеть → NAT GW-A (в публичной подсети AZ-A) → IGW
AZ-B приватная подсеть → NAT GW-B (в публичной подсети AZ-B) → IGW
```

> **📌 Tip:** На экзамене для новых архитектур почти всегда предпочтителен NAT Gateway. NAT Instance выбирайте только если вопрос упоминает экономию, port forwarding или совмещение с bastion host.

## Security Groups

Stateful-фаервол на уровне инстанса (ENI). **Только правила разрешения — явного запрета нет**.

| Свойство | Значение |
|---|---|
| Состояние | **Stateful** — обратный трафик разрешается автоматически |
| Правила | Только Allow (входящие и исходящие раздельно) |
| Уровень | Инстанс (ENI) |
| По умолчанию входящий | Весь трафик **запрещён** (без добавленных правил) |
| По умолчанию исходящий | Весь трафик **разрешён** |
| Оценка правил | Все правила оцениваются; применяется самое разрешительное |

**Цепочка SG (ссылка по ID вместо CIDR):**

```
Web-SG  входящий: разрешить порт 80 с 0.0.0.0/0
App-SG  входящий: разрешить порт 8080 с Web-SG  ← ссылка на SG
DB-SG   входящий: разрешить порт 3306 с App-SG
```

## Сетевые ACL (NACL)

Stateless-фаервол на уровне подсети. Поддерживает правила **разрешения и запрета**.

| Свойство | Значение |
|---|---|
| Состояние | **Stateless** — обратный трафик требует явного правила |
| Правила | Allow И Deny |
| Уровень | Подсеть |
| Оценка правил | **По возрастанию номера; останавливается на первом совпадении** |
| NACL по умолчанию | Разрешает весь входящий и исходящий |
| Пользовательский NACL | Запрещает всё по умолчанию до добавления правил |

**Порядок оценки правил:**
```
Правило 100: Allow TCP 443 с 0.0.0.0/0  ← оценивается первым
Правило 200: Allow TCP 80  с 0.0.0.0/0
Правило *:   Deny all                    ← оценивается последним (catch-all)
```

**Эфемерные порты** — NACL должен учитывать обратный трафик:
- Linux: `1024–65535`
- Windows: `49152–65535`
- AWS рекомендует: разрешить `1024–65535` на исходящий в NACL

| Параметр | Security Group | NACL |
|---|---|---|
| Уровень | Инстанс | Подсеть |
| Состояние | Stateful | Stateless |
| Правила | Только Allow | Allow + Deny |
| Оценка | Все правила | По порядку, первое совпадение |
| По умолчанию | Allow исходящий | Allow all (NACL по умолчанию) |

> **📌 Tip:** Ключевое отличие на экзамене: SG stateful — обратный трафик автоматический. NACL stateless — нужно явно разрешить обратный трафик включая эфемерные порты. Правила NACL оцениваются по порядку — первое совпадение побеждает.

## DNS в VPC

Два параметра VPC управляют поведением DNS:

| Параметр | По умолчанию | Эффект |
|---|---|---|
| `enableDnsSupport` | `true` | Включает DNS-разрешение через Route 53 Resolver по адресу `169.254.169.253` |
| `enableDnsHostnames` | `false` (пользовательские VPC) | Назначает публичные DNS-имена инстансам с публичными IP |

Оба должны быть `true`, чтобы EC2-инстансы получали публичные DNS-имена.

**Route 53 Resolver:** обеспечивает DNS для ресурсов VPC. Для гибридных архитектур используются входящие/исходящие **Resolver Endpoints** (см. статью по Route 53).

**Приватные Hosted Zones:** привязываются к VPC для разрешения приватных доменных имён. Требуют `enableDnsSupport = true`.

## VPC Flow Logs

Захватывают метаданные IP-трафика на уровне VPC, подсети или ENI.

**Назначения:** CloudWatch Logs, S3, Kinesis Data Firehose

**Что фиксируется:**
- IP-адреса и порты источника/назначения
- Протокол, количество пакетов, объём данных
- Действие (ACCEPT или REJECT)
- Время начала/окончания, статус логирования

**Что НЕ фиксируется:**
- Трафик к/от сервиса метаданных AWS (`169.254.169.254`)
- DHCP-трафик
- DNS-запросы к **Route 53 Resolver** внутри VPC (`169.254.169.253`)
- Трафик активации лицензии Windows
- Трафик между VPC Endpoints и сервисами AWS

```
version account-id interface-id srcaddr dstaddr srcport dstport
protocol packets bytes start end action log-status
2 123456789012 eni-abc12345 10.0.0.5 10.0.1.6 443 52412 6 10 840 ... ACCEPT OK
```

> **📌 Tip:** Flow Logs фиксируют метаданные потоков, а не содержимое пакетов. DNS-запросы к Route 53 Resolver не попадают в Flow Logs — для них используйте отдельные логи запросов Route 53 Resolver.

## Сводка ловушек экзамена

| Ловушка | Правильный ответ |
|---|---|
| SG — stateless | SG **stateful** — обратный трафик разрешается автоматически |
| NACL по умолчанию запрещает всё | Только **пользовательский** NACL запрещает всё; NACL по умолчанию разрешает всё |
| Правила NACL оцениваются произвольно | Правила оцениваются **по возрастанию номера** — первое совпадение |
| NAT Gateway в приватной подсети | NAT GW должен быть в **публичной подсети** |
| Один NAT GW покрывает все AZ | Для HA нужен **один NAT GW в каждой AZ** |
| Flow Logs захватывают DNS к резолверу | Трафик к Route 53 Resolver **НЕ** фиксируется в Flow Logs |
| IGW обрабатывает только исходящий IPv6 | Для исходящего IPv6 нужен **Egress-Only IGW**; обычный IGW двунаправленный |
