## Иерархия файловой системы Linux (FHS)

Linux представляет все файловые системы как единое дерево каталогов. FHS (Filesystem Hierarchy Standard, текущая версия 2.3) фиксирует расположение файлов и утилит так, чтобы разные дистрибутивы оставались совместимыми. Для подключения нового устройства его необходимо смонтировать в существующий пустой каталог, называемый **точкой монтирования**.

| Каталог | Назначение |
|---|---|
| `/` | Корень — не должен содержать лишних файлов |
| `/bin/` | Бинарные файлы, необходимые при загрузке |
| `/boot/` | Файлы загрузчика и образы ядра |
| `/dev/` | Файлы устройств |
| `/etc/` | Системные файлы конфигурации и скрипты |
| `/home/` | Домашние каталоги пользователей |
| `/lib/` | Разделяемые библиотеки ОС и модули ядра |
| `/lost+found/` | Файлы, восстановленные fsck |
| `/media/` | Точки монтирования сменных носителей (CD-ROM, USB) |
| `/mnt/` | Временные точки монтирования для администраторов |
| `/opt/` | Крупные сторонние приложения |
| `/proc/` | Виртуальная файловая система ядра (процессы, ресурсы) |
| `/root/` | Домашний каталог пользователя root |
| `/sbin/` | Бинарные файлы системного администрирования |
| `/tmp/` | Временные файлы |
| `/usr/` | Данные только для чтения: пользовательские команды, заголовочные файлы C |
| `/var/` | Изменяемые данные: журналы, почта, очереди печати |

> **Примечание:** Если точка монтирования не пуста, файлы в ней не исчезают — они просто становятся невидимы, пока устройство смонтировано. После `umount` они снова появляются.

## Внутренняя структура файловой системы

Перед использованием раздел необходимо инициализировать: на диск записываются служебные структуры. Этот процесс называется «созданием файловой системы».

Большинство UNIX-подобных файловых систем строится вокруг нескольких базовых концепций:

| Структура | Описание |
|---|---|
| **Суперблок** | Содержит информацию о файловой системе в целом: размер, количество блоков, тип |
| **Inode** | Хранит все атрибуты файла (права доступа, владелец, размер, временные метки) кроме имени |
| **Блок данных** | Блок, содержащий фактические данные файла |
| **Блок каталога** | Хранит имена файлов и соответствующие номера inode |
| **Косвенный блок** | Динамически выделяемый блок с указателями на блоки данных, используется когда прямые указатели inode исчерпаны |

Принцип работы: имя файла хранится в каталоге вместе с номером inode. Номер inode ведёт к блокам данных с содержимым файла. Для больших файлов inode указывает на косвенные блоки, которые в свою очередь указывают на блоки данных.

> **Важно:** Inode хранит все метаданные файла **кроме имени файла**. Имя хранится в записи каталога. Именно поэтому работают жёсткие ссылки: несколько имён в каталоге могут указывать на один и тот же inode.

## Создание файловой системы (mkfs)

`mkfs` — служебная утилита-обёртка для создания файловой системы на разделе или устройстве. Требует прав root.

```bash
# Общий синтаксис
mkfs [-c] [-t fstype] <device> [blocks]

# Создать ext4 на разделе
mkfs -t ext4 /dev/sdb1
# Краткая форма
mkfs.ext4 /dev/sdb1

# Создать ext2 (через mke2fs)
mkfs.ext2 /dev/sdb2
mke2fs /dev/sdb2         # mke2fs = mkfs.ext2

# Создать файловую систему FAT
mkfs -t vfat /dev/sdc1
mkfs.vfat /dev/sdc1
mkdosfs /dev/sdc1        # ещё один псевдоним

# Проверить наличие плохих блоков при создании
mkfs -c -t ext4 /dev/sdb1

# Просмотреть созданную файловую систему
parted -l
```

### Эквивалентные команды

| Полная форма | Краткая форма | Псевдонимы |
|---|---|---|
| `mkfs -t ext2 /dev/sdb1` | `mkfs.ext2 /dev/sdb1` | `mke2fs /dev/sdb1` |
| `mkfs -t ext3 /dev/sdb1` | `mkfs.ext3 /dev/sdb1` | `mke2fs -t ext3 /dev/sdb1` |
| `mkfs -t ext4 /dev/sdb1` | `mkfs.ext4 /dev/sdb1` | `mke2fs -t ext4 /dev/sdb1` |
| `mkfs -t vfat /dev/sdb1` | `mkfs.vfat /dev/sdb1` | `mkfs.msdos`, `mkdosfs` |

> **Внимание:** Создание файловой системы на разделе, уже содержащем данные, **безвозвратно уничтожает все данные**. Перед запуском `mkfs` тщательно проверьте имя устройства.

> **Важно:** `mkfs` — это обёртка. Под капотом она вызывает специализированную утилиту для каждого типа: `mkfs.ext4`, `mkfs.xfs`, `mkfs.btrfs` и т.д. Их можно вызывать напрямую.

## Монтирование файловых систем

### Временное монтирование

Команда `mount` подключает файловую систему к дереву каталогов. Точка монтирования должна уже существовать и желательно быть пустой.

```bash
# Базовая форма
mount /dev/sdb1 /mnt/data

# Явно указать тип файловой системы
mount -t ext4 /dev/sdb1 /mnt/data

# Смонтировать только для чтения
mount -o ro /dev/sdb1 /mnt/data

# Смонтировать по UUID
mount -U 652b786e-b87f-49d2-af23-8087ced0c828 /mnt/data

# Смонтировать по метке
mount -L MyData /mnt/data

# Смонтировать всё из /etc/fstab (кроме записей с noauto)
mount -a

# Симулировать монтирование без реального выполнения
mount -f -v /dev/sdb1 /mnt/data
```

### Основные опции -o

| Опция | Описание |
|---|---|
| `ro` | Только для чтения |
| `rw` | Чтение и запись |
| `exec` | Разрешить выполнение бинарных файлов |
| `noexec` | Запретить выполнение бинарных файлов |
| `user` | Разрешить монтирование конкретному пользователю |
| `users` | Разрешить монтирование всем пользователям |
| `noauto` | Не монтировать при `mount -a` |
| `sync` | Записывать напрямую на диск без буферизации (медленнее) |
| `defaults` | rw, suid, dev, exec, auto, nouser, async |

### Параметры mount

| Параметр | Описание |
|---|---|
| `-a` | Смонтировать всё из `/etc/fstab` |
| `-F` | Совместно с `-a` монтировать параллельно |
| `-f` | Симулировать без реального монтирования |
| `-L label` | Монтировать по метке |
| `-n` | Не записывать в `/etc/mtab` |
| `-r` | Монтировать только для чтения |
| `-t fstype` | Указать тип файловой системы |
| `-U uuid` | Монтировать по UUID |
| `-v` | Подробный вывод |
| `-w` | Монтировать для чтения и записи |
| `-s` | Игнорировать неподдерживаемые опции |

### Размонтирование

```bash
# По имени устройства
umount /dev/sdb1

# По точке монтирования
umount /mnt/data
```

> **Внимание:** Нельзя размонтировать устройство, которое используется процессом или пользователем. Также нельзя размонтировать устройство, на которое поверх него смонтировано другое устройство.

## Настройка /etc/fstab

Файл `/etc/fstab` (Filesystem Table) управляет монтированием файловых систем при загрузке. При старте система выполняет `mount -a`, что монтирует всё из fstab без флага `noauto`.

### Формат записи

```
<устройство>  <точка-монтирования>  <тип-фс>  <опции>  <dump>  <fsck>
```

Пример конфигурации:

```
# /etc/fstab — образец реальной конфигурации

# Корневой раздел по UUID
UUID=652b786e-b87f-49d2-af23-8087ced0c828  /          ext4    errors=remount-ro,noatime  0  1

# Домашний раздел по метке
LABEL=HomeDir                              /home      ext4    defaults                   0  2

# Раздел по имени устройства
/dev/sdb1                                  /mnt/data  xfs     defaults,noatime           0  2

# Раздел подкачки
/dev/sda3                                  none       swap    sw                         0  0

# Файл подкачки
/swapfile                                  none       swap    sw                         0  0

# Сетевая файловая система NFS
server:/export/share                       /mnt/nfs   nfs     defaults,_netdev           0  0
```

### Поля /etc/fstab

| Поле | Значение |
|---|---|
| 1. Устройство | `/dev/sda1`, `UUID=...`, `LABEL=...` или `server:/path` для NFS |
| 2. Точка монтирования | Абсолютный путь; `none` для swap |
| 3. Тип файловой системы | `ext4`, `xfs`, `btrfs`, `swap`, `nfs`, `vfat` |
| 4. Опции | `defaults`, `ro`, `noauto`, `_netdev` и т.д., через запятую |
| 5. Dump | `0` — пропустить резервное копирование dump, `1` — включить в dump |
| 6. fsck | `0` — пропустить, `1` — проверять первым (корневой), `2` — проверять после корневого |

> **Важно:** Файловые системы AutoFS **не должны** иметь записей в `/etc/fstab`. Если добавить такую запись, AutoFS не сможет управлять монтированием.

> **Совет:** Используйте UUID вместо имён устройств типа `/dev/sda1`. Если переставить диск в другой слот, имя может измениться, а UUID останется прежним.

## /etc/mtab и /proc/mounts

`/etc/mtab` содержит список смонтированных в данный момент файловых систем в формате fstab. Если файловая система смонтирована с флагом `-n`, запись в `/etc/mtab` не появляется.

`/proc/mounts` — виртуальный файл ядра с той же информацией, но более надёжный. На практике `/proc/mounts` предпочтительнее, поскольку он обновляется ядром напрямую.

```bash
# Просмотреть смонтированные файловые системы
cat /proc/mounts
mount
```

> **Примечание:** Разница между `/etc/mtab` и `/proc/mounts`: `/etc/mtab` не обновляется при монтировании с `-n`. `/proc/mounts` всегда актуален, поскольку поддерживается ядром.

## UUID и blkid

UUID (Universal Unique Identifier) — 128-битное число, однозначно идентифицирующее файловую систему. UUID присваивается при форматировании командой `mkfs` и не меняется при перестановке диска между слотами или машинами — только при повторном форматировании раздела.

Стандартный формат UUID: 32 шестнадцатеричных символа в группах `8-4-4-4-12`:

```
3297cded-69e9-4d35-b29f-c50cf263fb8b
```

### Команда blkid

```bash
# Показать UUID всех устройств
blkid

# UUID конкретного устройства
blkid /dev/sda5

# Пример вывода:
# /dev/sda5: UUID="24df5f2a-a23f-4130-ae45-90e1016031bc" TYPE="swap"
# /dev/sda1: UUID="652b786e-b87f-49d2-af23-8087ced0c828" TYPE="ext4"
```

### Работа с UUID

```bash
# Сгенерировать новый UUID
uuidgen

# Присвоить новый UUID файловой системе ext2/3/4
tune2fs -U $(uuidgen) /dev/sda5

# Альтернативный просмотр UUID
lsblk -f

# Временное монтирование по UUID
mount -U 652b786e-b87f-49d2-af23-8087ced0c828 /mnt/data
```

> **Внимание:** После форматирования раздела UUID меняется. Если используете UUID в `/etc/fstab`, обновите запись — иначе система не загрузится.

## Раздел подкачки и файл подкачки

Пространство подкачки — это раздел или файл, который ядро использует как расширение оперативной памяти. При заполнении физической памяти ядро перемещает неактивные страницы в swap.

### Создание раздела подкачки

```bash
# 1. Создать раздел (через fdisk/parted, тип 82 для Linux swap)
fdisk /dev/sdd

# 2. Инициализировать раздел как swap
mkswap /dev/sdd1
# Вывод:
# Setting up swapspace version 1, size = 838652 KiB
# no label, UUID=3297cded-69e9-4d35-b29f-c50cf263fb8b

# 3. Активировать swap
swapon /dev/sdd1

# 4. Проверить статус
swapon -s
free -m
```

### Создание файла подкачки

```bash
# 1. Создать файл нужного размера (512 МБ)
dd if=/dev/zero of=/swapfile bs=1024 count=524288

# 2. Установить права доступа
chmod 600 /swapfile

# 3. Инициализировать как swap
mkswap /swapfile

# 4. Активировать
swapon /swapfile
```

### Управление swap

```bash
# Деактивировать раздел или файл подкачки
swapoff /dev/sdd1
swapoff /swapfile

# Изменить приоритет swap (сначала нужен swapoff)
swapoff /dev/sdd1
swapon -p 10 /dev/sdd1

# Активировать всё пространство подкачки из /etc/fstab
swapon -a

# Мониторинг swap
swapon -s
cat /proc/swaps
free -m
```

### Приоритеты swap

При наличии нескольких пространств подкачки ядро выбирает по приоритету: чем выше число, тем выше приоритет. При ручной установке приоритет — положительное число (0–32767). Если не задан, ядро назначает отрицательное значение автоматически.

### Записи /etc/fstab для swap

```
# Раздел подкачки
/dev/sda3   none   swap   sw   0   0

# Swap по UUID
UUID=3297cded-69e9-4d35-b29f-c50cf263fb8b   none   swap   sw   0   0

# Файл подкачки
/swapfile   none   swap   sw   0   0
```

> **Внимание:** Не выполняйте `swapoff` текущего swap до тех пор, пока не будет готов новый раздел подкачки — система может зависнуть из-за нехватки виртуальной памяти.

> **Совет:** Выделенный раздел подкачки быстрее файла подкачки: нет фрагментации, и его можно разместить в начале диска (самое быстрое место на HDD). Файл подкачки проще изменить по размеру без переразбивки диска.

## Юниты монтирования systemd

В systemd-системах файловые системы можно монтировать через файлы юнитов вместо `/etc/fstab`. Тем не менее `/etc/fstab` остаётся предпочтительным методом: systemd автоматически преобразует записи fstab в нативные юниты через `systemd-fstab-generator` при загрузке.

### Соглашение об именовании

Имя файла отражает путь точки монтирования:

- Убрать ведущий и завершающий `/`
- Заменить оставшиеся `/` на `-`
- Добавить расширение `.mount`

Примеры:

| Точка монтирования | Имя файла юнита |
|---|---|
| `/home/temp/` | `home-temp.mount` |
| `/home/user/data/` | `home-user-data.mount` |
| `/mnt/backup` | `mnt-backup.mount` |

### Структура файла юнита монтирования

```ini
# /etc/systemd/system/home-user-data.mount

[Unit]
Description=Data for User

[Mount]
What=/dev/sda2
Where=/home/user/data
Type=ext4
Options=defaults
SloppyOptions=on
TimeoutSec=4

[Install]
WantedBy=multi-user.target
```

`What` принимает имя устройства или UUID: `What=/dev/disk/by-uuid/UUID`.

`SloppyOptions=on` игнорирует опции монтирования, не поддерживаемые типом файловой системы. По умолчанию отключено.

`TimeoutSec` задаёт таймаут в секундах. Если `mount` не завершится за это время, systemd считает операцию неудачной.

### Управление юнитами монтирования

```bash
# Перезагрузить конфигурацию systemd
systemctl daemon-reload

# Запустить монтирование вручную
systemctl start home-user-data.mount

# Проверить статус
systemctl status home-user-data.mount

# Сделать монтирование постоянным (включить при загрузке)
systemctl enable home-user-data.mount

# Убедиться, что смонтировано
mount | grep /home/user/data
```

> **Примечание:** Даже если вы используете только `/etc/fstab`, systemd всё равно управляет этими файловыми системами: он автоматически генерирует файлы юнитов через `systemd-fstab-generator`. Это стандартное и рекомендуемое поведение.

## Команда sync

Linux кэширует операции записи в памяти (буферы RAM) для повышения производительности. Команда `sync` принудительно сбрасывает эти буферы на диск.

```bash
sync
```

`sync` вызывается автоматически при завершении работы и перезагрузке системы. Ручное использование полезно перед извлечением USB-накопителя, когда `umount` по какой-то причине недоступен. Команда не имеет значимых опций — просто запускайте `sync` без аргументов.

> **Внимание:** Опция `sync` в `/etc/fstab` (четвёртое поле) заставляет каждую операцию записи идти напрямую на диск без буферизации. Это снижает производительность. Не путайте её с отдельной командой `sync`.

## Шпаргалка для экзамена

### Файлы и пути

| Файл / Путь | Назначение |
|---|---|
| `/etc/fstab` | Таблица монтирования, управляет монтированием при загрузке |
| `/etc/mtab` | Текущий список монтирований (может быть устаревшим) |
| `/proc/mounts` | Актуальный список монтирований от ядра |
| `/proc/swaps` | Состояние пространств подкачки |
| `/etc/systemd/system/*.mount` | Файлы юнитов монтирования systemd |

### Команды

| Команда | Действие |
|---|---|
| `mkfs -t ext4 /dev/sdb1` | Создать ext4 на разделе |
| `mkfs.ext4 /dev/sdb1` | То же, краткая форма |
| `mkfs -c -t ext4 /dev/sdb1` | Создать с проверкой плохих блоков |
| `mke2fs /dev/sdb1` | Создать ext2 (псевдоним mkfs.ext2) |
| `mount -a` | Смонтировать всё из `/etc/fstab` |
| `mount -t ext4 /dev/sdb1 /mnt` | Смонтировать с явным указанием типа |
| `mount -U UUID /mnt` | Смонтировать по UUID |
| `umount /mnt/data` | Размонтировать |
| `blkid` | Показать UUID всех устройств |
| `blkid /dev/sda1` | UUID конкретного устройства |
| `lsblk -f` | UUID и тип файловой системы |
| `mkswap /dev/sdd1` | Инициализировать раздел подкачки |
| `swapon /dev/sdd1` | Активировать swap |
| `swapoff /dev/sdd1` | Деактивировать swap |
| `swapon -a` | Активировать весь swap из fstab |
| `swapon -s` | Статистика swap |
| `swapon -p 10 /dev/sdd1` | Установить приоритет swap |
| `free -m` | Использование памяти и swap в МБ |
| `sync` | Сбросить буферы на диск |
| `uuidgen` | Сгенерировать новый UUID |
| `tune2fs -U UUID /dev/sda1` | Установить UUID для ext-файловой системы |
| `systemctl daemon-reload` | Перезагрузить файлы юнитов systemd |
| `systemctl enable home-tmp.mount` | Включить юнит монтирования при загрузке |

### Типичные ловушки на экзамене

- `/etc/mtab` и `/proc/mounts` почти идентичны, но `/proc/mounts` актуальнее и не зависит от флага `-n`
- `mkswap` только инициализирует; `swapon` активирует
- UUID меняется при переформатировании раздела
- Для изменения приоритета swap: сначала `swapoff`, затем `swapon -p`
- Имена файлов юнитов монтирования: убрать внешние `/`, внутренние заменить на `-`, добавить `.mount`
- Поле 5 fstab (dump): `0` или `1`. Поле 6 (fsck): `0` = отключено, `1` = корневой, `2` = остальные
- `noauto` предотвращает монтирование при запуске `mount -a`

## Практические вопросы

**В1.** Какой файл содержит наиболее актуальный список смонтированных файловых систем?

**Ответ:** `/proc/mounts` — поддерживается напрямую ядром. `/etc/mtab` может не обновляться при использовании `mount -n`.

---

**В2.** Нужно временно смонтировать раздел по UUID `abc123`. Какая команда правильная?

**Ответ:** `mount -U abc123 /mnt` — флаг `-U` указывает UUID для временного монтирования.

---

**В3.** Создан новый раздел `/dev/sde3`, который нужно использовать как swap. Какова правильная последовательность команд?

**Ответ:** `mkswap /dev/sde3` → `swapon /dev/sde3`. Сначала `mkswap` инициализирует раздел как swap, затем `swapon` его активирует.

---

**В4.** Как должен называться файл юнита для монтирования пути `/home/user/data/`?

**Ответ:** `home-user-data.mount` — убрать внешние `/`, внутренние заменить на `-`, добавить `.mount`.

---

**В5.** Что делает команда `sync`?

**Ответ:** Принудительно сбрасывает буферы записи из RAM на диск.

---

**В6.** В каком поле `/etc/fstab` указывается приоритет fsck?

**Ответ:** **Поле 6**: `0` = пропустить, `1` = проверять первым (корневой), `2` = проверять после корневого.

---

**В7.** Как проверить статус всех активных пространств подкачки?

**Ответ:** `swapon -s` — показывает имя, тип, размер, использование и приоритет каждого пространства подкачки. Также подходит `cat /proc/swaps`.

---

**В8.** Что хранит inode?

**Ответ:** Все метаданные файла — права доступа, владелец, размер, временные метки и указатели на блоки данных — **кроме имени файла**. Имя файла хранится в записи каталога вместе с номером inode.

---

**В9.** Нужно обновить UUID `/dev/sda2` (ext4) без потери данных. Какая команда это делает?

**Ответ:** `tune2fs -U $(uuidgen) /dev/sda2` — изменяет UUID без уничтожения данных. Запуск `mkfs.ext4 -U new-uuid /dev/sda2` переформатирует диск и сотрёт всё.

---

**В10.** Являются ли `mkfs.ext4 /dev/sdb1` и `mkfs -t ext4 /dev/sdb1` эквивалентными?

**Ответ:** **Да**, они полностью эквивалентны. `mkfs` — обёртка, которая вызывает `mkfs.ext4` при указании `-t ext4`.
