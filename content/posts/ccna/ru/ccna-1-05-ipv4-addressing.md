## Структура IPv4-адреса

IPv4-адрес — 32-битное число, записанное в виде четырёх десятичных октетов:

```
192    .   168    .    1    .    1
11000000.10101000.00000001.00000001
```

Адрес делится на две части:
- **Сетевая часть (N)** — биты, покрытые единицами маски
- **Хостовая часть (H)** — биты, покрытые нулями маски

В адресе:
- Все H-биты = 0 → **адрес сети**
- Все H-биты = 1 → **широковещательный адрес**
- H-биты = комбинация → **адрес хоста**

## Классы адресов

| Класс | Диапазон 1-го октета | Маска по умолч. | Кол-во сетей | Хостов/сеть |
|---|---|---|---|---|
| A | 1–126 | /8 (255.0.0.0) | 126 | 16 777 214 |
| B | 128–191 | /16 (255.255.0.0) | 16 384 | 65 534 |
| C | 192–223 | /24 (255.255.255.0) | 2 097 152 | 254 |
| D | 224–239 | — | Multicast | — |
| E | 240–255 | — | Зарезервировано | — |

> **📌 Обратите внимание:** 127.x.x.x — loopback (петля); 0.0.0.0 и 255.x.x.x — недопустимы для назначения хостам.

## Приватные адреса RFC 1918

| Класс | Диапазон | Префикс |
|---|---|---|
| A | 10.0.0.0 – 10.255.255.255 | /8 |
| B | 172.16.0.0 – 172.31.255.255 | /12 |
| C | 192.168.0.0 – 192.168.255.255 | /16 |

Также важные специальные адреса:

| Адрес/диапазон | Назначение |
|---|---|
| 127.0.0.1 | Loopback (петля обратной связи) |
| 169.254.0.0/16 | APIPA (автоматический адрес без DHCP) |
| 0.0.0.0 | Источник «любой» / маршрут по умолчанию |
| 255.255.255.255 | Ограниченный broadcast |

## Маски подсетей

Три способа записи одной маски:

| Запись | Пример |
|---|---|
| Двоичная | 11111111.11111111.11111111.00000000 |
| Десятичная (dotted decimal) | 255.255.255.0 |
| Префиксная (CIDR) | /24 |

### Таблица масок CIDR

| Префикс | Маска | Хостов |
|:---:|---|:---:|
| /8 | 255.0.0.0 | 16 777 214 |
| /16 | 255.255.0.0 | 65 534 |
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /29 | 255.255.255.248 | 6 |
| /30 | 255.255.255.252 | 2 |
| /31 | 255.255.255.254 | 0* |
| /32 | 255.255.255.255 | 1 (хост) |

> **💡 Совет:** /30 (2 хоста) — стандарт для WAN point-to-point-линков. /31 — по RFC 3021 для p2p без сетевого и широковещательного адреса. /32 — loopback или host-route.

## Формулы подсети

| Задача | Формула | Пример (/26) |
|---|---|---|
| Количество подсетей | 2ˢ | 2² = 4 (если заняли 2 бита у класса C) |
| Всего адресов | 2ʰ | 2⁶ = 64 |
| Допустимых хостов | 2ʰ − 2 | 64 − 2 = 62 |
| Шаг между сетями | 256 − (маска последнего октета) | 256 − 192 = 64 |

---

## Пошаговый расчёт подсети (на примере)

**Задача:** адрес 192.168.100.70/26. Определить сеть, broadcast, диапазон хостов.

**Шаг 1** — Маска /26 = 255.255.255.192, шаг = 256 − 192 = **64**

**Шаг 2** — Сети: .0, .64, .128, .192

**Шаг 3** — 70 попадает в подсеть **.64** (64 ≤ 70 < 128)

| Адрес сети | 192.168.100.64 |
|---|---|
| Первый хост | 192.168.100.65 |
| Последний хост | 192.168.100.126 |
| Broadcast | 192.168.100.127 |
| Маска | 255.255.255.192 / /26 |

## VLSM — Variable Length Subnet Masking

VLSM позволяет использовать маски разной длины в пределах одного адресного пространства.

**Принцип:** выделяйте подсети от большей к меньшей.

**Пример:** сеть 10.0.0.0/24, нужно:
- Сеть A: 50 хостов → /26 (62 хоста) → 10.0.0.0/26
- Сеть B: 25 хостов → /27 (30 хостов) → 10.0.0.64/27
- Сеть C: 10 хостов → /28 (14 хостов) → 10.0.0.96/28
- Линк WAN → /30 → 10.0.0.112/30

## Команды IOS

```bash
# Назначение IP-адреса на маршрутизатор
Router(config)# interface gigabitethernet 0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown

# Назначение IP-адреса на коммутатор (управление)
Switch(config)# interface vlan 1
Switch(config-if)# ip address 192.168.1.2 255.255.255.0
Switch(config-if)# no shutdown
Switch(config)# ip default-gateway 192.168.1.1

# Проверка
Router# show ip interface brief
Router# show interfaces gigabitethernet 0/0
Router# show ip route

# Диагностика
Router# ping 192.168.1.1
Router# ping 192.168.1.1 source loopback 0
Router# traceroute 10.0.0.1
```
