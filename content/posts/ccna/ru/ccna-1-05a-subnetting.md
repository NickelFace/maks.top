## Быстрый алгоритм (на экзамене)

### Шаг 1 — Определить параметры сети

Дано: `192.168.1.0/26`

```
/26 = 255.255.255.192

Хостов: 2^(32-26) - 2 = 2^6 - 2 = 64 - 2 = 62
Шаг подсетей: 256 - 192 = 64
```

### Шаг 2 — Список подсетей (считаем блоки шага)

```
Network:      192.168.1.0    Broadcast: 192.168.1.63
Hosts:        .1 – .62

Network:      192.168.1.64   Broadcast: 192.168.1.127
Hosts:        .65 – .126

Network:      192.168.1.128  Broadcast: 192.168.1.191
Hosts:        .129 – .190

Network:      192.168.1.192  Broadcast: 192.168.1.255
Hosts:        .193 – .254
```

### Таблица шагов по маске

| Маска | CIDR | Шаг | Хостов | Подсетей из /24 |
|---|---|---|---|---|
| 255.255.255.0 | /24 | 1 (из /24) | 254 | 1 |
| 255.255.255.128 | /25 | 128 | 126 | 2 |
| 255.255.255.192 | /26 | 64 | 62 | 4 |
| 255.255.255.224 | /27 | 32 | 30 | 8 |
| 255.255.255.240 | /28 | 16 | 14 | 16 |
| 255.255.255.248 | /29 | 8 | 6 | 32 |
| 255.255.255.252 | /30 | 4 | 2 | 64 |
| 255.255.255.254 | /31 | 2 | 0 (point-to-point) | 128 |
| 255.255.255.255 | /32 | 1 | 0 (loopback/host) | 256 |

## Набор задач — Уровень 1 (базовый)

### Задача 1.1

Дана сеть `10.0.0.0/8`. Сколько хостов в этой сети?

<details>
<summary>Ответ</summary>

`/8` → `32 - 8 = 24 бита для хостов`
`2^24 - 2 = 16.777.214 хостов`

</details>

---

### Задача 1.2

Хост имеет адрес `172.16.45.200/20`. Определи:
- Адрес сети
- Broadcast
- Диапазон хостов
- Маску подсети

<details>
<summary>Ответ</summary>

`/20` → маска `255.255.240.0`
Шаг в третьем октете: `256 - 240 = 16`

Третий октет: `45 / 16 = 2` (остаток 13) → блок начинается с `2 × 16 = 32`

```
Адрес сети:  172.16.32.0
Broadcast:   172.16.47.255
Хосты:       172.16.32.1 – 172.16.47.254
Маска:       255.255.240.0
```

</details>

---

### Задача 1.3

Хост `192.168.10.57/27`. Определи:
- Адрес сети
- Broadcast
- Первый/последний хост

<details>
<summary>Ответ</summary>

`/27` → маска `255.255.255.224`
Шаг: `256 - 224 = 32`

`57 / 32 = 1` (остаток 25) → блок: `1 × 32 = 32`

```
Адрес сети:   192.168.10.32
Broadcast:    192.168.10.63
Первый хост:  192.168.10.33
Последний:    192.168.10.62
```

</details>

---

### Задача 1.4

Нужно разместить 50 хостов в одной подсети. Какая минимальная маска?

<details>
<summary>Ответ</summary>

50 хостов → нужно `2^n - 2 ≥ 50`
- `/26` → `2^6 - 2 = 62` ✓ (минимальная маска)
- `/27` → `2^5 - 2 = 30` ✗ (мало)

**Ответ: /26 (255.255.255.192)**

</details>

### Задача 1.5

Какие из следующих адресов находятся в одной подсети `192.168.5.0/28`?
- A) `192.168.5.14`
- B) `192.168.5.16`
- C) `192.168.5.12`
- D) `192.168.5.17`

<details>
<summary>Ответ</summary>

`/28` → шаг 16

Подсети: `.0–.15`, `.16–.31`, `.32–.47`...

В `192.168.5.0/28` находятся хосты `.1–.14` (`.0` = сеть, `.15` = broadcast)

- A) `.14` ✓
- B) `.16` ✗ (следующая подсеть)
- C) `.12` ✓
- D) `.17` ✗ (следующая подсеть)

**Ответ: A и C**

</details>

## Набор задач — Уровень 2 (средний)

### Задача 2.1

Разбить `192.168.1.0/24` на 4 равные подсети. Укажи адрес сети, маску и broadcast каждой.

<details>
<summary>Ответ</summary>

4 подсети из /24 → нужно 2 бита для подсетей → `/24 + 2 = /26`

Шаг: 64

```
Подсеть 1:  192.168.1.0/26    → Broadcast: 192.168.1.63
Подсеть 2:  192.168.1.64/26   → Broadcast: 192.168.1.127
Подсеть 3:  192.168.1.128/26  → Broadcast: 192.168.1.191
Подсеть 4:  192.168.1.192/26  → Broadcast: 192.168.1.255
```

</details>

### Задача 2.2

Хост `172.30.100.100/18`. Это Private адрес? Укажи:
- Класс и диапазон приватных
- Адрес сети
- Количество хостов

<details>
<summary>Ответ</summary>

`172.30.x.x` → Класс B, приватный диапазон `172.16.0.0 – 172.31.255.255` ✓

`/18` → маска `255.255.192.0`
Шаг во втором октете: `256 - 192 = 64`

`100 / 64 = 1` (остаток 36) → блок: `1 × 64 = 64`

```
Адрес сети: 172.30.64.0/18
Broadcast:  172.30.127.255
Хостов:     2^14 - 2 = 16.382
```

</details>

### Задача 2.3

Определи, правильно ли настроены два хоста в одной подсети:
- Хост A: `10.1.50.130/26`
- Хост B: `10.1.50.190/26`

<details>
<summary>Ответ</summary>

`/26` → шаг 64

Подсети: `10.1.50.0/26` (хосты .1–.62), `10.1.50.64/26` (хосты .65–.126), `10.1.50.128/26` (хосты .129–.190), `10.1.50.192/26`...

- Хост A `.130` → подсеть `10.1.50.128/26`
- Хост B `.190` → подсеть `10.1.50.128/26`

**.130 и .190 — в одной подсети `10.1.50.128/26` ✓**

Broadcast этой подсети: `10.1.50.191`

</details>

### Задача 2.4 (формат экзамена)

Роутер R1 имеет интерфейс `192.168.10.65/27`. Клиент должен иметь IP в той же подсети. Какой из вариантов **не подойдёт**?

- A) `192.168.10.70`
- B) `192.168.10.90`
- C) `192.168.10.94`
- D) `192.168.10.63`

<details>
<summary>Ответ</summary>

`/27` → шаг 32

`.65` → блок: `64` (2 × 32 = 64)
Подсеть: `192.168.10.64/27`, хосты `.65–.94`, broadcast `.95`

- A) `.70` ✓ (в диапазоне)
- B) `.90` ✓ (в диапазоне)
- C) `.94` ✓ (последний хост)
- D) `.63` ✗ (это broadcast предыдущей подсети `192.168.10.32/27`)

## VLSM — задачи

### Что такое VLSM

**VLSM** (Variable Length Subnet Masking) — разные подсети имеют разные маски. Позволяет не тратить адреса зря.

**Правило**: всегда начинай с **самой большой** подсети.

---

### Задача VLSM 3.1

Дана сеть `192.168.20.0/24`. Создать подсети для:
- Отдел A: 60 хостов
- Отдел B: 28 хостов
- Отдел C: 12 хостов
- WAN линк: 2 хоста

<details>
<summary>Ответ</summary>

**Шаг 1: Сортируем по убыванию**
60 → 28 → 12 → 2

**Шаг 2: Подбираем маски**

| Отдел | Хостов | Маска | Хостов в подсети |
|---|---|---|---|
| A | 60 | /26 (62 хоста) | 62 |
| B | 28 | /27 (30 хостов) | 30 |
| C | 12 | /28 (14 хостов) | 14 |
| WAN | 2 | /30 (2 хоста) | 2 |

**Шаг 3: Распределяем адреса**

```
Отдел A:  192.168.20.0/26    → хосты .1–.62,  broadcast .63
Отдел B:  192.168.20.64/27   → хосты .65–.94, broadcast .95
Отдел C:  192.168.20.96/28   → хосты .97–.110, broadcast .111
WAN:      192.168.20.112/30  → хосты .113–.114, broadcast .115
```

Осталось свободно: `192.168.20.116 – 192.168.20.255`

</details>

---

### Задача VLSM 3.2

Сеть компании: `10.0.0.0/22`. Нужно:
- Производство: 400 хостов
- Офис: 200 хостов
- Склад: 50 хостов
- Управление: 10 хостов

<details>
<summary>Ответ</summary>

`/22` = `10.0.0.0 – 10.0.3.255` (1022 хоста всего)

**Маски:**

| Сеть | Хостов | Маска | Размер |
|---|---|---|---|
| Производство | 400 | /23 (510 хостов) | 512 адресов |
| Офис | 200 | /24 (254 хоста) | 256 адресов |
| Склад | 50 | /26 (62 хоста) | 64 адреса |
| Управление | 10 | /28 (14 хостов) | 16 адресов |

**Распределение:**

```
Производство: 10.0.0.0/23   → 10.0.0.1 – 10.0.1.254, broadcast 10.0.1.255
Офис:         10.0.2.0/24   → 10.0.2.1 – 10.0.2.254, broadcast 10.0.2.255
Склад:        10.0.3.0/26   → 10.0.3.1 – 10.0.3.62,  broadcast 10.0.3.63
Управление:   10.0.3.64/28  → 10.0.3.65 – 10.0.3.78, broadcast 10.0.3.79
```

Свободно: `10.0.3.80 – 10.0.3.255`

</details>

## Wildcard маска — практика

**Wildcard** = инверсия маски подсети. Используется в ACL и OSPF.

| Маска | Wildcard | Как получить |
|---|---|---|
| /24 = 255.255.255.0 | 0.0.0.255 | 255-255=0, 255-255=0, 255-255=0, 255-0=255 |
| /25 = 255.255.255.128 | 0.0.0.127 | 255-128=127 |
| /26 = 255.255.255.192 | 0.0.0.63 | 255-192=63 |
| /27 = 255.255.255.224 | 0.0.0.31 | 255-224=31 |
| /28 = 255.255.255.240 | 0.0.0.15 | 255-240=15 |
| /30 = 255.255.255.252 | 0.0.0.3 | 255-252=3 |
| /16 = 255.255.0.0 | 0.0.255.255 | |
| /8 = 255.0.0.0 | 0.255.255.255 | |

### Специальные wildcard

| Wildcard | Значение | Пример использования |
|---|---|---|
| `0.0.0.0` | Точный хост (host) | `permit host 10.0.0.1` |
| `255.255.255.255` | Любой адрес (any) | `permit any` |

### Задача по wildcard

ACL должен разрешать только сеть `172.16.16.0/20`. Какой wildcard использовать?

<details>
<summary>Ответ</summary>

`/20` = `255.255.240.0`
Wildcard = `255.255.255.255 - 255.255.240.0 = 0.0.15.255`

```
access-list 10 permit 172.16.16.0 0.0.15.255
```

</details>

## Подсчёт подсетей

**Сколько подсетей из /X можно создать с маской /Y?**

Формула: `2^(Y-X)` — количество подсетей

| Исходная | Новая | Подсетей |
|---|---|---|
| /24 | /25 | 2^1 = 2 |
| /24 | /26 | 2^2 = 4 |
| /24 | /27 | 2^3 = 8 |
| /24 | /28 | 2^4 = 16 |
| /24 | /30 | 2^6 = 64 |
| /16 | /24 | 2^8 = 256 |
| /8 | /16 | 2^8 = 256 |

## Частые ошибки на экзамене

| Ошибка | Правильно |
|---|---|
| Включить адрес сети в диапазон хостов | `.0` — адрес сети, хосты начинаются с `.1` |
| Включить broadcast в диапазон хостов | Broadcast — последний адрес блока, не хост |
| Использовать маску вместо wildcard в ACL | `permit 10.0.0.0 255.0.0.0` → неверно; нужно `0.255.255.255` |
| Забыть `/31` для point-to-point | /31 = 2 адреса, нет сети/broadcast (RFC 3021) |
| Считать хосты как 2^n вместо 2^n-2 | Всегда `-2` для обычных подсетей |
