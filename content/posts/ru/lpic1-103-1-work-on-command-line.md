## Получение информации о системе

### pwd и uname

Чтобы узнать, в какой директории ты сейчас находишься, запусти `pwd` (print working directory):

```bash
$ pwd
/home/frank
```

Для получения информации о ядре и архитектуре системы используй `uname`. Флаг `-a` выводит всё сразу:

```bash
$ uname -a
Linux base 4.18.0-18-generic #19~18.04.1-Ubuntu SMP Fri Apr 5 10:22:13 UTC 2019 x86_64 x86_64 x86_64 GNU/Linux
```

Отдельные флаги `uname`:

| Флаг | Что выводит |
|------|-------------|
| `-s` | Имя ядра |
| `-n` | Hostname |
| `-r` | Релиз ядра |
| `-v` | Версия ядра |
| `-m` | Архитектура |
| `-i` | Аппаратная платформа |
| `-o` | Операционная система |
| `-a` | Всё вместе |

### man и apropos

`man` открывает страницу руководства для команды:

```bash
$ man uname
```

Если не помнишь точное имя команды, используй `apropos`. Он ищет по именам и описаниям всех man-страниц:

```bash
$ apropos kernel
uname (2) - get name and information about current kernel
```

### type и which

`type` показывает тип команды и её расположение. Он умеет проверять несколько команд сразу:

```bash
$ type uname cp kill which
uname is hashed (/bin/uname)
cp is /bin/cp
kill is a shell builtin
which is /usr/bin/which
```

Обрати внимание на "hashed": это значит, что команда была недавно использована и попала в хеш-таблицу Bash для ускорения поиска. Очистить хеш-таблицу можно командой `hash -d`.

Три возможных типа команд в выводе `type`:
- Обычный бинарник (путь к файлу)
- Shell builtin (встроен в сам Bash)
- Hashed (закеширован в текущей сессии)

`which` возвращает только путь к исполняемому файлу, без лишних слов:

```bash
$ which uname
/bin/uname
```

Для встроенных команд Bash вместо `which` используй `help`.

## История команд

### Команда history

`history` выводит последние выполненные команды с порядковыми номерами. Последние команды идут в конце списка. Удобно сочетать с `grep`:

```bash
$ history | grep bash_history
1605 sudo find /home -name ".bash_history" | xargs grep sudo
```

Просмотр только последних N команд:

```bash
$ history 20
```

Навигация по истории с помощью клавиш:
- Стрелка вверх/вниз — перебирает команды в истории.
- `Ctrl+R` — интерактивный поиск по истории.

### Файл .bash_history

История сохраняется в скрытый файл `~/.bash_history`. Он видим только при `ls -a`:

```bash
$ ls -a /home/frank
. .. .bash_history .bash_logout .bashrc .profile .ssh newfile
```

Важный нюанс: команды добавляются в динамическую базу данных истории немедленно, но в файл `.bash_history` записываются только при выходе из сессии. Поэтому самые свежие команды в файле могут отсутствовать.

Сравнить динамическую историю с файлом:

```bash
$ history 20
$ tail -n 20 ~/.bash_history
```

## Переменные окружения

### Просмотр переменных: env и set

`env` выводит переменные окружения. Среди них всегда есть `PATH` — список директорий, где оболочка ищет исполняемые файлы:

```bash
$ env
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

Вывести только значение конкретной переменной через `echo`:

```bash
$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

`set` выводит больше, включая локальные переменные и функции оболочки. Разница между `env` и `set`:

| Команда | Показывает |
|---------|-----------|
| `env`   | Только экспортированные переменные окружения |
| `set`   | Все переменные, включая локальные, плюс функции |

Пример: переменная `mynewvar`, созданная без `export`, видна через `set`, но не через `env`:

```bash
$ mynewvar=goodbye
$ env | grep mynewvar
(нет вывода)
$ set | grep mynewvar
mynewvar=goodbye
```

### Создание и экспорт переменных

Создать переменную просто:

```bash
$ myvar=hello
$ echo $myvar
hello
```

Пробелов вокруг `=` быть не должно. Такая переменная живёт только в текущей сессии. Дочерние оболочки её не увидят:

```bash
$ bash
$ echo $myvar
(пусто)
```

Чтобы переменная передавалась в дочерние оболочки, её нужно экспортировать:

```bash
$ export myvar
$ bash
$ echo $myvar
hello
```

Добавить директорию в PATH временно (без перезагрузки):

```bash
$ export PATH="/home/frank/myfiles:$PATH"
```

### Удаление переменных: unset

`unset` удаляет переменную. Знак `$` перед именем не нужен:

```bash
$ unset myvar
$ echo $myvar
(пусто)
```

Если удалить переменную `PATH`, оболочка перестанет находить команды по имени. Придётся вводить полные пути, например `/usr/bin/sudo`. Выход из оболочки восстанавливает PATH.

## Экранирование специальных символов

Bash интерпретирует специальные символы и пробелы по-своему. Создание файла с пробелами в имени без кавычек создаст несколько файлов:

```bash
$ touch my big file
$ ls
my  big  file
```

Три способа экранировать специальные символы:

**Двойные кавычки** сохраняют буквальный смысл большинства символов, но `$`, `` ` ``, `\` и `!` всё равно интерпретируются:

```bash
$ touch "my big file"
$ echo "$myvar"
hello
```

**Одинарные кавычки** делают всё буквальным, без исключений:

```bash
$ touch 'my big file'
$ echo '$myvar'
$myvar
```

**Обратный слеш** экранирует один символ:

```bash
$ touch my\ big\ file
```

Полный список специальных символов в Bash: `& ; | * ? " ' [ ] ( ) $ < > { } # / \ ! ~`

## Полезные команды для экзамена

```bash
pwd                         # текущая директория
uname -a                    # вся информация о системе
uname -v                    # версия ядра
uname -i                    # аппаратная платформа
man <команда>               # страница руководства
apropos <ключевое слово>    # поиск по man-страницам
type <команда>              # тип и расположение команды
which <команда>             # путь к исполняемому файлу
history                     # история команд
history 20                  # последние 20 команд
history | grep <строка>     # поиск в истории
tail -n 20 ~/.bash_history  # последние 20 строк файла истории
env                         # переменные окружения
set                         # все переменные и функции
echo $VARIABLE              # значение переменной
export VAR=value            # создать и экспортировать переменную
export PATH="dir:$PATH"     # добавить директорию в PATH
unset VAR                   # удалить переменную
hash -d                     # очистить хеш-таблицу команд
```

## Типичные вопросы экзамена

**Какая команда выводит текущую рабочую директорию?**
`pwd`

**Чем отличается `set` от `env`?**
`env` показывает только экспортированные переменные. `set` показывает все переменные, включая локальные, и функции оболочки.

**Переменная создана без `export`. Будет ли она доступна в дочерней оболочке?**
Нет. Переменная без `export` доступна только в текущей сессии.

**Команда `type` показала `kill is a shell builtin`. Что это означает?**
Команда `kill` встроена в оболочку Bash и не является отдельным исполняемым файлом.

**В чём разница между одинарными и двойными кавычками?**
Одинарные кавычки делают все символы буквальными. Двойные кавычки сохраняют интерпретацию `$`, `` ` ``, `\` и `!`.

**Почему самые последние команды отсутствуют в `~/.bash_history`?**
Команды попадают в файл только при завершении сессии. В течение сессии они хранятся в динамической базе данных в памяти.

**Как временно добавить директорию `/opt/mytools` в PATH?**
`export PATH="/opt/mytools:$PATH"`

## Упражнения

### Урок 1: Практические упражнения

**1. Используя систему `man`, выяснить, как заставить `apropos` вывести краткое сообщение об использовании и завершить работу.**

<details>
<summary>Ответ</summary>

```bash
$ man apropos
```

В разделе "Options" найти параметр `--usage`. Он выводит краткое сообщение о синтаксисе команды и завершает работу.

```bash
$ apropos --usage
```

> Параметр `--usage` есть у многих GNU-утилит. Это стандартный способ получить минимальную справку без открытия полной man-страницы.

</details>

**2. Используя систему `man`, определить, под какой лицензией распространяется команда `grep`.**

<details>
<summary>Ответ</summary>

```bash
$ man grep
```

Нужно прокрутить страницу до раздела "Copyright". Там указано, что программа использует лицензию Free Software Foundation (GPL).

</details>

### Урок 1: Исследовательские упражнения

**1. Определить аппаратную архитектуру и версию ядра Linux в удобочитаемом формате.**

<details>
<summary>Ответ</summary>

Сначала изучить man-страницу `uname`, чтобы найти нужные флаги:

```bash
$ man uname
```

Флаг `-v` выводит версию ядра, флаг `-i` показывает аппаратную платформу:

```bash
$ uname -v
$ uname -i
```

> Можно также использовать `uname -a`, чтобы получить всё сразу. Но если нужна чистая строка без лишнего — `-v` и `-i` точнее.

</details>

**2. Вывести последние 20 строк динамической базы истории и файла `.bash_history` и сравнить их.**

<details>
<summary>Ответ</summary>

```bash
$ history 20
$ tail -n 20 ~/.bash_history
```

Динамическая история (вывод `history`) будет содержать команды текущей сессии, которых ещё нет в файле `.bash_history`. В файл они попадут только при завершении сессии.

</details>

**3. С помощью `apropos` найти man-страницу команды, которая показывает размер блочного устройства в байтах, а не в мегабайтах или гигабайтах.**

<details>
<summary>Ответ</summary>

```bash
$ apropos block
```

В выводе найти `lsblk`, которая выводит список блочных устройств. Затем открыть её man-страницу и найти нужный флаг:

```bash
$ man lsblk
```

В разделе "Description" флаг `-b` выводит размер в байтах. Запустить:

```bash
$ lsblk -b
```

</details>

### Урок 2: Практические упражнения

**1. Используя `export`, добавить новую директорию в PATH. Изменение не должно сохраняться после перезагрузки.**

<details>
<summary>Ответ</summary>

```bash
$ export PATH="/home/yourname/myfiles:$PATH"
```

Проверить, что директория добавилась:

```bash
$ echo $PATH
```

Для проверки работы нового пути создать скрипт, сделать его исполняемым и запустить из другой директории:

```bash
$ mkdir -p ~/myfiles
$ touch ~/myfiles/myscript.sh
$ echo '#!/bin/bash' >> ~/myfiles/myscript.sh
$ echo 'echo Hello' >> ~/myfiles/myscript.sh
$ chmod +x ~/myfiles/myscript.sh
$ myscript.sh
Hello
```

> Такое изменение PATH существует только в текущей сессии. Чтобы сделать его постоянным, нужно добавить строку экспорта в `~/.bashrc` или `~/.bash_profile`.

</details>

---

**2. Удалить переменную PATH с помощью `unset`. Попробовать выполнить команду через `sudo`. Что произошло и почему? Выход из оболочки восстановит исходное состояние.**

<details>
<summary>Ответ</summary>

```bash
$ unset PATH
$ sudo cat /etc/shadow
```

Команда завершится с ошибкой: оболочка не найдёт `sudo`, потому что не знает, где его искать. `sudo` — это обычный бинарник в `/usr/bin/sudo`, и без PATH оболочка не может его найти по одному имени.

Запустить через абсолютный путь всё же получится:

```bash
$ /usr/bin/sudo /bin/cat /etc/shadow
```

Восстановить PATH можно, выйдя из текущей оболочки:

```bash
$ exit
```

> Это упражнение наглядно объясняет, почему PATH настолько важна. Без неё оболочка не может найти ни одну команду по имени.

</details>

### Урок 2: Исследовательские упражнения

**1. Найти полный список специальных символов Bash.**

<details>
<summary>Ответ</summary>

Полный список специальных символов, которые Bash интерпретирует нелитерально:

```
& ; | * ? " ' [ ] ( ) $ < > { } # / \ ! ~
```

Каждый из них имеет особый смысл в оболочке. Например, `|` создаёт пайп, `$` раскрывает переменную, `*` служит метасимволом для имён файлов.

</details>

---

**2. Попробовать использовать различные методы экранирования специальных символов. Есть ли между ними разница?**

<details>
<summary>Ответ</summary>

Да, разница есть. Двойные кавычки экранируют большинство символов, но `$`, `` ` ``, `\` и `!` сохраняют своё значение:

```bash
$ mynewvar=goodbye
$ echo "$mynewvar"
goodbye
```

Одинарные кавычки делают абсолютно всё буквальным, включая `$`:

```bash
$ echo '$mynewvar'
$mynewvar
```

Обратный слеш экранирует один следующий символ:

```bash
$ echo \$mynewvar
$mynewvar
```

Выбор метода зависит от задачи: если нужно раскрыть переменные внутри строки, используй двойные кавычки. Если нужно передать строку буквально, используй одинарные.

</details>
