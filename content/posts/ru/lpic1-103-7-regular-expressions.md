## Что нужно знать

- Составлять простые регулярные выражения из нескольких элементов нотации.
- Понимать разницу между базовыми (BRE) и расширенными (ERE) регулярными выражениями.
- Разбираться в специальных символах, классах символов, квантификаторах и якорях.
- Использовать инструменты для поиска по файловой системе или содержимому файлов.
- Применять регулярные выражения для удаления, изменения и замены текста.

## Регулярные выражения: основы

Регулярное выражение (regex) описывает шаблон поиска. Инструменты вроде `grep` и `sed` используют этот шаблон, чтобы находить совпадения в тексте.

Базовый строительный блок регулярного выражения называется **атом** — один символ, который может иметь или не иметь специального значения. Большинство обычных символов сохраняют буквальное значение, а несколько зарезервированных символов работают как метасимволы.

Подстрока, которая соответствует атому с квантификатором, называется **piece**. Если подстрока совпадает с выражением и более длинная подстрока с той же начальной позиции тоже совпадает, берётся более длинная.

### Специальные символы

| Символ | Значение |
|--------|----------|
| `.` | Любой одиночный символ (кроме новой строки) |
| `*` | Ноль или больше повторений предыдущего элемента |
| `^` | Начало строки (якорь) |
| `$` | Конец строки (якорь) |
| `\` | Экранирование следующего символа |
| `[]` | Класс символов |
| `[^]` | Отрицательный класс символов |

Точка означает буквально любой символ. Чтобы найти именно точку, её нужно экранировать: `\.`.

### Классы символов

Класс символов задаётся в квадратных скобках и совпадает с любым одним символом из перечисленных.

```
[abc]      — a, b или c
[a-z]      — любая строчная буква
[A-Z]      — любая прописная буква
[0-9]      — любая цифра
[a-zA-Z]   — любая буква
[^0-9]     — любой символ, кроме цифры
```

POSIX-классы работают внутри `[]` и переносимы между локалями. Классы нельзя использовать как самостоятельный атом вне квадратных скобок и нельзя задавать ими границы диапазона.

| Класс | Описание |
|-------|---------|
| `[:alpha:]` | Буквы |
| `[:digit:]` | Цифры 0–9 |
| `[:alnum:]` | Буквы и цифры |
| `[:upper:]` | Прописные буквы |
| `[:lower:]` | Строчные буквы |
| `[:space:]` | Пробельные символы: пробел, `\t`, `\n`, `\r`, `\f`, `\v` |
| `[:blank:]` | Пробел и табуляция |
| `[:punct:]` | Печатаемые символы, не являющиеся пробелом или буквой/цифрой |
| `[:print:]` | Все печатаемые символы, включая пробел |
| `[:graph:]` | Все печатаемые символы, кроме пробела |
| `[:cntrl:]` | Управляющие символы |
| `[:ascii:]` | Символы ASCII (0–127) |
| `[:xdigit:]` | Шестнадцатеричные цифры 0–9, a–f, A–F |

Пример: `[[:alpha:]]` совпадает с любой буквой независимо от локали. `[[:blank:]]{2}` совпадает ровно с двумя пробельными символами.

### Квантификаторы

Квантификатор указывает, сколько раз должен встретиться предыдущий элемент.

| Квантификатор | BRE | ERE | Значение |
|--------------|-----|-----|---------|
| `*` | да | да | 0 или более |
| `\+` | да | `+` | 1 или более |
| `\?` | да | `?` | 0 или 1 |
| `\{n\}` | да | `{n}` | ровно n раз |
| `\{n,\}` | да | `{n,}` | n или более раз |
| `\{n,m\}` | да | `{n,m}` | от n до m раз |

В BRE символы `+`, `?`, `{` и `}` нужно экранировать обратным слешем. В ERE они работают без экранирования.

### Якоря

Якоря не совпадают с символом, а фиксируют позицию в строке.

- `^` совпадает с началом строки: `^root` найдёт строки, начинающиеся с «root». Вне начала выражения `^` воспринимается буквально.
- `$` совпадает с концом строки: `bash$` найдёт строки, заканчивающиеся на «bash». В середине выражения `$` воспринимается буквально.
- `^$` совпадает с пустой строкой.

## BRE и ERE: в чём разница

GNU-утилиты поддерживают два диалекта регулярных выражений.

**BRE (Basic Regular Expressions)** используется в `grep` и `sed` по умолчанию. Метасимволы `+`, `?`, `|`, `(`, `)`, `{`, `}` нужно экранировать, чтобы они работали как операторы.

**ERE (Extended Regular Expressions)** используется в `grep -E` и `egrep`. Те же метасимволы работают сразу без экранирования.

Сравнение: чтобы найти «colour» или «color», в BRE пишут `colou\?r`, в ERE — `colou?r`. Оба выражения эквивалентны по смыслу, но синтаксис разный.

Операция «или» через `|` доступна только в ERE: `cat|dog` найдёт строки с «cat» или «dog». В BRE `|` воспринимается буквально, однако большинство программ GNU поддерживают `\|` как расширение.

Группировка через скобки тоже ведёт себя иначе. В BRE: `\(abc\)*`, в ERE: `(abc)*`. Скобки без экранирования в BRE являются обычными символами.

**Обратные ссылки (back references)** позволяют ссылаться на уже найденную группу внутри того же выражения. Группа задаётся скобками, а ссылка — `\1`, `\2` и т.д. В ERE: `([[:digit:]])\1` совпадёт с любой цифрой, повторённой дважды подряд. В BRE скобки экранируются: `\([[:digit:]]\)\1`.

## grep, egrep, fgrep

### grep

`grep` ищет строки, совпадающие с шаблоном, по умолчанию использует BRE.

```bash
grep 'pattern' file.txt
grep 'root' /etc/passwd
grep '^#' /etc/fstab        # строки, начинающиеся с #
grep 'bash$' /etc/passwd    # строки, заканчивающиеся на bash
grep '.' /etc/hosts         # строки с любым символом (все непустые)
```

Для поиска по нескольким файлам или рекурсивно:

```bash
grep -r 'error' /var/log/
grep 'warning' /var/log/*.log
```

### egrep

`egrep` использует ERE. Полностью эквивалентен `grep -E`.

```bash
egrep 'colou?r' file.txt            # colour или color
egrep 'cat|dog' file.txt            # cat или dog
egrep '(err|warn)' /var/log/syslog  # err или warn
egrep '[0-9]{3}-[0-9]{4}' phones    # телефонный номер
```

На современных системах `egrep` помечен устаревшим, предпочтительнее `grep -E`.

### fgrep

`fgrep` ищет строки с фиксированной строкой, без интерпретации регулярных выражений. Эквивалентен `grep -F`. Работает быстрее на больших файлах, когда шаблон простой.

```bash
fgrep '192.168.1.1' /var/log/auth.log
fgrep '*.conf' Makefile              # * здесь буквальная звёздочка
```

Аналогично `egrep`, утилита `fgrep` считается устаревшей, лучше использовать `grep -F`.

### Полезные флаги

| Флаг | Значение |
|------|---------|
| `-i` | Игнорировать регистр |
| `-v` | Инвертировать выборку (строки без совпадения) |
| `-n` | Показывать номера строк |
| `-c` | Выводить только количество совпадений |
| `-l` | Выводить только имена файлов |
| `-H` | Всегда выводить имя файла перед каждой строкой |
| `-r` | Рекурсивный поиск по каталогам |
| `-E` | Использовать ERE |
| `-F` | Фиксированная строка (без regex) |
| `-f FILE` | Читать шаблон из файла |
| `-o` | Выводить только совпавшую часть строки |
| `-w` | Совпадение только с целым словом |
| `-z` | Разделять строки нулевым байтом вместо `\n` (для `find -print0`) |
| `-A n` | Показывать n строк после совпадения |
| `-B n` | Показывать n строк до совпадения |
| `-C n` | Показывать n строк вокруг совпадения |

Флаг `-H` особенно важен при вызове `grep` из `find -exec`: без него имя файла не выводится, когда файл один. Контекстные строки в выводе помечаются `-` вместо `:` после имени файла.

При использовании `find -print0` нужно передавать `-z` в `grep`, чтобы оба инструмента работали с нулевым разделителем:

```bash
find . -print0 | grep -z 'pattern'
```

Примеры:

```bash
grep -i 'error' /var/log/syslog                    # ошибки в любом регистре
grep -v '^#' /etc/fstab                            # строки без комментариев
grep -n 'TODO' *.py                                # номера строк с TODO
grep -c 'failed' /var/log/auth.log                 # количество неудачных попыток
grep -H 'root' /etc/*                              # с именем файла
grep -l 'root' /etc/*                              # только имена файлов
grep -w 'cat' /usr/share/words                     # только слово "cat", не "catch"
find /usr/share/doc -type f -exec grep -i -H '3d modeling' "{}" \;
```

## sed: редактирование потока

`sed` читает текст построчно, применяет команды и выводит результат. По умолчанию не изменяет исходный файл.

### Синтаксис команды substitute

Команда `s` заменяет текст по регулярному выражению:

```
s/regexp/replacement/flags
```

Разделитель `/` можно заменить на любой другой символ, что удобно при работе с путями:

```bash
sed 's/old/new/' file.txt             # заменить первое вхождение в строке
sed 's/old/new/g' file.txt            # заменить все вхождения
sed 's/old/new/2' file.txt            # заменить второе вхождение
sed 's/old/new/gi' file.txt           # все вхождения, без учёта регистра
sed 's|/usr/bin|/usr/local/bin|g' f   # использование | как разделителя
```

В строке замены:
- `&` ссылается на всё совпавшее выражение.
- `\1`, `\2` и т.д. ссылаются на группы в BRE: `\(group\)`.

```bash
# Обернуть число в скобки
sed 's/[0-9]\+/(&)/' file.txt

# Поменять первое и второе слово местами
sed 's/\(\w\+\) \(\w\+\)/\2 \1/' file.txt
```

Флаг `-i` редактирует файл на месте:

```bash
sed -i 's/http/https/g' config.txt
sed -i.bak 's/http/https/g' config.txt   # сохранить резервную копию
```

### Удаление строк

Команда `d` удаляет строки:

```bash
sed '/^#/d' /etc/fstab          # удалить строки-комментарии
sed '/^$/d' file.txt            # удалить пустые строки
sed '5d' file.txt               # удалить пятую строку
sed '1,3d' file.txt             # удалить строки 1–3
```

### Другие команды sed

| Команда | Значение |
|---------|---------|
| `p` | Напечатать строку (часто с `-n`) |
| `q` | Выйти после текущей строки |
| `c TEXT` | Заменить всю строку текстом TEXT |
| `a\text` | Добавить строку после текущей |
| `i\text` | Вставить строку перед текущей |
| `r FILE` | Вставить содержимое файла FILE после текущей строки |
| `w FILE` | Записать текущую строку в файл FILE |
| `y/abc/xyz/` | Транслитерация символов (как tr) |

Адреса в sed могут быть номером строки, регулярным выражением или диапазоном. Диапазон с регулярными выражениями задаётся через `/start/,/end/`:

```bash
sed -n '/error/p' /var/log/syslog          # вывести только строки с error
sed -n '10,20p' file.txt                   # вывести строки 10–20
sed '3a\new line' file.txt                 # добавить строку после третьей
sed '/^#/c REMOVED' file.txt              # заменить комментарии на «REMOVED»
sed -n '/<body>/,/<\/body>/p' page.html   # вывести блок body
```

Несколько команд можно передать через `-e`, разделить `;` внутри кавычек или сохранить в файле скрипта:

```bash
sed -e 's/foo/bar/g' -e '/^#/d' file.txt
sed "1,7d;11d" file.txt
sed -f script.sed file.txt
```

Флаг `-r` (или `-E`) включает ERE в sed. Флаг `e` в конце команды `s` выполняет результат замены как команду оболочки:

```bash
sed -r -e 's/(^[^.]*)\.(crt|key)$/cat \1.\2/e' < template.ovpn > client.ovpn
```

## Совместное использование grep и sed

`grep` и `sed` хорошо работают в конвейере: `grep` отбирает нужные строки, `sed` преобразует их форму. Типичный пример из практики: анализ неудачных попыток входа.

```bash
# Показать только записи с hostname (без IP в конце)
lastb -d -a --time-format notime | grep -v '[0-9]$'

# Оставить только доменные имена
lastb -d -a --time-format notime | grep -v '[0-9]$' \
  | sed -e 's/.* \(.*\)$/\1/'

# Уникальные домены, отсортированные
lastb -d -a --time-format notime | grep -v '[0-9]$' \
  | sed -e 's/.* \(.*\)$/\1/' | sort | uniq
```

Регулярное выражение `[0-9]$` в `grep -v` отбирает строки, не заканчивающиеся цифрой, то есть строки с hostname вместо IP. В `sed` паттерн `.* \(.*\)$` запоминает всё от последнего пробела до конца строки, а `\1` подставляет это обратно вместо всей строки.

## Поиск по файловой системе: find с regex

Команда `find` поддерживает опцию `-regex`, которая проверяет каждый путь в иерархии каталогов на соответствие регулярному выражению. По умолчанию `find` использует диалект `findutils-default`, очень близкий к BRE. ERE подключается через `-regextype posix-extended` или `-regextype egrep`.

```bash
# Найти скрытые файлы или файлы в скрытых каталогах размером более 100 МБ
find $HOME -regex '.*/\..*' -size +100M

# Найти шрифты с поддержкой ERE и ветвлением
find /usr/share/fonts -regextype posix-extended \
  -iregex '.*(dejavu|liberation).*sans.*(italic|oblique).*'
```

Опция `-iregex` работает без учёта регистра. Паттерн в `-regex` проверяется по всему пути, а не только по имени файла, поэтому обычно нужен `.*` в начале.

## Поиск в less

`less` позволяет выполнять поиск по регулярным выражениям прямо при просмотре файла или вывода команды.

| Клавиша | Действие |
|---------|---------|
| `/` | Открыть строку поиска, перейти к первому совпадению |
| `Ctrl+K` перед вводом | Только подсветить совпадения, не перемещаться |
| `&` | Фильтр: показывать только строки с совпадением |
| `n` | Следующее совпадение |
| `N` | Предыдущее совпадение |

Пример: при просмотре man-страницы нажать `/` и ввести `^[[:blank:]]*-o` или проще `^ *-o`, чтобы сразу перейти к описанию опции `-o`.

## Шпаргалка

```bash
# grep: поиск с BRE
grep 'root' /etc/passwd
grep '^root' /etc/passwd
grep 'bash$' /etc/passwd
grep -v ^# /etc/services
grep -i 'error' /var/log/syslog
grep -n 'TODO' *.py
grep -c 'failed' /var/log/auth.log
grep -r 'pattern' /etc/
grep -w 'word' file.txt

# grep -E / egrep: поиск с ERE
grep -E 'colou?r' file.txt
grep -E 'err|warn' /var/log/syslog
grep -E '[0-9]{2,4}' file.txt
egrep "\S+@\S+\.\S+"                                              # email
egrep "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}"          # IPv4
egrep ".org$|.com$" domains.txt

# grep -F / fgrep: точная строка
grep -F '192.168.1.1' file.txt
fgrep '*.conf' Makefile

# find с regex
find $HOME -regex '.*/\..*' -size +100M
find . -type f -regextype egrep -not -regex '.*\.[[:alnum:]]{1,}$'
find /path -regextype posix-extended -iregex '.*pattern.*'

# sed: замена
sed 's/old/new/g' file.txt
sed 's/old/new/gi' file.txt
sed -i 's/old/new/g' file.txt
sed -i.bak 's/old/new/g' file.txt

# sed: удаление строк
sed '/^#/d' file.txt
sed '/^$/d' file.txt
sed '1,5d' file.txt

# sed: вывод строк
sed -n '10,20p' file.txt
sed -n '/error/p' /var/log/syslog

# Справка по regex
man 7 regex
```

## Типичные вопросы экзамена

**Какой инструмент используется для поиска фиксированной строки без интерпретации regex?**
`grep -F` или `fgrep`.

**Какой флаг grep выводит только количество совпадающих строк?**
`-c`.

**Как в ERE обозначается «один или более»?**
`+`. В BRE нужно экранировать: `\+`.

**Как sed отредактировать файл на месте с резервной копией?**
`sed -i.bak 's/old/new/g' file.txt`. Суффикс после `-i` идёт без пробела.

**Что делает `^` внутри класса символов `[^abc]`?**
Отрицает класс: совпадает с любым символом, кроме a, b, c.

**Что означает `&` в строке замены sed?**
Весь совпавший текст. Например, `sed 's/[0-9]*/[&]/'` обернёт число в квадратные скобки.

**Чем отличается `grep -E` от `grep`?**
`grep -E` использует ERE, где `+`, `?`, `|`, `()`, `{}` работают без экранирования.

**Как найти пустые строки с помощью grep?**
`grep '^$' file.txt`.

**Как вывести строки без комментариев и без пустых строк?**
`grep -v '^#' file.txt | grep -v '^$'` или `grep -Ev '^(#|$)' file.txt`.

**Что делает `sed -n '5,10p'`?**
Выводит только строки с 5 по 10.

**Как в find включить ERE для -regex?**
`find . -regextype posix-extended` или `-regextype egrep`.

**Как в less показать только строки, совпадающие с regex, без перемещения по файлу?**
Нажать `&` и ввести выражение. Для подсветки без перемещения нажать `Ctrl+K` перед вводом в строке `/`.

**Что произойдёт при выполнении `uptime -s | sed -e 's/(.*) (.*)/\1/'`?**
Строка выведется без изменений. По умолчанию `sed` использует BRE, где скобки воспринимаются буквально. Нужно использовать `\(.*\) \(.*\)` в BRE или добавить `-r` для ERE.

**Что делает флаг `-z` в grep?**
Переключает разделитель строк с `\n` на нулевой байт. Нужен при совместной работе с `find -print0`.

**Как с помощью sed вывести только блок `<body>` HTML-файла?**
`sed -n '/<body>/,/<\/body>/p' file.html`. Диапазон адресов из двух regex и флаг `-n` с командой `p`.

**Как sed удалить все HTML-теги, оставив только текст?**
`sed 's/<[^>]*>//g'`. Шаблон `<[^>]*>` совпадает с `<`, любым количеством символов кроме `>`, и `>`.

#### 1. Регулярное выражение для email

**Какое расширенное регулярное выражение совпадёт с любым email-адресом, например `info@example.org`?**

<details><summary>Ответ</summary>

```bash
egrep "\S+@\S+\.\S+"
```

`\S+` совпадает с одним или более непробельными символами. Символ `@` буквальный. Точка перед доменом верхнего уровня экранирована `\.`, иначе она совпала бы с любым символом. Выражение не валидирует адрес строго, но покрывает стандартный формат.

</details>

#### 2. Регулярное выражение для IPv4

**Какое расширенное регулярное выражение совпадёт с любым IPv4-адресом в формате dotted-quad, например `192.168.15.1`?**

<details><summary>Ответ</summary>

```bash
egrep "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}"
```

Каждый октет представлен как `[0-9]{1,3}`: от одной до трёх цифр. Точки между октетами экранированы `\.`, чтобы совпадать буквально. Выражение не ограничивает диапазон значений до 0–255, но покрывает формат записи.

</details>

#### 3. Отфильтровать комментарии из /etc/services

**Как с помощью `grep` вывести содержимое `/etc/services`, отфильтровав все комментарии (строки, начинающиеся с `#`)?**

<details><summary>Ответ</summary>

Флаг `-v` инвертирует выборку: выводятся только строки, которые не совпадают с шаблоном. Якорь `^` фиксирует шаблон в начале строки, так что фильтруются только строки-комментарии, а не строки, содержащие `#` где-то в середине.

</details>

#### 7. sed, BRE и группы захвата

**Команда `uptime -s` выводит дату последней загрузки в формате `2019-08-05 20:13:22`. Что произойдёт при выполнении `uptime -s | sed -e 's/(.*) (.*)/\1/'`?**

<details><summary>Ответ</summary>

Строка выведется без изменений. По умолчанию `sed` использует BRE, где скобки `(` и `)` не являются метасимволами и воспринимаются буквально. Правильные варианты:

```bash
# BRE — скобки экранированы
uptime -s | sed -e 's/\(.*\) \(.*\)/\1/'

# ERE через флаг -r
uptime -s | sed -r -e 's/(.*) (.*)/\1/'
```

Оба варианта выведут только дату, отбросив время.

</details>

#### 1. Найти файлы без расширения

**Как с помощью `find` и расширенных регулярных выражений найти все файлы в текущем каталоге, у которых нет стандартного расширения?**

#### 4. Вывести блок body HTML-файла через sed

Адрес `/<body>/,/<\/body>/` задаёт диапазон: от строки, совпадающей с `<body>`, до строки с `<\/body>`. Обратный слеш перед `/` нужен для экранирования внутри адреса sed. Флаг `-n` подавляет автоматический вывод всех строк, команда `p` явно печатает только строки в диапазоне.

#### 6. Подстановка содержимого файлов через sed

**В шаблоне `.ovpn` имена файлов сертификатов нужно заменить содержимым этих файлов. Как это сделать одной командой `sed`?**

<details><summary>Ответ</summary>

```bash
sed -r -e 's/(^[^.]*)\.(crt|key)$/cat \1.\2/e' < client.template > client.ovpn
```

Флаг `-r` включает ERE. Шаблон `(^[^.]*)` захватывает имя файла без расширения (первая группа), `\.(crt|key)$` совпадает с расширением в конце строки (вторая группа). Строка замены `cat \1.\2` формирует команду `cat имяфайла.расширение`. Флаг `e` после выражения выполняет эту строку как команду оболочки и подставляет её вывод обратно в поток.

</details>
