В Linux практически всё хранится в текстовых файлах: конфиги сервисов, логи приложений, скрипты автоматизации, пользовательские данные. Это не случайность — текстовый формат читается любым редактором, легко передаётся по сети, хорошо обрабатывается программами.

Представь реальную рабочую ситуацию: твой сервис упал в 3 часа ночи, мониторинг разбудил тебя. Лог-файл — 500 тысяч строк. Что произошло? Открывать в текстовом редакторе и листать — это пытка. В терминале — одна команда за несколько секунд находит всё, что нужно.

Именно поэтому работа с текстом — один из ключевых навыков DevOps-инженера.

## Просмотр файлов

Прежде чем искать по файлу, нужно уметь его просто прочитать. В Linux для этого есть несколько команд — каждая со своим характером.

| Команда | Когда использовать |
|---|---|
| `cat` | Маленькие файлы, быстро вывести содержимое |
| `head` | Заголовок файла, первые N строк |
| `tail` | Последние события в логе, последние N строк |
| `tail -f` | Наблюдение за логом в реальном времени |
| `less` | Любой файл длиннее экрана — листать и искать |

### cat — вывести файл целиком

`cat` (сокращение от concatenate) выводит содержимое файла прямо в терминал.

```bash
cat /etc/hostname
cat /etc/hosts
cat /etc/passwd
```

Если файл большой — тысячи строк — cat выведет всё разом и экран «улетит» вверх. В таких случаях лучше использовать `less`.

`cat` умеет выводить несколько файлов подряд (отсюда и название):

```bash
cat file1.txt file2.txt file3.txt
```

Полезный флаг `-n` добавляет номера строк — удобно при изучении конфигов:

```bash
cat -n /etc/nginx/nginx.conf
```

### head — первые строки файла

`head` выводит начало файла. По умолчанию — первые 10 строк.

```bash
head /var/log/syslog
head -n 20 /var/log/syslog
head -n 3 /etc/passwd
```

Флаг `-n` задаёт количество строк. Многие форматы файлов начинаются с заголовка — версии, схемы, описания. `head` позволяет быстро понять, что за файл перед тобой, не загружая его весь.

### tail — последние строки файла

`tail` выводит конец файла. По умолчанию — последние 10 строк.

```bash
tail /var/log/syslog
tail -n 50 /var/log/nginx/error.log
tail -n 100 /var/log/auth.log
```

Логи растут снизу — новые записи добавляются в конец файла. Именно поэтому `tail` — это первая команда, которую запускают при разборе инцидента. Последние 50 строк лога почти всегда содержат то, что нужно.

### tail -f — следить за файлом в реальном времени

Флаг `-f` (follow) — особенно полезный режим: `tail` не завершается, а продолжает следить за файлом и выводит новые строки по мере их появления.

```bash
tail -f /var/log/nginx/access.log
tail -f /var/log/syslog
```

Терминал «живёт» — каждый новый запрос к серверу, каждое системное сообщение появляется на экране мгновенно. Чтобы остановить — нажми `Ctrl+C`.

> **Совет:** Открой `tail -f` в одном терминале, а во втором выполняй действия и смотри, что пишется в лог. Незаменимо при отладке.

### less — постраничный просмотр

`less` — интерактивный просмотрщик. Он открывает файл и позволяет им управлять: листать вверх и вниз, искать по содержимому — не загружая весь файл в память сразу.

```bash
less /var/log/syslog
less /etc/nginx/nginx.conf
```

Основные клавиши внутри `less`:

| Клавиша | Действие |
|---|---|
| стрелки / `j` / `k` | Движение по строкам |
| пробел / PgDn | Следующая страница |
| `b` / PgUp | Предыдущая страница |
| `G` | Перейти в конец файла |
| `g` | Перейти в начало файла |
| `/слово` | Поиск вперёд |
| `?слово` | Поиск назад |
| `n` | Следующее совпадение |
| `q` | Выйти |

## grep — поиск по содержимому

`grep` (Global Regular Expression Print) — вероятно, самая используемая команда в арсенале DevOps. Она читает файл строка за строкой и выводит только те строки, которые содержат заданный паттерн.

```bash
grep "что_ищем" имя_файла
```

Как это работает: grep проходит по каждой строке файла и проверяет, содержит ли она заданный текст. Если да — выводит строку целиком. Если нет — пропускает. Это как фильтр: из тысяч строк остаются только нужные.

### Флаги grep

| Флаг | Что делает | Пример |
|---|---|---|
| (без флагов) | Выводит строки, содержащие паттерн | `grep "ERROR" app.log` |
| `-i` | Поиск без учёта регистра | `grep -i "error" app.log` |
| `-n` | Добавляет номер строки | `grep -n "ERROR" app.log` |
| `-c` | Только число совпадений | `grep -c "ERROR" app.log` |
| `-v` | Инверсия: строки без паттерна | `grep -v "INFO" app.log` |
| `-r` | Рекурсивный поиск в директории | `grep -r "TODO" ./src/` |
| `-l` | Только имена файлов с совпадениями | `grep -l "ERROR" /var/log/*.log` |
| `-o` | Только совпадающая часть строки | `grep -o "ERROR.*" app.log` |
| `-E` | Расширенные регулярные выражения | `grep -E "ERROR\|WARN" app.log` |

> **Совет для новичков:** начни с простого — `grep "слово" файл`. Флаги добавляй по мере необходимости. `-i` и `-n` — самые частые.

### Регулярные выражения в grep

`grep` поддерживает регулярные выражения — специальный язык для описания паттернов:

```bash
grep "^ERROR" app.log            # строка начинается с ERROR
grep "\.py$" filelist.txt        # строка заканчивается на .py
grep "ERROR\|WARN" app.log       # OR в базовых регулярных выражениях
grep -E "ERROR|WARN|CRIT" app.log  # то же самое с флагом -E
```

Символы `^` и `$` — это якоря:
- `^ERROR` — строка начинается с ERROR (без `^` нашло бы и `some ERROR here`)
- `\.py$` — строка заканчивается на `.py` (точка экранируется `\.`, иначе она означает «любой символ»)

### Практические сценарии grep

**Найти неудачные попытки входа по SSH:**
```bash
grep "Failed password" /var/log/auth.log
```

**Найти IP-адреса, с которых идут атаки:**
```bash
grep "Failed password" /var/log/auth.log | grep -oE "[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+"
```

**Найти ошибки только за конкретный час:**
```bash
grep "2024-01-15 14:" /var/log/app.log | grep "ERROR"
```

**Найти конфиги, где упоминается старый адрес сервера:**
```bash
grep -r "old-server.company.com" /etc/
```

**Исключить шумные строки из лога:**
```bash
grep -v "DEBUG" /var/log/app.log | grep -v "TRACE"
```

**Проверить, слушает ли порт нужный процесс:**
```bash
ss -tlnp | grep ":80"
```

## Pipe | — конвейер команд

Pipe (знак `|`) — одна из самых мощных концепций Unix/Linux. Идея проста: вывод одной команды становится входом следующей.

```bash
команда1 | команда2 | команда3 | ...
```

Каждая утилита Linux делает одну вещь, но хорошо. Pipe позволяет комбинировать простые инструменты в сложные решения — без написания кода, прямо в терминале.

> **Аналогия:** представь завод. Первый рабочий вырезает детали, передаёт второму — тот шлифует, передаёт третьему — тот красит. Никто не ждёт, пока предыдущий закончит всю партию — работа течёт непрерывно. Так же работает pipe: данные проходят через команды поэтапно.

### Пример шаг за шагом: топ-5 частых ошибок

Нужно найти 5 самых частых ошибок в логе. Строим pipeline по шагам:

```bash
# Шаг 1: строки с ошибками
grep "ERROR" /var/log/app.log

# Шаг 2: сортируем — одинаковые строки окажутся рядом
grep "ERROR" /var/log/app.log | sort

# Шаг 3: подсчитываем количество повторений каждой уникальной строки
grep "ERROR" /var/log/app.log | sort | uniq -c

# Шаг 4: сортируем по убыванию числа повторений
grep "ERROR" /var/log/app.log | sort | uniq -c | sort -rn

# Шаг 5: оставляем только первые 5
grep "ERROR" /var/log/app.log | sort | uniq -c | sort -rn | head -n 5
```

Каждый шаг легко понять и проверить по отдельности — просто убери `|` и всё что после него.

### Практические pipeline

**Найти процесс по имени:**
```bash
ps aux | grep nginx
```

**Топ-10 самых тяжёлых директорий:**
```bash
du -sh /var/* | sort -rh | head -n 10
```

**Список уникальных пользователей, которые заходили по SSH:**
```bash
grep "Accepted" /var/log/auth.log | awk '{print $9}' | sort -u
```

**Количество активных соединений по каждому порту:**
```bash
ss -tn | awk 'NR>1 {print $4}' | cut -d: -f2 | sort | uniq -c | sort -rn
```

## wc — подсчёт строк, слов и байт

`wc` (word count) — простая, но незаменимая утилита. В DevOps почти всегда используется с pipe, чтобы получить число строк.

| Флаг | Что считает |
|---|---|
| `-l` | Количество строк |
| `-w` | Количество слов |
| `-c` | Количество байт (размер файла) |

`wc -l` в конце pipeline — это «сколько?». Вместо всех строк ты получаешь одно число:

```bash
grep "ERROR" app.log | wc -l          # сколько строк с ошибками?
ls /etc | wc -l                        # сколько файлов в /etc?
grep "Failed password" /var/log/auth.log | wc -l  # сколько неудачных попыток входа?
find /var/log -name "*.log" | wc -l   # сколько log-файлов на сервере?
```

## sort и uniq — сортировка и дедупликация

### sort — отсортировать строки

`sort` читает строки и выводит их в отсортированном порядке.

| Флаг | Поведение |
|---|---|
| (без флагов) | Алфавитная сортировка |
| `-n` | Числовая сортировка |
| `-r` | Обратный порядок |
| `-rn` | Числовая от большего к меньшему |
| `-u` | Убрать дубликаты за один проход |
| `-h` | Понимает суффиксы K, M, G (для размеров) |
| `-k2` | Сортировать по второму полю |

> **Важно:** при алфавитной сортировке числа сравниваются как строки — `1, 10, 100, 2, 20, 3`. При числовой (`-n`) — `1, 2, 3, 10, 20, 100`. Когда `uniq -c` добавляет счётчики вида `120 ERROR` и `9 WARNING`, всегда используй `-n`, иначе `9` окажется после `120`.

### uniq — убрать дублирующиеся строки

`uniq` убирает **соседние** одинаковые строки. Именно поэтому перед ним почти всегда стоит `sort` — чтобы одинаковые строки оказались рядом.

| Флаг | Поведение |
|---|---|
| (без флагов) | Убрать дубликаты, оставить одну копию |
| `-c` | Добавить счётчик повторений |
| `-d` | Показать только дублирующиеся строки |
| `-u` | Показать только уникальные (встречаются ровно один раз) |

### Классический паттерн: частотный анализ

Комбинация `sort | uniq -c | sort -rn` встречается снова и снова при работе с логами и данными:

**Самые частые URL в Nginx:**
```bash
awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20
```

**IP-адреса, которые чаще всего ломятся в SSH:**
```bash
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn | head -10
```

## find — поиск файлов

`find` — мощный инструмент для поиска файлов по самым разным критериям.

> **Ключевое отличие от grep:** `grep` ищет текст **внутри** файлов. `find` ищет **сами файлы** в файловой системе.

### Поиск по имени

```bash
find /etc -name "*.conf"
find /var/log -name "*.log"
find . -name "config.yaml"
find /home -iname ".bashrc"       # -iname — без учёта регистра
```

Паттерн в кавычках — это glob: `*` означает «любое количество любых символов». `find` ищет рекурсивно во всех поддиректориях.

### Поиск по типу

```bash
find . -type f    # только файлы
find . -type d    # только директории
find . -type l    # только символические ссылки
```

Указывай тип явно — иначе можно случайно получить директории с именем `something.conf`.

### Поиск по времени и размеру

```bash
find /var/log -name "*.log" -mtime -1    # изменены за последние 24 часа
find /var/log -name "*.log" -mtime +30   # изменены более 30 дней назад
find /tmp -size +100M                     # размер больше 100 МБ
find /home -size +1G                      # размер больше 1 ГБ
```

Условия объединяются по AND — файл должен соответствовать всем сразу:

```bash
find /var/log -name "*.log" -mtime +30 -size +10M   # старые большие логи
```

### Действия с найденными файлами

```bash
find /tmp -name "*.tmp" -delete                    # удалить найденные файлы
find . -name "*.log" -exec wc -l {} \;             # выполнить команду для каждого файла
find . -name "*.py" -exec grep -l "import os" {} \;
```

> **Осторожно с `-delete`:** сначала запусти `find` без этого флага и убедись, что находит нужные файлы — только потом добавляй `-delete`.

## Перенаправление ввода-вывода

У каждой команды есть три стандартных потока данных:

| Поток | Номер | Описание | По умолчанию |
|---|---|---|---|
| stdin | 0 | Стандартный ввод | Клавиатура |
| stdout | 1 | Стандартный вывод | Экран |
| stderr | 2 | Стандартный вывод ошибок | Экран |

Перенаправление меняет это поведение — ты говоришь системе, куда писать вместо экрана.

### Перенаправление stdout в файл

```bash
ls -la /etc > etc_files.txt      # записать в файл (перезаписать)
ls -la /var >> all_files.txt     # добавить в конец файла
```

> **Важно:** `>` перезапишет существующий файл без предупреждения. `>>` безопасно добавляет в конец.

### Перенаправление stderr

```bash
./deploy.sh 2> deploy_errors.log              # только ошибки в файл
./deploy.sh > output.log 2> errors.log        # вывод и ошибки в разные файлы
./deploy.sh > all.log 2>&1                    # всё в один файл
find /etc -name "*.conf" 2>/dev/null          # выбросить ошибки
```

`2>&1` — перенаправить поток ошибок туда же, куда идёт stdout.

`2>/dev/null` — выбросить ошибки. `/dev/null` — специальный файл-«чёрная дыра», всё записанное в него исчезает. Полезно, когда `find` ругается на директории без доступа: `Permission denied` — эти сообщения засоряют вывод.

### tee — и на экран, и в файл

Иногда нужно видеть вывод на экране и одновременно сохранять его в файл:

```bash
./deploy.sh | tee deploy.log       # перезаписывает файл
./deploy.sh | tee -a deploy.log    # добавляет в конец
```

Вывод отображается на экране как обычно и одновременно пишется в файл. Удобно для долгих операций — видишь прогресс и сохраняешь лог.

## Практические примеры

### Топ-10 IP-адресов в Nginx-логе

Nginx пишет IP клиента первым полем в access.log:

```bash
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -n 10
```

### Подсчитать 500-е ошибки за сегодня

```bash
grep " 500 " /var/log/nginx/access.log | grep "$(date +%d/%b/%Y)" | wc -l
```

`$(date +%d/%b/%Y)` подставляет сегодняшнюю дату в формат Nginx-лога, например `15/Jan/2024`.

### Найти конфиги с упоминанием паролей

```bash
grep -r "password" /etc/ 2>/dev/null
```

### Мониторинг ошибок в реальном времени

```bash
tail -f /var/log/nginx/error.log | grep --line-buffered "error"
```

`--line-buffered` нужен, чтобы строки появлялись сразу, не накапливаясь в буфере.

### Найти и удалить временные файлы старше 7 дней

```bash
find /tmp -type f -mtime +7 -delete
```

### Сохранить статус сервисов в файл

```bash
systemctl list-units --state=running > running_services.txt
```

### Найти самые большие файлы в домашней директории

```bash
find /home -type f -size +50M | xargs du -sh | sort -rh | head -10
```

### Посчитать строки во всех Python-файлах проекта

```bash
find . -name "*.py" -type f | xargs wc -l | tail -1
```

`tail -1` оставляет только итоговую строку, которую `wc` выводит для нескольких файлов.

## Быстрый справочник

### Просмотр файлов

| Команда | Описание |
|---|---|
| `cat файл` | Вывести файл целиком |
| `cat -n файл` | Вывести с номерами строк |
| `head -n N файл` | Первые N строк |
| `tail -n N файл` | Последние N строк |
| `tail -f файл` | Следить за файлом в реальном времени |
| `less файл` | Интерактивный просмотр |

### grep

| Команда | Описание |
|---|---|
| `grep "паттерн" файл` | Строки, содержащие паттерн |
| `grep -i "паттерн" файл` | Без учёта регистра |
| `grep -n "паттерн" файл` | С номерами строк |
| `grep -c "паттерн" файл` | Только число совпадений |
| `grep -v "паттерн" файл` | Строки без паттерна |
| `grep -r "паттерн" dir/` | Рекурсивный поиск |
| `grep -E "A\|B" файл` | Паттерн A или B |

### Pipeline и перенаправление

| Конструкция | Описание |
|---|---|
| `cmd1 \| cmd2` | Вывод cmd1 → ввод cmd2 |
| `cmd > файл` | Перезаписать файл |
| `cmd >> файл` | Добавить в конец файла |
| `cmd 2>/dev/null` | Выбросить ошибки |
| `cmd > all.log 2>&1` | Всё в один файл |
| `cmd \| tee файл` | И на экран, и в файл |

### Анализ данных

| Команда | Описание |
|---|---|
| `wc -l` | Подсчитать количество строк |
| `sort` | Алфавитная сортировка |
| `sort -n` | Числовая сортировка |
| `sort -rn` | Числовая, от большего к меньшему |
| `sort -rh` | По размеру файла, от большего к меньшему |
| `uniq -c` | Убрать дубликаты + счётчик |
| `sort \| uniq -c \| sort -rn` | Частотный анализ |

### find

| Команда | Описание |
|---|---|
| `find dir -name "*.conf"` | Поиск по имени |
| `find dir -type f` | Только файлы |
| `find dir -type d` | Только директории |
| `find dir -mtime -1` | Изменены за последние 24 ч |
| `find dir -mtime +30` | Изменены более 30 дней назад |
| `find dir -size +100M` | Размер больше 100 МБ |
| `find dir -name "*.tmp" -delete` | Найти и удалить |
