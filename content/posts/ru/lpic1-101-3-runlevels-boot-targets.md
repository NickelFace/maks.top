## Менеджеры служб: три подхода

Когда ядро завершает загрузку, оно запускает первый процесс в системе. Этот процесс всегда получает PID 1. В зависимости от дистрибутива это может быть SysVinit, systemd или Upstart. Все три умеют запускать и останавливать службы, но делают это по-разному.

## SysVinit и уровни запуска

SysVinit делит состояния системы на уровни запуска с номерами от 0 до 6.

| Уровень запуска | Назначение |
|----------|---------|
| 0 | Останов |
| 1, s, single | Однопользовательский режим, без сети (режим обслуживания) |
| 2, 3, 4 | Многопользовательский режим с консольным или сетевым входом |
| 5 | То же, что 3, плюс графический вход |
| 6 | Перезагрузка |

Единственные уровни с одинаковым значением во всех дистрибутивах — 0, 1 и 6.

### /etc/inittab

`/etc/inittab` управляет поведением init. Формат строки:

```
id:runlevels:action:process
```

Уровень запуска по умолчанию задаётся так:

```
id:3:initdefault:
```

Никогда не устанавливайте 0 или 6 как уровень по умолчанию — система немедленно остановится или перезагрузится после загрузки.

Доступные значения действий:

| Действие | Поведение |
|--------|-----------|
| `boot` | Выполняется при инициализации системы; поле уровней запуска игнорируется |
| `bootwait` | То же, но init ждёт завершения процесса |
| `sysinit` | Выполняется после инициализации, независимо от уровня запуска |
| `wait` | Выполняется для указанных уровней запуска; init ждёт завершения |
| `respawn` | Процесс перезапускается при завершении |
| `ctrlaltdel` | Выполняется при нажатии Ctrl+Alt+Del |

### Скрипты служб

Скрипты хранятся в `/etc/init.d/`. Для каждого уровня запуска существует каталог `/etc/rcN.d/` (N — номер уровня), содержащий символические ссылки на скрипты из `/etc/init.d/`. Имя ссылки начинается с `K` (kill, остановить) или `S` (start, запустить), за которым следует двузначный приоритет и имя службы. Например, `/etc/rc1.d/K90network` останавливает сетевую службу при переходе на уровень запуска 1.

### Команды SysVinit

Показать текущий уровень запуска:

```bash
runlevel
# Вывод: N 3  (N означает, что уровень запуска не менялся с момента загрузки)
```

Переключить уровень запуска без перезагрузки:

```bash
telinit 3
telinit 1   # однопользовательский режим
telinit 6   # перезагрузка
telinit 0   # останов
```

`init N` делает то же самое, что `telinit N`.

После каждого изменения `/etc/inittab` выполните:

```bash
telinit q   # или telinit Q
```

Это заставляет init перечитать конфигурацию. Без этого шага плохая запись в `/etc/inittab` может привести к зависанию системы при следующей перезагрузке.

## systemd и цели загрузки

systemd используется большинством современных дистрибутивов. Вместо уровней запуска он работает с юнитами и целями.

### Типы юнитов

| Тип | Назначение |
|------|---------|
| `service` | Активная служба, которую можно запустить и остановить |
| `socket` | Сетевой или файловый сокет |
| `device` | Аппаратное устройство, зарегистрированное через udev |
| `mount` | Точка монтирования (аналогично записи в `/etc/fstab`) |
| `automount` | То же, но монтируется автоматически при обращении |
| `target` | Группа юнитов, управляемых совместно |
| `snapshot` | Сохранённое состояние менеджера systemd |

### Соответствие уровней запуска и целей

| Уровень SysV | Цель systemd |
|---|---|
| 0 | poweroff.target |
| 1 | rescue.target |
| 3 | multi-user.target |
| 5 | graphical.target |
| 6 | reboot.target |

### Управление юнитами с помощью systemctl

```bash
systemctl start unit.service      # запустить
systemctl stop unit.service       # остановить
systemctl restart unit.service    # перезапустить
systemctl status unit.service     # подробный статус
systemctl is-active unit.service  # активна или нет
systemctl enable unit.service     # включить при загрузке
systemctl disable unit.service    # отключить при загрузке
systemctl is-enabled unit.service # проверить статус автозапуска
```

Если в системе есть только один юнит с данным именем, суффикс `.service` можно опустить. Например, `systemctl status sshd` работает так же, как `systemctl status sshd.service`.

### Переключение целей

```bash
systemctl isolate multi-user.target
systemctl isolate rescue.target    # однопользовательский режим
```

### Цель по умолчанию

Показать текущую цель по умолчанию:

```bash
systemctl get-default
```

Изменить её:

```bash
systemctl set-default multi-user.target
```

Цель по умолчанию хранится как символическая ссылка в `/etc/systemd/system/default.target`. Никогда не устанавливайте `shutdown.target` как цель по умолчанию — это эквивалент уровня запуска 0. Также можно задать цель через параметр ядра при загрузке:

```
systemd.unit=multi-user.target
```

### Файлы конфигурации systemd

- `/lib/systemd/system/` — файлы юнитов от производителя (основное расположение)
- `/usr/lib/systemd/` — альтернативное расположение в некоторых дистрибутивах
- `/etc/systemd/` — пользовательские переопределения
- `/etc/systemd/logind.conf` — настройки событий питания (сон, гибернация и т.д.)
- `/etc/systemd/logind.conf.d/` — дополнительные файлы конфигурации (альтернатива единому logind.conf)

Список юнитов:

```bash
systemctl list-unit-files
systemctl list-unit-files --type=service
systemctl list-unit-files --type=target
systemctl list-units
```

На системах с systemd `/sbin/init` является символической ссылкой на `/lib/systemd/systemd`.

## Выключение и перезагрузка

### Команда shutdown

```bash
shutdown [options] time [message]
```

Аргумент `time` обязателен. Форматы:

| Формат | Значение |
|--------|---------|
| `hh:mm` | Конкретное время, например `23:00` |
| `+m` | Через m минут, например `+10` |
| `now` или `+0` | Немедленно |

Параметры:

| Параметр | Действие |
|--------|--------|
| `-h` | Остановить после выключения |
| `-r` | Перезагрузить |
| `-c` | Отменить запланированное выключение |

Примеры:

```bash
shutdown -h now
shutdown -r +5 "Rebooting in 5 minutes"
shutdown -r 23:00
shutdown -c
```

После запуска `shutdown` все процессы сначала получают SIGTERM, затем SIGKILL, после чего система переключает уровень запуска или выключается. Запуск `shutdown` без `-h` или `-r` переводит систему в однопользовательский режим (уровень запуска 1).

На системах SysV можно ограничить список пользователей, которые могут перезагружать систему через Ctrl+Alt+Del. Добавьте параметр `-a` к команде `shutdown` в строке `ctrlaltdel` в `/etc/inittab`. Тогда инициировать перезагрузку таким способом смогут только пользователи, перечисленные в `/etc/shutdown.allow`.

### systemctl для управления питанием

```bash
systemctl reboot
systemctl poweroff
systemctl suspend    # режим сна, данные остаются в ОЗУ
systemctl hibernate  # гибернация, данные записываются на диск
```

В некоторых дистрибутивах `poweroff` и `reboot` являются символическими ссылками на `systemctl`.

## Уведомление пользователей: wall

Команда `wall` рассылает сообщение в терминальные сессии всех вошедших в систему пользователей.

```bash
wall "Server going down for maintenance in 10 minutes"
wall /path/to/message.txt
```

Команда `shutdown` автоматически отправляет предупреждение через `wall`, если задан аргумент с сообщением.

---

## acpid

`acpid` (Advanced Configuration and Power Interface daemon) — основной демон управления питанием в Linux. Он обрабатывает аппаратные события: закрытие крышки, изменение уровня заряда батареи, нажатие кнопки питания.

Если `acpid` запущен, функции управления питанием systemd (`suspend`, `hibernate`), настроенные в `logind.conf`, работать не будут, поскольку `acpid` перехватывает эти события первым.

## Upstart (для справки)

Upstart был разработан для Ubuntu как замена SysVinit с параллельным запуском служб. Ubuntu перешла на systemd в 2015 году, и Upstart теперь встречается редко.

Скрипты Upstart хранились в `/etc/init/`. Основные команды:

```bash
initctl list         # список служб
start tty6           # запустить
stop tty6            # остановить
status tty6          # статус
```

## Типовые вопросы экзамена

**В: Как перезагрузить систему с помощью `telinit`?**
`telinit 6` — уровень запуска 6 означает перезагрузку.

**В: Что происходит со службой, связанной как `/etc/rc1.d/K90network`, при переходе в уровень запуска 1?**
Служба останавливается. Имена, начинающиеся с K, вызывают скрипт с аргументом `stop`.

**В: Как проверить, работает ли `sshd.service`?**
`systemctl status sshd.service` для подробного вывода, или `systemctl is-active sshd.service` для однословного ответа.

**В: Как включить `sshd.service` для запуска при загрузке?**
`systemctl enable sshd.service` (от имени root). Создаёт символическую ссылку, которую systemd подхватывает при загрузке.

**В: Система SysV всегда загружается в уровень запуска 1, хотя в `/etc/inittab` указан 3. Почему?**
В конфигурации загрузчика задан параметр ядра `1` или `S`. Параметры ядра имеют приоритет над значением `initdefault` в `/etc/inittab`.

**В: На что указывает `/sbin/init` в системе с systemd?**
`/lib/systemd/systemd`.

**В: Как отменить запланированное выключение?**
`shutdown -c`

**В: Как проверить цель загрузки по умолчанию в systemd?**
`systemctl get-default` или проверить символическую ссылку: `ls -l /etc/systemd/system/default.target`.

### Упражнение 2 — K90network на уровне запуска 1

Что происходит со службой, стоящей за `/etc/rc1.d/K90network`, когда система переходит в уровень запуска 1?

<details>
<summary>Ответ</summary>

Служба останавливается. Имя файла начинается с `K` (kill), поэтому при переходе на уровень запуска 1 скрипт вызывается с аргументом `stop`.

> Уровень запуска 1 — это однопользовательский режим без сетевого подключения, поэтому все сетевые службы в `/etc/rc1.d/` имеют имена, начинающиеся с K.

</details>

---

### Упражнение 3 — Проверка статуса sshd

Как использовать `systemctl` для проверки, работает ли `sshd.service`?

<details>
<summary>Ответ</summary>

```bash
systemctl status sshd.service
```

Или лаконичнее:

```bash
systemctl is-active sshd.service
```

> `is-active` выводит `active`, если служба запущена, и `inactive` в противном случае. `status` даёт полную информацию: PID, последние строки журнала, время запуска.

</details>

### Упражнение 4 — Включение sshd при загрузке

Какая команда включает автоматический запуск `sshd.service` в системе с systemd?

<details>
<summary>Ответ</summary>

```bash
systemctl enable sshd.service
```

Выполнять от имени root. Команда создаёт символическую ссылку, которую systemd подхватывает при загрузке.

</details>

### Упражнение 5 — Неправильный уровень запуска вопреки inittab

Система SysV всегда загружается в уровень запуска 1, хотя в `/etc/inittab` указан уровень 3. Какова наиболее вероятная причина?

<details>
<summary>Ответ</summary>

В конфигурации загрузчика задан параметр ядра `1` или `S`. Параметры ядра имеют приоритет над значением `initdefault` в `/etc/inittab`.

> Проверьте параметры текущей загрузки с помощью `cat /proc/cmdline`. Постоянные параметры задаются в конфигурации загрузчика, например в `GRUB_CMDLINE_LINUX` в `/etc/default/grub`.

</details>
