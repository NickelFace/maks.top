> **Вес на экзамене: 3** — LPIC-1 v5, Экзамен 101

## Что нужно знать

- Управлять правами доступа на файлы, специальные файлы и каталоги.
- Использовать SUID, SGID и sticky bit.
- Менять маску создания файлов.
- Использовать поле группы для предоставления доступа.

Утилиты: `chmod`, `umask`, `chown`, `chgrp`.

---

## Просмотр прав через ls

```bash
ls -l
# -rw-rw-r-- 1 carol carol 1881 Dec 10 15:57 text.txt
# drwxr-xr-x 2 carol carol 4096 Dec 10 17:59 Another_Directory
```

Первый символ — тип файла, следующие 9 — права доступа (три группы по три), далее: число жёстких ссылок, владелец, группа, размер, дата, имя.

- `ls -a` — показать скрытые файлы (начинаются с `.`)
- `ls -ld DIR` — показать атрибуты самого каталога, не его содержимое
- `ls -lh` — размер в удобочитаемом формате

### Типы файлов (первый символ)

| Символ | Тип |
|---|---|
| `-` | обычный файл |
| `d` | каталог |
| `l` | символическая ссылка |
| `b` | блочное устройство (диски) |
| `c` | символьное устройство (терминалы, последовательные порты) |
| `s` | сокет |

### Расшифровка прав

9 символов прав разбиты на три группы по три:

- **u** (user/owner) — права владельца
- **g** (group) — права группы-владельца
- **o** (others/world) — права всех остальных

В каждой группе: `r` (чтение), `w` (запись), `x` (исполнение). Тире `-` — право отсутствует.

**Права для файла:**

| Право | Что позволяет |
|---|---|
| `r` | читать содержимое |
| `w` | изменять или удалять |
| `x` | запускать как программу |

**Права для каталога:**

| Право | Что позволяет |
|---|---|
| `r` | читать список файлов (`ls`) |
| `w` | создавать, удалять, переименовывать файлы (только вместе с `x`) |
| `x` | входить в каталог (`cd`), обращаться к файлам по имени |

### Как система проверяет права

Проверка строго по очереди — применяется только один набор:

1. Пользователь — владелец → применяются права владельца.
2. Пользователь в группе-владельце → применяются права группы.
3. Иначе → применяются права остальных.

Если ты владелец, но у группы прав больше — тебе всё равно достаются только твои права как владельца.

## Изменение прав: chmod

```
chmod ПРАВА ФАЙЛ
```

Менять права может только владелец файла или root.

### Символьный режим

Кто: `u` (владелец), `g` (группа), `o` (остальные), `a` (все).

Действие: `+` (добавить), `-` (убрать), `=` (установить точно).

Права: `r`, `w`, `x`.

```bash
chmod u+x script.sh           # добавить владельцу исполнение
chmod go-w text.txt           # убрать запись у группы и остальных
chmod a=r notes.txt           # всем — только чтение
chmod u+x,go-wx file          # несколько изменений через запятую
```

### Октальный режим

Каждое право имеет числовое значение: `r=4`, `w=2`, `x=1`. Сложением получаем итог для группы:

| Число | Права |
|---|---|
| 7 | rwx |
| 6 | rw- |
| 5 | r-x |
| 4 | r-- |
| 3 | -wx |
| 2 | -w- |
| 1 | --x |
| 0 | --- |

Три цифры: владелец — группа — остальные:

```bash
chmod 660 text.txt      # rw-rw----
chmod 755 script.sh     # rwxr-xr-x
chmod 640 secret.txt    # rw-r-----
```

Подсказка: нечётное число — файл гарантированно исполняемый.

### Рекурсивное изменение

```bash
chmod -R u+rwx Another_Directory/
```

Без `-R` меняются права только самого каталога. С `-R` — всё дерево.

### Символьный vs октальный

- **Октальный** — когда нужно задать все права сразу в конкретное значение.
- **Символьный** — когда меняешь одно право, не затрагивая остальные.

## Изменение владельца: chown и chgrp

```bash
chown ПОЛЬЗОВАТЕЛЬ:ГРУППА ФАЙЛ
```

```bash
chown carol:students text.txt   # владелец и группа
chown carol text.txt            # только владелец
chown carol: text.txt           # владелец + основная группа carol
chown :students text.txt        # только группа
```

Для смены только группы — отдельная команда:

```bash
chgrp students text.txt
```

Менять владельца на другого пользователя или чужую группу может только root. Оба поддерживают `-R` для рекурсивного применения.

## Работа с группами

```bash
getent group                    # список всех групп системы
groups carol                    # группы конкретного пользователя
groupmems -g cdrom -l           # члены группы (требует root)
```

## Маска по умолчанию: umask

При создании файла или каталога система берёт максимальные права и вычитает маску:

- Каталоги: `0777 - umask`
- Файлы: `0666 - umask` (файлы по умолчанию не получают `x`)

```bash
umask          # текущее значение (например, 0022)
umask -S       # в символьной форме (u=rwx,g=rx,o=rx)
```

При `umask 022`: каталоги → `755`, файлы → `644`.

Изменить маску на текущую сессию:

```bash
umask 027
umask u=rwx,g=rx,o=    # в символьной форме
```

Для постоянного применения — прописать в `~/.bashrc` или `/etc/profile`.

### Таблица значений umask

| umask | Файл | Каталог |
|---|---|---|
| 022 | rw-r--r-- (644) | rwxr-xr-x (755) |
| 027 | rw-r----- (640) | rwxr-x--- (750) |
| 077 | rw------- (600) | rwx------ (700) |
| 007 | rw-rw---- (660) | rwxrwx--- (770) |

## Специальные права

### Sticky bit

Октальное значение: **1** (4-я цифра). Символ: `t` на месте `x` у остальных.

На каталоге: файл может удалить только его владелец или владелец каталога. Применяется к каталогам — на файлах не работает.

```bash
chmod 1755 /tmp           # октальный режим
chmod o+t /tmp            # символьный режим
```

Пример — `/tmp`:

```
drwxrwxrwt 19 root root 16K /tmp
```

### SGID

Октальное значение: **2** (4-я цифра). Символ: `s` на месте `x` у группы.

На исполняемом файле: процесс работает с привилегиями группы-владельца файла.

На каталоге: новые файлы и подкаталоги внутри наследуют группу родителя.

```bash
chmod 2755 Sample_Directory    # октальный режим
chmod g+s Sample_Directory     # символьный режим
```

### SUID

Октальное значение: **4** (4-я цифра). Символ: `s` на месте `x` у владельца.

На исполняемом файле: процесс запускается с привилегиями владельца файла. Классический пример — `/usr/bin/passwd` (работает как root, хотя запускает обычный пользователь). На каталогах не работает.

```bash
chmod 4755 /bin/foo      # октальный режим
chmod u+s /bin/foo       # символьный режим
```

### Четырёхзначная октальная нотация

Специальные права — первая (4-я) цифра:

| 4-я цифра | Право |
|---|---|
| 0 | нет специальных прав |
| 1 | sticky bit |
| 2 | SGID |
| 4 | SUID |
| 6 | SUID + SGID |
| 7 | SUID + SGID + sticky |

```bash
chmod 6755 test.sh       # SUID + SGID
chmod 0755 test.sh       # убрать все специальные права
```

### Заглавные S и T

Если специальный бит установлен, а право `x` для той же группы отсутствует — буква заглавная:

| Символ | Значение |
|---|---|
| `s` (строчная) | спецбит + `x` есть |
| `S` (заглавная) | спецбит + `x` отсутствует |
| `t` (строчная) | sticky + `x` у остальных есть |
| `T` (заглавная) | sticky + `x` у остальных отсутствует |

Заглавная буква — диагностический сигнал: спецбит установлен, но без `x` скорее всего бесполезен.

## Шпаргалка

### Биты прав

| Бит | Октал | Символ | Работает на |
|---|---|---|---|
| read | 4 | r | файл, каталог |
| write | 2 | w | файл, каталог |
| execute | 1 | x | файл, каталог |
| sticky | 1 (4-я цифра) | t (у o) | только каталог |
| SGID | 2 (4-я цифра) | s (у g) | файл, каталог |
| SUID | 4 (4-я цифра) | s (у u) | только файл |

### Команды

| Команда | Что делает |
|---|---|
| `ls -l` | права, владелец, группа, размер |
| `ls -ld DIR` | атрибуты самого каталога |
| `chmod MODE FILE` | изменить права |
| `chmod -R MODE DIR` | рекурсивно |
| `chown USER:GROUP FILE` | изменить владельца и группу |
| `chgrp GROUP FILE` | изменить только группу |
| `umask` | показать или установить маску |
| `umask -S` | маска в символьной форме |
| `getent group` | список всех групп |
| `groups USER` | группы пользователя |

## Типичные вопросы экзамена

1. Какое umask даёт файлам `rw-r-----`? → **`027`** (666 − 027 = 640).
2. Как сделать так, чтобы файлы в каталоге `sales` принадлежали группе `sales`? → `chmod g+s sales` или `chmod 2755 sales`.
3. Как установить SUID на `/bin/foo`? → `chmod 4755 /bin/foo` или `chmod u+s /bin/foo`.
4. Октальные значения специальных битов? → SUID=4, SGID=2, sticky=1.
5. Что означает `t` в `/tmp` (`drwxrwxrwt`)? → Sticky bit: удалить файл может только его владелец.
6. Что означает заглавная `S` в `-rwSr-xr-x`? → SUID установлен, но у владельца нет права `x`.
7. Чем `chown` отличается от `chgrp`? → `chown` меняет владельца и/или группу; `chgrp` — только группу.
8. Где прописать `umask` для постоянного действия? → `~/.bashrc`, `~/.profile` или `/etc/profile`.
9. Что происходит с правами каталога при `chmod` без `-R`? → Меняются права только самого каталога, файлы внутри не затрагиваются.
10. Как убрать все специальные права октальной нотацией? → Добавить `0` первой цифрой: `chmod 0755 file`.
