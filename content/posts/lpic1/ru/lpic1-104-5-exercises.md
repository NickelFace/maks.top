> **Тема 104.5** — Управление правами доступа и владением файлов. Практические и исследовательские разборы.

---

## Практические упражнения (Guided)

### 1. Права на пустой каталог

**Задача:** создай каталог `emptydir` командой `mkdir emptydir`. Покажи права на сам каталог (не его содержимое).

**Ответ:**

```bash
ls -ld emptydir
# или
ls -l -d emptydir
```

Без `-d` команда `ls` выведет содержимое каталога. Параметр `-d` переключает на вывод атрибутов самого каталога.

---

### 2. chmod в символьном режиме

**Задача:** создай файл `emptyfile` через `touch emptyfile`. Одной командой `chmod` в символьном режиме: добавь право исполнения владельцу и убери права записи и исполнения у группы и остальных.

**Ответ:**

```bash
chmod u+x,go-wx emptyfile
```

Разбор по частям: `u+x` — владельцу добавить `x`. `go-wx` — группе и остальным убрать `w` и `x`. Несколько изменений объединяются через запятую без пробелов.

---

### 3. Расчёт прав по umask

**Задача:** какие права будут у новых файлов, если umask равна `027`?

**Ответ:** `rw-r-----` (640).

Расчёт: `666 - 027 = 640`. Файлы не получают `x` по умолчанию, поэтому цифра `x` в маске на файлы не влияет.

---

### 4. Разбор прав и снятие SGID

**Задача:** у файла `test.sh` такие права:

```
-rwxr-sr-x 1 carol root 33 Dec 11 10:36 test.sh
```

Какие права у владельца? Как снять специальное право октальной нотацией?

**Ответ:**

Права владельца — символы 2–4 в выводе `ls -l`. Здесь `rwx`: можно читать, писать и исполнять.

Обычные права: `rwx` у владельца = 7, `r-x` у группы (буква `s` стоит на месте `x`, значит `x` есть) = 5, `r-x` у остальных = 5. Итого `755`.

Чтобы убрать специальное право, передаём `0` как четвёртую цифру:

```bash
chmod 0755 test.sh
```

---

### 5. Блочное устройство

**Задача:**

```
$ ls -l /dev/sdb1
brw-rw---- 1 root disk 8, 17 Dec 21 18:51 /dev/sdb1
```

Какой тип файла? Кто может писать?

**Ответ:** первый символ `b` — блочное устройство (обычно диск).

Писать могут: владелец (`root`) и любые члены группы `disk`. У остальных прав нет.

---

### 6. Октальная нотация для четырёх файлов

**Задача:** запиши права в четырёхзначной октальной нотации:

```
drwxr-xr-t 2 carol carol  4,0K Dec 20 18:46 Another_Directory
----r--r-- 1 carol carol     0 Dec 11 10:55 foo.bar
-rw-rw-r-- 1 carol carol  1,2G Dec 20 18:22 HugeFile.zip
drwxr-sr-x 2 carol users 4,0K Jan 18 17:26 Sample_Directory
```

**Ответ:**

| Файл | Октал | Пояснение |
|---|---|---|
| `Another_Directory` | **1755** | sticky=1; `rwx`=7, `r-x`=5, `r-x`=5 |
| `foo.bar` | **0044** | нет спецбитов; `---`=0, `r--`=4, `r--`=4 |
| `HugeFile.zip` | **0664** | нет спецбитов; `rw-`=6, `rw-`=6, `r--`=4 |
| `Sample_Directory` | **2755** | SGID=2; `rwx`=7, `r-x`=5, `r-x`=5 |

## Исследовательские упражнения (Explorational)

### 1. chmod с одной и двумя цифрами

**Задача:** сбрось все права: `chmod 000 emptyfile`. Что произойдёт при `chmod 4 emptyfile`? При `chmod 44 emptyfile`? Что это говорит о том, как `chmod` читает числа?

**Ответ:**

После `chmod 4 emptyfile`:

```
-------r-- 1 carol carol 0 emptyfile
```

Поменялись права **остальных**.

После `chmod 44 emptyfile`:

```
----r--r-- 1 carol carol 0 emptyfile
```

Поменялись права **группы и остальных**.

**Вывод:** `chmod` читает цифры справа налево:

| Цифр | Что меняется |
|---|---|
| 1 | только остальные |
| 2 | группа + остальные |
| 3 | владелец + группа + остальные |
| 4 | спецбиты + все три группы |

---

### 2. Права на /tmp и удаление файлов

**Задача:**

```
drwxrwxrwt 19 root root 16K /tmp
```

Владелец, группа и остальные имеют полные права. Может ли обычный пользователь удалить чужой файл?

**Ответ:** нет. `/tmp` — world-writable каталог (писать может любой), но на нём установлен **sticky bit** (буква `t`). Он означает: удалить или переименовать файл может только владелец файла или владелец каталога.

Обычный пользователь может удалять только свои собственные файлы в `/tmp`.

---

### 3. SUID и заглавная S

**Задача:** у `test.sh` права `-rwsr-xr-x` (SUID установлен). Выполни:

```bash
chmod u-x test.sh
ls -l test.sh
# -rwSr-xr-x 1 carol carol 33 Dec 11 10:36 test.sh
```

Что сделали? Что означает заглавная `S`?

**Ответ:** убрали право исполнения у владельца. Буква `s` занимает позицию `x`, поэтому система кодирует наличие `x` регистром:

| Символ | SUID/SGID/sticky | Право `x` |
|---|---|---|
| `s` (строчная) | установлен | есть |
| `S` (заглавная) | установлен | **нет** |
| `t` (строчная) | sticky установлен | есть |
| `T` (заглавная) | sticky установлен | **нет** |

Заглавная буква — диагностический сигнал: спецбит есть, но без `x` он скорее всего бесполезен.

---

### 4. Общий каталог Box с SGID и sticky bit

**Задача:** создать каталог `Box`, где все файлы автоматически принадлежат группе `users`, и удалить файл может только его создатель.

**Ответ:**

Шаг 1 — создать каталог:

```bash
mkdir Box
```

Шаг 2 — назначить группу-владельца и установить SGID (новые файлы наследуют группу родителя):

```bash
chown :users Box/
chmod g+wxs Box/
```

Шаг 3 — установить sticky bit (удалять может только владелец файла):

```bash
chmod o+t Box/
```

Или оба шага одной командой:

```bash
chmod g+wxs,o+t Box/
```

Итоговые права:

```
drwxrwsr-t 2 carol users 4,0K Jan 18 19:09 Box
```
