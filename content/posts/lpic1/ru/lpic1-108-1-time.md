## Что нужно знать

Согласно официальным целям LPIC-1:

- Устанавливать системную дату и время.
- Синхронизировать аппаратные часы с правильным временем UTC.
- Настраивать временную зону.
- Базовая настройка NTP с ntpd и chrony.
- Знание сервиса pool.ntp.org.
- Общее представление о командах ntpq, chronyc и timedatectl.

Ключевые файлы и команды: `/usr/share/zoneinfo/`, `/etc/timezone`, `/etc/localtime`, `/etc/ntp.conf`, `/etc/chrony.conf`, `date`, `hwclock`, `timedatectl`, `ntpd`, `ntpq`, `chronyd`, `chronyc`, `pool.ntp.org`.

## Системные часы и аппаратные часы

В Linux существуют два независимых источника времени:

| Часы | Описание |
|---|---|
| Системные часы | Программные часы, поддерживаемые ядром; сбрасываются при выключении питания |
| Аппаратные часы (RTC) | Часы реального времени на материнской плате с питанием от батарейки; работают без питания |

При загрузке ядро считывает RTC для инициализации системных часов. После этого оба источника времени работают независимо.

Аппаратные часы могут хранить время в местном формате или UTC. В Linux по умолчанию принято использовать UTC для аппаратных часов, что позволяет избежать сложностей с переходом на летнее время.

## date — системные часы

`date` отображает или устанавливает системное время.

```bash
$ date
Mon Oct 21 10:45:21 -03 2019
```

### Полезные параметры date

| Параметр | Описание |
|---|---|
| `-u` | Показать или установить время в UTC |
| `-I` | Вывод в формате ISO 8601 (`YYYY-MM-DD`) |
| `-R` | Вывод в формате RFC 2822 (используется в заголовках email) |
| `+%s` | Вывести время Unix epoch (секунды с 1970-01-01 00:00:00 UTC) |
| `--date=@EPOCH` | Преобразовать epoch в читаемый формат |
| `--debug` | Показать подробности разбора строки даты |
| `-s "СТРОКА"` | Установить системные часы на указанное время |

```bash
$ date -u
Mon Oct 21 13:45:21 UTC 2019

$ date +%s
1571661921

$ date --date=@1571661921
Mon Oct 21 10:45:21 -03 2019

$ date -s "2019-10-21 10:45:00"
```

## hwclock — аппаратные часы

`hwclock` читает или устанавливает аппаратные часы (RTC) и синхронизирует их с системными часами.

| Параметр | Описание |
|---|---|
| `--systohc` | Установить аппаратные часы по системным |
| `--hctosys` | Установить системные часы по аппаратным |
| `--set --date="СТРОКА"` | Установить аппаратные часы на указанную дату/время |
| `--verbose` | Подробный вывод |

```bash
# Прочитать аппаратные часы
$ hwclock

# Синхронизировать аппаратные часы с системными
$ hwclock --systohc

# Установить аппаратные часы напрямую
$ hwclock --set --date="2019-10-21 10:45:00"
```

## timedatectl — управление временем в systemd

`timedatectl` — инструмент systemd для просмотра и настройки времени, временной зоны и синхронизации NTP.

```bash
$ timedatectl
               Local time: Mon 2019-10-21 10:45:21 -03
           Universal time: Mon 2019-10-21 13:45:21 UTC
                 RTC time: Mon 2019-10-21 13:45:21
                Time zone: America/Sao_Paulo (-03, -0300)
System clock synchronized: yes
              NTP service: active
          RTC in local TZ: no
```

### Команды timedatectl

| Команда | Описание |
|---|---|
| `timedatectl set-time "ГГГГ-ММ-ДД ЧЧ:ММ:СС"` | Установить системное время (NTP должен быть отключён) |
| `timedatectl list-timezones` | Показать все доступные временные зоны |
| `timedatectl set-timezone ЗОНА` | Установить системную временную зону |
| `timedatectl set-ntp true/false` | Включить или отключить синхронизацию NTP |
| `timedatectl show-timesync --all` | Подробная информация о синхронизации (systemd-timesyncd) |

## Концепции NTP

NTP (Network Time Protocol) синхронизирует часы по сети. Серверы организованы в иерархию, называемую **стратами**:

- **Страта 0**: Эталонные часы (атомные, GPS-приёмники) — не подключены к сети напрямую
- **Страта 1**: Серверы, напрямую подключённые к источникам страты 0
- **Страта 2**: Серверы, синхронизированные со страты 1
- Более высокие страты вносят дополнительную погрешность

`pool.ntp.org` предоставляет пул публичных NTP-серверов, распределяя нагрузку между множеством хостов.

### Термины NTP

| Термин | Описание |
|---|---|
| Offset (смещение) | Разница между локальными часами и временем NTP |
| Step (шаг) | Немедленная крупная корректировка времени (при большом смещении) |
| Slew (плавная корректировка) | Постепенное изменение скорости хода часов для уменьшения смещения |
| Insane Time (безумное время) | Смещение слишком велико для безопасной корректировки (порог по умолчанию в ntpd: 1000 секунд) |
| Drift (дрейф) | Систематическое отклонение скорости хода часов |
| Jitter (джиттер) | Изменчивость измерений смещения |

## systemd-timesyncd

`systemd-timesyncd` — лёгкий SNTP-клиент, входящий в состав systemd. Он проще, чем ntpd или chrony, и подходит для большинства систем, которым не нужно раздавать время другим хостам.

Настройка: `/etc/systemd/timesyncd.conf`. Статус: `timedatectl show-timesync --all`.

---

## ntpd — демон NTP

`ntpd` (пакет ntp) — классическая полная реализация NTP. Может как синхронизироваться с вышестоящими серверами, так и раздавать время другим хостам.

Файл конфигурации: `/etc/ntp.conf`

Типичный `/etc/ntp.conf`:

```
server 0.pool.ntp.org iburst
server 1.pool.ntp.org iburst
server 2.pool.ntp.org iburst
driftfile /var/lib/ntp/ntp.drift
```

Параметр `iburst` отправляет несколько пакетов при запуске для быстрой синхронизации.

### ntpq -p — статус пиров

`ntpq -p` выводит список NTP-пиров и их статус:

```
     remote           refid      st t when poll reach   delay   offset  jitter
==============================================================================
*0.pool.ntp.org  10.0.0.1        2 u   15   64  377    1.234   -0.456   0.123
```

| Столбец | Значение |
|---|---|
| `remote` | Имя NTP-сервера (префикс: `*` = выбран, `+` = допустим, `-` = отброшен) |
| `refid` | Идентификатор источника времени сервера |
| `st` | Страта удалённого сервера |
| `when` | Секунд с момента получения последнего пакета |
| `poll` | Интервал опроса в секундах |
| `reach` | Восьмеричная маска последних 8 опросов (377 = все успешны) |
| `delay` | Время приёма-передачи до сервера (мс) |
| `offset` | Разница времени между локальными часами и сервером (мс) |
| `jitter` | Изменчивость смещения по последним измерениям (мс) |

## chrony

`chrony` — современная реализация NTP с быстрой сходимостью. Хорошо работает при нестабильном соединении; используется по умолчанию в RHEL/Fedora.

- Демон: `chronyd`
- Клиент: `chronyc`
- Конфигурация: `/etc/chrony.conf`

Типичный `/etc/chrony.conf`:

```
pool pool.ntp.org iburst
driftfile /var/lib/chrony/drift
makestep 1.0 3
rtcsync
```

`makestep 1.0 3` разрешает шаговую коррекцию при смещении более 1 секунды, но только в течение первых 3 обновлений часов.

### Команды chronyc

| Команда | Описание |
|---|---|
| `chronyc tracking` | Сводка производительности часов (смещение, дрейф и т.д.) |
| `chronyc sources` | Список источников NTP и их статус |
| `chronyc ntpdata` | Подробные данные NTP по каждому источнику |
| `chronyc makestep` | Принудительная немедленная шаговая корректировка часов |

## Типичные вопросы экзамена

1. В чём разница между системными и аппаратными часами? → Системные часы поддерживаются ядром в памяти (теряются при выключении); аппаратные часы — RTC на батарейке, работают без питания.
2. Какая команда устанавливает аппаратные часы по системным? → `hwclock --systohc`
3. Какая команда устанавливает системные часы по аппаратным? → `hwclock --hctosys`
4. Что выводит `date +%s`? → Текущее время в формате Unix epoch (секунды с 1970-01-01 00:00:00 UTC).
5. Как установить системные часы с помощью `date`? → `date -s "ГГГГ-ММ-ДД ЧЧ:ММ:СС"`
6. Что делает `date --date=@1571661921`? → Преобразует Unix epoch 1571661921 в читаемую дату.
7. Что делает `timedatectl set-ntp true`? → Включает синхронизацию NTP через systemd-timesyncd.
8. Какая команда выводит все доступные временные зоны на systemd-системе? → `timedatectl list-timezones`
9. Что такое сервер NTP страты 1? → Сервер, напрямую подключённый к эталонному источнику страты 0 (атомные часы, GPS).
10. Что означает префикс `*` в выводе `ntpq -p`? → Этот сервер выбран текущим источником синхронизации.
11. Что показывает столбец `reach` в `ntpq -p`? → Восьмеричную маску последних 8 попыток опроса; 377 означает, что все 8 успешны.
12. Какой параметр NTP в `ntp.conf` ускоряет начальную синхронизацию? → `iburst` — отправляет пакеты пакетами при запуске.
13. В каком файле ntpd хранит данные о дрейфе часов? → `/var/lib/ntp/ntp.drift` (задаётся директивой `driftfile` в `ntp.conf`).
14. Что делает `chronyc tracking`? → Показывает сводку: текущее смещение, дрейф, страту и источник.
15. Что означает `makestep 1.0 3` в `chrony.conf`? → Разрешает шаговую коррекцию при смещении более 1 секунды, но только в течение первых 3 обновлений часов.
16. Чем `step` отличается от `slew` в NTP? → Step — немедленная крупная корректировка; slew — постепенное изменение скорости хода часов для снижения смещения.
17. Что такое `pool.ntp.org`? → DNS-пул публичных NTP-серверов, распределяющий нагрузку между множеством хостов.
18. Что такое `systemd-timesyncd`? → Лёгкий SNTP-клиент в составе systemd, подходящий для синхронизации времени, но не для раздачи времени другим хостам.

## Упражнения

### Практическое упражнение 1 — Чтение аппаратных часов

Какая команда читает аппаратные часы с подробным выводом?

<details>
<summary>Ответ</summary>

```bash
hwclock --verbose
```

</details>

### Практическое упражнение 2 — Преобразование epoch

В логе встречается значение epoch `1571661921`. Как его преобразовать в читаемую дату?

<details>
<summary>Ответ</summary>

```bash
date --date=@1571661921
```

</details>

---

### Практическое упражнение 3 — Отключение NTP перед ручной установкой времени

Нужно вручную установить системное время `2019-10-21 15:00:00`. Какие две команды следует выполнить?

<details>
<summary>Ответ</summary>

```bash
timedatectl set-ntp false
timedatectl set-time "2019-10-21 15:00:00"
```

NTP необходимо отключить перед ручной установкой времени через `timedatectl`.

</details>

---

### Исследовательское упражнение 1 — Чтение вывода ntpq

В выводе `ntpq -p` у сервера значение `reach` равно `377`. Что это означает?

<details>
<summary>Ответ</summary>

Все 8 последних попыток опроса прошли успешно. `377` в восьмеричной системе равно `11111111` в двоичной — все биты установлены.

</details>

---

### Исследовательское упражнение 2 — Принудительная шаговая корректировка chrony

Системные часы значительно отстают, chrony запущен. Как принудительно применить немедленную шаговую корректировку?

<details>
<summary>Ответ</summary>

```bash
chronyc makestep
```

Команда выполняет немедленную крупную корректировку вместо ожидания постепенного процесса slew.

</details>
