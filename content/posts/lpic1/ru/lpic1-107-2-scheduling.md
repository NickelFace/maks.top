## Что нужно знать

Согласно официальным целям LPIC-1:

- Управлять заданиями cron и at.
- Настраивать доступ пользователей к планировщикам.
- Использовать таймеры systemd как альтернативу cron.

Ключевые файлы и команды: `/etc/cron.d/`, `/etc/cron.{daily,hourly,monthly,weekly}/`, `/etc/at.deny`, `/etc/at.allow`, `/etc/crontab`, `/etc/cron.allow`, `/etc/cron.deny`, `/var/spool/cron/`, `crontab`, `at`, `atq`, `atrm`, `systemctl`, `systemd-run`.

## Cron — повторяющиеся задания

Демон `cron` просыпается каждую минуту и проверяет файлы crontab на наличие заданий. Пользовательские crontab хранятся в `/var/spool/cron/`.

### Формат пользовательского crontab

Пользовательский crontab содержит **5 временных полей** и команду:

```
мин  час  день_месяца  месяц  день_недели  команда
```

| Поле | Диапазон | Описание |
|---|---|---|
| мин | 0-59 | Минута |
| час | 0-23 | Час |
| день_месяца | 1-31 | День месяца |
| месяц | 1-12 | Месяц |
| день_недели | 0-7 | День недели (0 и 7 — воскресенье) |
| команда | — | Команда оболочки |

### Операторы временных полей

| Оператор | Смысл | Пример |
|---|---|---|
| `*` | Любое значение | `* * * * *` — каждую минуту |
| `,` | Список | `1,15,30` — на 1-й, 15-й, 30-й минуте |
| `-` | Диапазон | `1-5` — с понедельника по пятницу |
| `/` | Шаг | `*/20` — каждые 20 единиц |

### Примеры crontab

```
# Каждый день в 10:00
0 10 * * * /home/frank/foo.sh

# Каждый вторник в 08:00, 08:15, 08:30, 08:45
0,15,30,45 08 * * 2 /home/frank/bar.sh

# Пн-Пт, январь и июнь, дни 1-15, в 20:30
30 20 1-15 1,6 1-5 /home/frank/foobar.sh

# Каждые 20 минут
*/20 * * * * /path/to/script.sh
```

### Сокращения crontab

| Сокращение | Эквивалент | Смысл |
|---|---|---|
| `@reboot` | — | Один раз при запуске системы |
| `@hourly` | `0 * * * *` | Каждый час в :00 |
| `@daily` / `@midnight` | `0 0 * * *` | Каждый день в полночь |
| `@weekly` | `0 0 * * 0` | Каждую неделю в воскресенье в полночь |
| `@monthly` | `0 0 1 * *` | Каждый месяц 1-го числа в полночь |
| `@yearly` / `@annually` | `0 0 1 1 *` | Каждый год 1 января в полночь |

### Переменные crontab

Задаются в начале файла:

| Переменная | По умолчанию | Действие |
|---|---|---|
| `HOME` | Домашний каталог | Рабочий каталог для заданий cron |
| `MAILTO` | Имя пользователя | Получатель вывода; `""` отключает отправку |
| `PATH` | Системный путь | Путь поиска команд |
| `SHELL` | `/bin/sh` | Оболочка для выполнения команд |

### Команда crontab

| Параметр | Описание |
|---|---|
| `-e` | Редактировать crontab текущего пользователя (открывает `$EDITOR`) |
| `-l` | Вывести текущий crontab |
| `-r` | Удалить текущий crontab |
| `-u пользователь` | Работать с crontab другого пользователя (только root) |

### Системные crontab

В системных crontab добавляется **6-е поле** (имя пользователя) перед командой:

```bash
# /etc/crontab или файлы в /etc/cron.d/
30 01 * * * root /root/barfoo.sh >>/root/output.log 2>>/root/error.log
```

Каталоги для скриптов (без формата crontab, просто исполняемые файлы):
- `/etc/cron.hourly/`
- `/etc/cron.daily/`
- `/etc/cron.weekly/`
- `/etc/cron.monthly/`

### Управление доступом к cron

`/etc/cron.allow` и `/etc/cron.deny` определяют, кто из обычных пользователей может использовать `crontab`:

- Если `/etc/cron.allow` существует: только перечисленные пользователи могут планировать задания.
- Если есть только `/etc/cron.deny`: перечисленные пользователи лишены доступа, пустой файл означает, что все разрешены.
- Если ни один файл не существует: доступ зависит от дистрибутива.

Оба файла содержат по одному имени пользователя в строке.

## Таймеры systemd — альтернатива cron

Таймеры systemd являются юнит-файлами с суффиксом `.timer`. Каждый таймер активирует соответствующий юнит `.service` с тем же именем, но другим суффиксом.

### Структура файла таймера

```ini
[Unit]
Description=Run the foobar service

[Timer]
OnCalendar=Mon *-*-1..7 05:30:00
Persistent=true

[Install]
WantedBy=timers.target
```

### Синтаксис OnCalendar

```
ДеньНедели Год-Месяц-День Час:Минута:Секунда
```

`ДеньНедели` необязателен. Используйте `*` для любого значения, `,` для списка, `..` для диапазона (не `-`), `/` для шага.

### Примеры OnCalendar

| Выражение | Смысл |
|---|---|
| `*-*-* 08:30:00` | Каждый день в 08:30 |
| `Sat,Sun *-*-* 05:00:00` | В субботу и воскресенье в 05:00 |
| `*-*-01 13:15,30,45:00` | 1-го числа каждого месяца в 13:15, 13:30, 13:45 |
| `Fri *-09..12-* 16:20:00` | Каждую пятницу с сентября по декабрь в 16:20 |
| `*-*-* *:00/05:00` | Каждые 5 минут |
| `Mon *-*-* 00:00:00` | Каждый понедельник в полночь |

### Сокращения OnCalendar

| Сокращение | Эквивалент |
|---|---|
| `hourly` | `*-*-* *:00:00` |
| `daily` | `*-*-* 00:00:00` |
| `weekly` | `Mon *-*-* 00:00:00` |
| `monthly` | `*-*-01 00:00:00` |
| `yearly` | `*-01-01 00:00:00` |

### Управление таймерами

```bash
systemctl enable foobar.timer
systemctl start foobar.timer
systemctl daemon-reload          # после редактирования файлов таймеров
systemctl list-timers            # активные таймеры по времени следующего срабатывания
systemctl list-timers --all      # включая неактивные
```

Журналы таймеров пишутся в systemd journal, просмотр через `journalctl`. Обычные пользователи используют флаг `--user` с `systemctl` и `journalctl`.

## at — однократные задания

`at` планирует выполнение команды один раз в заданное время в будущем. Требует работающего демона `atd`.

```bash
$ at now +5 minutes
warning: commands will be executed using /bin/sh
at> date
at> Ctrl+D
job 12 at Sat Sep 14 09:15:00 2019
```

### Параметры at

| Параметр | Описание |
|---|---|
| `-c ID` | Вывести команды задания на stdout |
| `-d ID` | Удалить задание (псевдоним atrm) |
| `-f файл` | Читать команды из файла |
| `-l` | Вывести список ожидающих заданий (псевдоним atq) |
| `-m` | Отправить почту даже при отсутствии вывода |
| `-q очередь` | Указать очередь (a-z, A-Z); прописные буквы работают как batch |
| `-v` | Показать время выполнения до чтения задания |

### Форматы времени для at

- `ЧЧ:ММ` — 24-часовой или 12-часовой с AM/PM (если указанное время уже прошло, берётся следующий день)
- `midnight`, `noon`, `teatime` (16:00)
- `now +N minutes/hours/days/weeks`
- `today`, `tomorrow`
- Форматы даты: `month-name day`, `MMDDYY`, `MM/DD/YY`, `DD.MM.YY`, `YYYY-MM-DD`

### atq и atrm

```bash
atq                    # список ожидающих заданий (root видит все)
atrm 14                # удалить задание 14
atrm 50 51 52          # удалить несколько заданий
at -l                  # псевдоним atq
at -d 14               # псевдоним atrm 14
```

### Управление доступом к at

Логика аналогична cron:
- Если `/etc/at.allow` существует: только перечисленные пользователи могут планировать задания `at`.
- Если есть только `/etc/at.deny`: перечисленные пользователи лишены доступа, пустой файл разрешает всем.
- Если ни один файл не существует: доступ зависит от дистрибутива.

### systemd-run — альтернатива at

```bash
# Запустить команду в заданное время
systemd-run --on-calendar='2019-10-06 11:30' date

# Запустить скрипт через 2 минуты
systemd-run --on-active="2m" ./foo.sh
```

Обычные пользователи используют флаг `--user`. Просмотр журналов через `journalctl --user`.

## Полезные команды для экзамена

```
Пользовательский crontab: 5 временных полей + команда (мин час день_мес месяц день_нед команда)
Системный crontab: 6 полей (добавляется имя пользователя перед командой)
  /etc/crontab, /etc/cron.d/

Каталоги для скриптов (без формата crontab):
  /etc/cron.hourly/  /etc/cron.daily/  /etc/cron.weekly/  /etc/cron.monthly/

Пользовательские crontab хранятся в: /var/spool/cron/

Флаги crontab:
  -e  редактировать    -l  вывести    -r  удалить    -u пользователь (только root)

Сокращения cron:
  @reboot  @hourly  @daily/@midnight  @weekly  @monthly  @yearly/@annually

Вывод cron: по умолчанию отправляется почтой; MAILTO="" отключает; MAILTO="user" перенаправляет

Управление доступом cron:
  /etc/cron.allow   белый список (разрешены только перечисленные)
  /etc/cron.deny    чёрный список (запрещены перечисленные; пустой файл разрешает всем)

Таймер systemd:
  OnCalendar=ДеньНедели Год-Месяц-День Час:Минута:Секунда
  Persistent=true   наверстать пропущенные запуски
  systemctl enable/start ИМЯ.timer
  systemctl list-timers [--all]

at — однократное планирование (требует atd):
  at ЧЧ:ММ / at now +N minutes / at midnight / at tomorrow
  atq                 список ожидающих заданий
  atrm ID             удалить задание
  at -c ID            показать команды задания
  at -l = atq    at -d = atrm

Управление доступом at:
  /etc/at.allow   /etc/at.deny   (та же логика, что и cron)

systemd-run (альтернатива at):
  systemd-run --on-calendar='ГГГГ-ММ-ДД ЧЧ:ММ' команда
  systemd-run --on-active="Nм" команда
```

## Типичные вопросы экзамена

1. Каков формат записи в пользовательском crontab? → Пять временных полей (мин час день_мес месяц день_нед) и команда.
2. Что означает `*/20 * * * *` в crontab? → Запускать каждые 20 минут.
3. Какое сокращение заменяет `0 0 * * *` в crontab? → `@daily` или `@midnight`
4. В чём отличие `/etc/crontab` от пользовательских crontab? → Системные crontab содержат 6-е поле с именем пользователя, от которого выполняется задание.
5. Где хранятся пользовательские crontab? → `/var/spool/cron/`
6. Что делает `crontab -e`? → Открывает crontab текущего пользователя для редактирования в `$EDITOR`.
7. Что делает переменная `MAILTO=""` в crontab? → Отключает отправку почты с выводом заданий cron.
8. Что такое `/etc/cron.allow`? → Белый список: только перечисленные обычные пользователи могут использовать `crontab`. Имеет приоритет над `cron.deny`.
9. Какая команда запустит задание один раз через 30 минут? → `at now +30 minutes`
10. Что такое `atq`? → Выводит список ожидающих заданий `at`. Для root отображает задания всех пользователей.
11. Что такое `atrm`? → Удаляет ожидающие задания `at` по ID. `at -d` является псевдонимом.
12. Какой демон должен работать для использования `at`? → `atd`
13. Как записать `OnCalendar` для каждой пятницы с сентября по декабрь в 16:20? → `Fri *-09..12-* 16:20:00`
14. Как посмотреть активные таймеры systemd? → `systemctl list-timers`
15. Какой параметр `systemd-run` планирует однократное задание по календарному времени? → `--on-calendar=`
16. Что делает `Persistent=true` в таймере systemd? → Немедленно запускает таймер при следующем старте, если плановый запуск был пропущен.
17. Как запланировать задание cron на 1-е и 15-е число каждого месяца в 08:30? → `30 08 1,15 * * команда`
18. Какое сокращение `OnCalendar` означает "каждый понедельник в полночь"? → `weekly` (эквивалент `Mon *-*-* 00:00:00`)

## Упражнения

### Практическое упражнение 1 — Сокращения crontab

Какой временной спецификации crontab соответствует сокращение `@weekly`?

<details>
<summary>Ответ</summary>

`0 0 * * 0` — запуск в полночь в воскресенье (день недели 0).

</details>

---

### Практическое упражнение 2 — Чтение выражений cron

Объясните, что делают следующие записи crontab:

```
30 13 * * 1-5
00 09-18 * * *
0,20,40 11 * * Sun
*/20 * * * *
```

<details>
<summary>Ответ</summary>

- `30 13 * * 1-5` — в 13:30 каждый будний день (пн-пт)
- `00 09-18 * * *` — в начале каждого часа с 09:00 до 18:00, каждый день
- `0,20,40 11 * * Sun` — каждое воскресенье в 11:00, 11:20 и 11:40
- `*/20 * * * *` — каждые 20 минут

</details>

---

### Практическое упражнение 3 — Отправка вывода cron по почте

Как перенаправить вывод всех заданий cron пользователю `emma` по почте? И как полностью отключить отправку почты?

<details>
<summary>Ответ</summary>

Задать переменную `MAILTO` в начале файла crontab:

```
MAILTO="emma"      # отправлять вывод пользователю emma
MAILTO=""          # отключить любую почту
```

</details>
