- Следить за активными процессами.
- Выбирать и сортировать процессы для вывода на экран.
- Отправлять сигналы процессам.

## Процессы: основные понятия

Каждая запущенная программа в Linux живёт как процесс. Ядро выдаёт каждому процессу уникальный числовой идентификатор.

### PID, PPID и состояния процесса

**PID** (Process ID) — номер процесса, от 1 до 32768 по умолчанию.  
**PPID** (Parent Process ID) — PID родительского процесса, который его породил.

Процесс с PID 1 всегда `init` или `systemd`, он родитель всего.

Возможные состояния процесса:

| Символ | Состояние |
|--------|-----------|
| R | Running / Runnable (выполняется или готов к выполнению) |
| S | Sleeping (ждёт события) |
| D | Uninterruptible sleep (ждёт I/O, нельзя прервать) |
| T | Stopped (остановлен сигналом SIGSTOP) |
| Z | Zombie (завершился, родитель ещё не принял статус) |

## Управление заданиями (job control)

Оболочка bash умеет управлять несколькими заданиями в одном терминале.

### Запуск в фоне

Добавь `&` в конец команды, и она уйдёт в фон сразу:

```bash
sleep 300 &
# [1] 4521   <- номер задания и PID
```

Чтобы отправить уже запущенный процесс в фон: нажми `Ctrl+Z` (это остановит процесс сигналом SIGTSTP), затем введи `bg`.

### fg, bg и jobs

**jobs** показывает все задания текущей оболочки:

```bash
jobs
# [1]-  Running    sleep 300 &
# [2]+  Stopped    vim file.txt
```

Знак `+` означает текущее задание по умолчанию, `-` — предыдущее.

Опции `jobs`:

| Опция | Действие |
|-------|----------|
| `-l` | Показать PID рядом со статусом |
| `-n` | Только задания, изменившие статус с последнего уведомления |
| `-p` | Только PID заданий |
| `-r` | Только работающие задания |
| `-s` | Только остановленные задания |

**bg** возобновляет остановленное задание в фоне:

```bash
bg %2    # отправить задание №2 в фон
bg       # отправить текущее задание (со знаком +)
```

**fg** возвращает задание на передний план:

```bash
fg %1    # вернуть задание №1
fg       # вернуть текущее задание
```

Если задание не указано, `fg` и `bg` действуют на текущее задание (со знаком `+`). Команде `kill`, в отличие от них, jobspec нужен всегда.

### Jobspec: способы обращаться к заданию

Jobspec (job specification) — это способ обратиться к конкретному заданию в командах `fg`, `bg`, `kill` и `jobs`.

| Форма | Описание |
|-------|----------|
| `%n` | Задание с номером n |
| `%str` | Задание, команда которого начинается со str |
| `%?str` | Задание, команда которого содержит str |
| `%+` или `%%` | Текущее задание (последнее запущенное или остановленное) |
| `%-` | Предыдущее задание |

Примеры:

```bash
jobs %1        # задание №1
jobs %sl       # задание, команда которого начинается с «sl»
jobs %?leep    # задание, команда которого содержит «leep»
fg %+          # вернуть текущее задание
kill %-        # убить предыдущее задание
```

## Выживание после logout: nohup, screen, tmux

По умолчанию при выходе из терминала оболочка отправляет сигнал **SIGHUP** всем своим дочерним процессам, и они завершаются. Три инструмента решают эту проблему по-разному.

### nohup

`nohup` запускает команду с игнорированием SIGHUP. Вывод по умолчанию уходит в файл `nohup.out` в текущей директории (или в `$HOME/nohup.out`, если текущая директория недоступна для записи).

```bash
nohup ./long_script.sh &
# вывод -> nohup.out

nohup ./script.sh > /var/log/script.log 2>&1 &
# вывод -> указанный файл
```

`nohup` защищает только от SIGHUP и не создаёт постоянного сеанса.

### screen

GNU Screen создан в 1987 году для эмуляции нескольких независимых VT100-терминалов на одном физическом терминале. Структура: сессия содержит окна (windows), окна можно делить на регионы (regions).

Префикс команд screen: `Ctrl+A`.

**Запуск и сессии:**

```bash
screen              # новая сессия
screen -S mywork    # новая сессия с именем
screen -ls          # список сессий
screen -r           # переподключиться (если сессия одна)
screen -r mywork    # переподключиться по имени или PID
screen -S PID -X quit   # завершить сессию по PID
```

Опции для повторного подключения:

| Опция | Действие |
|-------|----------|
| `-d -r` | Подключиться, при необходимости сначала отсоединить |
| `-d -R` | То же, но создать сессию если её нет |
| `-d -m` | Запустить сессию в отсоединённом режиме (для скриптов запуска) |

**Горячие клавиши (после `Ctrl+A`):**

| Клавиша | Действие |
|---------|----------|
| `c` | Создать новое окно |
| `n` | Следующее окно |
| `p` | Предыдущее окно |
| `число` | Перейти к окну по номеру |
| `"` | Список всех окон для выбора |
| `w` | Список окон в нижней строке |
| `A` | Переименовать текущее окно |
| `k` | Закрыть текущее окно (с подтверждением) |
| `d` | Detach (отсоединиться от сессии) |
| `S` | Разделить горизонтально (создать регион) |
| `\|` | Разделить вертикально (создать регион) |
| `Tab` | Перейти к следующему региону |
| `Q` | Закрыть все регионы кроме текущего |
| `X` | Закрыть текущий регион |
| `[` | Войти в режим copy/scrollback |
| `]` | Вставить скопированный текст |
| `?` | Справка по всем клавишам |
| `:` | Командная строка screen |

Закрытие региона не закрывает его окно, окно продолжает работать.

**Copy mode в screen:**

1. Войти: `Ctrl+A [`
2. Перемещаться стрелками или PgUp/PgDown.
3. Отметить начало: `Space`.
4. Перемещаться к концу.
5. Отметить конец: `Space`.
6. Вставить в нужном окне: `Ctrl+A ]`.

**Конфигурация:** `/etc/screenrc` (системная) и `~/.screenrc` (пользовательская).

### tmux

tmux вышел в 2007 году. Работает по модели клиент-сервер. Структура: сессия содержит окна (windows), окна делятся на панели (panes). Панели, в отличие от регионов screen, являются полноценными псевдотерминалами: закрытие панели завершает её процесс.

Префикс команд tmux: `Ctrl+B`.

**Запуск и сессии:**

```bash
tmux                            # новая сессия
tmux new -s "LPI" -n "Window"   # сессия с именем, первое окно с именем
tmux ls                         # список сессий
tmux a                          # подключиться (если сессия одна)
tmux attach -t mywork           # подключиться по имени
tmux attach -d -t mywork        # подключиться, сначала отсоединив от других терминалов
tmux kill-session -t mywork     # завершить сессию
```

**Горячие клавиши (после `Ctrl+B`):**

| Клавиша | Действие |
|---------|----------|
| `c` | Создать новое окно |
| `n` | Следующее окно |
| `p` | Предыдущее окно |
| `число` | Перейти к окну по номеру |
| `w` | Список всех окон для выбора |
| `f` | Найти окно по имени |
| `,` | Переименовать текущее окно |
| `&` | Закрыть текущее окно (с подтверждением) |
| `d` | Detach |
| `D` | Выбрать клиента для отсоединения |
| `s` | Список сессий для переключения |
| `$` | Переименовать текущую сессию |
| `"` | Разделить панель горизонтально |
| `%` | Разделить панель вертикально |
| `x` | Закрыть текущую панель (с подтверждением) |
| `!` | Преобразовать панель в отдельное окно |
| `z` | Увеличить/уменьшить панель на весь экран |
| стрелки | Переключаться между панелями |
| `;` | Перейти к последней активной панели |
| `Ctrl+стрелка` | Изменить размер панели на 1 строку |
| `Alt+стрелка` | Изменить размер панели на 5 строк |
| `{` | Поменять текущую панель с предыдущей |
| `}` | Поменять текущую панель со следующей |
| `t` | Показать часы в панели |
| `r` | Обновить терминал клиента |
| `?` | Справка |
| `:` | Командная строка tmux |

**Copy mode в tmux:**

1. Войти: `Ctrl+B [`
2. Перемещаться стрелками.
3. Отметить начало: `Ctrl+Space`.
4. Перемещаться к концу.
5. Скопировать: `Alt+W`.
6. Вставить: `Ctrl+B ]`.

**Конфигурация:** `/etc/tmux.conf` и `~/.tmux.conf`. Пример запуска с альтернативным файлом: `tmux -f /path/to/config`.

### Отличия screen от tmux

| | screen | tmux |
|-|--------|------|
| Префикс | `Ctrl+A` | `Ctrl+B` |
| Модель | Монолитная | Клиент-сервер |
| Деление окна | Регионы | Панели (псевдотерминалы) |
| Закрытие раздела | Не закрывает окно | Закрывает процесс панели |
| Год выпуска | 1987 | 2007 |

Ключевое отличие от `nohup`: в `screen` и `tmux` процессы живут внутри сервера, который продолжает работать независимо от сетевого подключения.

## Мониторинг процессов

### ps

`ps` выводит снимок процессов на момент запуска. Принимает параметры в трёх стилях:

| Стиль | Пример | Особенность |
|-------|--------|-------------|
| BSD | `ps aux` | Без ведущего дефиса |
| UNIX | `ps -ef` | С одним ведущим дефисом |
| GNU | `ps --pid 811` | С двумя ведущими дефисами |

Самые распространённые варианты:

```bash
ps aux               # все процессы, расширенный формат (BSD-стиль)
ps -ef               # все процессы, полный формат (System V)
ps a                 # процессы с терминалом (tty)
ps -e --forest       # дерево процессов
ps -p 1234           # конкретный PID
ps aux --sort=-%cpu  # сортировка по CPU убыванию
ps aux --sort=-%mem  # сортировка по памяти убыванию
```

Фильтрация по пользователю (все три варианта эквивалентны):

```bash
ps U www-data        # BSD
ps -u www-data       # UNIX
ps --user www-data   # GNU
```

Выбор столбцов через `-o`:

```bash
ps -eo pid,user,cmd             # только нужные поля
ps o user,%mem,%cpu,cmd         # пользователь, память, CPU, команда
ps o user,comm                  # пользователь и имя программы
```

Флаги `a`, `u`, `x` в BSD-стиле:

| Флаг | Смысл |
|------|-------|
| `a` | Процессы с терминалом (все пользователи) |
| `u` | Формат с владельцем и деталями памяти/CPU |
| `x` | Процессы без терминала |

Колонки в `ps aux`:

| Колонка | Смысл |
|---------|-------|
| USER | Владелец процесса |
| PID | Идентификатор |
| %CPU | Загрузка CPU |
| %MEM | Доля памяти |
| VSZ | Виртуальная память, включая swap (KB) |
| RSS | Физическая память без swap (KB) |
| STAT | Состояние процесса |
| START | Время запуска |
| COMMAND | Команда |

Расширенные коды в столбце STAT:

| Код | Значение |
|-----|----------|
| `S` | Прерываемый сон (ждёт события) |
| `R` | Выполняется или в очереди |
| `D` | Непрерываемый сон (обычно I/O) |
| `T` | Остановлен сигналом управления |
| `Z` | Зомби |
| `<` | Высокий приоритет (не уступает другим) |
| `N` | Низкий приоритет (уступает другим) |
| `+` | В группе переднего плана |

### top

`top` показывает процессы в реальном времени и обновляется каждые 3 секунды.

```bash
top
top -u alice        # только процессы пользователя
top -p 1234,5678    # конкретные PID
top -n 5            # обновить 5 раз и выйти
top -b -n 1         # batch-режим, одно обновление (удобно для скриптов)
```

Интерактивные клавиши внутри `top`:

| Клавиша | Действие |
|---------|----------|
| `q` | Выйти |
| `k` | Убить процесс (запросит PID и сигнал, по умолчанию SIGTERM) |
| `r` | Renice (изменить приоритет; только root может понижать значение) |
| `M` | Сортировать по памяти |
| `P` | Сортировать по CPU |
| `N` | Сортировать по PID |
| `T` | Сортировать по времени работы |
| `R` | Переключить порядок сортировки: по убыванию / по возрастанию |
| `u` | Фильтр по пользователю |
| `c` | Показать полные пути и разделить userspace / kernelspace (в квадратных скобках) |
| `V` | Дерево процессов (forest view) |
| `1` | Показать каждое ядро CPU отдельно |
| `t` | Изменить вид строки CPU (прогресс-бар, числа, скрыть) |
| `m` | Изменить вид строки памяти |
| `x` | Подсветить столбец сортировки |
| `W` | Сохранить настройки в `~/.toprc` |
| `?` или `h` | Помощь |

Запустить `top` с сортировкой по нужному полю сразу:

```bash
top -o %MEM    # сортировать по памяти при запуске
```

Строки заголовка в top:

- `load average` — три числа: нагрузка за 1, 5 и 15 минут. На одноядерной системе значение 1.0 означает полную загрузку.
- `Tasks` — итого, запущено, спящих, остановлено, зомби.
- `%Cpu(s)` — разбивка времени CPU: us (user), sy (system), ni (nice), id (idle), wa (iowait), hi (hardware interrupts), si (software interrupts), st (stolen).
- `MiB Mem` / `MiB Swap` — использование памяти.

### pgrep и pidof

`pgrep` ищет процессы по имени и возвращает PID:

```bash
pgrep firefox           # найти все PID процессов firefox
pgrep -l firefox        # с именами
pgrep -u alice          # процессы пользователя alice
pgrep -a sshd           # PID и полная командная строка
```

`pidof` делает то же самое, но принимает точное имя программы:

```bash
pidof apache2
```

Оба подходят для подстановки в `kill`: `kill $(pgrep sleep)` и `kill $(pidof sleep)`.

### watch

`watch` запускает команду повторно с заданным интервалом и выводит результат на экран:

```bash
watch ps aux            # обновлять каждые 2 секунды (по умолчанию)
watch -n 5 free -h      # каждые 5 секунд
watch -d ls /tmp        # подсвечивать изменения
```

### uptime и free

`uptime` выводит время работы системы и среднюю нагрузку:

```bash
uptime
# 14:32:11 up 5 days, 3:12,  2 users,  load average: 0.15, 0.10, 0.09
```

`free` показывает использование RAM и swap:

```bash
free -h     # в читабельных единицах (K, M, G)
free -m     # в мегабайтах
free -s 2   # обновлять каждые 2 секунды
```

## Сигналы и завершение процессов

Сигналы — это механизм уведомления процессов о событиях. Процесс может поймать сигнал, проигнорировать или обработать его по умолчанию.

Наиболее важные сигналы:

| Номер | Имя | Описание |
|-------|-----|----------|
| 1 | SIGHUP | Повесить трубку / перечитать конфиг |
| 2 | SIGINT | Прервать (как Ctrl+C) |
| 9 | SIGKILL | Принудительно завершить (нельзя поймать) |
| 15 | SIGTERM | Завершить корректно (по умолчанию в kill) |
| 18 | SIGCONT | Продолжить остановленный процесс |
| 19 | SIGSTOP | Остановить (нельзя поймать) |
| 20 | SIGTSTP | Остановить с терминала (Ctrl+Z) |

Посмотреть все сигналы: `kill -l`.

### kill

`kill` отправляет сигнал процессу по PID:

```bash
kill 1234           # SIGTERM (15) по умолчанию
kill -9 1234        # SIGKILL
kill -SIGKILL 1234  # то же самое
kill -15 1234       # явно SIGTERM
kill -HUP 1234      # SIGHUP (перечитать конфиг)
```

Сигнал можно указать тремя способами:

```bash
kill -SIGHUP 1234    # по имени
kill -1 1234         # по номеру
kill -s SIGHUP 1234  # через опцию -s
```

Чтобы не искать PID вручную, используй подстановку команды:

```bash
kill $(pgrep apache2)   # современный синтаксис
kill `pgrep apache2`    # старый синтаксис
```

Использовать SIGKILL стоит в последнюю очередь: процесс не успевает корректно завершиться, сохранить данные или освободить ресурсы.

### pkill и killall

**pkill** отправляет сигнал процессам по имени (не требует PID):

```bash
pkill firefox           # SIGTERM всем firefox
pkill -9 firefox        # SIGKILL
pkill -u alice          # завершить все процессы пользователя alice
pkill -STOP firefox     # остановить firefox
```

**killall** работает аналогично, но требует точного совпадения имени:

```bash
killall httpd           # SIGTERM всем httpd
killall -9 httpd        # SIGKILL
killall -v httpd        # verbose (вывести PID)
killall -i httpd        # интерактивный режим (спрашивать подтверждение)
```

Разница: `pkill` поддерживает регулярные выражения и частичное совпадение, `killall` ищет точное имя.

## Шпаргалка

```bash
# Запуск и управление заданиями
command &               # запустить в фоне
Ctrl+Z                  # остановить текущий процесс
bg [%n]                 # возобновить задание в фоне
fg [%n]                 # вернуть задание на передний план
jobs                    # список заданий текущей оболочки
jobs -l                 # с PID

# Защита от logout
nohup command &                  # игнорировать SIGHUP
nohup cmd > file.log 2>&1 &      # с перенаправлением
screen -S name                   # новая сессия screen
screen -r name                   # переподключиться
tmux new -s name                 # новая сессия tmux
tmux attach -t name              # переподключиться

# Мониторинг
ps aux                  # все процессы
ps -ef                  # все процессы (другой формат)
ps aux --sort=-%mem     # сортировка по памяти
top                     # интерактивный мониторинг
top -b -n 1             # batch режим
pgrep -l name           # найти PID по имени
watch -n 5 free -h      # наблюдать за командой
uptime                  # нагрузка системы
free -h                 # память

# Сигналы
kill -l                 # список сигналов
kill PID                # SIGTERM
kill -9 PID             # SIGKILL
pkill name              # по имени
pkill -u user           # по пользователю
killall name            # точное имя
```

## Типичные вопросы экзамена

**Какой сигнал kill отправляет по умолчанию?**
SIGTERM (15). Можно поймать и обработать, в отличие от SIGKILL (9).

**Чем pkill отличается от killall?**
`pkill` поддерживает частичное совпадение и регулярные выражения. `killall` требует точного имени.

**Что делает Ctrl+Z?**
Отправляет сигнал SIGTSTP, который приостанавливает процесс. После этого `bg` отправляет его в фон, `fg` возвращает на передний план.

**Как nohup отличается от screen?**
`nohup` защищает только от SIGHUP. `screen` создаёт постоянную сессию, к которой можно вернуться.

**Куда nohup направляет вывод?**
В `nohup.out` в текущей директории, или в `$HOME/nohup.out`.

**Как отсоединиться от сессии tmux?**
`Ctrl+B`, затем `d`.

**Что означает load average 2.0 на двухядерном процессоре?**
Полную загрузку всех ядер (1.0 на ядро).

**Как посмотреть все сигналы?**
`kill -l`.

**Какой сигнал нельзя поймать или проигнорировать?**
SIGKILL (9) и SIGSTOP (19).

#### Упражнение 8. watch + ps для мониторинга соединений

Статический вывод `ps` превращается в динамический, если обернуть его в `watch`. Мониторим новые подключения к Apache.

**Как добавить директиву MaxConnectionsPerChild в конфиг?**

<details><summary>Ответ</summary>

Вставить строку в конфиг файл:

```
MaxConnectionsPerChild 1
```

На Debian/Ubuntu: `/etc/apache2/apache2.conf`. На CentOS/RHEL: `/etc/httpd/conf/httpd.conf`.

После изменений перезапустить:

```bash
sudo systemctl restart apache2
```

</details>

**Как запустить watch с ps и grep для мониторинга apache2?**

<details><summary>Ответ</summary>

```bash
watch 'ps aux | grep apache2'
```

Команда передаётся в одинарных кавычках, чтобы оболочка не раскрывала конструкцию `ps aux | grep` до передачи её в watch.

</details>

**Что происходит в выводе watch когда браузер подключается к серверу?**

<details><summary>Ответ</summary>

Один из дочерних процессов `www-data` исчезает из списка: `MaxConnectionsPerChild 1` означает, что после одного запроса дочерний процесс завершается.

</details>
