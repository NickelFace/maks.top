- Перемещаться по документу в vi.
- Понимать и использовать режимы vi.
- Вставлять, редактировать, удалять, копировать и искать текст в vi.
- Знать о существовании Emacs, nano и vim.
- Настраивать редактор по умолчанию через переменную `EDITOR`.

## Режимы vi

vi работает в нескольких режимах. Режим определяет, что происходит при нажатии клавиш.

### Нормальный режим

Нормальный режим (он же командный) — режим по умолчанию при открытии файла. В нём клавиши выполняют команды навигации и редактирования, а не вводят текст. Из любого режима вернуться сюда можно клавишей `Esc`.

### Режим вставки

В режиме вставки текст вводится как обычно, символ за символом. Для входа используй команды `i`, `a`, `o` и другие. Выход через `Esc`.

### Режим команд с двоеточием

Этот режим запускается нажатием `:` в нормальном режиме. Здесь вводятся команды сохранения, выхода, поиска с заменой и запуска shell. Для возврата в нормальный режим нажми `Enter` без команды или выполни `:visual`.

## Навигация в нормальном режиме

Базовое перемещение курсора:

| Клавиша | Действие |
|---------|----------|
| `h` | влево |
| `j` | вниз |
| `k` | вверх |
| `l` | вправо |
| `0` | начало строки |
| `$` | конец строки |
| `w` / `W` | следующее слово / следующее СЛОВО (с пунктуацией) |
| `e` / `E` | конец текущего слова |
| `b` | назад на одно слово |
| `(` / `)` | начало / конец предложения |
| `{` / `}` | начало / конец абзаца |
| `1G` или `gg` | начало файла |
| `G` | конец файла |
| `:N` | перейти на строку N |

Для перехода в конец файла при открытии используй `vi + filename`. Знак `+` без числа означает последнюю строку. Для конкретной строки N пиши `vi +N filename`.

## Вставка текста

Все команды ниже переводят vi из нормального режима в режим вставки:

| Клавиша | Действие |
|---------|----------|
| `i` | вставить перед курсором |
| `I` | вставить в начало строки |
| `a` | добавить после курсора |
| `A` | добавить в конец строки |
| `o` | открыть новую строку снизу и войти в режим вставки |
| `O` | открыть новую строку сверху и войти в режим вставки |
| `s` | удалить символ под курсором и войти в режим вставки |
| `S` | удалить всю строку и войти в режим вставки |

---

## Удаление и изменение

| Клавиша | Действие |
|---------|----------|
| `x` | удалить символ под курсором |
| `dd` | удалить строку |
| `dw` | удалить слово |
| `d$` или `D` | удалить до конца строки |
| `dt.` | удалить от курсора до символа `.` (не включая его) |
| `c` | заменить выделенное: удалить и войти в режим вставки |
| `r` | заменить один символ под курсором |
| `u` | отменить последнее действие |
| `Ctrl-R` | повторить отменённое действие |

Команда `d` принимает движение в качестве аргумента. `d5w` удаляет текущее слово и следующие 4. `dt.` удаляет от курсора до ближайшей точки, не затрагивая саму точку.

## Копирование и вставка

| Клавиша | Действие |
|---------|----------|
| `yy` | скопировать строку |
| `y` | скопировать выделенное |
| `yw` | скопировать слово |
| `p` | вставить после курсора |
| `P` | вставить перед курсором |

---

## Поиск

| Команда | Действие |
|---------|----------|
| `/pattern` | поиск вперёд |
| `?pattern` | поиск назад |
| `n` | следующее совпадение |
| `N` | предыдущее совпадение |

После ввода паттерна нажми `Enter` для запуска поиска. Клавиши `n` и `N` продолжают поиск в прямом и обратном направлениях.

---

## Числовые префиксы

Перед большинством команд нормального режима можно указать число. Команда выполнится ровно столько раз. Примеры:

- `3yy` — скопировать текущую строку и следующие 2.
- `2dd` — удалить текущую строку и следующую.
- `5j` — переместиться на 5 строк вниз.
- `d5w` — удалить текущее слово и следующие 4.
- `3<` — сдвинуть выделение на 3 позиции влево.

Число вводится прямо перед командой, без пробела.

## Команды с двоеточием

| Команда | Действие |
|---------|----------|
| `:w` | сохранить файл |
| `:w filename` | сохранить под именем filename |
| `:w!` | принудительно сохранить |
| `:q` | выйти |
| `:q!` | выйти без сохранения |
| `:wq` | сохранить и выйти |
| `:x`, `:exit` или `:e` | сохранить и выйти (только если были изменения) |
| `ZZ` | сохранить и выйти (нормальный режим, без `:`) |
| `ZQ` | выйти без сохранения (нормальный режим) |
| `:!cmd` | выполнить команду shell |
| `:s/REGEX/TEXT/g` | заменить REGEX на TEXT во всей текущей строке |
| `:visual` | вернуться в нормальный режим |

`ZZ` и `ZQ` вводятся в нормальном режиме заглавными буквами, без двоеточия.

## Регистры и макросы

vi хранит скопированный текст в регистрах. Регистр указывается символом `"` перед его именем. Например, `"ly` копирует выделенное в регистр `l`, а `"lp` вставляет содержимое регистра `l`.

Макрос записывается командой `q` плюс буква регистра. Запись идёт до повторного нажатия `q`. После этого макрос запускается командой `@` и именем регистра. Пример: `qa` начинает запись в регистр `a`, второй `q` заканчивает запись, а `@a` воспроизводит всю записанную последовательность.

Метки позволяют отмечать позиции в тексте. `m` плюс буква создаёт метку в текущей позиции курсора, а `'` плюс та же буква возвращает к ней.

## Переменная EDITOR

Bash использует переменные `VISUAL` и `EDITOR`, чтобы определить редактор по умолчанию. Этот редактор открывается, например, при `crontab -e` или при написании commit-сообщения в git.

Установить редактор для текущей сессии:

```bash
export EDITOR=nano
```

Чтобы настройка сохранялась между сессиями, добавь эту строку в `~/.bash_profile`. Если заданы обе переменные, `VISUAL` имеет приоритет над `EDITOR`.

## Альтернативные редакторы

### vim

vim (Vi IMproved) — расширенная версия vi. В большинстве дистрибутивов команда `vi` фактически запускает vim. Из отличий: подсветка синтаксиса, поддержка плагинов, разделение экрана и улучшенный поиск. Для знакомства с vim есть встроенный интерактивный учебник `vimtutor`.

### nano

GNU nano подходит тем, кто не хочет разбираться с режимами vi. Текст вводится напрямую, без переключения режимов. Команды управляются клавишами `Ctrl` и `Meta` (Alt или Command в зависимости от системы), а их список отображается прямо внизу экрана. Nano поддерживает подсветку синтаксиса, поиск с заменой, автоотступ и нумерацию строк.

| Клавиша | Действие |
|---------|----------|
| `Ctrl-6` или `Meta-A` | начать выделение |
| `Meta-6` | скопировать выделенное |
| `Ctrl-K` | вырезать выделенное |
| `Ctrl-U` | вставить |
| `Meta-U` | отменить действие |
| `Meta-E` | повторить действие |
| `Ctrl-\` | заменить текст |
| `Ctrl-T` | проверка орфографии |

### Emacs

Emacs — мощный редактор с командными сочетаниями через `Ctrl` и `Meta`. Текст вводится напрямую, как в nano. Emacs умеет компилировать код, работать с почтой и RSS-лентами, что превращает его в полноценную среду разработки. Для экзамена LPIC-1 достаточно знать о его существовании.

## Шпаргалка

```
# Открытие файла
vi filename           открыть файл
vi + filename         открыть, перейти на последнюю строку
vi +N filename        открыть, перейти на строку N

# Сохранение и выход
ZZ                    сохранить и выйти (нормальный режим)
ZQ                    выйти без сохранения (нормальный режим)
:w                    сохранить
:w filename           сохранить под другим именем
:w!                   принудительно сохранить
:wq                   сохранить и выйти
:q!                   выйти без сохранения

# Навигация
h j k l               влево / вниз / вверх / вправо
0  $                  начало / конец строки
gg  G                 начало / конец файла
/pattern              поиск вперёд
?pattern              поиск назад
n  N                  следующее / предыдущее совпадение

# Режим вставки
i  a  o               вставить / добавить / новая строка снизу
I  A  O               начало строки / конец строки / новая строка сверху

# Удаление
x                     символ под курсором
dd                    строка
2dd                   две строки
dw  d5w               слово / пять слов
dt.                   до символа .

# Копирование и вставка
yy  3yy               скопировать строку / три строки
p  P                  вставить после / до курсора

# Редактор по умолчанию
export EDITOR=vi
export VISUAL=nano
```

## Типичные вопросы экзамена

**Как удалить текущую строку и следующую?**
`2dd`

**Как сохранить файл и выйти из vi? (два варианта)**
`:wq` в режиме двоеточия или `ZZ` в нормальном режиме.

**Как сохранить файл под новым именем?**
`:w filea.txt`

**Как выполнить команду N раз?**
Указать число прямо перед командой: `4l`, `2yj`, `3dd`.

**Как задать nano редактором по умолчанию?**
`export EDITOR=nano`; для постоянного эффекта добавить в `~/.bash_profile`.

**Что делает команда `vi + ~/.bash_profile`?**
Открывает файл и сразу помещает курсор на последнюю строку.

#### Упражнение 1

**vi часто используют для редактирования конфигурационных файлов и исходного кода, где отступы помогают выделить блоки текста. Выделение можно сдвигать влево клавишей `<` и вправо клавишей `>`. Какие клавиши нужно нажать в нормальном режиме, чтобы сдвинуть текущее выделение на три позиции влево?**

<details><summary>Ответ</summary>

`3<`

</details>

---

#### Упражнение 2

**Всю строку целиком можно выделить клавишей `V` в нормальном режиме, но символ перевода строки при этом тоже попадает в выделение. Какие клавиши нужно нажать в нормальном режиме, чтобы выделить строку от первого символа до конца, не включая символ перевода строки?**

<details><summary>Ответ</summary>

`0v$h`

`0` переходит в начало строки, `v` запускает режим выделения символов, `$` перемещает курсор в конец строки, `h` отступает на один символ назад и тем самым исключает символ перевода строки из выделения.

</details>

---

#### Упражнение 3

**Как запустить vi из командной строки, чтобы открыть `~/.bash_profile` и сразу перейти на последнюю строку?**

<details><summary>Ответ</summary>

```bash
vi + ~/.bash_profile
```

Флаг `+` без числа означает "перейти на последнюю строку". Для перехода на конкретную строку N используй `vi +N filename`.

</details>

---

#### Упражнение 4

**Какие клавиши нужно нажать в нормальном режиме vi, чтобы удалить символы от текущей позиции курсора до ближайшей точки, не включая саму точку?**

<details><summary>Ответ</summary>

`dt.`

`d` начинает удаление, `t` означает "до следующего вхождения символа" (не включая его), `.` задаёт целевой символ.

</details>

### Исследовательские упражнения (Explorational Exercises)

#### Упражнение 1

**vim позволяет выделять прямоугольные блоки текста произвольной ширины, а не только строки целиком. Нажатие `Ctrl+V` в нормальном режиме запускает блочное выделение, которое расширяется движениями курсора. Как выделить и удалить блок, начиная с первого символа текущей строки, шириной 8 столбцов и высотой 5 строк?**

<details><summary>Ответ</summary>

`0`, затем `Ctrl-V`, затем `8l5jd`

`0` переходит в начало строки, `Ctrl-V` запускает блочное выделение, `8l` сдвигает правую границу на 8 символов вправо, `5j` опускает нижнюю границу на 5 строк вниз, `d` удаляет выделенный блок.

</details>

#### Упражнение 2

**Сессия vi прервалась из-за неожиданного отключения питания. При повторном открытии файла vi предлагает восстановить swap-файл (автоматическую резервную копию). Что нужно сделать, чтобы отказаться от swap-файла?**

<details><summary>Ответ</summary>

Нажать `d` в ответ на запрос vi.

</details>

---

#### Упражнение 3

**В сессии vim строка была скопирована в регистр `l`. Какая последовательность клавиш запишет макрос в регистр `a`, который вставляет содержимое регистра `l` перед текущей строкой?**

<details><summary>Ответ</summary>

`qa"lPq`

`q` начинает запись макроса, `a` привязывает его к регистру `a`, `"l` указывает на регистр `l` как источник вставки, `P` вставляет текст перед текущей строкой, второй `q` завершает запись. Чтобы воспроизвести макрос, нажми `@a`.

</details>
