> **Тема 104.2** — Поддержание целостности файловых систем. Практические и исследовательские разборы.

---

## Практические упражнения (Guided)

### 1. Размер только файлов текущего каталога через du

**Задача:** с помощью `du` проверить, сколько места занимают именно файлы в текущем каталоге, без подкаталогов.

**Ответ:**

```bash
du -S -h -d 0
# или в коротком виде
du -Shd 0
```

Параметр `-S` разделяет учёт текущего каталога и его подкаталогов. Параметр `-d 0` ограничивает глубину рекурсии нулём — никаких подкаталогов в выводе. Без `-S` итог всё равно включал бы вложенные файлы, поэтому оба параметра нужны вместе.

---

### 2. df с фильтром по ext4 и кастомными колонками

**Задача:** с помощью `df` вывести информацию по каждой ext4-файловой системе. Колонки в порядке: устройство, точка монтирования, общее число inodes, число доступных inodes, процент использованного места.

**Ответ:**

```bash
df -t ext4 --output=source,target,itotal,iavail,pcent
```

`-t ext4` — фильтр по типу. `--output=...` — задаёт список и порядок колонок.

Поле `pcent` показывает процент использованного места. Отдельного поля «процент свободного» у `df` нет.

---

### 3. e2fsck в неинтерактивном режиме с автоисправлением

**Задача:** запустить `e2fsck` для `/dev/sdc1` в неинтерактивном режиме с автоматическим исправлением большинства ошибок.

**Ответ:**

```bash
e2fsck -p /dev/sdc1
```

При `-p` утилита чинит то, что может без участия человека. При серьёзной ошибке — описывает её и завершается. Альтернативы: `-y` отвечает yes на всё, `-n` отвечает no и открывает ФС только для чтения.

---

### 4. Конвертация ext2 в ext3 со сбросом счётчика и меткой

**Задача:** `/dev/sdb1` — ext2. Конвертировать в ext3, сбросить счётчик монтирований, поменять метку на `UserData`.

**Ответ:**

```bash
tune2fs -j -C 0 -L UserData /dev/sdb1
```

- `-j` — добавить журнал (конвертация ext2 → ext3)
- `-C 0` (заглавная) — сбросить текущий счётчик монтирований
- `-L UserData` — установить метку

**Регистр важен:** малое `-c` задаёт максимум монтирований, заглавное `-C` устанавливает текущее значение счётчика.

После конвертации при монтировании нужно указать тип `ext3`, иначе журнал не задействуется.

---

### 5. Проверка XFS без внесения изменений

**Задача:** проверить XFS-файловую систему на ошибки без реального ремонта.

**Ответ:**

```bash
xfs_repair -n /dev/sdb1
```

Параметр `-n` — no modify: ФС проверяется, ошибки выводятся, на диск ничего не пишется.

## Исследовательские упражнения (Explorational)

### 1. Эффект `tune2fs -c 9` при mount count = 8

**Задача:** ext4-ФС на `/dev/sda1`. По выводу `tune2fs -l`:

```
Mount count: 8
Maximum mount count: -1
```

Что произойдёт на следующей загрузке, если выполнить `tune2fs -c 9 /dev/sda1`?

**Ответ:** команда установит максимум монтирований в 9. Поскольку текущее значение — 8, на следующей загрузке счётчик станет 9, достигнет лимита, и система автоматически запустит проверку ФС.

Изначальное значение `-1` означало, что проверка по счётчику была отключена.

---

### 2. Разбор вывода `du -h` и расчёт размера файлов

**Задача:** вывод команды `du -h`:

```
216K ./somedir/anotherdir
224K ./somedir
232K .
```

Сколько места занимают только файлы в текущем каталоге? Как переписать команду, чтобы это было видно явно?

**Ответ:** из суммарных 232 КБ подкаталог `somedir` со всем содержимым весит 224 КБ. В текущем каталоге без подкаталогов: 232 − 224 = **8 КБ**.

Чтобы видеть это явно:

```bash
du -Sh
```

Параметр `-S` разделяет каталоги в учёте — строка для текущего каталога покажет только его собственные файлы. Дополнительно `-d 0` уберёт строки про подкаталоги.

---

### 3. Эффект `tune2fs -j -J device= -i 30d` на ext2

**Задача:** что произойдёт с ext2-ФС `/dev/sdb1`, если выполнить:

```bash
tune2fs -j /dev/sdb1 -J device=/dev/sdc1 -i 30d
```

**Ответ:** к `/dev/sdb1` добавится журнал — ФС конвертируется в ext3. Журнал будет храниться на отдельном устройстве `/dev/sdc1`. Автоматическая проверка ФС будет запускаться раз в 30 дней.

- `-j` — создаёт журнал, поднимает версию до ext3
- `-J device=` — размещает журнал на отдельном диске
- `-i 30d` — задаёт интервал автоматической проверки

---

### 4. Проверка XFS с внешним log section без правок

**Задача:** проверить XFS `/dev/sda1`, у которой log section находится на `/dev/sdc1`, без реального ремонта.

**Ответ:**

```bash
xfs_repair -l /dev/sdc1 -n /dev/sda1
```

Параметр `-l` (строчная L) указывает устройство с внешним журналом XFS. Не путать с `-L` (заглавная), которая **обнуляет** повреждённый журнал.

Параметр `-n` — всегда безопасен, подходит для первичной диагностики.

---

### 5. Разница между `-T` и `-t` у df

**Задача:** в чём разница между параметрами `-T` и `-t` у команды `df`?

**Ответ:**

- `-T` (заглавная) — добавляет колонку с типом ФС в вывод. Все ФС остаются в выводе, просто появляется новая колонка.
- `-t TYPE` (строчная) — работает как **фильтр**: в выводе остаются только ФС указанного типа.

Аналогичная пара: `-x TYPE` исключает ФС указанного типа.

Способ запомнить: заглавная `-T` (Type) **показывает** тип, строчная `-t` **принимает** аргумент-тип и фильтрует.
