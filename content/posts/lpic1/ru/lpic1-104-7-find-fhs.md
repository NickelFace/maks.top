- Понимать правильное расположение файлов согласно FHS.

Утилиты и файлы: `find`, `locate`, `updatedb`, `whereis`, `which`, `type`, `/etc/updatedb.conf`.

## Стандарт FHS

FHS (Filesystem Hierarchy Standard) — инициатива Linux Foundation, задающая стандартную раскладку каталогов. Соответствие не обязательно, но почти все дистрибутивы его придерживаются.

Полная спецификация: http://refspecs.linuxfoundation.org/fhs.shtml

### Корневые каталоги

| Каталог | Назначение |
|---|---|
| `/` | Корневой — верхушка иерархии |
| `/bin` | Основные исполняемые файлы для всех пользователей |
| `/boot` | Файлы загрузки: ядро, initrd, GRUB |
| `/dev` | Файлы устройств (физических и виртуальных): `/dev/sda`, `/dev/null` |
| `/etc` | Конфигурационные файлы, специфичные для хоста |
| `/home` | Домашние каталоги пользователей (`/home/USER`) |
| `/lib` | Разделяемые библиотеки для `/bin` и `/sbin` |
| `/media` | Точки монтирования съёмных носителей (USB, CD, карты) |
| `/mnt` | Точка монтирования временно подключаемых ФС |
| `/opt` | Прикладные пакеты программ |
| `/proc` | Виртуальная ФС с данными о процессах и ядре |
| `/root` | Домашний каталог суперпользователя |
| `/run` | Переменные данные времени выполнения; очищается при загрузке |
| `/sbin` | Системные исполняемые файлы |
| `/srv` | Данные, отдаваемые системой (например, `/srv/www`) |
| `/tmp` | Временные файлы |
| `/usr` | Пользовательские данные только для чтения: утилиты, приложения |
| `/var` | Переменные данные: логи, почта, очереди печати |

**Для экзамена:** скомпилированные администратором бинарники для всех пользователей кладутся в `/usr/local/bin`.

### Временные файлы

FHS 3.0 определяет три места под временные файлы:

| Каталог | Очищается при загрузке? | Назначение |
|---|---|---|
| `/tmp` | рекомендуется | Временные данные программ; могут не пережить перезапуск |
| `/var/tmp` | нет | Временные файлы, которые должны пережить перезагрузку |
| `/run` | да | Данные времени выполнения (`.pid`-файлы); старые системы используют `/var/run` → симлинк на `/run` |

## Команда find

`find` рекурсивно обходит каталоги и проверяет файлы по заданным условиям.

### Базовый синтаксис

```
find STARTING_PATH OPTIONS EXPRESSION
```

```bash
find . -name '*.jpg'
```

Шаблон всегда оборачивай в одинарные кавычки — иначе шелл раскроет его до запуска `find`.

### Поиск по имени

| Параметр | Что делает |
|---|---|
| `-name PATTERN` | поиск по имени, регистр важен |
| `-iname PATTERN` | поиск по имени, регистр не важен |
| `-not EXPR` | инвертирует условие |

`*.jpg` — только имена, оканчивающиеся на `.jpg`; `*.jpg*` — имена, где `.jpg` встречается в любом месте.

### Глубина и границы ФС

| Параметр | Что делает |
|---|---|
| `-maxdepth N` | не глубже N уровней (текущий каталог = уровень 1) |
| `-mindepth N` | начинать с уровня N |
| `-mount` | не пересекать границы файловых систем |
| `-fstype TYPE` | поиск только на ФС указанного типа |

```bash
find /mnt -fstype exfat -iname "*report*"
```

### Поиск по атрибутам

| Параметр | Что делает |
|---|---|
| `-user NAME` | файлы пользователя |
| `-group NAME` | файлы группы |
| `-readable` / `-writable` / `-executable` | доступны текущему пользователю |
| `-perm NNNN` | точное совпадение прав |
| `-perm -NNNN` | права включают как минимум эти биты |
| `-empty` | пустые файлы и каталоги |
| `-size N[cKMG]` | по размеру; `c`=байты, `K`=КиБ, `M`=МиБ, `G`=ГиБ; `+`/`-` — больше/меньше |
| `-type X` | `f`=файл, `d`=каталог, `l`=симлинк, `b`=блочное, `c`=символьное, `s`=сокет, `p`=канал |

```bash
find ~ -iname "*report*" -perm 0644 -atime 10 -size +1M
```

### Поиск по времени

| Параметр | Что делает |
|---|---|
| `-amin N` / `-cmin N` / `-mmin N` | обращение / смена атрибутов / изменение содержимого N минут назад |
| `-atime N` / `-ctime N` / `-mtime N` | то же, в единицах N×24 часа |

`-cmin` и `-ctime` срабатывают на любое изменение метаданных, включая смену прав.

Префикс `+` означает «больше N», `-` — «меньше N»:

```bash
find . -mtime -1 -size +100M    # изменено менее суток назад, больше 100 МБ
```

### Действия над результатом

| Параметр | Что делает |
|---|---|
| `-exec CMD {} \;` | запустить команду для каждого файла; `{}` подставляется путь |
| `-delete` | удалить найденные файлы |
| `-print0` | разделитель NUL для безопасной передачи в `xargs -0` |

```bash
find . -name "*.conf" -exec chmod 644 {} \;
find . -name "*.bak" -delete
```

## locate и updatedb

`locate` ищет в заранее собранной базе данных, а не сканирует диск — мгновенные результаты, но могут быть устаревшими.

База: `/var/lib/mlocate.db`.

### Устаревание базы

```bash
locate jpg
# /home/carol/Downloads/Expert.jpg
# /home/carol/Downloads/jpg_specs.doc     ← совпадает, потому что содержит "jpg"
```

Обновить базу вручную:

```bash
sudo updatedb
```

### Параметры locate

| Параметр | Что делает |
|---|---|
| `-i` | без учёта регистра |
| `-A "p1" "p2"` | совпадение со ВСЕМИ шаблонами (по умолчанию — с любым) |
| `-c` | подсчитать совпадения вместо вывода путей |
| `-e` | выводить только реально существующие файлы |
| `-r` / `--regex` | поиск по регулярному выражению |

### /etc/updatedb.conf

| Переменная | Что исключает |
|---|---|
| `PRUNEFS=` | типы ФС (через пробел, регистр не важен) |
| `PRUNENAMES=` | имена каталогов |
| `PRUNEPATHS=` | абсолютные пути |
| `PRUNE_BIND_MOUNTS=yes\|no` | bind-монтирования |

## Поиск исполняемых файлов и справки

### which

Показывает полный путь к исполняемому файлу через `PATH`.

```bash
which bash        # /usr/bin/bash
which -a mkfs     # все совпадения в PATH
```

### type

Встроенная команда шелла. Сообщает, как шелл интерпретирует имя — различает алиасы, функции, встроенные команды и файлы.

```bash
type locate        # locate is /usr/bin/locate
type -a locate     # все совпадения
type -t locate     # "file"
type -t ll         # "alias"
type -t cd         # "builtin"
```

Используй `type`, когда экзамен спрашивает «как шелл обработает эту команду».

### whereis

Ищет бинарник, страницу справки и исходный код программы.

```bash
whereis locate
# locate: /usr/bin/locate /usr/share/man/man1/locate.1.gz
```

| Параметр | Что делает |
|---|---|
| `-b` | только бинарник |
| `-m` | только man-страница |
| `-s` | только исходный код |

## Шпаргалка

```bash
# Поиск по имени
find PATH -name "PATTERN"
find PATH -iname "PATTERN"           # без учёта регистра
find PATH -not -name "PATTERN"

# По типу и атрибутам
find PATH -type f                    # только файлы
find PATH -type d                    # только каталоги
find PATH -size +10M                 # больше 10 МиБ
find PATH -empty                     # пустые файлы/каталоги
find PATH -perm 0644                 # точные права
find PATH -perm -644                 # не меньше 0644
find PATH -user NAME -group NAME
find PATH -writable

# По времени
find PATH -mtime -7                  # изменено за последние 7 дней
find PATH -mmin -30                  # изменено за последние 30 минут

# Ограничение области
find PATH -maxdepth 2
find PATH -mount                     # не выходить за пределы ФС
find PATH -fstype ext4

# Действия
find PATH -name "*.bak" -exec rm {} \;
find PATH -name "*.bak" -delete

# locate
locate PATTERN
locate -i PATTERN
locate -A "p1" "p2"
locate -c PATTERN                    # счётчик
locate -e PATTERN                    # только существующие
locate -r 'REGEX'
sudo updatedb

# Поиск команд
which COMMAND
which -a COMMAND
type COMMAND
type -a COMMAND
type -t COMMAND                      # alias|keyword|function|builtin|file
whereis COMMAND
whereis -b COMMAND
whereis -m COMMAND
```

## Типичные вопросы экзамена

1. Куда положить скомпилированные администратором бинарники для всех пользователей? → **`/usr/local/bin`**.
2. Какой временный каталог очищается при загрузке? → **`/run`** (в старых системах `/var/run`, теперь симлинк).
3. Куда монтируются съёмные носители? → **`/media`**. Ручные временные монтирования — **`/mnt`**.
4. Какая команда показывает, как шелл интерпретирует имя (алиас/встроенная/файл)? → **`type`**.
5. Чем `which` отличается от `type`? → `which` ищет только в `PATH`; `type` различает алиасы, функции и встроенные команды.
6. Что показывает `whereis` в отличие от `which`? → Man-страницы и исходный код.
7. Что делает `-maxdepth N` у `find`? → Ограничивает поиск N уровнями вглубь; текущий каталог = уровень 1.
8. Чем `-perm NNNN` отличается от `-perm -NNNN`? → Без минуса — точное совпадение; с минусом — права включают как минимум эти биты.
9. `-mtime` vs `-mmin`? → `-mtime` в единицах 24 часа; `-mmin` в минутах.
10. Как исключить тип ФС из индексации `updatedb`? → Добавить в **`PRUNEFS=`** в `/etc/updatedb.conf`.
11. Как исключить путь из `updatedb`? → Добавить в **`PRUNEPATHS=`**.
12. Как исключить имя каталога из `updatedb`? → Добавить в **`PRUNENAMES=`**.
13. Как отключить индексацию bind-монтирований? → Установить **`PRUNE_BIND_MOUNTS=yes`**.
14. `find -name` использует glob или regex? → **Glob** (`*`, `?`, `[]`). Для regex — `find -regex`.
15. Как включить regex в `locate`? → Опция **`-r`**.

## Упражнения

### Упражнение 1 — Куда положить одноразовый временный файл

Программе нужен одноразовый временный файл, который не понадобится после её завершения. Какой каталог по FHS подходит для такого файла?

<details>
<summary>Ответ</summary>

`/tmp`. Раз судьба файла после завершения программы не важна, подходит именно этот каталог.

`/tmp` может очищаться при загрузке системы — для одноразового файла это безвредно. Для файлов, которые должны пережить перезагрузку, используется `/var/tmp`.

</details>

---

### Упражнение 2 — Какой временный каталог очищается при загрузке

Какой временный каталог обязан очищаться во время загрузки системы?

<details>
<summary>Ответ</summary>

**`/run`**, в некоторых системах `/var/run` (теперь симлинк на `/run`).

Для `/tmp` очистка при загрузке только рекомендуется, у `/run` это требование стандарта.

</details>

---

### Упражнение 3 — find с фильтрами по правам, времени и размеру

Найти в текущем каталоге файлы, доступные текущему пользователю на запись, изменённые за последние 10 дней и крупнее 4 GiB.

<details>
<summary>Ответ</summary>

```bash
find . -writable -mtime -10 -size +4G
```

`-writable` проверяет право записи для текущего пользователя. `-mtime -10` — изменено не более 10 суток назад. `-size +4G` — строго больше 4 ГиБ.

</details>

---

### Упражнение 4 — locate с несколькими шаблонами одновременно

Найти через `locate` файлы, в имени которых есть подстрока `report` и одна из: `update`, `updated`, `updating`.

<details>
<summary>Ответ</summary>

```bash
locate -A "report" "updat"
```

`-A` переключает `locate` на логическое И — должны совпасть все шаблоны. Шаблон `updat` покрывает все три варианта окончания.

Без `-A` `locate` с несколькими шаблонами использует ИЛИ — выводит файлы, совпадающие хотя бы с одним.

</details>

### Упражнение 5 — Где лежит man-страница для ifconfig

Как узнать путь к man-странице команды `ifconfig`?

<details>
<summary>Ответ</summary>

```bash
whereis -m ifconfig
```

`whereis` ищет бинарники, страницы справки и исходники. Опция `-m` сужает вывод только до man-страниц.

</details>

### Упражнение 6 — Исключить ntfs из индексации updatedb

Какую переменную добавить в `/etc/updatedb.conf`, чтобы `updatedb` пропускал файловые системы типа ntfs?

<details>
<summary>Ответ</summary>

```
PRUNEFS=ntfs
```

`PRUNEFS=` принимает список типов ФС через пробел, регистр не важен. Запись `PRUNEFS=ntfs vfat` тоже работает.

</details>

---

### Упражнение 7 — Куда монтировать внутренний диск по FHS

Системный администратор хочет смонтировать внутренний диск (`/dev/sdc1`). Куда по FHS?

<details>
<summary>Ответ</summary>

**`/mnt`** — FHS рекомендует этот каталог для временных ручных монтирований внутренних ФС.

`/media` предназначен для съёмных носителей (флешки, CD/DVD, карты памяти).

</details>

### Упражнение 8 — Как заставить locate показывать только существующие файлы

База `mlocate` обновляется не мгновенно, поэтому `locate` иногда выдаёт пути к уже удалённым файлам. Как показывать только реально существующие?

<details>
<summary>Ответ</summary>

```bash
locate -e PATTERN
```

`-e` проверяет существование файла в момент запуска `locate`. Файлы, удалённые позже, всё равно попадут в вывод до следующего обновления базы.

</details>

### Упражнение 9 — Поиск с ограничением глубины и одной ФС

Найти в текущем каталоге и подкаталогах не глубже двух уровней файлы, в имени которых есть подстрока `Status` или `statute`, без учёта регистра. Не заходить в смонтированные ФС.

<details>
<summary>Ответ</summary>

```bash
find . -maxdepth 3 -mount -iname "*statu*"
```

`Status` и `statute` оба содержат подстроку `statu`, поэтому `*statu*` покрывает оба варианта. `-mount` останавливает обход на границах ФС. Текущий каталог = уровень 1, поэтому «два уровня вниз» = `-maxdepth 3`.

</details>

---

### Упражнение 10 — Поиск с фильтрами по типу ФС, правам и изменению атрибутов

Найти под `/mnt` файлы: только на ext4-разделах, у которых есть как минимум право исполнения для группы и чтения для владельца, читаемые текущим пользователем, и у которых менялись атрибуты за последние 2 часа.

<details>
<summary>Ответ</summary>

```bash
find /mnt -fstype ext4 -perm -410 -cmin -120
```

`-fstype ext4` ограничивает поиск. Октальная маска `410`: чтение владельца (`4`) + исполнение группы (`1`); минус-префикс — «не меньше этих битов». `-cmin -120` — изменение метаданных менее 120 минут назад.

</details>

---

### Упражнение 11 — Пустые файлы, изменённые давно, на заданной глубине

Найти пустые файлы, изменённые более 30 дней назад, расположенные минимум на два уровня ниже текущего каталога.

<details>
<summary>Ответ</summary>

```bash
find . -empty -mtime +30 -mindepth 3
```

`-empty` ловит пустые файлы и каталоги. `-mtime +30` — изменено более 30 дней назад. Текущий каталог = уровень 1, поэтому «минимум два уровня вниз» = `-mindepth 3`.

Уточнение: `-ctime` в `find` означает change time inode (изменение метаданных), а не creation time. Время создания (`birth time`) доступно только через `stat`, но не через стандартный `find`.

</details>

---

### Упражнение 12 — Файлы в чужом домашнем каталоге, доступные через общую группу

Пользователи `carol` и `john` входят в одну группу `mkt`. Найти в домашнем каталоге `john` файлы, которые `carol` может прочитать через общую группу.

<details>
<summary>Ответ</summary>

```bash
find /home/john -perm -040
```

Для чтения `carol` через группу нужен бит group-read. Октальный код `040` = чтение для группы. Минус-префикс означает «не меньше этих битов» — другие биты могут быть любыми.

</details>
