## Архитектура X Window System

*X Window System* (X11, или просто X) представляет собой программный стек для отображения текста и графики на экране. Он разделён на **клиент** и **сервер**:

- **X-сервер** управляет физическим дисплеем и устройствами ввода (мышь, клавиатура). Он рисует на экране всё, что запрашивают клиенты.
- **X-клиент** — любое графическое приложение: терминал, браузер, игра. Каждый клиент сообщает серверу о местоположении и содержимом своего окна.

X-протокол (X11) является механизмом связи между клиентом и сервером. Основная реализация в современных дистрибутивах Linux — это **Xorg** версии 11.

X **поддерживает сеть**: несколько X-клиентов с разных компьютеров могут обращаться к одному удалённому X-серверу, позволяя запускать графические приложения удалённо с отображением локально.

X является **модульной** системой: новые возможности добавляются как **расширения** без изменения ядра протокола X11. Примеры библиотек расширений Xorg: `libXrandr`, `libXcursor`, `libX11`, `libxkbfile`.

**Менеджер дисплея** предоставляет графический экран входа в систему. Запускается при старте, инициирует X-сессию для аутентифицированного пользователя и поддерживает работу X-сервера. Распространённые менеджеры дисплея: **GDM**, **SDDM**, **LightDM**.

## Переменная DISPLAY

Каждый запущенный экземпляр X-сервера имеет *имя дисплея*:

```
hostname:displaynumber.screennumber
```

- **hostname** — система, на которой отображается приложение (для localhost опускается).
- **displaynumber** — идентифицирует набор экранов сессии (начинается с `0`).
- **screennumber** — номер отдельного экрана внутри дисплея (по умолчанию `0`, можно опустить).

Имя дисплея текущей X-сессии хранится в переменной окружения `DISPLAY`:

```bash
$ echo $DISPLAY
:0
```

Вывод означает:
1. X-сервер работает локально (слева от двоеточия ничего нет).
2. Первая X-сессия (дисплей `0`).
3. Один логический экран (номер экрана не указан).

**Примеры конфигураций с несколькими мониторами:**

- **(A)** Один монитор, один дисплей `:0`, один экран.
- **(B)** Два физических монитора как один логический экран `:0` — окна свободно перемещаются между мониторами.
- **(C)** Два монитора как независимые экраны `:0.0` и `:0.1` — окна нельзя перемещать между ними.

Запуск приложения на конкретном экране:

```bash
$ DISPLAY=:0.1 firefox &
```

## Конфигурация X-сервера

Основным конфигурационным файлом является `/etc/X11/xorg.conf`. На современных дистрибутивах X настраивается автоматически при запуске, поэтому этот файл может отсутствовать. Файл разделён на **секции**: `Section "имя"` … `EndSection`.

### InputDevice

Настройка конкретной модели клавиатуры или мыши.

### InputClass

Настройка *класса* устройств (например, всех клавиатур). Обычно размещается в `/etc/X11/xorg.conf.d/`. Пример `/etc/X11/xorg.conf.d/00-keyboard.conf`:

```
Section "InputClass"
        Identifier "system-keyboard"
        MatchIsKeyboard "on"
        Option "XkbLayout" "us"
        Option "XkbModel" "pc105"
EndSection
```

`XkbLayout` задаёт раскладку (язык, Dvorak и т. д.), а `XkbModel` задаёт тип клавиатуры. Файлы раскладок находятся в `/usr/share/X11/xkb`.

### Monitor

Описывает физический монитор и порт подключения:

```
Section "Monitor"
        Identifier  "DP2"
        Option      "Primary" "true"
EndSection
```

### Device

Описывает видеокарту, драйвер ядра и адрес на материнской плате:

```
Section "Device"
        Identifier  "Device0"
        Driver      "i915"
        BusID       "PCI:0:2:0"
EndSection
```

### Screen

Связывает монитор и видеокарту:

```
Section "Screen"
        Identifier "Screen0"
        Device     "Device0"
        Monitor    "DP2"
EndSection
```

### ServerLayout

Объединяет все секции в единый интерфейс X Window System:

```
Section "ServerLayout"
    Identifier   "Layout-1"
    Screen       "Screen0" 0 0
    InputDevice  "mouse1" "CorePointer"
    InputDevice  "system-keyboard" "CoreKeyboard"
EndSection
```

**Приоритет файлов конфигурации:**
- `/etc/X11/xorg.conf.d/` — пользовательские drop-in файлы (обрабатываются **первыми**)
- `/usr/share/X11/xorg.conf.d/` — конфигурации от дистрибутива
- `/etc/X11/xorg.conf` — основной файл (обрабатывается после `.conf.d/`)

## Раскладка клавиатуры

**Только для текущей сессии** — изменить раскладку в работающей X-сессии:

```bash
$ setxkbmap -model chromebook -layout "gr(polytonic)"
```

`setxkbmap` использует расширение *X Keyboard Extension* (XKB). Изменение действует до конца сессии.

**Постоянно** — через `localectl` (systemd), который автоматически создаёт `/etc/X11/xorg.conf.d/00-keyboard.conf`:

```bash
$ localectl --no-convert set-x11-keymap "gr(polytonic)" chromebook
```

`--no-convert` предотвращает изменение консольной раскладки.

Команда `xdpyinfo` выводит информацию о работающем X-сервере: имя дисплея, версию, расширения, размеры экрана и другое.


## Создание базового файла конфигурации Xorg

Для генерации постоянного файла `/etc/X11/xorg.conf`:

```bash
$ sudo Xorg -configure
```

Если X-сессия уже запущена, указать другой дисплей:

```bash
$ sudo Xorg :1 -configure
```

Создаётся файл `xorg.conf.new` в текущей директории на основе обнаруженного оборудования. Переместить в итоговое место:

```bash
$ sudo mv xorg.conf.new /etc/X11/xorg.conf
```

На некоторых дистрибутивах `X` является символической ссылкой на `Xorg`.

Связанные man-страницы: `xorg.conf(5)`, `Xserver(1)`, `X(1)`, `Xorg(1)`.

## Wayland

**Wayland** является новым протоколом отображения, разработанным на замену X. Многие современные дистрибутивы используют его по умолчанию. Wayland легче на ресурсы и имеет меньший размер установки.

Ключевые отличия от X:
- **Нет отдельного сервера** — между клиентом и ядром нет X-сервера.
- Клиентское окно рендерит само (через GTK+ 3, Qt 5 и т. д.) и отправляет запросы ядру через протокол Wayland.
- **Wayland-композитор** обрабатывает ввод устройств, управление окнами и компоновку (объединяет отрендеренные элементы в изображение на экране).

Современные тулкиты (GTK+ 3, Qt 5) поддерживают и X, и Wayland. Приложения, написанные только под X, можно запустить через **XWayland**, представляющий собой отдельный X-сервер внутри Wayland-клиента.

Аналогом переменной `DISPLAY` для Wayland служит `WAYLAND_DISPLAY`:

```bash
$ echo $WAYLAND_DISPLAY
wayland-0
```

Эта переменная не установлена в системах, работающих на чистом X.

## Полезные команды для экзамена

```bash
# Проверить текущий дисплей
echo $DISPLAY          # :0  (локальный, первая сессия, один экран)
                       # lab01:0.2  (хост lab01, дисплей 0, третий независимый экран)
echo $WAYLAND_DISPLAY  # wayland-0  (только в системах с Wayland)

# Запустить приложение на конкретном экране
DISPLAY=:0.1 firefox &

# Показать информацию о X-сервере (расширения, размер экрана, версия)
xdpyinfo

# Сгенерировать xorg.conf из обнаруженного оборудования
sudo Xorg -configure                      # создаёт xorg.conf.new в текущей директории
sudo Xorg :1 -configure                   # если X-сессия уже запущена
sudo mv xorg.conf.new /etc/X11/xorg.conf

# Раскладка клавиатуры — только для текущей сессии (использует расширение XKB)
setxkbmap -model pc105 -layout us
setxkbmap -model chromebook -layout "gr(polytonic)"

# Раскладка клавиатуры — постоянно (создаёт /etc/X11/xorg.conf.d/00-keyboard.conf)
localectl --no-convert set-x11-keymap us pc105
localectl --no-convert set-x11-keymap "gr(polytonic)" chromebook
# --no-convert: не изменять консольную раскладку

# Порядок обработки файлов конфигурации
# 1-й: /etc/X11/xorg.conf.d/       (пользовательские drop-in файлы)
# 2-й: /usr/share/X11/xorg.conf.d/ (файлы дистрибутива)
# 3-й: /etc/X11/xorg.conf           (основной файл)

# Секции xorg.conf
Section "InputDevice"    # конкретная клавиатура или мышь
Section "InputClass"     # класс устройств (например, все клавиатуры)
Section "Monitor"        # физический монитор и порт
Section "Device"         # видеокарта, драйвер, BusID
Section "Screen"         # связывает Monitor + Device
Section "ServerLayout"   # объединяет все секции в один интерфейс

# Связанные man-страницы
man xorg.conf    # xorg.conf(5)
man Xserver      # Xserver(1)
man Xorg         # Xorg(1)
```

## Типичные вопросы экзамена

1. Каков формат переменной DISPLAY? → `hostname:displaynumber.screennumber`
2. Что означает `:0` в DISPLAY? → Локальный X-сервер, первая сессия, один логический экран.
3. Что означает `lab01:0.2`? → Хост `lab01`, дисплей `0`, третий независимый экран (индекс `2`).
4. Как запустить приложение на втором независимом экране? → `DISPLAY=:0.1 app &`
5. Какая команда показывает информацию X-сервера, включая расширения? → `xdpyinfo`
6. Какая секция xorg.conf настраивает *класс* устройств ввода? → `InputClass`
7. Какая секция настраивает *конкретную* модель клавиатуры или мыши? → `InputDevice`
8. Какая секция связывает Monitor и Device? → `Screen`
9. Какая секция объединяет все секции в один интерфейс? → `ServerLayout`
10. Куда помещать пользовательские drop-in файлы конфигурации Xorg? → `/etc/X11/xorg.conf.d/`
11. Что обрабатывается первым — `xorg.conf` или `xorg.conf.d/`? → `/etc/X11/xorg.conf.d/` обрабатывается первым.
12. Как изменить раскладку только для текущей X-сессии? → `setxkbmap -layout us`
13. Как постоянно задать раскладку X11? → `localectl --no-convert set-x11-keymap`
14. Что делает `--no-convert` в `localectl`? → Предотвращает одновременное изменение консольной раскладки.
15. Какая команда генерирует новый `xorg.conf` из обнаруженного оборудования? → `sudo Xorg -configure`
16. Какой файл создаёт `Xorg -configure` и где? → `xorg.conf.new` в текущей рабочей директории.
17. Какой компонент отвечает за поддержание работы X-сервера после входа в систему? → Менеджер дисплея (GDM, SDDM, LightDM).
18. Что такое XWayland? → Отдельный X-сервер, работающий внутри Wayland-клиента для запуска X-только приложений.
19. Какая переменная окружения идентифицирует дисплей в системе Wayland? → `WAYLAND_DISPLAY`
20. Какие библиотеки Xorg расширяют функциональность X-сервера? → `libXrandr`, `libXcursor`, `libX11`, `libxkbfile`.

## Упражнения

### Практическое упражнение 1 — Определение расширений Xorg

Какую команду следует использовать для определения, какие расширения Xorg доступны в системе?

<details>
<summary>Ответ</summary>

```bash
$ xdpyinfo
```

`xdpyinfo` выводит информацию о работающем X-сервере, включая полный список загруженных расширений, имя дисплея, версию сервера и параметры экрана.

</details>

### Практическое упражнение 2 — Настройка новой мыши

Получена новая мышь с 10 кнопками, требующая дополнительной конфигурации. Без изменения остальной конфигурации X-сервера — в какой директории создать файл конфигурации и какую секцию в нём использовать?

<details>
<summary>Ответ</summary>

Директория: `/etc/X11/xorg.conf.d/`

Секция: `InputDevice`

Секция `InputDevice` настраивает конкретное аппаратное устройство. Размещение файла в `/etc/X11/xorg.conf.d/` позволяет не трогать основной `xorg.conf`, при этом drop-in файл обрабатывается первым.

</details>

### Практическое упражнение 3 — Что поддерживает работу X?

Какой компонент Linux отвечает за поддержание работы X-сервера?

<details>
<summary>Ответ</summary>

**Менеджер дисплея** (например, GDM, SDDM, LightDM). Он запускается при загрузке системы, инициирует X-сессию для аутентифицированного пользователя и поддерживает её работу.

</details>

### Практическое упражнение 4 — Генерация xorg.conf

Какой ключ командной строки используется с командой `X` (или `Xorg`) для создания нового файла конфигурации `xorg.conf`?

<details>
<summary>Ответ</summary>

`-configure`

```bash
$ sudo Xorg -configure
```

`X` во многих дистрибутивах является символической ссылкой на `Xorg`. Команда создаёт `xorg.conf.new` в текущей директории на основе обнаруженного оборудования. Для активации переместить в `/etc/X11/xorg.conf`.

</details>

### Исследовательское упражнение 2 — Постоянная конфигурация клавиатуры

Какую команду можно использовать для создания постоянного файла конфигурации клавиатуры для X Window System?
