## Что нужно знать

Согласно официальным целям LPIC-1:

- Создавать псевдонимы электронной почты.
- Настраивать переадресацию почты.
- Знать популярные программы MTA (postfix, sendmail, exim).

Ключевые файлы и команды: `~/.forward`, `/etc/aliases`, `sendmail`, `mailq`, `mail`, `postfix`, `sendmail`, `exim`, `newaliases`.

---

## Концепции почты

**MTA (Mail Transfer Agent)** — агент пересылки почты — маршрутизирует электронную почту между серверами по протоколу SMTP (Simple Mail Transfer Protocol) на порту 25. При поступлении сообщения MTA доставляет его в локальный **почтовый ящик** пользователя: `/var/spool/mail/имя_пользователя`.

DNS-записи **MX (Mail Exchanger)** указывают на почтовый сервер домена.

### Популярные MTA

| MTA | Описание |
|---|---|
| Sendmail | Классический Unix MTA; мощный, но сложная конфигурация |
| Postfix | Замена Sendmail; самый распространённый сегодня |
| Exim | MTA по умолчанию в Debian |
| qmail | Безопасная архитектура; не устанавливается по умолчанию |

Postfix предоставляет обёртку совместимости с `sendmail`, поэтому скрипты, использующие `sendmail`, продолжают работать без изменений.

## Команда sendmail

Команда `sendmail` (доступная во всех MTA как интерфейс совместимости) отправляет письмо из командной строки. Тело письма читается из stdin; завершается строкой, содержащей только `.`:

```bash
$ sendmail user@example.com
Subject: Тест

Привет, это тестовое письмо.
.
```

Полезные флаги:

| Флаг | Описание |
|---|---|
| `-bp` | Показать очередь почты (аналог `mailq`) |
| `-q` | Сбросить очередь (попытаться доставить сообщения из очереди) |
| `-bi` / `-I` | Перестроить базу данных псевдонимов (аналог `newaliases`) |

---

## Команда mail (MUA)

`mail` — простой почтовый клиент (Mail User Agent) для чтения и отправки почты из терминала.

### Режим отправки

```bash
mail -s "Тема" получатель@example.com
```

Команда читает тело письма из stdin.

### Режим чтения

Запуск `mail` без аргументов открывает почтовый ящик и показывает список писем. Команды в режиме чтения:

| Команда | Действие |
|---|---|
| `N` (номер) | Прочитать письмо N |
| `d N` | Удалить письмо N |
| `r N` | Ответить на письмо N |
| `q` | Выйти и сохранить изменения |

## Очередь почты

Если сообщение не удалось доставить немедленно, MTA помещает его в очередь. Расположение очереди:

- Sendmail: `/var/spool/mqueue/`
- Postfix: `/var/spool/postfix/`

### Команды управления очередью

```bash
mailq              # показать сообщения в очереди
sendmail -bp       # аналог mailq
sendmail -q        # принудительная повторная попытка доставки
```

## /etc/aliases — общесистемные псевдонимы

В файле `/etc/aliases` задаются псевдонимы электронной почты для системы. Псевдонимы сопоставляются с одним или несколькими адресатами:

```
# Формат: псевдоним: адресат
postmaster:   root
root:         admin@example.com
helpdesk:     alice, bob, charlie
bugs:         /var/log/bugs.txt
complaints:   |/usr/local/bin/ticket-handler
```

### Типы адресатов

| Адресат | Описание |
|---|---|
| `имя_пользователя` | Локальный пользователь |
| `user@domain` | Удалённый адрес электронной почты |
| `user1, user2` | Список через запятую (список рассылки) |
| `/путь/к/файлу` | Добавить письмо в файл |
| `\|/путь/к/команде` | Передать тело письма программе |
| `:include:/путь/к/файлу` | Читать адресатов из файла |

После изменения `/etc/aliases` нужно перестроить бинарную базу данных псевдонимов:

```bash
newaliases          # предпочтительная команда
sendmail -bi        # аналог
sendmail -I         # аналог
```

Скомпилированная база данных хранится в `/etc/aliases.db`.

## ~/.forward — пользовательская переадресация

Каждый пользователь может создать файл `~/.forward` для переадресации почты без участия администратора и без необходимости запускать `newaliases`.

```
# Переадресовать на другой адрес
user@otherdomain.com

# Доставить локально И переадресовать
\имя_пользователя, user@otherdomain.com

# Передать программе
|/usr/bin/vacation
```

Правила для `~/.forward`:

- `newaliases` не требуется — MTA читает файл напрямую.
- Файл должен принадлежать пользователю и не иметь прав на запись для группы и остальных.
- Синтаксис `\имя_пользователя` с обратной косой чертой доставляет копию локально, предотвращая зацикливание переадресации.

## Полезные команды для экзамена

```
MTA = агент пересылки почты
  Протокол SMTP, порт 25
  Локальный почтовый ящик: /var/spool/mail/имя_пользователя
  Популярные MTA: Postfix (наиболее распространён), Sendmail (классический),
                  Exim (по умолчанию в Debian), qmail

sendmail:
  sendmail addr         отправить письмо (тело из stdin, завершить строкой .)
  sendmail -bp          показать очередь почты
  sendmail -q           сбросить/повторить очередь
  sendmail -bi / -I     перестроить базу псевдонимов (= newaliases)

Команда mail:
  mail -s "Тема" addr    отправить письмо (тело из stdin)
  mail                   читать почтовый ящик

Очередь почты:
  mailq                 показать сообщения в очереди
  sendmail -q           принудительная повторная попытка

/etc/aliases:
  псевдоним: адресат(ы)
  Адресаты: имя_пользователя, user@domain, список через запятую,
            /файл, |команда, :include:/файл
  После изменений: выполнить newaliases

~/.forward:
  user@domain           переадресовать на адрес
  \имя_пользователя     сохранить локальную копию (предотвращает зацикливание)
  |/путь/к/команде      передать программе
  newaliases не нужен; файл не должен быть доступен на запись группе и остальным
```

## Типичные вопросы экзамена

1. Какой протокол использует MTA для маршрутизации почты? → SMTP на порту 25.
2. Где хранится локальная почта пользователя? → `/var/spool/mail/имя_пользователя`
3. Какой тип DNS-записи направляет почту на почтовый сервер? → Запись MX (Mail Exchanger).
4. Какой MTA наиболее распространён сегодня и служит заменой Sendmail? → Postfix.
5. Какой MTA используется по умолчанию в Debian? → Exim.
6. Как отправить письмо через `sendmail` из командной строки? → Запустить `sendmail user@domain`, ввести тело письма, затем завершить строкой, содержащей только `.`.
7. Что делает `sendmail -bp`? → Показывает очередь почты (аналог `mailq`).
8. Что делает `sendmail -q`? → Заставляет MTA повторить попытку доставки всех сообщений из очереди.
9. Как отправить письмо с темой через команду `mail`? → `mail -s "Тема" получатель@example.com`
10. В каком файле задаются общесистемные псевдонимы электронной почты? → `/etc/aliases`
11. Какую команду нужно выполнить после изменения `/etc/aliases`? → `newaliases` (или `sendmail -bi` / `sendmail -I`).
12. В каком файле хранится скомпилированная база данных псевдонимов? → `/etc/aliases.db`
13. Что означает адресат `|/путь/к/скрипту` в `/etc/aliases`? → Тело письма передаётся на стандартный ввод указанной программы.
14. Для чего служит `~/.forward`? → Позволяет пользователю настроить переадресацию входящей почты без участия администратора.
15. Нужно ли запускать `newaliases` после изменения `~/.forward`? → Нет, MTA читает этот файл напрямую.
16. Что означает `\имя_пользователя` в `~/.forward`? → Доставить локальную копию письма этому пользователю (обратная косая черта предотвращает зацикливание переадресации).
17. Каким требованиям должны отвечать права доступа к `~/.forward`? → Файл должен принадлежать пользователю и не иметь прав на запись для группы и остальных.
18. Где Postfix хранит очередь почты? → `/var/spool/postfix/`

## Упражнения

### Практическое упражнение 1 — Системный псевдоним

Добавьте псевдоним `webmaster`, чтобы письма на этот адрес доставлялись `alice` и `bob@remote.example.com`. Какая команда перестраивает базу данных?

<details>
<summary>Ответ</summary>

В `/etc/aliases`:

```
webmaster:   alice, bob@remote.example.com
```

Затем:

```bash
newaliases
```

</details>

---

### Практическое упражнение 2 — Доставка в файл

Создайте псевдоним `bugs`, который добавляет входящие письма в файл `/var/log/bugs.txt`.

<details>
<summary>Ответ</summary>

В `/etc/aliases`:

```
bugs:   /var/log/bugs.txt
```

После изменений запустить `newaliases`.

</details>

---

### Практическое упражнение 3 — Пользовательская переадресация

Пользователь `carol` хочет переадресовывать всю входящую почту на `carol@gmail.com`, сохраняя локальную копию. Напишите содержимое её `~/.forward`.

<details>
<summary>Ответ</summary>

```
\carol, carol@gmail.com
```

`\carol` обеспечивает локальную доставку; `carol@gmail.com` — внешнюю переадресацию. Обратная косая черта предотвращает бесконечный цикл переадресации.

</details>
