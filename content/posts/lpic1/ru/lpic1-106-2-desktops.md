## Что нужно знать

Согласно официальным целям LPIC-1:

- Знакомство с основными рабочими средами.

Ключевые термины: KDE, Gnome, Xfce, X11, XDMCP, VNC, Spice, RDP.

## Рабочая среда

X Window System предоставляет только низкоуровневые механизмы: обработку событий ввода, буфер обмена между приложениями и программный интерфейс для рисования. Сложные визуальные эффекты — цвета, формы, анимацию — генерируют сами приложения.

**Рабочая среда** (desktop environment) расширяет X, объединяя библиотеки, интерфейсы и вспомогательные программы для целостного графического опыта. Ключевые компоненты:

- **Менеджер окон** — управляет расположением окон, заголовками, кнопками сворачивания/разворачивания/закрытия и переключением между окнами.
- **Тулкит виджетов** — предоставляет переиспользуемые визуальные элементы (кнопки, текстовые поля, диалоги). Распространённые тулкиты: **GTK+** и **Qt**.
- **Запуск приложений** — отображает список доступных программ.
- **Менеджер сессий** — управляет входом, выходом и работающими службами.

Рабочие среды также интегрируют сторонние службы, такие как PulseAudio (звук) и CUPS (печать), предоставляя для них графические интерфейсы управления.

## Популярные рабочие среды

Linux поддерживает несколько рабочих сред. Все разделяют общие концепции: запуск приложений, правила приложений по умолчанию для типов файлов и инструменты настройки.

### GNOME

Одна из самых популярных рабочих сред, используется по умолчанию в Fedora, Debian, Ubuntu, SUSE, RHEL, CentOS и других. GNOME 3 заменил традиционный лаунчер и панель задач на **Gnome Shell Activities** — полноэкранный запускатель. Классический вид доступен через опцию **Gnome Classic** на экране входа. GNOME использует тулкит **GTK+**.

### KDE Plasma

Крупная экосистема приложений и платформа разработки. По умолчанию в openSUSE, Mageia, Kubuntu и других. KDE построен на библиотеке **Qt**, что придаёт ему узнаваемый внешний вид. Включает инструмент для обеспечения визуальной согласованности с GTK+-приложениями.

### Xfce

Рабочая среда, ориентированная на эстетику при минимальном потреблении ресурсов. Модульная структура позволяет активировать или отключать компоненты по необходимости.

### Другие

- **LXDE** — оптимизирована для экономии ресурсов, подходит для старого оборудования и одноплатных компьютеров.
- **Cinnamon** — форк GNOME 3, разрабатывается командой Linux Mint.
- **MATE** — форк GNOME 2.

Приложения, написанные с разными тулкитами (GTK+ или Qt), могут работать одновременно вне зависимости от используемой рабочей среды.

## Совместимость рабочих сред

Разнообразие Linux-сред создаёт проблемы с кросс-средовой совместимостью. Организация **freedesktop.org** поддерживает набор общих спецификаций, частично или полностью принятых большинством сред:

| Спецификация | Назначение |
|---|---|
| Расположение директорий | Где хранятся личные настройки и пользовательские файлы |
| Записи рабочего стола | Файлы `.desktop` — описание приложений и способ их запуска |
| Автозапуск приложений | Файлы `.desktop` для программ, запускаемых при входе в систему |
| Перетаскивание (drag and drop) | Правила обработки drag-and-drop событий |
| Корзина | Общее расположение удалённых файлов и методы управления ими |
| Темы иконок | Общий формат взаимозаменяемых библиотек иконок |

## Удалённый доступ

X Window System поддерживает удалённые графические сессии через протокол **XDMCP** (X Display Manager Control Protocol). Менеджер X-дисплея может управлять несколькими сессиями одновременно. Возможные сценарии:

- Переключение между несколькими графическими сессиями на одной машине.
- Несколько наборов устройств ввода/вывода на одной машине, каждый со своей сессией.
- Удалённые графические сессии, передаваемые по сети.

### XDMCP

Нативный протокол удалённого рабочего стола X. Недостатки:
- Высокое потребление полосы пропускания: редко используется через интернет или медленные сети.
- Проблемы безопасности: удалённый менеджер дисплея выполняет привилегированные процедуры, уязвимость может позволить выполнение произвольных команд.
- Требует X на **обеих** сторонах соединения.

### VNC

*Virtual Network Computing* использует протокол **Remote Frame Buffer (RFB)**. События клавиатуры и мыши отправляются на удалённый рабочий стол, который возвращает обновления экрана.

Ключевые факты:
- Каждый VNC-сервер требует отдельного TCP-порта; по соглашению первый — **порт 5900**, второй — **5901** и т. д.
- Обычный пользователь может запустить собственный VNC-сервер без привилегий.
- `~/.vnc/xstartup` — скрипт, выполняемый VNC-сервером при запуске, определяет, какую рабочую среду предоставить клиенту.
- VNC не обеспечивает современное шифрование и аутентификацию нативно, поэтому рекомендуется защита через VPN или SSH-туннель.

### RDP

*Remote Desktop Protocol* (RDP) используется преимущественно для доступа к **Microsoft Windows** через **TCP-порт 3389**. Клиентские реализации для Linux реализуют его как открытый исходный код под лицензией GPL.

### Spice

*Simple Protocol for Independent Computing Environments* предназначен для **виртуализированных** систем. Помимо трансляции экрана, Spice обеспечивает нативную интеграцию: доступ к локальным устройствам (аудио, USB) с удалённой машины и обмен файлами между системами.

### Remmina

**Remmina** является графическим клиентом удалённого рабочего стола для Linux, объединяющим все четыре протокола (XDMCP, VNC, RDP, Spice) через систему плагинов. Сохраняет настройки соединений для повторного использования.

## Типичные вопросы экзамена

1. Какой тулкит использует KDE? → **Qt**
2. Какой тулкит использует GNOME? → **GTK+**
3. Что такое менеджер окон? → Компонент рабочей среды, управляющий расположением окон, заголовками и переключением между ними.
4. Что такое `.desktop` файлы? → Текстовые файлы с описанием приложений, используемые лаунчером и системой автозапуска.
5. Какая организация поддерживает спецификации межсредовой совместимости? → **freedesktop.org**
6. Что такое XDMCP? → X Display Manager Control Protocol, нативный протокол удалённого рабочего стола X. Требует X на обоих концах.
7. Почему XDMCP редко используется через интернет? → Высокое потребление полосы и проблемы безопасности (привилегированные удалённые процедуры).
8. Какой протокол использует VNC? → **Remote Frame Buffer (RFB)**
9. Какой TCP-порт по умолчанию использует первый VNC-сервер? → **5900** (второй — 5901 и т. д.)
10. Для чего нужен `~/.vnc/xstartup`? → Скрипт, запускаемый VNC-сервером при старте; определяет, какую рабочую среду предоставить клиенту.
11. Обеспечивает ли VNC шифрование? → Нет, нужна внешняя защита через VPN или SSH-туннель.
12. Какой TCP-порт использует RDP? → **3389**
13. Для каких систем предназначен Spice? → **Виртуализированных** систем.
14. Какие дополнительные возможности есть у Spice? → Доступ к локальным USB-устройствам и аудио с удалённой машины, обмен файлами.
15. Что такое Remmina? → Графический клиент удалённого рабочего стола для Linux с плагинами для XDMCP, VNC, RDP и Spice.
16. Какой протокол лучше всего подходит для подключения к Windows-машине? → **RDP**

## Упражнения

### Практическое упражнение 1 — Терминал в рабочей среде

Какой тип приложения предоставляет оконные сессии командной строки в рабочей среде?

<details>
<summary>Ответ</summary>

**Эмулятор терминала** — например Konsole (KDE), GNOME Terminal, xterm и другие. Даёт доступ к интерактивной командной строке в графическом окне.

</details>

---

### Практическое упражнение 2 — Совместимость тулкитов

Торрент-клиент *Transmission* существует в двух версиях: `transmission-gtk` и `transmission-qt`. Какую установить для максимальной интеграции с KDE?

<details>
<summary>Ответ</summary>

**transmission-qt** — KDE построен на библиотеке Qt, поэтому Qt-версия лучше всего интегрируется с его стилем виджетов и темами.

</details>

---

### Практическое упражнение 3 — Лёгкие рабочие среды

Какие Linux-среды рекомендуются для дешёвых одноплатных компьютеров с ограниченными ресурсами?

<details>
<summary>Ответ</summary>

**Xfce** и **LXDE** — обе лёгкие, не используют тяжёлые визуальные эффекты и подходят для старого оборудования и одноплатных компьютеров (например, Raspberry Pi).

</details>

---

### Исследовательское упражнение 1 — Копирование в терминале

Существует два способа копировать и вставлять текст в X Window System: `Ctrl+C` / `Ctrl+V` и щелчок средней кнопкой мыши для вставки выделенного текста. Какой способ подходит для эмулятора терминала?

<details>
<summary>Ответ</summary>

Метод **средней кнопки мыши**. В интерактивных сессиях `Ctrl+C` отправляет SIGINT (останавливает запущенную программу) и не может использоваться для копирования. Выделить текст левой кнопкой, вставить — средней.

</details>

---

### Исследовательское упражнение 2 — Удалённый Windows-рабочий стол

Какой протокол лучше всего подходит для доступа к удалённому рабочему столу Windows из Linux?

<details>
<summary>Ответ</summary>

**RDP** (Remote Desktop Protocol) нативно поддерживается Windows. Linux-клиенты с открытым исходным кодом под лицензией GPL реализуют его без правовых ограничений.

</details>
