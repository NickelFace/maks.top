## Что нужно знать

Согласно официальным целям LPIC-1:

- Диагностировать проблемы сетевой конфигурации на клиентских хостах.
- Использовать системные утилиты для выявления конфигурации сети и решения проблем.

Ключевые файлы и команды: `ip`, `ifconfig`, `route`, `ping`, `ping6`, `traceroute`, `traceroute6`, `tracepath`, `tracepath6`, `netcat`, `netstat`, `ss`.

---

## Настройка интерфейсов

### ip — современный инструмент

```bash
# Просмотр всех интерфейсов
ip addr show
ip addr show eth0

# Добавить / удалить IP-адрес
ip addr add 192.168.1.10/24 dev eth0
ip addr del 192.168.1.10/24 dev eth0

# Поднять / опустить интерфейс
ip link set eth0 up
ip link set eth0 down
```

### ifconfig — устаревший инструмент

```bash
ifconfig                               # показать активные интерфейсы
ifconfig -a                            # все, включая неактивные
ifconfig eth0 192.168.1.10 netmask 255.255.255.0
ifconfig eth0 down
ifconfig eth0 up
```

## Маршрутизация

### Просмотр таблицы маршрутов

```bash
ip route show           # современный вариант
route -n                # устаревший (числовой вывод)
netstat -r              # устаревший (таблица маршрутов)
```

### Управление маршрутами

```bash
# Добавить маршрут
ip route add 192.168.2.0/24 via 192.168.1.1
ip route add default via 192.168.1.1

# Удалить маршрут
ip route del 192.168.2.0/24

# Устаревшая команда route
route add -net 192.168.2.0/24 gw 192.168.1.1 dev eth0
route del -net 192.168.2.0/24
```

### Маршрутизация IPv6

```bash
ip -6 route show
ip -6 route add 2001:db8::/32 via fe80::1 dev eth0
```

## ping и ping6

`ping` отправляет ICMP-запросы эхо для проверки доступности хоста.

```bash
ping 8.8.8.8                # непрерывно
ping -c 4 8.8.8.8           # отправить ровно 4 пакета
ping6 ::1                   # IPv6-пинг
ping -6 ::1                 # альтернативный синтаксис IPv6
```

Причины, по которым ping может не пройти:
- Удалённый хост недоступен.
- Брандмауэр (локальный или удалённый) блокирует ICMP.
- Неправильное имя хоста или ошибка DNS.
- Некорректная локальная сетевая конфигурация.
- Интерфейс отключён.
- ACL маршрутизатора блокирует ICMP.

## traceroute и traceroute6

`traceroute` трассирует путь пакетов до адресата, отправляя пакеты с последовательно увеличивающимся TTL и фиксируя каждый маршрутизатор, который возвращает сообщение ICMP Time Exceeded.

```bash
traceroute 8.8.8.8
traceroute6 2001:4860:4860::8888
```

| Параметр | Описание |
|---|---|
| `-I` | Использовать ICMP-эхо вместо UDP |
| `-T` | Использовать TCP SYN |
| `-i ИНТЕРФЕЙС` | Использовать указанный сетевой интерфейс |
| `--mtu` | Отображать MTU на каждом переходе |

По умолчанию `traceroute` использует UDP-пакеты. Параметры `-I` и `-T` требуют прав root.

---

## tracepath и tracepath6

`tracepath` похож на `traceroute`, но не требует прав root и автоматически обнаруживает MTU пути (Path MTU Discovery).

```bash
tracepath 8.8.8.8
tracepath6 ::1
tracepath -6 ::1
```

В отличие от `traceroute`, `tracepath` сообщает MTU на каждом переходе и не требует повышенных привилегий.

## nc — netcat

`nc` создаёт произвольные TCP- или UDP-соединения для тестирования и передачи данных.

```bash
# Подключиться к TCP-порту
nc hostname 80

# Слушать на порту
nc -l 1234

# Режим UDP
nc -u hostname 53

# Проверить, открыт ли порт
nc -z hostname 22
```

### Ручной HTTP-запрос через nc

```bash
nc learning.lpi.org 80
GET /index.html HTTP/1.1
HOST: learning.lpi.org

```

(Нажать Enter дважды после последнего заголовка.)

## netstat и ss

Обе команды показывают открытые сетевые соединения, таблицы маршрутизации и статистику интерфейсов.

### ss — современный инструмент

```bash
ss -tulnp           # TCP и UDP прослушивающие, числовой, с именами процессов
ss -a               # все сокеты (прослушивающие и установленные)
ss -ln | grep ":80" # проверить, используется ли порт 80
```

### netstat — устаревший инструмент

```bash
netstat -tulnp     # TCP и UDP прослушивающие, числовой, с именами процессов
netstat -r         # таблица маршрутов
```

### Общие параметры (обе команды)

| Параметр | Описание |
|---|---|
| `-a` | Все сокеты (прослушивающие и подключённые) |
| `-l` | Только прослушивающие сокеты |
| `-t` | TCP-сокеты |
| `-u` | UDP-сокеты |
| `-n` | Числовые адреса (без обратного DNS) |
| `-p` | Имя и PID процесса |

Утилита `ss` является современной заменой `netstat`. Обе команды принимают одинаковые параметры.

## Типичные вопросы экзамена

1. Какая команда добавляет шлюз по умолчанию с помощью `ip`? → `ip route add default via ШЛЮЗ`
2. Какой параметр `ping` отправляет ровно 5 ICMP-запросов? → `-c 5`
3. Как называется IPv6-аналог `ping`? → `ping6` или `ping -6`
4. Какой протокол `traceroute` использует по умолчанию? → UDP
5. Какой параметр `traceroute` переключает зонды на ICMP? → `-I`
6. В чём главное преимущество `tracepath` перед `traceroute`? → `tracepath` не требует прав root и автоматически обнаруживает Path MTU.
7. Какой параметр `nc` включает режим UDP? → `-u`
8. Какая команда `nc` проверяет, открыт ли порт 443 на хосте? → `nc -z hostname 443`
9. Что заменяет `netstat` в современных системах? → `ss`
10. Какая команда `ss` показывает все TCP-прослушивающие сокеты с именами процессов? → `ss -tlnp`
11. Какая устаревшая команда показывает таблицу маршрутов? → `route -n` или `netstat -r`
12. Какая команда поднимает интерфейс `ens3` с помощью `ip`? → `ip link set ens3 up`
13. Какая команда удаляет адрес `10.0.0.5/24` с интерфейса `eth0`? → `ip addr del 10.0.0.5/24 dev eth0`
14. Какие инструменты позволяют перехватывать сетевые пакеты? → `tcpdump` и `wireshark`
15. Какой параметр `traceroute` заставляет использовать конкретный интерфейс? → `-i ИНТЕРФЕЙС`
16. Как просмотреть таблицу маршрутов IPv6? → `ip -6 route show`
17. Что делает `ss -ln | grep ":80"`? → Показывает, прослушивает ли какой-либо сокет порт 80.
18. Чем отличаются `traceroute` и `tracepath`? → `traceroute` гибче (выбор протокола, флаги), но некоторые режимы требуют root. `tracepath` не требует прав и автоматически определяет MTU.
