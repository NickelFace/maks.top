## Что нужно знать

Из официальных objectives курса LPIC-1:

- Применять стандартный синтаксис sh: циклы, проверки.
- Использовать подстановку команд.
- Проверять коды возврата команд для определения успеха или ошибки.
- Соединять команды в цепочки.
- Отправлять условные уведомления администратору root.
- Правильно выбирать интерпретатор скрипта через строку шебанга `#!`.
- Управлять расположением, владельцем, правами на запуск и SUID скриптов.

Файлы, термины и утилиты из objectives: `for`, `while`, `test`, `if`, `read`, `seq`, `exec`, `||`, `&&`.

## Шебанг и выбор интерпретатора

Первая строка скрипта начинается с символов `#!` (шебанг), за которыми идёт абсолютный путь к интерпретатору. Ядро читает эту строку при запуске файла и передаёт его указанной программе. Без шебанга скрипт всё равно запустится, но через текущую оболочку, а это не всегда то поведение, которого ожидает автор.

```bash
#!/bin/bash
echo "Hello, world"
```

Распространённые варианты:

- `#!/bin/sh`: совместимость с POSIX, минимальный набор возможностей
- `#!/bin/bash`: расширения Bash, массивы, `[[ ]]`
- `#!/usr/bin/env python3`: поиск интерпретатора через `env` по `$PATH`

Запуск возможен двумя путями. Через имя файла (`./script.sh`), когда ядро смотрит на шебанг и запускает интерпретатор. Через явный вызов интерпретатора (`bash script.sh`), когда шебанг игнорируется и работает указанная команда.

## Запуск скрипта, комментарии, подоболочка

Внутри скрипта строки, начинающиеся с `#`, считаются комментариями и игнорируются интерпретатором. Исключение: первая строка с шебангом `#!`. Пустые строки тоже пропускаются. Несколько команд можно ставить в одну строку через точку с запятой, но в Bash перевод строки эквивалентен этому разделителю, поэтому привычнее писать каждую команду отдельной строкой.

Суффикс `.sh` ничего не меняет для интерпретатора. Имя свободное. Однако расширение помогает быстро находить скрипты при просмотре каталога.

Команда `echo` по умолчанию добавляет перевод строки в конце вывода. Опция `-n` подавляет это поведение, и следующий вывод печатается на той же строке:

```bash
echo -n "Сейчас: "
date +%H:%M
```

Чтобы скрипт могли запускать другие пользователи, им нужен доступ на чтение. Команда `chmod o+r script.sh` открывает чтение всем. Доступ на чтение нужен, когда скрипт запускают через `bash script.sh`. При запуске по имени файла дополнительно требуется бит `x` (см. [Расположение и права скриптов](#расположение-и-права-скриптов)).

При обычном запуске скрипт выполняется в новой оболочке-потомке (подоболочке). Это защищает текущую сессию: переменные и изменения каталога, сделанные внутри скрипта, не затрагивают родительский шелл. Поведение полезно для большинства задач.

Чтобы выполнить инструкции скрипта прямо в текущей сессии, без подоболочки, применяют:

```bash
source script.sh
. script.sh
```

Между точкой и именем файла обязателен пробел. Этот приём нужен, когда скрипт устанавливает переменные окружения или функции, которые должны остаться в текущем шелле (типичный случай, файлы профиля `.bashrc`, `.profile`).

Команда `exec` перед скриптом или другой командой полностью заменяет текущий процесс шелла. После завершения такого скрипта закрывается и сама сессия:

```bash
exec ./long_task.sh
# сюда управление уже не вернётся
```

После окончания обычного запуска код возврата скрипта остаётся в `$?`.

## Базовый синтаксис sh

### Переменные

Присваивание идёт без пробелов вокруг знака равно, обращение через `$`:

```bash
NAME="Maks"
echo "Hello, $NAME"
```

По соглашению имена переменных пишут заглавными буквами, но это не обязательно. Имя не может начинаться с цифры или другого неалфавитного символа.

Длина значения переменной (число символов) возвращается префиксом `#` внутри фигурных скобок:

```bash
OS=$(uname -o)
echo $OS         # GNU/Linux
echo ${#OS}      # 9
```

Кавычки меняют поведение. Двойные раскрывают переменные и подстановки команд, одинарные передают всё буквально.

### Параметры скрипта

Внутри скрипта позиционные параметры доступны как переменные:

`$0`: имя самого скрипта  
`$1`, `$2`, ..., `$9`: первый, второй и так далее аргументы  
`$#`: число переданных аргументов  
`$@`: все аргументы списком, каждый отдельной строкой при `"$@"`  
`$*`: все аргументы одной строкой  
`$?`: код возврата последней выполненной команды  
`$$`: PID текущего скрипта  
`$!`: PID последнего фонового процесса

```bash
#!/bin/bash
echo "Скрипт: $0"
echo "Первый аргумент: $1"
echo "Всего аргументов: $#"
```

Параметры с номером больше девяти требуют фигурных скобок: `${10}`, `${11}` и так далее. Запись `$10` интерпретируется как `$1` со следующим за ним символом `0`.

### Массивы Bash

Bash поддерживает одномерные массивы. Объявить массив можно явно или сразу заполнить:

```bash
declare -a SIZES
SIZES=( 1048576 1073741824 )
```

Индексация начинается с нуля. Чтение элемента требует фигурных скобок и квадратных индексов:

```bash
echo ${SIZES[0]}     # 1048576
echo ${SIZES[1]}     # 1073741824
```

Запись элемента идёт без фигурных скобок:

```bash
SIZES[0]=2097152
```

Длина отдельного элемента возвращается через `${#ARR[N]}`. Общее число элементов в массиве: `${#ARR[@]}` или `${#ARR[*]}`:

```bash
echo ${#SIZES[0]}    # 7 (длина строки "2097152")
echo ${#SIZES[@]}    # 2 (всего элементов)
```

Массив удобно создавать из вывода команды через подстановку. Тогда каждое слово, отделённое пробелом, табуляцией или переводом строки, становится элементом:

```bash
FS=( $(cut -f 2 < /proc/filesystems) )
echo ${FS[0]} ${FS[1]} ${FS[2]}
```

Разделитель полей задаётся переменной окружения `IFS` (Input Field Separator). По умолчанию это пробел, табуляция, перевод строки. Чтобы разбивать только по переводу строки:

```bash
IFS=$'\n'
```

## Подстановка команд

Подстановка команд берёт стандартный вывод программы и подставляет его в место вызова. Поддерживаются две формы. Обратные кавычки идут из POSIX, форма `$(...)` современнее, читается лучше и допускает вложенность.

```bash
TODAY=$(date +%Y-%m-%d)
echo "Сегодня: $TODAY"

# Эквивалент через обратные кавычки
TODAY=`date +%Y-%m-%d`

# Вложенные подстановки
KERNEL_DIR=/lib/modules/$(uname -r)
```

Результат подстановки часто кладут в переменную или сразу передают другой команде:

```bash
mkdir backup-$(date +%F)
USER_COUNT=$(wc -l < /etc/passwd)
```

## Арифметика

Bash считает целые числа двумя способами. Старый POSIX-вариант через `expr`, современный встроенный через `$(( ))`.

```bash
SUM=`expr $VAL1 + $VAL2`
SUM=$(( $VAL1 + $VAL2 ))
```

Между числами и оператором в `expr` обязательны пробелы. Внутри `$(( ))` пробелы необязательны и допускаются операторы `+`, `-`, `*`, `/`, `%`, `**` (степень):

```bash
echo $(( 1024 ** 2 ))    # 1048576
echo $(( 1024 ** 3 ))    # 1073741824
SIZES=( $((1024**2)) $((1024**3)) )
```

Подстановка команд комбинируется с арифметикой. Пример: получить число свободных байт RAM из второй строки `/proc/meminfo`:

```bash
FREE=$(( 1000 * `sed -nre '2s/[^[:digit:]]//gp' < /proc/meminfo` ))
echo "Свободно: $(( $FREE / 1024**2 )) МБ"
```

Внутри `$(( ))` имена переменных можно писать без `$`: `$(( VAL1 + VAL2 ))` работает так же.

## Вывод скрипта: echo и printf

### echo и спецсимволы

Команда `echo` печатает строку и добавляет перевод строки в конце. Опция `-n` отменяет финальный перевод строки. Опция `-e` включает интерпретацию escape-последовательностей с обратным слешем:

`\n`: перевод строки  
`\t`: табуляция  
`\\`: сам обратный слеш  
`\"`: двойная кавычка

```bash
echo -e "Имя:\tMaks\nГород:\tSydney"
```

При использовании `-e` строку обязательно обрамляют кавычками, иначе оболочка может съесть слеши до того, как `echo` их увидит.

### printf

Команда `printf` форматирует вывод по шаблону, как одноимённая функция в C. Первый аргумент: строка формата, остальные: значения для подстановки в маркеры.

`%s`: строка  
`%d`: целое число  
`%f`: число с плавающей точкой  
`%x`: шестнадцатеричное число  
`%%`: сам символ процента

```bash
OS=$(uname -o)
FREE_MB=1491
printf "Операционка:\t%s\nПамять:\t\t%d МБ\n" "$OS" "$FREE_MB"
```

Особенности `printf` против `echo`:

- Автоматический перевод строки не добавляется, нужен явный `\n` в шаблоне
- Маркеры заменяются по порядку из аргументов
- Если значений больше, чем маркеров, шаблон применяется повторно
- Шаблон можно сохранить в переменной и переключать форматы вывода

```bash
FMT_TXT='Имя: %s, возраст: %d\n'
FMT_CSV='%s,%d\n'
printf "$FMT_TXT" "Maks" 30
printf "$FMT_CSV" "Maks" 30
```

Полная справка по форматам доступна в `man 3 printf`.

## Возвращаемые значения и команда test

### Код выхода

Каждая команда после завершения возвращает целое число от 0 до 255. Ноль означает успех. Любое другое значение, ошибку. Это число лежит в переменной `$?` сразу после выполнения команды.

```bash
ls /etc > /dev/null
echo $?    # 0

ls /no_such_dir 2>/dev/null
echo $?    # ненулевое значение, типично 2
```

В скриптах код выхода задают командой `exit N`, где N от 0 до 255.

### Проверки файлов

Команда `test` проверяет условие и возвращает 0 при истине, 1 при лжи. Эквивалентная форма с квадратными скобками `[ ... ]` чаще используется в коде. Внутри скобок обязательны пробелы вокруг каждого элемента.

`-e ФАЙЛ`: файл или каталог существует  
`-f ФАЙЛ`: обычный файл  
`-d ФАЙЛ`: каталог  
`-L ФАЙЛ`: символическая ссылка  
`-r ФАЙЛ`: доступен на чтение  
`-w ФАЙЛ`: доступен на запись  
`-x ФАЙЛ`: исполняемый  
`-s ФАЙЛ`: ненулевой размер  
`ФАЙЛ1 -nt ФАЙЛ2`: первый новее второго  
`ФАЙЛ1 -ot ФАЙЛ2`: первый старше второго

```bash
test -f /etc/passwd
[ -f /etc/passwd ]
[ -d /etc ] && echo "Каталог найден"
```

### Сравнение строк

`СТР1 = СТР2`: строки совпадают  
`СТР1 != СТР2`: строки различаются  
`-z СТР`: строка пустая  
`-n СТР`: строка непустая

### Сравнение чисел

`-eq`: равно  
`-ne`: не равно  
`-lt`: меньше  
`-le`: меньше или равно  
`-gt`: больше  
`-ge`: больше или равно

```bash
[ "$AGE" -ge 18 ] && echo "Совершеннолетний"
```

В Bash доступна расширенная форма `[[ ... ]]`. Она безопаснее: не требует кавычек вокруг переменных, поддерживает шаблоны и регулярные выражения, имеет операторы `&&` и `||` внутри. Для арифметики удобна форма `(( ... ))`.

```bash
if [[ "$FILE" == *.log ]]; then echo "лог-файл"; fi
if (( COUNT > 10 )); then echo "много"; fi
```

## Условные конструкции

### if then elif else

```bash
if [ -f /etc/passwd ]; then
    echo "Файл найден"
elif [ -d /etc/passwd ]; then
    echo "Это каталог"
else
    echo "Не найдено"
fi
```

Условие `if` это любая команда. Используется её код возврата, а не вывод. Поэтому `if grep -q root /etc/passwd; then ... fi` работает корректно.

### case

```bash
case "$1" in
    start)
        echo "Запуск службы"
        ;;
    stop)
        echo "Остановка службы"
        ;;
    restart|reload)
        echo "Перезагрузка"
        ;;
    *)
        echo "Использование: $0 {start|stop|restart|reload}"
        exit 1
        ;;
esac
```

Сравнение в `case` идёт по шаблонам глоббинга, не по регулярным выражениям. Каждая ветка завершается `;;`. Конструкция закрывается `esac`.

## Циклы for, while, until

Цикл `for` обходит элементы списка:

```bash
for i in 1 2 3 4 5; do
    echo "Число $i"
done

for f in *.txt; do
    echo "Файл: $f"
done

for i in $(seq 1 10); do
    echo $i
done
```

Bash также поддерживает C-подобный синтаксис:

```bash
for ((i=0; i<10; i++)); do
    echo $i
done
```

Цикл `while` повторяет блок, пока условие истинно:

```bash
COUNT=0
while [ $COUNT -lt 5 ]; do
    echo $COUNT
    COUNT=$((COUNT + 1))
done

# Чтение файла построчно
while read LINE; do
    echo "$LINE"
done < input.txt
```

Цикл `until` работает зеркально, повторяет блок, пока условие ложно:

```bash
until ping -c1 -W1 host >/dev/null 2>&1; do
    sleep 5
done
echo "Хост доступен"
```

## Команды read, seq, exec

### read

Считывает строку со стандартного ввода и записывает в одну или несколько переменных. Если переменных несколько, слова разбиваются по `IFS`.

```bash
read -p "Введите имя: " NAME
echo "Привет, $NAME"

read FIRST LAST <<< "John Smith"
echo "$LAST, $FIRST"
```

Если в `read` не указано имя переменной, прочитанное значение попадает в переменную `REPLY`:

```bash
echo "Продолжить (y/n)?"
read
echo "Ответ: $REPLY"
```

При чтении нескольких переменных слов может оказаться больше, чем имён. Тогда лишние слова приклеиваются к последней переменной.

Полезные опции:

`-p ТЕКСТ`: показать приглашение  
`-s`: скрыть ввод (для паролей)  
`-t СЕК`: тайм-аут на ввод  
`-n N`: прочитать ровно N символов  
`-r`: не интерпретировать обратные слеши  
`-a МАССИВ`: прочитать слова в массив Bash

### seq

Печатает числовую последовательность.

```bash
seq 5         # 1 2 3 4 5
seq 2 8       # 2 3 4 5 6 7 8
seq 1 2 9     # 1 3 5 7 9 (шаг 2)
seq -w 1 10   # выравнивает ширину нулями: 01 02 ... 10
seq -s, 1 5   # 1,2,3,4,5 (свой разделитель)
```

Часто используется в связке с `for`:

```bash
for i in $(seq 1 100); do
    curl -s "https://example.com/page$i" -o "page$i.html"
done
```

### exec

Команда `exec` имеет два режима работы. С аргументом она заменяет текущий процесс на указанную программу. Скрипт после такого `exec` не продолжает выполнение.

```bash
exec ls /tmp
echo "эта строка не выполнится"
```

Без аргумента (только с перенаправлением) `exec` меняет файловые дескрипторы текущей оболочки. Это удобно, когда весь дальнейший вывод скрипта надо отправить в журнал:

```bash
#!/bin/bash
exec >> /var/log/myscript.log 2>&1
echo "Эта запись и всё дальше попадёт в лог"
date
```

## Цепочки команд

Несколько команд можно соединить в одной строке. Разделитель определяет логику запуска.

`cmd1 ; cmd2`: запускает обе команды последовательно, без проверки результата  
`cmd1 && cmd2`: запускает `cmd2` только при успехе `cmd1` (код 0)  
`cmd1 || cmd2`: запускает `cmd2` только при ошибке `cmd1` (ненулевой код)

```bash
mkdir backup && cp *.conf backup/
ping -c1 host >/dev/null || echo "Хост недоступен"
[ -f config ] && source config || echo "config отсутствует"
```

Эти операторы заменяют короткие `if` и читаются компактнее. Длинные цепочки лучше переписать через `if`, иначе теряется ясность.

## Условная отправка почты администратору

Стандартный приём: уведомить root, когда что-то пошло не так. Текст сообщения подаётся на стандартный вход команды `mail`.

```bash
backup.sh || echo "Сбой бэкапа на $(hostname)" | mail -s "Backup error" root
```

Вариант с проверкой состояния службы:

```bash
if ! systemctl is-active --quiet sshd; then
    echo "SSH остановлен на $(hostname) в $(date)" | mail -s "ALERT: sshd" root
fi
```

Вариант с порогом по диску:

```bash
USAGE=$(df / | awk 'NR==2 {print $5}' | tr -d %)
if [ "$USAGE" -gt 90 ]; then
    df -h | mail -s "Корень почти заполнен на $(hostname)" root
fi
```

Команда `mail` читает почту локального пользователя root, если настроен MTA (postfix, sendmail, exim) или хотя бы локальная доставка.

## Расположение и права скриптов

Где принято хранить скрипты:

`~/bin` или `~/.local/bin`: личные скрипты пользователя, многие дистрибутивы автоматически добавляют их в `$PATH`  
`/usr/local/bin`: общедоступные локальные скрипты для всех пользователей  
`/usr/local/sbin`: административные скрипты, доступные только root  
`/etc/cron.daily`, `/etc/cron.hourly`: скрипты, запускаемые периодически  
`/etc/init.d`: устаревшие SysV-скрипты инициализации служб

Сделать скрипт исполняемым:

```bash
chmod +x myscript.sh
chmod 755 myscript.sh   # rwxr-xr-x
chmod 750 myscript.sh   # rwxr-x--- (только владелец и группа)
```

Назначить владельца и группу:

```bash
sudo chown root:root /usr/local/bin/myscript.sh
sudo chmod 755 /usr/local/bin/myscript.sh
```

### SUID на скриптах

Бит SUID (Set User ID, восьмеричное значение 4000) при запуске даёт процессу права владельца файла, а не запустившего пользователя. Устанавливается так:

```bash
chmod u+s script.sh
chmod 4755 script.sh
```

Важная особенность: ядро Linux игнорирует SUID на интерпретируемых скриптах из соображений безопасности (классическая race-condition уязвимость). Бит можно поставить, но эффекта на запуск не будет. SUID работает только на скомпилированных бинарниках. Если скрипту нужны права root, правильный путь: настроить `sudo` через `/etc/sudoers`:

```
# /etc/sudoers
maks ALL=(root) NOPASSWD: /usr/local/sbin/backup.sh
```

## Типичные вопросы экзамена

1. Какая первая строка скрипта указывает интерпретатор и как она называется? → Строка шебанга `#!`, самая первая строка файла.
2. Чем `#!/bin/sh` отличается от `#!/bin/bash` на практике? → `sh` это POSIX-оболочка с минимальным набором возможностей; `bash` добавляет массивы, `[[ ]]`, `$(( ))`.
3. Как получить код возврата последней выполненной команды? → `$?`.
4. Что обозначает код выхода 0? → Успех.
5. Какой синтаксис подстановки команд предпочтительнее: обратные кавычки или `$()`? → `$()`: поддерживает вложенность, читается лучше.
6. В чём разница между `cmd1 && cmd2`, `cmd1 || cmd2` и `cmd1 ; cmd2`? → `&&` при успехе; `||` при ошибке; `;` в любом случае.
7. Как отправить письмо root, если команда завершилась с ошибкой? → `cmd || echo "сообщение" | mail -s "тема" root`
8. Какая команда читает строку со стандартного ввода в переменную? → `read`.
9. В какую переменную попадает значение, если у `read` не указано имя? → `REPLY`.
10. Какая опция команды `read` скрывает ввод (для пароля)? → `-s`.
11. Что выведет `seq 1 2 7`? → `1 3 5 7`.
12. Что делает `exec` с аргументом и без него? → С аргументом: заменяет текущий процесс. Без аргумента (с перенаправлением): меняет файловые дескрипторы оболочки.
13. Чем `bash script.sh` отличается от `source script.sh` и `. script.sh`? → `bash` запускает в подоболочке; `source`/`.` выполняют в текущей сессии.
14. Как сделать скрипт исполняемым в восьмеричной нотации? → `chmod 755 script.sh`.
15. Где хранят пользовательские, общедоступные и административные скрипты? → `~/bin`, `/usr/local/bin`, `/usr/local/sbin`.
16. Работает ли SUID на shell-скриптах в современном Linux? → Нет, ядро игнорирует его.
17. Как реализовать условный выбор без `if`, через `&&` и `||`? → `cmd1 && on_success || on_failure`.
18. Какие операторы `test` проверяют существование файла, каталога, исполняемости? → `-e`, `-d`, `-x`.
19. Чем отличается `[ ]` от `[[ ]]` в Bash? → `[[ ]]` это расширение Bash: не требует кавычек, поддерживает шаблоны и regex, допускает `&&`/`||` внутри.
20. Как обойти все файлы каталога циклом `for`? → `for f in /каталог/*; do ...; done`.
21. Как получить длину значения переменной? → `${#VAR}`.
22. Как обратиться к десятому позиционному параметру? → `${10}`.
23. Чем отличается `echo -e` от обычного `echo`? → `-e` интерпретирует `\n`, `\t` и другие escape-последовательности.
24. Какая разница между `echo` и `printf` в части переноса строки? → `echo` всегда добавляет перевод строки; `printf` только если в шаблоне есть `\n`.
25. Как объявить массив в Bash и обратиться к его элементам? → `ARR=( a b c )`; обращение через `${ARR[0]}`.
26. Как узнать общее число элементов массива? → `${#ARR[@]}`.
27. Что хранит переменная `IFS` и зачем её менять? → Разделитель полей для разбивки слов; устанавливают в `$'\n'`, чтобы не делить по пробелам.
28. Как посчитать арифметическое выражение через встроенный синтаксис Bash? → `$(( ВЫРАЖЕНИЕ ))`.
29. Что делает префикс `exec` перед запуском скрипта? → Заменяет текущую оболочку скриптом; сессия завершается вместе со скриптом.

## Упражнения

### Практическое упражнение 1 — Чтение пароля без эха

Опция `-s` команды `read` удобна для ввода паролей: набираемый текст не показывается на экране. Как с её помощью сохранить введённое пользователем значение в переменную `PASSWORD`?

<details>
<summary>Ответ</summary>

```bash
read -s PASSWORD
```

> Опцию `-s` часто комбинируют с `-p` для приглашения и с `-t` для тайм-аута. Полная форма для интерактивного ввода: `read -s -p "Пароль: " PASSWORD`. После такого ввода стоит сразу добавить `echo` для перевода строки, потому что Enter с подавленным эхом не печатает символ перевода.

</details>

### Практическое упражнение 2 — Сохранение вывода whoami в переменную

Команда `whoami` показывает имя пользователя, от которого она запущена. Как сохранить вывод `whoami` в переменную `WHO` внутри Bash-скрипта?

<details>
<summary>Ответ</summary>

```bash
WHO=`whoami`
```

или

```bash
WHO=$(whoami)
```

> Форма `$()` предпочтительнее: она читается яснее и допускает вложенность (например, `LOG="$(whoami)-$(date +%F).log"`). Обратные кавычки сохранены ради совместимости со старыми POSIX-скриптами.

</details>

---

### Практическое упражнение 3 — Условный запуск через оператор

Какой оператор Bash должен стоять между командами `apt-get dist-upgrade` и `systemctl reboot`, если root хочет выполнить `systemctl reboot` только при успешном завершении `apt-get dist-upgrade`?

<details>
<summary>Ответ</summary>

Оператор `&&`:

```bash
apt-get dist-upgrade && systemctl reboot
```

> Оператор `&&` запускает правую команду, только если левая вернула код 0 (успех). Симметричный оператор `||` запускает правую при ненулевом коде. Если нужно перезагрузиться независимо от результата обновления, используют `;` или перевод строки.

</details>

### Исследовательское упражнение 1 — Permission denied при запуске скрипта

При попытке запустить только что созданный Bash-скрипт пользователь получает сообщение:

```
bash: ./script.sh: Permission denied
```

Файл `./script.sh` создан тем же самым пользователем. Какова вероятная причина ошибки?

<details>
<summary>Ответ</summary>

У файла `./script.sh` не установлен бит исполнения.

> Запуск через `./script.sh` требует прав на чтение и исполнение. По умолчанию вновь созданный файл получает права без бита `x`. Исправляется через `chmod +x script.sh`. Альтернативный способ запуска без бита `x`, явный вызов интерпретатора: `bash script.sh`. В этом случае достаточно прав на чтение, потому что сам файл не выполняется ядром, а только читается интерпретатором.

</details>

### Исследовательское упражнение 2 — Определение имени вызова через символическую ссылку

Пусть скрипт `do.sh` исполняемый, а `undo.sh` символическая ссылка на него. Как изнутри скрипта определить, под каким из этих имён его вызвали?

<details>
<summary>Ответ</summary>

Специальная переменная `$0` содержит имя файла, под которым скрипт был запущен.

```bash
#!/bin/bash
case "$(basename "$0")" in
    do.sh)    echo "Режим: применить" ;;
    undo.sh)  echo "Режим: откатить" ;;
    *)        echo "Неизвестный режим запуска: $0" ;;
esac
```

> Этот приём (один скрипт, несколько символических ссылок с разными именами) широко применяется в Unix. Классический пример: BusyBox, единый бинарник, который ведёт себя как `ls`, `cat`, `grep` и десятки других команд в зависимости от значения `$0`. Функция `basename` срезает путь и оставляет только имя файла, потому что в `$0` может оказаться, например, `/usr/local/bin/do.sh`.

</details>

### Исследовательское упражнение 3 — Условная отправка письма root по коду возврата

В системе с настроенной почтовой службой команда

```bash
mail -s "Maintenance Error" root <<<"Scheduled task error"
```

отправляет уведомление root. Напишите конструкцию `if`, которая выполнит указанную команду `mail`, если код возврата предыдущей команды ненулевой.

<details>
<summary>Ответ</summary>

```bash
if [ "$?" -ne 0 ]; then mail -s "Maintenance Error" root <<<"Scheduled task error"; fi
```

> Проверка `[ "$?" -ne 0 ]` сравнивает значение `$?` с нулём (флаг `-ne`, not equal). Кавычки вокруг `"$?"` страхуют от ошибок при пустом значении, хотя `$?` всегда содержит число. Альтернативная компактная запись через оператор `||` даёт тот же эффект без `if`: `previous_command || mail -s "Maintenance Error" root <<<"Scheduled task error"`. Минус подхода с `||`: его нельзя поставить отдельной строкой после произвольной предыдущей команды, нужно приклеивать к ней на той же строке.

</details>
