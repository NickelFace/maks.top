Согласно официальным целям LPIC-1:

## Настройка и использование клиента OpenSSH

### Подключение к удалённому хосту

```bash
ssh user@host              # подключиться от имени 'user' на 'host'
ssh user@192.168.1.77      # по IP-адресу
ssh host                   # подключиться от имени текущего локального пользователя
ssh user@host ls .ssh      # выполнить одну команду и вернуться
```

При первом подключении к хосту SSH показывает отпечаток ключа сервера и просит подтверждения. После принятия отпечаток сохраняется в `~/.ssh/known_hosts`.

Если ключ хоста изменился (переустановка, смена IP через DHCP), SSH предупреждает о возможной атаке «человек посередине». Удалить старый ключ:

```bash
ssh-keygen -R имя_хоста           # удалить все ключи для хоста
ssh-keygen -f "~/.ssh/known_hosts" -R "192.168.1.77"   # явное указание файла
```

## Вход по ключам

### Генерация пары ключей

```bash
ssh-keygen -t ecdsa              # сгенерировать пару ключей ECDSA
ssh-keygen -t ed25519            # сгенерировать пару Ed25519 (наиболее безопасный)
ssh-keygen -t rsa                # сгенерировать пару RSA
ssh-keygen -t ecdsa -b 521       # указать размер ключа в битах
```

Без флага `-t` `ssh-keygen` по умолчанию генерирует RSA (2048 бит).

Файлы создаются в `~/.ssh/`:

| Файл | Содержимое |
|---|---|
| `id_ecdsa` | Закрытый ключ (защищайте этот файл) |
| `id_ecdsa.pub` | Открытый ключ (передаётся на серверы) |
| `known_hosts` | Отпечатки доверенных хостов |

### Алгоритмы ключей

| Алгоритм | Примечания |
|---|---|
| RSA | Широко используется; минимум 1024, по умолчанию 2048 бит |
| DSA | Устарел в OpenSSH 7.0; ровно 1024 бит |
| ecdsa | Эллиптическая кривая; 256, 384 или 521 бит |
| ed25519 | EdDSA / кривая Эдвардса; фиксированные 256 бит; наиболее безопасный |

### Копирование открытого ключа на сервер

```bash
ssh-copy-id user@host                          # рекомендованный способ
cat id_ecdsa.pub | ssh user@host 'cat >> .ssh/authorized_keys'   # вручную
```

Открытый ключ добавляется в `~/.ssh/authorized_keys` на удалённом хосте.

### SSH-агент аутентификации

Агент хранит расшифрованные закрытые ключи в памяти, поэтому пароль нужно вводить только один раз за сеанс.

```bash
ssh-agent /bin/bash              # запустить оболочку с агентом
ssh-add                          # добавить закрытый ключ по умолчанию; ввести пароль один раз
ssh-add ~/.ssh/id_ed25519        # добавить конкретный ключ
```

После загрузки ключа последующие SSH-соединения устанавливаются без запроса пароля.

## Ключи хоста сервера OpenSSH

Ключи хоста сервера хранятся в `/etc/ssh/`:

| Файл | Права | Назначение |
|---|---|---|
| `ssh_host_rsa_key` | `0600` | Закрытый ключ RSA |
| `ssh_host_rsa_key.pub` | `0644` | Открытый ключ RSA |
| `ssh_host_ecdsa_key` | `0600` | Закрытый ключ ECDSA |
| `ssh_host_ecdsa_key.pub` | `0644` | Открытый ключ ECDSA |
| `ssh_host_ed25519_key` | `0600` | Закрытый ключ Ed25519 |
| `ssh_host_ed25519_key.pub` | `0644` | Открытый ключ Ed25519 |
| `ssh_config` | — | Конфигурация клиента |
| `sshd_config` | — | Конфигурация сервера |

Просмотр отпечатка ключа:

```bash
ssh-keygen -l -f /etc/ssh/ssh_host_ed25519_key.pub    # отпечаток
ssh-keygen -lv -f /etc/ssh/ssh_host_ed25519_key.pub   # отпечаток + рисунок
```

### Ключевые директивы sshd_config

| Директива | Назначение |
|---|---|
| `PermitRootLogin` | Разрешить/запретить вход root по SSH |
| `AllowUsers` | Белый список пользователей |

## SSH-туннели

Туннелирование портов шифрует трафик с локального порта и передаёт его через SSH-соединение.

### Локальный туннель (`-L`)

Трафик на `ЛОКАЛЬНЫЙ_ПОРТ` локальной машины передаётся через SSH-сервер на `ХОСТ_НАЗНАЧЕНИЯ:ПОРТ_НАЗНАЧЕНИЯ`.

```bash
ssh -L 8585:www.gnu.org:80 debian
# Браузер на http://localhost:8585 → попадает на www.gnu.org:80 через 'debian'

ssh -L 8585:www.gnu.org:80 -Nf ina@192.168.1.77
# -N: не выполнять команду на сервере (только проброс порта)
# -f: запустить в фоновом режиме
```

### Удалённый (обратный) туннель (`-R`)

Трафик, поступающий на `УДАЛЁННЫЙ_ПОРТ` SSH-сервера, передаётся на `ХОСТ_НАЗНАЧЕНИЯ:ПОРТ_НАЗНАЧЕНИЯ` с точки зрения локальной машины.

```bash
ssh -R 8585:localhost:80 -Nf ina@192.168.1.77
# Любой, подключившийся к 192.168.1.77:8585, попадает на localhost:80
```

### Несколько туннелей в одной команде

```bash
ssh -L 8080:www.gnu.org:80 -L 8585:www.melpa.org:80 -Nf ina@halof
```

### X11-туннель (`-X`)

Перенаправляет дисплей X Window System удалённого хоста на локальную машину.

```bash
ssh -X user@host             # включить X11-перенаправление
ssh -x user@host             # отключить X11-перенаправление
```

### Сводная таблица туннелей

| Туннель | Параметр | Директива sshd_config |
|---|---|---|
| Локальный | `-L ЛОКАЛЬНЫЙ:ХОСТ:ПОРТ` | `AllowTcpForwarding` |
| Удалённый / обратный | `-R УДАЛЁННЫЙ:ХОСТ:ПОРТ` | `GatewayPorts` |
| X11 | `-X` | `X11Forwarding` |

## GnuPG (GPG)

GnuPG (*GNU Privacy Guard*) — свободная реализация стандарта OpenPGP. Использует асимметричную криптографию: закрытый ключ (хранится в тайне) и открытый ключ (передаётся адресатам).

### Генерация ключей

```bash
gpg --gen-key                # интерактивный режим; запрашивает имя, email, пароль
```

Создаётся каталог `~/.gnupg/` со следующим содержимым:

| Путь | Содержимое |
|---|---|
| `openpgp-revocs.d/` | Сертификаты отзыва |
| `private-keys-v1.d/` | Закрытые ключи |
| `pubring.kbx` | Связка открытых ключей |
| `trustdb.gpg` | База доверия |

### Управление ключами

```bash
gpg --list-keys              # список связки открытых ключей
gpg --fingerprint USER-ID    # показать отпечаток ключа
```

**KEY-ID** — последние 8 шестнадцатеричных символов полного отпечатка ключа.

### Экспорт и распространение ключей

```bash
gpg --export USER-ID > carol.pub.key                    # двоичный экспорт
gpg --export --armor USER-ID > carol.pub.key            # ASCII-broня (безопасно для email)
gpg --export --output carol.pub.key USER-ID             # то же с флагом -o
```

Распространение через серверы ключей:

```bash
gpg --keyserver ИМЯ_СЕРВЕРА --send-keys KEY-ID       # загрузить
gpg --keyserver ИМЯ_СЕРВЕРА --recv-keys KEY-ID       # скачать
```

### Импорт ключей

```bash
gpg --import carol.pub.key
```

### Отзыв ключа

Применяется при компрометации или выводе из использования закрытого ключа.

Шаг 1 — Создать сертификат отзыва:

```bash
gpg --output revocation_file.asc --gen-revoke USER-ID
```

Шаг 2 — Импортировать его, чтобы применить отзыв:

```bash
gpg --import revocation_file.asc
```

Шаг 3 — Распространить отозванный ключ среди адресатов и на серверах ключей.

## Шифрование и расшифровка файлов

### Шифрование

```bash
gpg --output encrypted-message --recipient carol --armor --encrypt unencrypted-message
```

| Параметр | Описание |
|---|---|
| `--output` / `-o` | Имя выходного файла |
| `--recipient` / `-r` | USER-ID получателя |
| `--armor` / `-a` | Вывод в ASCII-броне (безопасно для email) |
| `--encrypt` / `-e` | Зашифровать указанный файл |

### Расшифровка

```bash
gpg --decrypt encrypted-message                          # вывод в stdout
gpg --output unencrypted-message --decrypt encrypted-message   # сохранить в файл
```

---

## Подпись и проверка файлов

### Подпись

```bash
gpg --output message.sig --sign message       # двоичный подписанный файл
gpg --output message.asc --clearsign message  # подпись открытым текстом
```

### Проверка

```bash
gpg --verify message.sig
gpg --output message --decrypt message.sig    # проверить и извлечь содержимое
```

---

## Короткие параметры GPG

| Длинный | Короткий |
|---|---|
| `--armor` | `-a` |
| `--output` | `-o` |
| `--recipient` | `-r` |
| `--decrypt` | `-d` |
| `--encrypt` | `-e` |
| `--sign` | `-s` |

### Другие полезные параметры gpg

```bash
gpg --export-secret-keys --output all_private.key    # экспорт закрытых ключей
gpg --edit-key USER-ID                               # интерактивное меню управления ключами
gpg --clearsign message                              # подпись открытым текстом
```

### gpg-agent

`gpg-agent` — демон, управляющий закрытыми ключами GPG в памяти (запускается по требованию самим `gpg`).

```bash
gpg-agent --help
gpg-agent -h
```

## Типичные вопросы экзамена

1. Какая команда подключается к хосту `server01` от имени пользователя `alice`? → `ssh alice@server01`
2. Где на клиентской машине хранятся отпечатки доверенных хостов? → `~/.ssh/known_hosts`
3. Как удалить устаревший ключ хоста `server01` из `known_hosts`? → `ssh-keygen -R server01`
4. Какой тип ключа считается наиболее безопасным для `ssh-keygen`? → `ed25519`
5. Какой тип ключа генерируется по умолчанию при запуске `ssh-keygen` без флага `-t`? → RSA (2048 бит)
6. Какой файл на удалённом сервере должен содержать открытый ключ клиента для входа по ключу? → `~/.ssh/authorized_keys`
7. Для чего нужен `ssh-agent`? → Хранит расшифрованные закрытые ключи в памяти, чтобы пароль вводился только один раз за сеанс.
8. Какая команда добавляет закрытый ключ в агент? → `ssh-add`
9. Какие права установлены на файлы закрытых ключей хоста в `/etc/ssh/`? → `0600` (только владелец может читать и писать).
10. Какой параметр SSH создаёт локальный туннель? → `-L ЛОКАЛЬНЫЙ_ПОРТ:ХОСТ_НАЗНАЧЕНИЯ:ПОРТ_НАЗНАЧЕНИЯ`
11. Какой параметр SSH создаёт удалённый (обратный) туннель? → `-R УДАЛЁННЫЙ_ПОРТ:ХОСТ_НАЗНАЧЕНИЯ:ПОРТ_НАЗНАЧЕНИЯ`
12. Что делают параметры `-N` и `-f` в команде SSH-туннеля? → `-N` — не выполнять команду на сервере; `-f` — запустить SSH в фоне.
13. Какая директива `sshd_config` включает X11-перенаправление? → `X11Forwarding`
14. Какая команда GPG генерирует пару ключей? → `gpg --gen-key`
15. Что такое KEY-ID в GPG? → Последние 8 шестнадцатеричных цифр отпечатка открытого ключа.
16. Как экспортировать открытый ключ в ASCII-формате? → `gpg --export --armor USER-ID > файл`
17. Каков правильный порядок отзыва ключа GPG? → 1. Создать сертификат отзыва (`--gen-revoke`), 2. Импортировать его (`--import`), 3. Распространить отозванный ключ.
18. Какой параметр GPG создаёт подпись открытым текстом (читаемый файл)? → `--clearsign`

## Упражнения

### Практическое упражнение 1 — Настройка SSH-ключей

Сгенерируйте пару ключей Ed25519 и настройте вход по ключу на удалённый хост.

<details>
<summary>Ответ</summary>

```bash
ssh-keygen -t ed25519          # создаёт ~/.ssh/id_ed25519 и id_ed25519.pub
ssh-copy-id user@remote-host   # копирует открытый ключ в authorized_keys
ssh user@remote-host           # вход без пароля
```

</details>

### Исследовательское упражнение 2 — Отзыв ключа GPG

<details>
