## Шебанг и выбор интерпретатора

Первая строка скрипта начинается с символов `#!` (шебанг), за которыми идёт абсолютный путь к интерпретатору. Ядро читает эту строку при запуске файла и передаёт его указанной программе. Без шебанга скрипт всё равно запустится, но через текущую оболочку — что не всегда то поведение, которого ожидает автор.

Типичные варианты:

```bash
#!/bin/bash       # Bash — наиболее распространённый выбор
#!/bin/sh         # POSIX-совместимая оболочка (реализация зависит от дистрибутива)
#!/usr/bin/env python3   # Python через env — удобно для переносимости
```

## Базовый синтаксис sh

### Переменные

Присвоение — без пробелов вокруг знака равно:

```bash
NAME="Alice"
COUNT=42
```

Обращение через `$NAME` или `${NAME}`. Двойные кавычки сохраняют подстановки переменных; одинарные подавляют их.

### Параметры скрипта

Внутри скрипта аргументы доступны как позиционные параметры:

| Параметр | Значение |
|---|---|
| `$0` | Имя скрипта |
| `$1`, `$2` … | Первый, второй … аргумент |
| `$#` | Количество аргументов |
| `$@` | Все аргументы как отдельные слова |
| `$*` | Все аргументы как одна строка |
| `$?` | Код выхода последней команды |
| `$$` | PID текущего процесса |
| `$!` | PID последнего фонового процесса |

## Подстановка команд

Вывод команды можно подставить в строку или присвоить переменной:

```bash
TODAY=$(date +%Y-%m-%d)
TODAY=`date +%Y-%m-%d`    # старая форма
```

Современный синтаксис `$(...)` предпочтительнее: поддерживает вложение, обратные кавычки — нет.

## Возвращаемые значения и команда test

### Код выхода

Каждая команда возвращает целое число. Ноль означает успех, всё остальное — ошибку. Код доступен в `$?` сразу после выполнения:

```bash
ls /etc
echo $?       # 0
ls /несуществующий
echo $?       # 2
```

Скрипт может явно задать код через `exit N` (0–255).

### Две формы записи теста

Builtin команда `test` имеет две эквивалентные формы — как имя команды и как квадратные скобки, где `test` подразумевается:

```bash
test -d /etc && echo "да"
[ -d /etc ] && echo "да"
```

Пробелы внутри `[ ]` обязательны — после `[` и перед `]`.

### Проверки файлов и каталогов

Предполагается, что путь хранится в `$VAR`:

| Флаг | Истина когда |
|---|---|
| `-e "$VAR"` | путь существует |
| `-f "$VAR"` | путь — обычный файл |
| `-d "$VAR"` | путь — каталог |
| `-s "$VAR"` | путь существует и не пустой |
| `-r "$VAR"` | путь читаем текущим пользователем |
| `-w "$VAR"` | путь записываем |
| `-x "$VAR"` | путь исполняем |
| `-b "$VAR"` | блочное специальное устройство |
| `-c "$VAR"` | символьное специальное устройство |
| `-h` / `-L "$VAR"` | символическая ссылка |
| `-p "$VAR"` | именованный канал |
| `-S "$VAR"` | сокет |
| `-u "$VAR"` | установлен SUID-бит |
| `-g "$VAR"` | установлен SGID-бит |
| `-k "$VAR"` | установлен sticky-бит |
| `-O "$VAR"` | принадлежит текущему пользователю |
| `-G "$VAR"` | принадлежит эффективной группе |
| `-N "$VAR"` | изменялся с момента последнего обращения |

Двойные кавычки вокруг переменной защищают от синтаксической ошибки, если переменная пустая.

### Сравнение двух файлов

| Тест | Истина когда |
|---|---|
| `"$A" -nt "$B"` | A новее B (по времени изменения) |
| `"$A" -ot "$B"` | A старше B |
| `"$A" -ef "$B"` | A и B — один и тот же файл (один inode) |

### Сравнение строк

| Тест | Истина когда |
|---|---|
| `-z "$S"` | строка пустая |
| `-n "$S"` | строка не пустая |
| `"$A" = "$B"` | строки равны |
| `"$A" != "$B"` | строки различаются |
| `"$A" < "$B"` | A сортируется раньше B (зависит от локали) |
| `"$A" > "$B"` | A сортируется позже B |

Для устойчивого алфавитного порядка независимо от локали: установить `LANG=C` перед операцией.

### Сравнение чисел

| Оператор | Значение |
|---|---|
| `-lt` | меньше (less than) |
| `-gt` | больше (greater than) |
| `-le` | меньше или равно |
| `-ge` | больше или равно |
| `-eq` | равно |
| `-ne` | не равно |

Использовать только буквенные операторы для чисел. Операторы `<` и `>` сравнивают строки и заодно перенаправляют stdout — `test 1 > 2` создаёт файл `2` и возвращает истину.

### Модификаторы и логические операторы

| Оператор | Значение |
|---|---|
| `! EXPR` | истина, если EXPR ложно |
| `EXPR1 -a EXPR2` | оба выражения истинны (И) |
| `EXPR1 -o EXPR2` | хотя бы одно истинно (ИЛИ) |

Пример:

```bash
if [ ! -d "$FROM" -o ! -d "$TO" ]
then
    echo "Один из каталогов не существует"
    exit 1
fi
```

## Условные конструкции

### if / then / elif / else

```bash
if УСЛОВИЕ
then
    КОМАНДЫ
elif ДРУГОЕ_УСЛОВИЕ
then
    КОМАНДЫ
else
    КОМАНДЫ
fi
```

Условием служит любая команда — проверяется её код выхода. Закрывается ключевым словом `fi`.

### case

Используется когда нужно много ветвей:

```bash
#!/bin/bash
DISTRO=$1
echo -n "Distribution $DISTRO uses "
case "$DISTRO" in
  debian | ubuntu | mint)
    echo -n "the DEB"
    ;;
  centos | fedora | opensuse)
    echo -n "the RPM"
    ;;
  *)
    echo -n "an unknown"
    ;;
esac
echo " package format."
```

```
$ ./script.sh opensuse
Distribution opensuse uses the RPM package format.
```

Каждый блок шаблонов завершается `;;`, `;&` или `;;&`. Шаблон `*` срабатывает, если ничего не подошло. Закрывается ключевым словом `esac`.

### Опция nocasematch

Включить сопоставление без учёта регистра в `case` и `[[ ]]`:

```bash
shopt -s nocasematch    # включить
shopt -u nocasematch    # выключить
```

Изменения внутри скрипта не влияют на родительскую сессию — скрипт запускается в субшелле.

## Циклы

### for

```bash
for VARNAME in LIST
do
    КОМАНДЫ
done
```

`LIST` — последовательность разделённых элементов. Разделители задаются `IFS` (по умолчанию: пробел, табуляция, перевод строки).

Пример — проверка чётности:

```bash
#!/bin/bash
for NUM in 1 1 2 3 5 8 13
do
    echo -n "$NUM is "
    if [ $(( $NUM % 2 )) -ne 0 ]; then echo "odd."; else echo "even."; fi
done
```

### Стиль for из C

```bash
#!/bin/bash
SEQ=( 1 1 2 3 5 8 13 )
for (( IDX = 0; IDX < ${#SEQ[*]}; IDX++ ))
do
    echo "${SEQ[$IDX]}"
done
```

### until

Повторяет, пока тест возвращает ненулевой код (неуспех):

```bash
IDX=0
until [ $IDX -eq ${#SEQ[*]} ]
do
    echo "${SEQ[$IDX]}"
    IDX=$(( $IDX + 1 ))
done
```

### while

Повторяет, пока тест возвращает ноль (успех):

```bash
while [ $IDX -lt ${#SEQ[*]} ]
do
    echo "${SEQ[$IDX]}"
    IDX=$(( $IDX + 1 ))
done
```

`while` и `until` — зеркальные отражения: `while COND` эквивалентно `until ! COND`.

### Практический пример: sync.sh

Скрипт для синхронизации файлов из источника в каталог назначения. Список элементов хранится в `~/.sync.list` (по одному на строку, пробелы в именах допустимы):

```bash
#!/bin/bash
set -ef

FILE=~/.sync.list
FROM=$1
TO=$2

if [ ! -d "$FROM" -o ! -d "$TO" ]
then
    echo "Использование: $0 <ИСТОЧНИК> <НАЗНАЧЕНИЕ>"
    exit 1
fi

mapfile -t LIST < $FILE

for (( IDX = 0; IDX < ${#LIST[*]}; IDX++ ))
do
    echo -e "$FROM/${LIST[$IDX]} \u2192 $TO/${LIST[$IDX]}"
    rsync -qa --delete "$FROM/${LIST[$IDX]}" "$TO"
done
```

`set -ef`: `-e` выходит при любой ошибке, `-f` отключает раскрытие имён файлов через wildcards.

`mapfile -t LIST < $FILE`: загружает файл в массив `LIST`, по одной строке на элемент, убирая завершающие переводы строки.

`rsync -qa --delete`: тихий режим, архивный (сохраняет атрибуты), удаляет в назначении то, чего нет в источнике.

## Команды read, seq, exec

### read

```bash
read -p "Имя: " NAME           # приглашение и ввод
read -s PASSWORD                # тихий ввод (для паролей)
read -t 5 ANSWER                # таймаут 5 секунд
read FIRST SECOND REST          # разбить ввод на несколько переменных
```

### seq

Вывести последовательность чисел:

```bash
seq 5            # 1 2 3 4 5
seq 2 2 10       # 2 4 6 8 10  (шаг 2)
seq -w 1 10      # 01 02 ... 10  (с ведущими нулями)
```

В цикле `for`:

```bash
for i in $(seq 1 10); do echo "Шаг $i"; done
```

### exec

Заменяет текущий процесс оболочки указанной командой — без нового процесса, без возврата:

```bash
exec /usr/bin/python3 myscript.py
```

Без команды `exec` перенаправляет потоки всего скрипта:

```bash
exec > /tmp/log.txt 2>&1    # весь последующий вывод идёт в файл
```

## Цепочки команд

| Разделитель | Поведение |
|---|---|
| `;` | выполнять по порядку независимо от результата |
| `&&` | следующая команда только если предыдущая успешна (код 0) |
| `\|\|` | следующая команда только если предыдущая упала (код ≠ 0) |

```bash
make && make install                    # установить только при успешной сборке
test -f /tmp/lock || touch /tmp/lock    # создать lock-файл, если его нет
mkdir backup && cp -r data backup/      # скопировать только в созданный каталог
```

Цепочка `&&` заменяет простой `if`:

```bash
[ -d /backup ] && rsync -a /data/ /backup/
```

## Условная отправка почты администратору

Из objectives: уметь условно отправлять почту суперпользователю. Команды `mail` или `mailx`:

```bash
df -h | grep -E '9[0-9]%|100%' && \
    df -h | mail -s "Диск переполнен на $(hostname)" root
```

С использованием `if`:

```bash
if ! systemctl is-active --quiet apache2
then
    echo "Apache упал в $(date)" | mail -s "ALERT: apache2 down" root
fi
```

Для реальной отправки на сервере должна работать MTA (postfix, sendmail, exim).

## Расположение и права скриптов

Распространённые каталоги:

| Путь | Назначение |
|---|---|
| `~/bin` или `~/.local/bin` | личные скрипты пользователя |
| `/usr/local/bin` | скрипты для всех пользователей |
| `/usr/local/sbin` | административные скрипты, требующие root |
| `/etc/cron.daily/` и т.п. | скрипты, запускаемые cron |

Сделать скрипт исполняемым:

```bash
chmod +x myscript.sh         # для всех
chmod 755 myscript.sh        # rwxr-xr-x
chmod 700 myscript.sh        # только владельцу
```

Запуск по короткому имени работает только если каталог скрипта в `PATH`. Иначе — полный или относительный путь:

```bash
./myscript.sh
/usr/local/bin/myscript.sh
```

### SUID на скриптах

Ядро Linux игнорирует SUID-бит на shell-скриптах по соображениям безопасности (гонки состояния при интерпретации). Если нужно дать обычному пользователю выполнение чего-либо от root:

- Настроить `sudo` через `/etc/sudoers` для конкретной команды
- Написать обёртку на C
- Использовать `setcap` для конкретных capabilities

## Краткий справочник

```bash
# Шебанг (первая строка скрипта)
#!/bin/bash
#!/bin/sh
#!/usr/bin/env python3

# Параметры скрипта
$0  $1  $2  $#  $@  $*  $?  $$  $!

# Подстановка команд
result=$(command)
result=`command`      # старая форма

# test
test -f "$FILE"       # обычный файл
[ -d "$DIR" ]         # каталог
[ -z "$VAR" ]         # пустая строка
[ "$A" -lt "$B" ]     # числовое: меньше
[ "$A" -nt "$B" ]     # файл A новее B
echo $?               # проверить код выхода

# Логические операторы внутри test
[ ! EXPR ]                    # НЕ
[ EXPR1 -a EXPR2 ]            # И
[ EXPR1 -o EXPR2 ]            # ИЛИ

# if
if [ COND ]; then CMD; elif [ COND2 ]; then CMD2; else CMD3; fi

# case
case "$VAR" in
  pat1) CMDS ;;
  pat2|pat3) CMDS ;;
  *) CMDS ;;
esac
shopt -s nocasematch   # сопоставление без учёта регистра

# for
for VAR in LIST; do CMDS; done
for (( i=0; i<N; i++ )); do CMDS; done

# while / until
while [ COND ]; do CMDS; done
until [ COND ]; do CMDS; done

# read / seq / exec
read -p "Приглашение: " VAR
seq 1 10
for i in $(seq 1 10); do ...; done
exec CMD          # заменить оболочку командой
exec > file.txt   # перенаправить весь вывод скрипта

# Цепочки команд
CMD1 && CMD2      # CMD2 только если CMD1 успешна
CMD1 || CMD2      # CMD2 только если CMD1 упала
CMD1 ; CMD2       # всегда выполнить обе

# Права
chmod +x script.sh
chmod 755 script.sh

# Опции set
set -e    # выход при ошибке
set -f    # отключить globbing
set -x    # трассировка выполнения
```

## Типичные вопросы экзамена

1. Какой код выхода означает успех? → Ноль.
2. Чем `==` в `test` отличается от `-eq`? → `==` сравнивает строки; `-eq` сравнивает целые числа.
3. Как проверить, что переменная пустая? → `[ -z "$VAR" ]`.
4. Как проверить, что путь — это каталог? → `[ -d "$PATH" ]`.
5. Как проверить, что файл A новее файла B? → `[ "$A" -nt "$B" ]`.
6. Что такое шебанг и где он стоит? → Строка `#!интерпретатор` в самом начале скрипта.
7. Как сравнивать без учёта регистра в `case`? → `shopt -s nocasematch` перед конструкцией.
8. Как закрыть `case`? → Ключевым словом `esac`.
9. Как закрыть `if`? → Ключевым словом `fi`.
10. Разница между `&&` и `;`? → `&&` запускает следующую команду только при успехе предыдущей; `;` — в любом случае.
11. Почему `test 1 > 2` возвращает истину? → Оператор `>` предназначен для строкового сравнения, а сам символ `>` интерпретируется шеллом как перенаправление вывода в файл `2`.
12. Как обеспечить устойчивый порядок сортировки? → Установить `LANG=C` перед операцией.
13. Что делает `mapfile -t LIST < file`? → Читает файл в массив `LIST`, по одной строке на элемент, без завершающих переводов строки.
14. Какая опция `set` прерывает скрипт при ошибке? → `-e`.
15. Влияют ли изменения `shopt` внутри скрипта на родительскую оболочку? → Нет, скрипт запускается в субшелле.
16. Что делает `exec > /tmp/log.txt 2>&1` в скрипте? → Перенаправляет весь последующий stdout и stderr в файл, не заменяя оболочку.
17. Как отправить однострочный алерт суперпользователю? → `echo "сообщение" | mail -s "тема" root`.
18. Почему ядро игнорирует SUID на shell-скриптах? → Риск гонки состояния при интерпретации шебанга.

## Упражнения

### Упражнение 1 — test для сравнения файлов

Как командой `test` проверить, что путь к файлу в переменной `FROM` новее, чем путь к файлу в переменной `TO`?

<details>
<summary>Ответ</summary>

```bash
test "$FROM" -nt "$TO"
```

Вернёт код 0, если файл из `FROM` новее файла из `TO`. Флаг `-nt` — сокращение от "newer than". Двойные кавычки защищают от пустых переменных.

</details>

### Упражнение 2 — Бесконечный while и потерянный инкремент

Скрипт должен напечатать числа от 0 до 9, но вместо этого бесконечно печатает 0. Что нужно исправить?

```bash
#!/bin/bash
COUNTER=0
while [ $COUNTER -lt 10 ]
do
    echo $COUNTER
done
```

<details>
<summary>Ответ</summary>

В теле цикла нет инкремента `COUNTER`, поэтому условие остаётся истинным навсегда.

Исправленный скрипт:

```bash
#!/bin/bash
COUNTER=0
while [ $COUNTER -lt 10 ]
do
    echo $COUNTER
    COUNTER=$(( $COUNTER + 1 ))
done
```

Альтернативные формы инкремента: `(( COUNTER++ ))` или `let COUNTER++`.

</details>

### Упражнение 3 — Разный порядок сортировки на двух машинах

Один и тот же скрипт сортирует список имён по-разному на двух машинах:

Машина 1:
```
carol
Dave
emma
Frank
Grace
henry
```

Машина 2:
```
Dave
Frank
Grace
carol
emma
henry
```

В чём причина и как исправить?

<details>
<summary>Ответ</summary>

Сортировка опирается на текущую локаль. На разных машинах настроены разные локали, правила сравнения букв разного регистра отличаются. Для устойчивого результата:

```bash
LANG=C sort users.txt
```

На первой машине, скорее всего, активна UTF-8 локаль (например, `en_US.UTF-8`), где заглавные и строчные буквы упорядочиваются вперемешку. На второй работает `C`/`POSIX`, где сначала все заглавные ASCII-буквы, потом все строчные.

</details>

### Упражнение 4 — Аргументы скрипта в массив Bash

Как использовать все аргументы командной строки скрипта для инициализации массива Bash?

<details>
<summary>Ответ</summary>

```bash
PARAMS=( "$@" )
```

Или эквивалентно:

```bash
PARAMS=( $* )
```

`"$@"` предпочтительнее — сохраняет каждый аргумент как отдельный элемент даже при наличии пробелов внутри. `$*` (без кавычек) разбивает аргументы по `IFS` и теряет границы.

</details>

### Упражнение 5 — Почему `test 1 > 2` возвращает истину?

<details>
<summary>Ответ</summary>

Две причины:

1. Оператор `>` в `test` предназначен для **строкового** сравнения, а не числового. Строково "1" < "2", но тест проверяет "идёт ли 1 позже 2" — нет.
2. Шелл перехватывает `>` как **перенаправление** до того, как `test` его увидит. Фактически выполняется `test 1` с перенаправлением stdout в файл `2`. Команда `test 1` — непустая строка, истина, код 0.

Для правильного числового сравнения: `[ 1 -gt 2 ]`. Для правильного строкового: `test 1 \> 2`.

</details>

### Упражнение 6 — Временная смена IFS на символ перевода строки

Как временно изменить разделитель полей на символ перевода строки и потом вернуть исходное значение?

<details>
<summary>Ответ</summary>

```bash
OLDIFS=$IFS
IFS=$'\n'

# ... команды, которым нужен перевод строки как разделитель ...

IFS=$OLDIFS
```

Запись `$'\n'` — синтаксис ANSI-C quoting в Bash, где экранированные последовательности (`\n`, `\t` и т.д.) интерпретируются как управляющие символы. Без `$` переменной была бы присвоена буквальная строка из двух символов `\` и `n`.

</details>
