## Что нужно знать

- Устанавливать, обновлять и удалять бинарные пакеты Debian.
- Находить пакеты, содержащие конкретные файлы или библиотеки, в том числе не установленные.
- Получать информацию о пакете: версию, содержимое, зависимости, целостность, статус установки.
- Понимать назначение и возможности apt.

Утилиты и файлы: `/etc/apt/sources.list`, `dpkg`, `dpkg-reconfigure`, `apt-get`, `apt-cache`.

## Формат пакета .deb и инструмент dpkg

Формат `.deb` появился как стандарт распространения скомпилированного программного обеспечения на Debian-системах. Инструмент `dpkg` работает с этим форматом напрямую. Он используется на самом Debian и всех его производных, включая Ubuntu.

`dpkg` выполняет установку, настройку, обслуживание и удаление пакетов. APT работает поверх `dpkg`, добавляя разрешение зависимостей и работу с репозиториями, а не заменяя его.

### Установка и обновление пакетов

```bash
dpkg -i PACKAGENAME.deb
```

Перед установкой `dpkg` проверяет, есть ли уже более старая версия. Если да, пакет обновится. Если нет, установится с нуля.

### Зависимости в dpkg

`dpkg` проверяет наличие зависимостей при установке. Если нужные пакеты отсутствуют, установка завершится с ошибкой и списком недостающих зависимостей. Решать зависимости самостоятельно `dpkg` не умеет: нужные `.deb`-файлы придётся найти и установить вручную.

Пример ошибки зависимостей:

```
dpkg: dependency problems prevent configuration of openshot-qt:
 openshot-qt depends on fonts-cantarell; however:
  Package fonts-cantarell is not installed.
```

Флаг `--force` позволяет обойти предупреждение о зависимостях при удалении, но это опасно: система может оказаться в нерабочем состоянии.

### Удаление пакетов

```bash
dpkg -r PACKAGENAME
```

`dpkg` не позволит удалить пакет, если другой установленный пакет от него зависит. Чтобы удалить оба сразу, можно перечислить их через пробел:

```bash
dpkg -r unzip file-roller
```

При удалении через `-r` конфигурационные файлы остаются в системе. Чтобы удалить пакет вместе со всеми конфигами, используется флаг `-P` (purge):

```bash
dpkg -P PACKAGENAME
```

### Просмотр информации о пакете

```bash
dpkg -I PACKAGENAME.deb
```

Флаг `-I` (заглавная i) показывает детали `.deb`-файла: версию, зависимости, описание. Это работает с ещё не установленным файлом.

### Список установленных пакетов и их файлов

Показать все установленные пакеты:

```bash
dpkg --get-selections
```

Показать список файлов, установленных конкретным пакетом:

```bash
dpkg -L PACKAGENAME
```

Пример вывода:

```
# dpkg -L unrar
/.
/usr
/usr/bin
/usr/bin/unrar-nonfree
/usr/share/doc/unrar/changelog.Debian.gz
```

### Поиск владельца файла

Чтобы найти, какому пакету принадлежит конкретный файл в системе:

```bash
dpkg-query -S /usr/bin/unrar-nonfree
# unrar: /usr/bin/unrar-nonfree
```

`dpkg-query -S` работает только с уже установленными файлами. Для поиска по неустановленным пакетам нужен `apt-file search`.

### Перенастройка пакета

При установке пакет запускает post-install скрипт, который настраивает файлы конфигурации и задаёт вопросы пользователю. Если конфигурация испорчена или нужно изменить ответы, запускают:

```bash
dpkg-reconfigure PACKAGENAME
```

Утилита создаёт резервную копию старых конфигов, распаковывает новые и запускает post-install скрипт заново, как будто пакет устанавливается впервые. Пример:

```bash
dpkg-reconfigure tzdata
```

## Advanced Package Tool (apt)

APT упрощает установку, обновление и удаление пакетов, добавляя автоматическое разрешение зависимостей и работу с удалёнными репозиториями. Это надстройка над `dpkg`, а не его замена.

Основные утилиты APT:

- `apt-get` — установка, обновление, удаление пакетов.
- `apt-cache` — поиск и получение информации из индекса пакетов.
- `apt-file` — поиск файлов внутри пакетов.
- `apt` — объединяет наиболее часто используемые команды `apt-get` и `apt-cache`. Для экзамена нужно знать `apt-get` и `apt-cache`, поскольку `apt` может отсутствовать в системе.

APT требует сетевое соединение для загрузки пакетов и обновления индексов.

### Обновление индекса пакетов

Перед установкой или обновлением пакетов обновляют локальный индекс:

```bash
apt-get update
# или
apt update
```

Команда скачивает свежие списки пакетов из репозиториев, описанных в `/etc/apt/sources.list` и файлах в `/etc/apt/sources.list.d/`.

### Установка, обновление и удаление через apt-get

Установить пакет (с автоматическим разрешением зависимостей):

```bash
apt-get install PACKAGENAME
```

Обновить все установленные пакеты до последних версий из репозиториев:

```bash
apt-get upgrade
# или
apt upgrade
```

Обновить один конкретный пакет:

```bash
apt-get upgrade PACKAGENAME
```

Удалить пакет:

```bash
apt-get remove PACKAGENAME
# или
apt remove PACKAGENAME
```

При удалении через `remove` конфигурационные файлы остаются в системе. Чтобы удалить пакет вместе с конфигами, используется `purge`:

```bash
apt-get purge PACKAGENAME
# или
apt-get remove --purge PACKAGENAME
```

Исправить незавершённую установку из-за неудовлетворённых зависимостей:

```bash
apt-get install -f
```

### Локальный кеш пакетов

При загрузке пакета `.deb`-файл сохраняется в локальный кеш: `/var/cache/apt/archives/`. Частично загруженные файлы попадают в `/var/cache/apt/archives/partial/`. Кеш со временем занимает много места, очистить его можно так:

```bash
apt-get clean
# или
apt clean
```

### Поиск пакетов через apt-cache

Поиск пакета по имени, описанию или файлам:

```bash
apt-cache search PATTERN
# или
apt search PATTERN
```

Получить подробную информацию о пакете (версия, зависимости, описание), аналогично `dpkg -I`:

```bash
apt-cache show PACKAGENAME
# или
apt show PACKAGENAME
```

### Файл sources.list

APT берёт список источников пакетов из файла `/etc/apt/sources.list`. Редактировать его можно любым текстовым редактором.

Типичная строка:

```
deb http://us.archive.ubuntu.com/ubuntu/ disco main restricted universe multiverse
```

Синтаксис строки: `тип_архива URL дистрибутив компоненты`.

**Тип архива:**
- `deb` — бинарные пакеты (готовые к запуску).
- `deb-src` — пакеты с исходным кодом.

**Компоненты в Ubuntu:**
- `main` — официально поддерживаемые пакеты с открытым кодом.
- `restricted` — официально поддерживаемое проприетарное ПО (например, драйверы).
- `universe` — пакеты, поддерживаемые сообществом.
- `multiverse` — несвободное или патентованное ПО.

**Компоненты в Debian:**
- `main` — пакеты, соответствующие DFSG, считаются частью дистрибутива.
- `contrib` — DFSG-совместимые пакеты, зависящие от non-free компонентов.
- `non-free` — пакеты, не соответствующие DFSG.
- `security` — обновления безопасности.
- `backports` — свежие версии пакетов из `main` для стабильного выпуска.

Строки, начинающиеся с `#`, считаются комментариями и игнорируются. После добавления новой строки нужно выполнить `apt-get update`.

### Директория sources.list.d

Вместо редактирования основного файла можно добавлять отдельные `.list`-файлы в директорию `/etc/apt/sources.list.d/`. Синтаксис внутри такой же. Пример файла `buster-backports.list`:

```
deb http://deb.debian.org/debian buster-backports main contrib non-free
deb-src http://deb.debian.org/debian buster-backports main contrib non-free
```

### apt-file: содержимое пакетов и поиск файлов

`apt-file` может не быть установлен по умолчанию. Установить и подготовить к работе:

```bash
apt-get install apt-file
apt-file update
```

Показать все файлы, которые содержит пакет (включая неустановленные):

```bash
apt-file list PACKAGENAME
```

Найти, в каком пакете содержится файл (в том числе в неустановленных пакетах):

```bash
apt-file search FILENAME
```

Пример:

```bash
apt-file search libSDL2.so
# libsdl2-dev: /usr/lib/x86_64-linux-gnu/libSDL2.so
```

Разница между `apt-file search` и `dpkg-query -S`: первый ищет по всем пакетам, включая неустановленные, второй видит только файлы уже установленных пакетов.

## Полезные команды для экзамена

| Команда | Что делает |
|---|---|
| `dpkg -i pkg.deb` | Установить пакет из файла |
| `dpkg -r pkg` | Удалить пакет (конфиги остаются) |
| `dpkg -P pkg` | Удалить пакет вместе с конфигами (purge) |
| `dpkg -I pkg.deb` | Показать информацию о .deb-файле |
| `dpkg --get-selections` | Список всех установленных пакетов |
| `dpkg -L pkg` | Файлы, установленные пакетом |
| `dpkg-query -S /path/to/file` | Найти владельца файла (только из установленных) |
| `dpkg-reconfigure pkg` | Перезапустить настройку пакета |
| `apt-get update` | Обновить индекс пакетов |
| `apt-get install pkg` | Установить пакет из репозитория |
| `apt-get upgrade` | Обновить все пакеты |
| `apt-get remove pkg` | Удалить пакет (конфиги остаются) |
| `apt-get purge pkg` | Удалить пакет вместе с конфигами |
| `apt-get install -f` | Исправить неудовлетворённые зависимости |
| `apt-get clean` | Очистить локальный кеш пакетов |
| `apt-cache search pattern` | Поиск пакета по шаблону |
| `apt-cache show pkg` | Подробная информация о пакете |
| `apt-file update` | Обновить кеш для apt-file |
| `apt-file list pkg` | Файлы внутри пакета (включая неустановленные) |
| `apt-file search file` | Найти пакет по имени файла (включая неустановленные) |

## Типичные вопросы экзамена

**Чем `-r` отличается от `-P` в dpkg?**
`-r` удаляет пакет, но оставляет конфигурационные файлы. `-P` (purge) удаляет пакет вместе со всеми конфигами. То же различие у `apt-get remove` и `apt-get purge`.

**Чем отличается `dpkg-query -S` от `apt-file search`?**
`dpkg-query -S` работает только с установленными пакетами. `apt-file search` ищет по всем пакетам, в том числе тем, которые ещё не установлены.

**Можно ли удалить пакет, если от него зависит другой пакет?**
Нет. `dpkg -r` откажет. Нужно сначала удалить зависимый пакет или удалить оба сразу через `dpkg -r pkg1 pkg2`. Флаг `--force` обходит проверку, но это опасно.

**Что делает `apt-get install -f`?**
Исправляет прерванную или незавершённую установку, устанавливая недостающие зависимости. Флаг `-f` означает `--fix-broken`.

**Чем `deb` отличается от `deb-src` в sources.list?**
`deb` указывает на репозиторий с бинарными пакетами, `deb-src` — с исходным кодом.

**Зачем нужен `dpkg-reconfigure`?**
Чтобы заново запустить post-install настройку пакета, например после повреждения конфига или для изменения первоначальных настроек.

**Куда APT сохраняет скачанные .deb-файлы?**
В `/var/cache/apt/archives/`. Очистить кеш можно командой `apt-get clean`.

**2. С помощью `dpkg-query` найти, какой пакет содержит файл `7zr.1.gz`.**

<details>
<summary>Ответ</summary>

```bash
dpkg-query -S 7zr.1.gz
```

</details>

---

**3. Можно ли удалить пакет `unzip` через `dpkg -r unzip`, если пакет `file-roller` от него зависит? Если нет, как сделать правильно?**

<details>
<summary>Ответ</summary>

Нет. `dpkg` не разрешит удалить пакет, если другой установленный пакет от него зависит. Решение: удалить `file-roller` сначала (если от него ничего не зависит), затем `unzip`, или удалить оба сразу:

```bash
dpkg -r unzip file-roller
```

</details>

---

**4. Как с помощью `apt-file` найти, какой пакет содержит файл `/usr/bin/unrar`?**

<details>
<summary>Ответ</summary>

```bash
apt-file search /usr/bin/unrar
```

</details>

---

**5. Какая команда `apt-cache` показывает информацию о пакете `gimp`?**

<details>
<summary>Ответ</summary>

```bash
apt-cache show gimp
```

</details>

### Исследовательские упражнения (Explorational Exercises)

**1. Строка репозитория с исходниками в sources.list.**

Есть репозиторий с пакетами исходного кода Debian для дистрибутива `xenial`, расположенный по адресу `http://us.archive.ubuntu.com/ubuntu/`, компонент `universe`. Какую строку нужно добавить в `/etc/apt/sources.list`?

<details>
<summary>Ответ</summary>

```
deb-src http://us.archive.ubuntu.com/ubuntu/ xenial universe
```

Пакеты с исходным кодом имеют тип `deb-src`. Эту строку можно добавить в отдельный файл в `/etc/apt/sources.list.d/`, например `xenial_sources.list`.

</details>

---

**2. Найти пакет для заголовочного файла `zzip-io.h`.**

При компиляции программы появляется ошибка: заголовочный файл `zzip-io.h` отсутствует в системе. Как найти, какой пакет его предоставляет?

<details>
<summary>Ответ</summary>

Файл не установлен, поэтому `dpkg-query -S` не поможет. Нужен `apt-file search`, который ищет по всем пакетам, включая неустановленные:

```bash
apt-file search zzip-io.h
```

</details>

---

**3. Принудительное удаление пакета с зависимостями.**

Как с помощью `dpkg` проигнорировать предупреждение о зависимостях и удалить пакет, даже если от него зависят другие пакеты?

<details>
<summary>Ответ</summary>

Используется параметр `--force`, но это крайне опасная операция. Система может оказаться в нерабочем или нестабильном состоянии. Применять только если точно понимаешь последствия.

</details>

</details>

apt-get update
