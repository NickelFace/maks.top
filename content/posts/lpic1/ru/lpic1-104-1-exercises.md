> **Тема 104.1** — Создание разделов и файловых систем. Практические и исследовательские разборы.

---

## Практические упражнения (Guided)

### 1. Какую таблицу разделов выбрать для диска 3 ТБ под три раздела по 1 ГБ. Почему

**Ответ:** GPT. MBR ограничен дисками до 2 ТБ, поэтому 3-терабайтный диск в MBR-схеме не поместится целиком.

---

### 2. Как в gdisk узнать сколько свободного места осталось на диске

**Ответ:** команда `p` (print) выводит таблицу разделов, в шапке которой указан общий объём свободного места.

```
Total free space is 1282071 sectors (626.0 MiB)
```

---

### 3. Команда для создания ext3 на /dev/sdc1 с проверкой bad blocks, меткой MyDisk и случайным UUID

**Ответ:**

```bash
mkfs.ext3 -c -L MyDisk -U random /dev/sdc1
```

Альтернатива через `mke2fs`:

```bash
mke2fs -t ext3 -c -L MyDisk -U random /dev/sdc1
```

---

### 4. В parted: создать ext4-раздел на 300 МБ начиная с отметки 500 МБ

**Ответ:**

```
(parted) mkpart primary ext4 500m 800m
```

**Важно помнить:** parted ставит лишь флаг типа на разделе и сам файловую систему не создаёт. После выхода из parted нужно отдельно вызвать `mkfs.ext4 /dev/sdXN`.

---

### 5. Два раздела по 20 ГБ. Как объединить в одну Btrfs с зеркалированием. Какой будет размер

**Ответ:**

```bash
mkfs.btrfs /dev/sda1 /dev/sdb1 -m raid1
```

Размер итоговой ФС — **20 ГБ**, потому что один раздел работает как зеркало другого.

Для полного зеркалирования и данных, и метаданных:

```bash
mkfs.btrfs -d raid1 -m raid1 /dev/sda1 /dev/sdb1
```

## Исследовательские упражнения (Explorational)

### 1. Можно ли создать раздел на 600 МБ на MBR-диске с дырой между разделами

**Дано:** MBR-диск 1.9 ГБ, на нём `/dev/sdb1` (512 МБ) и `/dev/sdb3` (512 МБ). Между разделами и после третьего — свободное место.

**Ответ:** нельзя. Суммарно свободно около 1 ГБ, но наибольший **непрерывный** кусок — только 512 МБ. Раздел на 600 МБ не помещается, новый раздел не может «перепрыгнуть» через занятый.

```
Value out of range.
```

Команда `F` в fdisk покажет точную картину свободного места:

```
Unpartitioned space /dev/sdd: 881 MiB, 923841536 bytes, 1804378 sectors
  Start       End  Sectors  Size
1050624   2099199  1048576  512M
3147776   3903577   755802  369M
```

---

### 2. Как через parted ужать первый раздел /dev/sdc до минимально нужного размера

**Дано:** первый раздел 1 ГБ с ext4, в нём около 256 МБ файлов.

**Ответ:** операция в два шага — сначала ФС, потом раздел.

Шаг 1 — ужать файловую систему под фактический объём данных (`-M`):

```bash
resize2fs -M /dev/sdc1
```

Шаг 2 — сдвинуть конец раздела в parted:

```
(parted) resizepart 1 241M
```

**Важен порядок:** при уменьшении сначала файловая система, потом граница раздела. В обратном порядке — потеря данных.

---

### 3. swapon выдаёт ошибку "read swap header failed". Что не так

**Дано:** создан раздел `mkpart primary linux-swap 0 1024M`, потом `swapon /dev/sdb1` ругнулся ошибкой.

**Ответ:** `mkpart` в parted создаёт раздел, но **не записывает swap-структуру**. Перед `swapon` нужно запустить `mkswap`:

```bash
mkswap /dev/sdb1
swapon /dev/sdb1
```

---

### 4. Восстановить случайно удалённый третий раздел в parted

**Дано:** на диске была EFI-партиция 250 МБ, swap 4 ГБ, третий раздел 10 ГБ. Третий случайно удалили.

**Ответ:** команда `rescue` с примерными границами.

Считаем позиции: 250 МБ (EFI) + 4×1024 МБ (swap) = 4346 МБ — начало. Конец: 4346 + 10×1024 = 14 586 МБ.

```
(parted) rescue 4346m 14586m
```

Можно дать небольшой запас вперёд и назад — геометрия диска может слегка сдвигать границы. `rescue` находит только разделы, на которых была создана файловая система.

---

### 5. Превратить /dev/sda3 в активный swap через fdisk

**Дано:** на /dev/sda3 неиспользуемый раздел 4 ГБ.

**Ответ:** три шага.

Шаг 1 — в fdisk сменить тип раздела на Linux Swap:

```
Command (m for help): t
Partition number (1-3, default 3): 3
Hex code (type L to list all codes): 82

Changed type of partition 'Linux' to 'Linux swap / Solaris'.

Command (m for help): w
```

Шаг 2 — подготовить swap-структуру:

```bash
mkswap /dev/sda3
```

Шаг 3 — включить swap:

```bash
swapon /dev/sda3
```
