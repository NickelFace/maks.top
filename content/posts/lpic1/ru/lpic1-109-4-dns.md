> **Вес на экзамене: 2** — LPIC-1 v5, Exam 102

## Что нужно знать

Согласно официальным целям LPIC-1:

- Запрашивать удалённые DNS-серверы.
- Настраивать локальное разрешение имён и использовать удалённые DNS-серверы.
- Изменять порядок разрешения имён.
- Диагностировать ошибки разрешения имён.
- Знать о `systemd-resolved`.

Ключевые файлы и команды: `/etc/hosts`, `/etc/resolv.conf`, `/etc/nsswitch.conf`, `host`, `dig`, `getent`.

---

## Процесс разрешения имён

Когда программа разрешает имя хоста, она обращается к библиотеке GNU C (glibc), которая читает `/etc/nsswitch.conf`, чтобы определить, где и в каком порядке искать имя.

DNS имеет три класса записей:
- **IN** — Internet (TCP/IP-адреса, используется во всех стандартных запросах).
- **CH** — ChaosNet (устаревший, практически не встречается).
- **HS** — Hesiod (хранение данных passwd/group в DNS).

---

## /etc/nsswitch.conf

Управляет источниками и порядком поиска для каждого типа базы данных имён.

```
passwd:         compat
group:          compat
hosts:          dns [!UNAVAIL=return] files
networks:       nis [NOTFOUND=return] files
services:       nis [NOTFOUND=return] files
```

Строка `hosts` определяет способ разрешения имён хостов. Каждый столбец — источник. Столбцы в скобках задают условную логику:

| Запись | Значение |
|---|---|
| `[!UNAVAIL=return]` | Если служба ДОСТУПНА, остановиться здесь (даже если запись не найдена). |
| `[NOTFOUND=return]` | Если запрос выполнен, но запись не найдена, остановиться. |
| `[SUCCESS=continue]` | Продолжать обход источников даже при успешном нахождении. |

Примеры строки `hosts`:

```
hosts:    files dns          # сначала /etc/hosts, затем DNS
hosts:    dns files          # сначала DNS, затем /etc/hosts
```

## /etc/resolv.conf

Настройка DNS-разрешения. Может перезаписываться NetworkManager и другими инструментами (признак — комментарий `# Generated by NetworkManager` в начале файла).

```
search lpi.org
nameserver 10.0.0.53
nameserver fd00:ffff::2:53
```

| Директива | Описание |
|---|---|
| `nameserver IP` | DNS-сервер (не более 3 записей) |
| `search ДОМЕН` | Суффикс для коротких имён (не более 6) |
| `domain ДОМЕН` | Имя локального домена (несовместимо с `search`) |
| `options КЛЮЧ:ЗНАЧЕНИЕ` | Параметры резолвера, например `options timeout:3` |

Если указаны оба, `search` и `domain` несовместимы: действует последняя запись в файле.

## /etc/hosts

Локальное сопоставление IP-адресов с именами хостов. Проверяется раньше DNS, если в `/etc/nsswitch.conf` `files` стоит первым.

```
127.0.0.1       localhost
::1             localhost ip6-localhost ip6-loopback
10.0.0.1        gateway.lpi.org gateway gw
10.0.1.53       dns1.lpi.org
```

Формат: `IP-АДРЕС  основное-имя [псевдоним ...]`

IP-адрес всегда стоит в первом столбце.

## systemd-resolved

`systemd-resolved` является службой systemd, предоставляющей разрешение DNS на адресе `127.0.0.53`. Она проксирует запросы на серверы из `/etc/resolv.conf` или `/etc/systemd/resolv.conf`.

Чтобы использовать её, укажите `resolve` в строке `hosts` файла `/etc/nsswitch.conf`:

```
hosts:    resolve [!UNAVAIL=return] files
```

`systemd-resolved` не заменяет полноценный DNS-сервер: она только перенаправляет запросы.

## getent

`getent` получает записи из баз данных NSS, используя те же библиотеки-резолверы, что и приложения, с учётом `/etc/nsswitch.conf`.

```bash
getent hosts                       # вывести все записи из базы hosts
getent hosts dns1.lpi.org          # найти конкретный хост

getent -s files hosts learning.lpi.org   # принудительно из /etc/hosts
getent -s dns hosts learning.lpi.org     # принудительно через DNS
```

Параметр `-s` (glibc 2.2.5+) указывает источник данных.

**Ключевое отличие**: `getent` использует полный стек NSS (учитывает nsswitch.conf); `host` и `dig` обращаются к DNS напрямую.

## host

`host` представляет собой простую утилиту поиска DNS. По умолчанию возвращает записи A, AAAA и MX.

```bash
host wikipedia.org                     # записи A, AAAA, MX
host 208.80.154.224                    # обратный поиск (PTR)
host -t NS lpi.org                     # конкретный тип записи
host -t MX lpi.org dns1.easydns.com   # использовать указанный сервер имён
```

| Параметр | Описание |
|---|---|
| `-t ТИП` | Тип записи: A, AAAA, MX, NS, SOA, PTR и др. |
| Последний аргумент | IP-адрес или имя сервера имён — использовать вместо `/etc/resolv.conf` |

## dig

`dig` является подробным инструментом диагностики DNS. По умолчанию запрашивает записи A.

```bash
dig learning.lpi.org               # поиск записи A
dig -t SOA lpi.org                 # запрос SOA
dig +short lpi.org                 # краткий вывод (только IP)
dig +noall +answer +question lpi.org   # только секции вопроса и ответа
dig +nocookie -t MX lpi.org        # отключить EDNS-расширение cookie
```

### Секции вывода dig

| Секция | Содержимое |
|---|---|
| Header | Метаданные запроса (opcode, статус, флаги, счётчики) |
| Question | Отправленный запрос |
| Answer | Возвращённые ресурсные записи |
| Authority | NS-записи авторитетной зоны |
| Additional | A/AAAA-записи серверов из секции Authority |

### Параметры dig

| Параметр | Описание |
|---|---|
| `-t ТИП` | Тип запрашиваемой записи |
| `+short` | Только результат, без лишних секций |
| `+noall` | Отключить все секции вывода |
| `+answer` | Включить секцию Answer |
| `+question` | Включить секцию Question |

### ~/.digrc

Параметры `dig` по умолчанию задаются в файле `~/.digrc`. Например, чтобы всегда использовать `+short`:

```
+short
```

## Полезные команды для экзамена

```
/etc/nsswitch.conf:
  hosts: files dns        сначала /etc/hosts, затем DNS
  [!UNAVAIL=return]       остановиться, если служба доступна
  [NOTFOUND=return]       остановиться, если запись не найдена
  [SUCCESS=continue]      продолжать даже при успешном нахождении

/etc/resolv.conf:
  nameserver IP           DNS-сервер (до 3)
  search ДОМЕН            поисковый домен (до 6)
  domain ДОМЕН            локальный домен (несовместим с search)
  options timeout:3

/etc/hosts:
  IP  имя-хоста [псевдоним]   IP-адрес всегда в первом столбце

systemd-resolved:
  слушает на 127.0.0.53; проксирует на настроенные DNS
  использовать "resolve" в строке hosts nsswitch.conf

getent hosts [ИМЯ]             поиск через NSS (учитывает nsswitch.conf)
getent -s files hosts ИМЯ      принудительно из /etc/hosts
getent -s dns hosts ИМЯ        принудительно через DNS

host ИМЯ                 возвращает A, AAAA, MX
host -t ТИП ИМЯ          конкретный тип записи
host ИМЯ СЕРВЕР          использовать указанный сервер

dig ИМЯ                  подробный поиск A
dig -t ТИП ИМЯ           конкретный тип
dig +short ИМЯ           краткий вывод
dig +noall +answer ИМЯ   только секция Answer
~/.digrc                 параметры dig по умолчанию
```

## Типичные вопросы экзамена

1. Какой файл управляет порядком обращения к источникам баз данных имён? → `/etc/nsswitch.conf`
2. Что означает `[!UNAVAIL=return]` в `/etc/nsswitch.conf`? → Если служба доступна (не недоступна), остановить поиск и не переходить к следующему источнику.
3. Сколько записей `nameserver` может быть в `/etc/resolv.conf`? → До 3.
4. Сколько поисковых доменов можно указать в `/etc/resolv.conf`? → До 6.
5. Что произойдёт, если в `/etc/resolv.conf` указаны и `search`, и `domain`? → Они несовместимы, действует последняя запись в файле.
6. Каков формат записи в `/etc/hosts`? → IP-адрес в первом столбце, затем имя хоста и псевдонимы: `IP имя [псевдоним]`
7. На каком адресе слушает `systemd-resolved`? → `127.0.0.53`
8. Какое ключевое слово в nsswitch.conf подключает `systemd-resolved`? → `resolve`
9. Чем `getent` отличается от `host`/`dig`? → `getent` использует полный стек NSS (учитывает nsswitch.conf), тогда как `host` и `dig` обращаются к DNS напрямую.
10. Какой параметр `getent` принудительно использует `/etc/hosts`? → `-s files`
11. Что по умолчанию возвращает `host wikipedia.org`? → Записи A, AAAA и MX.
12. Какой параметр `host` задаёт тип DNS-записи? → `-t ТИП`
13. Как использовать `host` для запроса к конкретному серверу имён? → Указать IP-адрес или имя сервера последним аргументом: `host ИМЯ СЕРВЕР`.
14. Какой параметр `dig` выводит только IP-адрес? → `+short`
15. В каком файле хранятся параметры `dig` по умолчанию? → `~/.digrc`
16. Какая комбинация параметров `dig` выводит только секции вопроса и ответа? → `dig +noall +answer +question ИМЯ`
17. Что делает `dig -t MX lpi.org`? → Запрашивает MX-записи (Mail Exchanger) для `lpi.org`.
18. Почему изменения в `/etc/resolv.conf` могут не сохраниться? → NetworkManager перезаписывает этот файл при изменении конфигурации сети.

## Упражнения

### Практическое упражнение 1 — Порядок в nsswitch.conf

Измените порядок разрешения имён так, чтобы DNS опрашивался раньше `/etc/hosts`.

<details>
<summary>Ответ</summary>

Отредактируйте строку в `/etc/nsswitch.conf`:

```
hosts:    dns files
```

</details>

---

### Практическое упражнение 2 — Настройка resolv.conf

Настройте `/etc/resolv.conf` для использования DNS-серверов `1.1.1.1` и `8.8.8.8` с поисковым доменом `example.com`.

<details>
<summary>Ответ</summary>

```
nameserver 1.1.1.1
nameserver 8.8.8.8
search example.com
```

</details>

---

### Практическое упражнение 3 — getent и dig

Найдите адрес `myserver.local` через полный стек NSS, а затем только через DNS.

<details>
<summary>Ответ</summary>

```bash
getent hosts myserver.local             # через nsswitch.conf
getent -s dns hosts myserver.local      # только DNS
```

Для DNS-запроса также подойдёт:

```bash
dig myserver.local
```

</details>
