## Что нужно знать

- Перенаправлять stdin, stdout и stderr.
- Передавать вывод одной команды на вход другой через пайп.
- Использовать вывод команды как аргументы для другой команды.
- Одновременно отправлять вывод на экран и в файл.
- Утилиты: `tee`, `xargs`.

---

## Файловые дескрипторы и стандартные потоки

Каждый процесс Linux работает с тремя стандартными каналами обмена данными:

| Канал  | Имя    | Дескриптор | Устройство      |
|--------|--------|------------|-----------------|
| stdin  | ввод   | 0          | /dev/stdin      |
| stdout | вывод  | 1          | /dev/stdout     |
| stderr | ошибки | 2          | /dev/stderr     |

По умолчанию в терминальной сессии клавиатура связана со stdin, а экран монитора со stdout и stderr. Bash позволяет переназначить любой из этих каналов при запуске программы.

## Перенаправление вывода

Символ `>` перенаправляет stdout в файл:

```bash
cat /proc/cpuinfo > /tmp/cpu.txt
```

Это эквивалентно явной записи `1>`, потому что при отсутствии номера Bash подставляет дескриптор 1:

```bash
cat /proc/cpuinfo 1> /tmp/cpu.txt
```

### Перезапись и дополнение

Оператор `>` перезаписывает файл без подтверждения. Оператор `>>` добавляет данные в конец файла, не затрагивая существующее содержимое. Если файла ещё нет, оба оператора создадут его:

```bash
cat /proc/cpuinfo >> /tmp/cpu.txt    # данные добавятся в конец
```

### Перенаправление stderr

Для перехвата stderr перед символом `>` ставится цифра 2:

```bash
cat /proc/cpu_info 2> /tmp/error.txt
cat /tmp/error.txt
# cat: /proc/cpu_info: No such file or directory
```

### Перенаправление stdout и stderr вместе

Для одновременного перенаправления обоих потоков в один файл используют `&>` или `>&`:

```bash
cat /proc/cpuinfo &> /tmp/all.txt
```

Пробелы рядом с амперсандом недопустимы. Пробел перед `&` заставит Bash запустить команду в фоне.

Тот же результат через явную запись:

```bash
cat /proc/cpuinfo > /tmp/all.txt 2>&1
```

Порядок операторов здесь важен: сначала stdout перенаправляется в файл, затем stderr перенаправляется туда же, куда уже указывает stdout.

### Перенаправление между дескрипторами

`1>&2` перенаправляет stdout в stderr. `2>&1` делает обратное: stderr направляется туда, куда идёт stdout.

Это нужно, когда требуется передать сообщения stderr другой программе через пайп, потому что пайп захватывает только stdout. Решение: сначала `2>&1`, потом пайп.

### Дескрипторы выше 2

Программы могут открывать собственные дескрипторы с номерами от 3 и выше. Их можно переназначать той же нотацией:

```bash
./fd 3</tmp/error.txt 4>&1
```

Дескриптор `3<>/tmp/error.txt` открывает файл одновременно на чтение и запись.

### noclobber

По умолчанию `>` перезаписывает файл без предупреждения. Опция `noclobber` запрещает это:

```bash
set -o noclobber    # включить (или set -C)
set +o noclobber    # выключить (или set +C)
```

Даже при активном `noclobber` оператор `>>` работает. Также можно принудительно перезаписать файл через `>|`:

```bash
cat /proc/cpuinfo >| /tmp/cpu.txt
```

Для постоянного эффекта опцию добавляют в `~/.bashrc` или системный профиль. Перенаправление в `/dev/null` работает при включённом `noclobber`, потому что это устройство, а не обычный файл.

## Перенаправление ввода

Символ `<` перенаправляет содержимое файла в stdin процесса. Данные текут справа налево:

```bash
uniq -c < /tmp/error.txt
```

Без явного номера Bash подставляет дескриптор 0, поэтому это эквивалентно `uniq -c 0</tmp/error.txt`.

## Here Document и Here String

**Here Document** позволяет ввести многострочный текст прямо в командной строке и передать его в stdin:

```bash
wc -c <<EOF
How many characters
in this Here document?
EOF
# 43
```

Строка справа от `<<` задаёт завершающий маркер. Ввод заканчивается, когда на отдельной строке появляется этот маркер. Пробелов между `<<` и маркером быть не должно.

**Here String** делает то же самое для одной строки:

```bash
wc -c <<< "How many characters in this Here string?"
# 41
```

Строки с пробелами нужно брать в кавычки, иначе в stdin попадёт только первое слово, а остальные Bash передаст как аргументы.

## Пайпы

Вертикальная черта `|` соединяет stdout одной команды со stdin другой. В отличие от перенаправлений, данные текут слева направо, а цель не файловая система, а другой процесс. Bash запускает все команды конвейера одновременно:

```bash
cat /proc/cpuinfo | wc
# 208 1184 6096
```

Пайпов в одной команде может быть сколько угодно:

```bash
cat /proc/cpuinfo | grep 'model name' | uniq
```

Пайпы и перенаправления можно комбинировать в одной строке:

```bash
grep 'model name' < /proc/cpuinfo | uniq
```

### tee: вывод в файл и на экран одновременно

Пайп и перенаправление взаимно исключают друг друга: один поток, один получатель. Команда `tee` снимает это ограничение. Она принимает данные на stdin, выводит их на stdout и одновременно записывает в файл:

```bash
grep 'model name' < /proc/cpuinfo | uniq | tee cpu_model.txt
```

Результат виден на экране и сохраняется в файл. Флаг `-a` переключает `tee` в режим дополнения вместо перезаписи:

```bash
... | tee -a cpu_model.txt
```

### Пайпы и stderr

Пайп захватывает только stdout. Чтобы захватить и stderr, его нужно сначала перенаправить в stdout:

```bash
make 2>&1 | tee log.txt
```

Без `2>&1` сообщение об ошибке появится на экране, но в `log.txt` не попадёт.

## Подстановка команд

Command substitution позволяет подставить вывод команды как аргумент или значение переменной. Два синтаксиса дают одинаковый результат:

```bash
mkdir `date +%Y-%m-%d`
mkdir $(date +%Y-%m-%d)
```

Сохранение вывода в переменную:

```bash
OS=`uname -o`
echo $OS
# GNU/Linux
```

Синтаксис `$()` предпочтительнее обратных кавычек: он нагляднее при вложениях.

## xargs

Когда вывод команды нужно передать другой не через stdin, а как аргументы командной строки, используют `xargs`. Он читает данные из stdin и передаёт их как аргументы указанной команде:

```bash
find /usr/share/icons -name 'debian*' | xargs identify -format "%f: %wx%h\n"
```

`xargs` собрал все пути, найденные `find`, и передал их как аргументы команде `identify` за один вызов.

### Опции xargs

| Опция    | Смысл                                                  |
|----------|--------------------------------------------------------|
| `-n N`   | Запускать команду с N аргументами за раз               |
| `-L N`   | Использовать N строк ввода на один запуск команды      |
| `-I STR` | Задать строку-заменитель для позиционирования аргумента|
| `-0`     | Использовать null-символ как разделитель               |

```bash
find . -name '*.log' | xargs -n 1 rm
```

### Пути с пробелами: -print0 и -0

Если имена файлов содержат пробелы, стандартный разбор по пробелам сломается. `find -print0` разделяет записи нулевым байтом, а `xargs -0` корректно их читает:

```bash
find . -name '*avi' -print0 | xargs -0 du | sort -n
```

### Позиция аргументов: -I

По умолчанию `xargs` добавляет аргументы в конец команды. Опция `-I` задаёт строку-заменитель, которую можно разместить в любом месте:

```bash
find . -mindepth 2 -name '*.mp4' -print0 | xargs -0 -I PATH mv PATH ./
```

Здесь `PATH` встаёт на место источника в команде `mv`, после чего идёт целевая директория.

## Шпаргалка

```bash
# Перенаправление stdout в файл
command > file.txt

# Добавление stdout в файл
command >> file.txt

# Перенаправление stderr в файл
command 2> errors.txt

# Оба потока в один файл
command &> all.txt
command > all.txt 2>&1

# Только stderr в /dev/null (игнорировать ошибки)
command 2> /dev/null

# Перенаправление stdin из файла
command < input.txt

# Here Document
command <<END
строка 1
строка 2
END

# Here String
command <<< "строка"

# Пайп
cmd1 | cmd2 | cmd3

# tee: экран + файл
cmd | tee file.txt
cmd | tee -a file.txt     # дополнять, не перезаписывать

# stderr через пайп
cmd 2>&1 | tee log.txt

# noclobber
set -o noclobber          # включить
set +o noclobber          # выключить
command >| file.txt       # принудительная перезапись при noclobber

# Command substitution
VAR=$(command)
VAR=`command`

# xargs
find . -name '*.txt' | xargs grep "pattern"
find . -name '*.txt' -print0 | xargs -0 rm
find . -name '*.jpg' -print0 | xargs -0 -I F mv F ./photos/
```

## Типичные вопросы экзамена

**Какой оператор перенаправляет stderr в файл?**
`2>`

**Как перенаправить и stdout, и stderr в один файл?**
`&>` или `> file 2>&1`

**Как записать вывод в файл и одновременно вывести на экран?**
Через `tee`: `cmd | tee file.txt`

**Как добавить данные в существующий файл, не перезаписывая его?**
Через `>>` или `tee -a`

**Что делает `2>&1`?**
Перенаправляет stderr туда, куда в данный момент указывает stdout.

**Чем `xargs` отличается от command substitution?**
Command substitution подставляет вывод как часть командной строки (аргумент или значение). `xargs` читает stdin и явно передаёт его содержимое как аргументы команде, с контролем количества аргументов на вызов.

**Зачем нужна связка `-print0` / `-0`?**
Чтобы корректно обрабатывать имена файлов с пробелами и специальными символами.

**Что такое Here Document?**
Способ передать многострочный текст в stdin команды прямо из командной строки, не создавая временный файл. Обозначается `<<МАРКЕР`.

### Практические упражнения (Guided Exercises)

#### Урок 1: Перенаправления

**1. Команда `cat` умеет работать с бинарными данными. Как с помощью перенаправления отправить содержимое устройства `/dev/sdc` в файл `sdc.img` в текущей директории?**

<details><summary>Ответ</summary>

```bash
cat /dev/sdc > sdc.img
```

`cat` читает `/dev/sdc` как обычный поток данных и направляет его в файл через stdout. Это стандартный способ создать побитовый образ диска.

</details>

---

**2. Как называется стандартный канал, перенаправляемый командой `date 1> now.txt`?**

<details><summary>Ответ</summary>

Стандартный вывод, stdout.

Дескриптор 1 соответствует stdout. Запись `1>` явная, но полностью эквивалентна просто `>`.

</details>

---

**3. При попытке перезаписать файл через перенаправление пользователь получает ошибку об активной опции `noclobber`. Как отключить её для текущей сессии?**

<details><summary>Ответ</summary>

```bash
set +C
# или
set +o noclobber
```

Флаг `+` отменяет опцию, флаг `-` включает. `set -C` и `set -o noclobber` включают защиту.

</details>

---

**4. Каков результат команды `cat <<.>/dev/stdout`?**

<details><summary>Ответ</summary>

Bash войдёт в режим ввода Here Document и будет ждать строк до тех пор, пока на отдельной строке не появится точка `.`. После этого весь введённый текст уйдёт на stdout и появится на экране.

</details>

---

#### Урок 2: Пайпы и command substitution

**1. Команда `date +%Y-%m-%d` показывает текущую дату в формате год-месяц-день. Как сохранить её вывод в переменную `TODAY` с помощью command substitution?**

<details><summary>Ответ</summary>

```bash
TODAY=`date +%Y-%m-%d`
# или
TODAY=$(date +%Y-%m-%d)
```

Синтаксис `$()` предпочтительнее при вложениях.

</details>

---

**2. Как с помощью `echo` отправить содержимое переменной `TODAY` на stdin команды `sed s/-/./g`?**

<details><summary>Ответ</summary>

```bash
echo $TODAY | sed s/-/./g
```

`echo` выводит значение переменной на stdout, пайп передаёт его в stdin `sed`, который заменяет дефисы на точки.

</details>

---

**3. Как использовать вывод команды `date +%Y-%m-%d` как Here String для команды `sed s/-/./g`?**

<details><summary>Ответ</summary>

```bash
sed s/-/./g <<< `date +%Y-%m-%d`
# или
sed s/-/./g <<< $(date +%Y-%m-%d)
```

Here String `<<<` передаёт строку прямо в stdin, без запуска `echo`.

</details>

---

**4. Команда `convert image.jpeg -resize 25% small/image.jpeg` создаёт уменьшенную копию файла и помещает её в подкаталог `small`. Как с помощью `xargs` выполнить эту команду для каждого файла из списка `filelist.txt`?**

<details><summary>Ответ</summary>

```bash
xargs -I IMG convert IMG -resize 25% small/IMG < filelist.txt
# или
cat filelist.txt | xargs -I IMG convert IMG -resize 25% small/IMG
```

Опция `-I IMG` задаёт строку-заменитель `IMG`. `xargs` подставляет каждое имя файла из списка на место `IMG` в двух позициях: источник и путь назначения.

</details>

### Исследовательские упражнения (Explorational Exercises)

#### Урок 1: Перенаправления

**1. Команда `cat /proc/cpu_info 2>1` перенаправляет сообщение об ошибке куда?**

<details><summary>Ответ</summary>

В файл с именем `1` в текущей директории.

Это не перенаправление в дескриптор 1, потому что после оператора `2>` нет амперсанда. Конструкция `2>&1` перенаправляет в дескриптор, а `2>1` создаёт файл с именем `1`.

</details>

---

**2. Будет ли работать перенаправление в `/dev/null` при активной опции `noclobber`?**

<details><summary>Ответ</summary>

Да. `/dev/null` это символьное устройство, а `noclobber` защищает от перезаписи только обычные файлы.

</details>

---

**3. Как передать содержимое переменной `$USER` в stdin команды `sha1sum` без использования `echo`?**

<details><summary>Ответ</summary>

```bash
sha1sum <<< $USER
```

Here String напрямую передаёт строку в stdin без внешней команды.

</details>

---

**4. Ядро Linux хранит символические ссылки на открытые файлы процесса в `/proc/PID/fd/`. Как с помощью этого каталога проверить, в какие файлы пишет nginx с PID 1234?**

<details><summary>Ответ</summary>

```bash
ls -l /proc/1234/fd
```

Каждая ссылка в этом каталоге указывает на реальный файл. Те, что ведут в директории логов, и покажут, куда nginx пишет.

</details>

---

**5. Программа `bc` принимает операции только через stdin. Как отправить в неё операцию `scale=6; 1/3` с помощью Here String?**

<details><summary>Ответ</summary>

```bash
bc <<< "scale=6; 1/3"
```

Here String передаёт строку в stdin `bc`, и тот сразу выполняет вычисление и выводит результат.

</details>

#### Урок 2: Пайпы и xargs

**1. Команды `dd < /dev/sda1 > sda1.img` и `sha1sum < sda1.img > sda1.sha1` запускаются отдельно. Как объединить их в одну команду с помощью `tee`?**

<details><summary>Ответ</summary>

```bash
dd < /dev/sda1 | tee sda1.img | sha1sum > sda1.sha1
```

`dd` читает раздел и передаёт данные через пайп в `tee`. `tee` одновременно записывает поток в `sda1.img` и пропускает его дальше на stdin `sha1sum`, которая считает хэш и сохраняет в `sda1.sha1`.

</details>

---

**2. Команда `find /etc -type f | tar -cJ -f /srv/backup/etc.tar.xz -T -` может давать сбои при путях с пробелами. Какие опции нужно добавить для `find` и `tar`?**

<details><summary>Ответ</summary>

```bash
find /etc -type f -print0 | tar -cJ -f /srv/backup/etc.tar.xz --null -T -
```

`find -print0` разделяет записи нулевым байтом вместо символа новой строки. Опция `--null` для `tar` говорит читать список файлов из `-T` с нулевым разделителем.

</details>

---

**3. Как передать локальный файл `etc.tar.gz` на удалённый хост через `ssh`, не копируя временный файл?**

<details><summary>Ответ</summary>

```bash
cat etc.tar.gz | ssh user@storage "cat > /srv/backup/etc.tar.gz"
# или
ssh user@storage "cat > /srv/backup/etc.tar.gz" < etc.tar.gz
```

`cat` отправляет содержимое файла в stdout, пайп передаёт его в stdin `ssh`. Второй вариант делает то же самое через перенаправление stdin: `ssh` читает локальный файл напрямую и передаёт его потоком в команду на удалённой стороне.

</details>
