> **Вес на экзамене: 5** — LPIC-1 v5, Exam 102

## Что нужно знать

Согласно официальным целям LPIC-1:

- Добавлять, изменять и удалять пользователей и группы.
- Управлять информацией в базах данных паролей и групп.
- Создавать специализированные и ограниченные учётные записи.

Ключевые файлы и команды: `/etc/passwd`, `/etc/shadow`, `/etc/group`, `/etc/gshadow`, `/etc/skel`, `/etc/login.defs`, `useradd`, `usermod`, `userdel`, `groupadd`, `groupmod`, `groupdel`, `passwd`, `chage`, `getent`.

---

## Файлы учётных записей

### /etc/passwd

Каждый локальный пользователь занимает одну строку в `/etc/passwd` с семью полями, разделёнными двоеточием:

```
frank:x:1001:1001:Frank User:/home/frank:/bin/bash
```

| Поле | Пример | Описание |
|---|---|---|
| Имя пользователя | `frank` | Имя для входа |
| Пароль | `x` | `x` означает хранение пароля в `/etc/shadow` |
| UID | `1001` | Числовой идентификатор пользователя |
| GID | `1001` | Идентификатор основной группы |
| GECOS | `Frank User` | Полное имя / комментарий |
| Домашний каталог | `/home/frank` | Домашний каталог пользователя |
| Оболочка | `/bin/bash` | Оболочка по умолчанию при входе |

Системные учётные записи обычно имеют UID ниже 1000, обычные пользователи получают UID от 1000 и выше.

### /etc/shadow

Хранит хэши паролей и настройки устаревания (доступен только для root):

```
frank:$6$...:18600:0:99999:7:::
```

| Поле | Описание |
|---|---|
| Имя пользователя | Имя для входа |
| Зашифрованный пароль | Хэш; префикс `!` означает заблокированную учётную запись |
| Дата последнего изменения | Дней с 01.01.1970; `0` означает смену пароля при следующем входе |
| Минимальный срок | Минимальное число дней до смены пароля |
| Максимальный срок | Максимальное число дней до обязательной смены |
| Период предупреждения | Дней до истечения, когда выводится предупреждение |
| Период неактивности | Дней после истечения до блокировки учётной записи |
| Дата истечения | Дней с 01.01.1970 до блокировки; пустое значение означает бессрочность |
| Зарезервировано | Не используется |

### /etc/group

Каждая группа занимает одну строку с четырьмя полями:

```
db-admin:x:1050:frank,emma
```

| Поле | Описание |
|---|---|
| Имя группы | Название группы |
| Пароль группы | `x` означает хранение в `/etc/gshadow` |
| GID | Числовой идентификатор группы |
| Список участников | Пользователи с дополнительным членством через запятую (основные участники не перечисляются) |

### /etc/gshadow

Теневой файл групп с четырьмя полями: имя группы, зашифрованный пароль (`!` означает запрет `newgrp`), администраторы группы, участники группы.

## Управление пользователями

### useradd

Создаёт новую учётную запись. Основные параметры:

| Параметр | Описание |
|---|---|
| `-c комментарий` | Поле GECOS (полное имя и т. д.) |
| `-d каталог` | Путь к домашнему каталогу |
| `-e ГГГГ-ММ-ДД` | Дата истечения учётной записи |
| `-f дни` | Дней после истечения пароля до блокировки |
| `-g GID` | Основная группа (имя или GID) |
| `-G группа,...` | Дополнительные группы |
| `-k каталог` | Каталог-шаблон (требует `-m`) |
| `-m` | Создать домашний каталог |
| `-M` | Не создавать домашний каталог |
| `-s оболочка` | Оболочка для входа |
| `-u UID` | Задать UID |

Пример: создание пользователя `emma` с домашним каталогом и bash:

```bash
useradd -m -s /bin/bash -c "Emma User" emma
```

### usermod

Изменяет существующую учётную запись. Параметры аналогичны `useradd` с дополнениями:

| Параметр | Описание |
|---|---|
| `-d каталог [-m]` | Сменить домашний каталог; `-m` перемещает содержимое |
| `-G группы [-a]` | Задать дополнительные группы; `-a` добавляет к существующим |
| `-l новое_имя` | Переименовать логин |
| `-L` | Заблокировать учётную запись (добавляет `!` к хэшу пароля в shadow) |
| `-U` | Разблокировать учётную запись (убирает `!`) |

### userdel

Удаляет учётную запись пользователя. Параметр `-r` удаляет также домашний каталог и почтовый ящик:

```bash
userdel -r emma
```

## Управление группами

| Команда | Описание |
|---|---|
| `groupadd -g GID имя` | Создать новую группу с необязательным GID |
| `groupmod -n новое_имя имя` | Переименовать группу |
| `groupmod -g GID имя` | Изменить GID группы |
| `groupdel имя` | Удалить группу |

## Пароли и устаревание — passwd и chage

### passwd

Управляет паролями и блокировкой учётных записей:

| Параметр | Описание |
|---|---|
| `-d` | Удалить пароль (вход без пароля) |
| `-e` | Принудительная смена пароля при следующем входе |
| `-i дни` | Период неактивности после истечения |
| `-l` | Заблокировать учётную запись |
| `-n дни` | Минимальный срок действия пароля |
| `-S` | Показать статус учётной записи |
| `-u` | Разблокировать учётную запись |
| `-w дни` | Дней предупреждения до истечения |
| `-x дни` | Максимальный срок действия пароля |

### chage

Управляет устареванием пароля и сроком действия учётной записи:

| Параметр | Описание |
|---|---|
| `-d ГГГГ-ММ-ДД` | Установить дату последней смены пароля |
| `-E ГГГГ-ММ-ДД` | Установить дату истечения учётной записи |
| `-I дни` | Установить период неактивности |
| `-l` | Показать сведения об устаревании (можно без прав root для своей записи) |
| `-m дни` | Минимальный срок действия пароля |
| `-M дни` | Максимальный срок действия пароля |
| `-W дни` | Дней предупреждения до истечения |

### Соответствия passwd и chage

| passwd | chage | Смысл |
|---|---|---|
| `-n` | `-m` | Минимальный срок действия пароля |
| `-x` | `-M` | Максимальный срок действия пароля |
| `-w` | `-W` | Дни предупреждения |
| `-i` | `-I` | Дни неактивности |
| `-S` | `-l` | Показать статус |

**Заблокировать** учётную запись: `usermod -L имя` или `passwd -l имя`.
**Разблокировать** учётную запись: `usermod -U имя` или `passwd -u имя`.

## Шаблон и настройки по умолчанию

### /etc/skel

При создании домашнего каталога командой `useradd -m` содержимое `/etc/skel` копируется в него. Здесь хранятся файлы конфигурации по умолчанию (`.bashrc`, `.profile` и т. д.).

### /etc/login.defs

Системные значения по умолчанию для создания пользователей и групп:

| Директива | Смысл |
|---|---|
| `UID_MIN` / `UID_MAX` | Диапазон UID для обычных пользователей |
| `GID_MIN` / `GID_MAX` | Диапазон GID для обычных групп |
| `CREATE_HOME` | Создавать ли домашний каталог по умолчанию |
| `USERGROUPS_ENAB` | Создавать ли одноимённую группу для каждого нового пользователя |
| `MAIL_DIR` | Каталог почтового ящика |
| `PASS_MAX_DAYS` | Максимальный срок действия пароля по умолчанию |
| `PASS_MIN_DAYS` | Минимальный срок действия пароля по умолчанию |
| `PASS_MIN_LEN` | Минимальная длина пароля |
| `PASS_WARN_AGE` | Дней предупреждения по умолчанию |

## Запросы к базам данных — getent

`getent` извлекает записи из баз данных NSS (Name Service Switch) и работает как с локальными файлами, так и с сетевыми каталогами (LDAP, NIS):

```bash
getent passwd emma
# emma:x:1020:1020:User Emma:/home/emma:/bin/bash

getent group db-admin
# db-admin:x:1050:frank,emma
```

## Типичные вопросы экзамена

1. Какая команда создаёт пользователя `frank` с домашним каталогом и оболочкой bash? → `useradd -m -s /bin/bash frank`
2. Что делает `usermod -L пользователь`? → Блокирует учётную запись, добавляя `!` перед хэшем пароля в `/etc/shadow`.
3. Что эквивалентно `passwd -l`? → `usermod -L`, оба блокируют учётную запись.
4. Как добавить пользователя в группу `staff`, не удаляя существующие группы? → `usermod -aG staff пользователь`
5. Какая команда устанавливает максимальный срок действия пароля в 90 дней? → `chage -M 90 пользователь` или `passwd -x 90 пользователь`
6. В каком файле хранятся хэши паролей на Linux? → `/etc/shadow`
7. Что означает префикс `!` в поле пароля `/etc/shadow`? → Учётная запись заблокирована.
8. Что такое `/etc/skel`? → Каталог, содержимое которого копируется в домашний каталог нового пользователя при его создании.
9. Какая команда удаляет пользователя `emma` вместе с домашним каталогом? → `userdel -r emma`
10. Что делает `getent passwd emma`? → Запрашивает базу данных NSS passwd для пользователя emma, работает с локальными файлами и сетевыми каталогами.
11. Сколько полей в `/etc/passwd`? → **7**: имя пользователя, заглушка пароля, UID, GID, GECOS, домашний каталог, оболочка.
12. Сколько полей в `/etc/shadow`? → **9**: имя пользователя, хэш, дата изменения, мин, макс, предупреждение, неактивность, истечение, зарезервировано.
13. Какой диапазон UID обычно используется для обычных пользователей? → **1000 и выше** (системные учётные записи имеют UID ниже 1000).
14. В каком файле задаются системные значения по умолчанию для создания пользователей, например `UID_MIN`? → `/etc/login.defs`
15. Какая команда выводит сведения об устаревании пароля учётной записи? → `chage -l пользователь`
16. Какой параметр `chage` задаёт дату истечения учётной записи? → `-E ГГГГ-ММ-ДД`
17. Что делает `userdel` без флага `-r`? → Удаляет учётную запись, оставляя домашний каталог и почтовый ящик нетронутыми.
18. Как заставить пользователя сменить пароль при следующем входе? → `passwd -e пользователь` или `chage -d 0 пользователь`

## Упражнения

### Практическое упражнение 1 — Создание учётной записи разработчика

Создайте пользователя `dev1` с UID 2001, домашним каталогом `/home/dev1`, оболочкой bash и дополнительной группой `developers`. Какая команда решает эту задачу?

<details>
<summary>Ответ</summary>

```bash
useradd -u 2001 -m -s /bin/bash -G developers dev1
```

Флаги: `-u 2001` задаёт UID, `-m` создаёт домашний каталог, `-s` указывает оболочку, `-G developers` назначает дополнительную группу.

</details>

### Практическое упражнение 2 — Блокировка и разблокировка учётной записи

Учётная запись сотрудника должна быть временно отключена на период отпуска. Назовите два способа её заблокировать и объясните, как разблокировать.

<details>
<summary>Ответ</summary>

**Блокировка:**
- `usermod -L пользователь` — добавляет `!` перед хэшем пароля
- `passwd -l пользователь` — аналогичный эффект

**Разблокировка:**
- `usermod -U пользователь`
- `passwd -u пользователь`

Оба метода изменяют поле пароля в `/etc/shadow`.

</details>

### Практическое упражнение 3 — Политика устаревания пароля

Задайте для пользователя `frank` политику: максимальный срок 60 дней, минимальный 2 дня, предупреждение за 7 дней, блокировка через 14 дней после истечения.

<details>
<summary>Ответ</summary>

```bash
chage -M 60 -m 2 -W 7 -I 14 frank
```

Эквивалентно через `passwd`:
```bash
passwd -x 60 -n 2 -w 7 -i 14 frank
```

</details>

### Исследовательское упражнение 1 — Чтение /etc/shadow

Дана запись в shadow: `emma:$6$abc:19000:5:90:7:30:19365:`

Объясните каждое поле.

<details>
<summary>Ответ</summary>

| Поле | Значение | Смысл |
|---|---|---|
| Имя | `emma` | Имя для входа |
| Пароль | `$6$abc` | Хэш SHA-512 |
| Дата изменения | `19000` | День 19000 с 01.01.1970 |
| Мин. срок | `5` | Нельзя менять 5 дней |
| Макс. срок | `90` | Смена обязательна каждые 90 дней |
| Предупреждение | `7` | Предупреждение за 7 дней |
| Неактивность | `30` | Блокировка через 30 дней после истечения |
| Истечение | `19365` | Учётная запись истекает в день 19365 от начала эпохи |
| Зарезервировано | (пусто) | Не используется |

</details>

---

### Исследовательское упражнение 2 — Членство в группе

Пользователя `kevin` нужно добавить в группу `sysadmin` как дополнительную, не затронув существующие группы. Какая команда решает эту задачу?

<details>
<summary>Ответ</summary>

```bash
usermod -aG sysadmin kevin
```

Флаг `-a` обязателен: без него `-G` заменит все текущие дополнительные группы только группой `sysadmin`.

</details>
