## Что нужно знать

Согласно официальным целям LPIC-1:

- Проверять систему на наличие файлов с установленными битами SUID/SGID.
- Устанавливать и изменять пароли и параметры срока их действия.
- Настраивать `su`.
- Использовать `sudo` для повышения привилегий.
- Управлять входами в систему с помощью `lsof`, `fuser`, `nmap`, `netstat`, `last`, `who`, `w`.
- Устанавливать ограничения ресурсов с помощью `ulimit` и `/etc/security/limits.conf`.

Ключевые файлы и команды: `find`, `passwd`, `fuser`, `lsof`, `nmap`, `chage`, `netstat`, `sudo`, `su`, `usermod`, `ulimit`, `who`, `last`, `lastb`, `w`.

## Специальные биты прав: SUID и SGID

### SUID (Set User ID) — восьмеричное 4000

При установке на исполняемый файл процесс запускается с правами владельца файла, а не вызывающего пользователя.

```bash
chmod u+s /usr/bin/program     # символьная запись
chmod 4755 /usr/bin/program    # восьмеричная запись
```

Пример: `/usr/bin/passwd` имеет бит SUID, поэтому любой пользователь может менять свой пароль.

### SGID (Set Group ID) — восьмеричное 2000

На исполняемом файле: запускается с правами группы-владельца.
На каталоге: новые файлы наследуют группу каталога.

```bash
chmod g+s /shared/dir
chmod 2755 /shared/dir
```

### Поиск файлов с SUID/SGID

```bash
find / -perm -4000 -type f        # все файлы с SUID
find / -perm -2000 -type f        # все файлы с SGID
find / -perm /6000 -type f        # SUID или SGID (хотя бы один бит)
find / -perm -6000 -type f        # SUID и SGID (оба бита)
```

Префикс `-` означает «установлены как минимум эти биты», а `/` означает «любой из этих битов».

## Управление паролями

### passwd

```bash
passwd                       # изменить собственный пароль
passwd emma                  # изменить пароль другому пользователю (только root)
passwd -S emma               # показать состояние пароля
passwd -l emma               # заблокировать учётную запись (добавить ! перед хешем)
passwd -u emma               # разблокировать учётную запись
passwd -e emma               # устаревить пароль (принудительная смена при входе)
passwd -d emma               # удалить пароль (беспарольный вход)
```

Формат вывода `passwd -S`: `ИМЯ СТАТУС ДАТА МИН МАКС ПРЕДУПРЕЖДЕНИЕ НЕАКТИВНОСТЬ`

Коды статуса: `P` — пароль установлен, `L` — заблокирован, `NP` — пароля нет.

### chage — изменение параметров срока действия

```bash
chage -l emma                # показать параметры срока действия
chage -m 7 emma              # минимальное число дней между сменами
chage -M 90 emma             # максимальное число дней до обязательной смены
chage -d 0 emma              # принудительная смена при следующем входе
chage -d 2024-01-01 emma     # установить дату последней смены
chage -W 14 emma             # число дней предупреждения до истечения срока
chage -I 30 emma             # дней неактивности после истечения до блокировки
chage -E 2025-12-31 emma     # дата истечения учётной записи (ГГГГ-ММ-ДД)
chage -E -1 emma             # дата истечения: никогда
```

### usermod — блокировка/разблокировка

```bash
usermod -L emma              # заблокировать (аналог passwd -l)
usermod -U emma              # разблокировать
usermod -f 30 emma           # дней неактивности (аналог chage -I)
usermod --inactive 30 emma   # длинная форма флага -f
```

## su — смена пользователя

```bash
su emma                      # переключиться на emma, сохранить текущую среду
su - emma                    # переключиться на emma, загрузить полную среду входа
su                           # переключиться на root, сохранить среду
su -                         # переключиться на root с полной средой входа
```

Флаг `-` (или `-l` / `--login`) загружает профиль целевого пользователя и устанавливает `HOME`, `SHELL`, `USER`, `LOGNAME`.

## Просмотр открытых файлов и портов

### lsof — список открытых файлов

```bash
lsof -i                      # все сетевые соединения
lsof -i4                     # только IPv4
lsof -i6                     # только IPv6
lsof -i :22                  # процессы, использующие порт 22
lsof -i :22 -P               # то же, показывать номер порта, а не имя
lsof -i @192.168.1.1         # соединения с конкретным IP
lsof -i TCP                  # только TCP-соединения
```

### fuser — процессы, использующие файл или порт

```bash
fuser -v /var/log/syslog     # подробно: какой процесс использует файл
fuser -n tcp 22              # какой процесс слушает TCP-порт 22
fuser -n udp 53              # UDP-порт 53
fuser -k /var/log/syslog     # отправить SIGKILL процессам, использующим файл
```

Коды доступа в выводе `fuser -v`: `c` (текущий каталог), `e` (исполняемый файл), `f` (открытый файл), `r` (корневой каталог), `m` (файл, отображённый в память).

## netstat

```bash
netstat -l                   # только слушающие сокеты
netstat -t                   # TCP-сокеты
netstat -u                   # UDP-сокеты
netstat -n                   # числовые адреса/порты (без DNS)
netstat -e                   # расширенная информация
netstat -tulnp               # TCP/UDP слушающие, числовые, с именами процессов
netstat -r                   # таблица маршрутов
```

Без флага `-l` `netstat` показывает только установленные соединения.

## nmap — сетевой сканер

```bash
nmap 192.168.1.1             # сканировать один хост
nmap 192.168.1.1 192.168.1.2 # несколько хостов
nmap 192.168.1.1-254         # диапазон адресов
nmap 192.168.1.0/24          # подсеть
nmap 192.168.1.*             # диапазон с маской

nmap -p 22 192.168.1.1       # конкретный порт
nmap -p 22,80,443 host       # список портов
nmap -p 1-1024 host          # диапазон портов
nmap -p- host                # все 65535 портов
nmap -F host                 # быстрое сканирование (100 портов)
nmap -v host                 # подробный вывод
nmap --exclude 192.168.1.5 192.168.1.0/24   # исключить хост
```

## ulimit — ограничения ресурсов

`ulimit` является встроенной командой bash для установки ограничений ресурсов на процесс.

```bash
ulimit -a                    # показать все текущие мягкие ограничения
ulimit -Ha                   # показать все жёсткие ограничения

ulimit -S -n 1024            # установить мягкий лимит на открытые файлы
ulimit -H -n 4096            # установить жёсткий лимит на открытые файлы
ulimit -f 100                # максимальный размер файла в 512-байтных блоках
ulimit -b                    # максимальный размер буфера сокета
ulimit -l                    # максимальный объём заблокированной памяти
ulimit -m                    # максимальный объём резидентной памяти
ulimit -v                    # максимальный объём виртуальной памяти
ulimit -u                    # максимальное число процессов пользователя
```

Мягкие ограничения пользователь может повысить до жёстких. Жёсткие ограничения может повысить только root.

### /etc/security/limits.conf

Постоянные ограничения, сохраняемые после перезагрузки. Применяются PAM при входе в систему.

```
emma    hard    nproc   50
@staff  soft    nofile  4096
*       hard    core    0
```

Формат: `ПОЛЬЗОВАТЕЛЬ/ГРУППА  ТИП  ПАРАМЕТР  ЗНАЧЕНИЕ`
Типы: `soft`, `hard`, `both`.

## История входов и активные сеансы

### last и lastb

```bash
last                         # успешные входы (из /var/log/wtmp)
last emma                    # входы конкретного пользователя
last reboot                  # перезагрузки системы
lastb                        # неудачные попытки входа (из /var/log/btmp)
```

### who

```bash
who                          # текущие пользователи системы
who -b                       # время последней загрузки системы
who -r                       # текущий уровень запуска
who -H                       # вывести заголовки столбцов
```

### w

```bash
w                            # кто вошёл и чем занимается
```

Столбцы вывода `w`: `USER`, `TTY`, `FROM` (удалённый хост), `LOGIN@`, `IDLE`, `JCPU` (суммарное CPU задания), `PCPU` (CPU текущего процесса), `WHAT` (текущая команда).

## sudo — повышение привилегий

```bash
sudo command                 # выполнить команду от имени root
sudo -u emma command         # выполнить команду от имени emma
sudo -l                      # вывести разрешённые команды для текущего пользователя
sudo -s                      # открыть оболочку root
```

Кэш учётных данных: по умолчанию sudo не запрашивает пароль повторно в течение 15 минут (`timestamp_timeout = 15`).

### /etc/sudoers

Редактировать только через `visudo`, который проверяет синтаксис перед сохранением.

```
# Основной формат:
# ПОЛЬЗОВАТЕЛЬ  ХОСТ=(КЕМ)  КОМАНДА

root    ALL=(ALL:ALL) ALL
emma    ALL=(ALL:ALL) ALL
%sudo   ALL=(ALL:ALL) ALL          # группа sudo (Debian)
%wheel  ALL=(ALL:ALL) ALL          # группа wheel (RHEL)

# Без запроса пароля:
emma    ALL=(ALL) NOPASSWD: ALL

# Конкретная команда:
emma    ALL=(root) /usr/bin/systemctl restart nginx
```

### Псевдонимы в sudoers

```
Host_Alias   WEBSERVERS = web1, web2
User_Alias   ADMINS = alice, bob
Cmnd_Alias   SERVICES = /usr/bin/systemctl, /usr/sbin/service
Runas_Alias  WEBUSERS = www-data, nginx

ADMINS  WEBSERVERS=(WEBUSERS) SERVICES
```

| Тип псевдонима | Назначение |
|---|---|
| `Host_Alias` | Группы хостов |
| `User_Alias` | Группы пользователей |
| `Cmnd_Alias` | Группы команд |
| `Runas_Alias` | Группы пользователей для запуска (по UID) |

### /etc/sudoers.d/

Каталог для дополнительных правил sudoers. Файлы из него включаются автоматически.

```bash
visudo -f /etc/sudoers.d/emma
```

### Добавление пользователя в группу sudo

```bash
usermod -aG sudo emma        # Debian/Ubuntu
usermod -aG wheel emma       # RHEL/CentOS/Fedora
```

## Типичные вопросы экзамена

1. Какое восьмеричное значение обозначает бит SUID? → `4000` (символьно: `u+s`)
2. Какая команда `find` находит все SUID-файлы в системе? → `find / -perm -4000 -type f`
3. Что означает префикс `-` в `find -perm -4000`? → Установлены как минимум указанные биты (бит SUID должен быть установлен, другие биты могут быть любыми).
4. Что выводит `passwd -S emma` и что означает каждое поле? → Имя пользователя, статус (P/L/NP), дата последней смены, мин. дни, макс. дни, дни предупреждения, дни неактивности.
5. Как принудить пользователя сменить пароль при следующем входе через `chage`? → `chage -d 0 emma`
6. Как установить дату истечения учётной записи в «никогда» через `chage`? → `chage -E -1 emma`
7. В чём разница между `su emma` и `su - emma`? → `su - emma` загружает полную среду входа (HOME, SHELL, профиль), тогда как `su emma` сохраняет текущую среду.
8. Какой параметр `lsof` показывает соединения на конкретном порту? → `-i :ПОРТ` (например, `lsof -i :22`)
9. Как найти с помощью `fuser`, какой процесс слушает TCP-порт 80? → `fuser -n tcp 80`
10. Какой параметр `nmap` сканирует только 100 основных портов? → `-F`
11. В чём разница между мягким и жёстким ограничением `ulimit`? → Мягкое ограничение пользователь может повысить до жёсткого. Жёсткое ограничение может повысить только root.
12. Где настраиваются постоянные ограничения ресурсов? → `/etc/security/limits.conf`
13. Откуда `last` берёт данные? → `/var/log/wtmp`
14. Какая команда показывает неудачные попытки входа? → `lastb` (данные из `/var/log/btmp`)
15. Какие столбцы показывает `w`? → USER, TTY, FROM, LOGIN@, IDLE, JCPU, PCPU, WHAT
16. Как выполнить команду от имени конкретного пользователя через `sudo`? → `sudo -u имя_пользователя команда`
17. Каким инструментом нужно редактировать `/etc/sudoers`? → `visudo`, он проверяет синтаксис перед сохранением.
18. Как разрешить группе `ops` выполнять все команды без пароля в sudoers? → `%ops ALL=(ALL) NOPASSWD: ALL`

## Упражнения

### Практическое упражнение 1 — Поиск файлов SUID/SGID

Найдите все файлы в системе, у которых установлены оба бита: SUID и SGID.

<details>
<summary>Ответ</summary>

```bash
find / -perm -6000 -type f
```

`-6000` означает, что оба бита должны быть установлены. Используйте `/6000`, если нужны файлы хотя бы с одним из битов.

</details>

### Практическое упражнение 2 — Параметры срока действия пароля

Настройте пользователя `alice` так, чтобы её пароль истекал через 60 дней, предупреждение появлялось за 7 дней, а учётная запись блокировалась через 14 дней после истечения.

<details>
<summary>Ответ</summary>

```bash
chage -M 60 alice
chage -W 7 alice
chage -I 14 alice
chage -l alice
```

</details>

### Практическое упражнение 3 — Ограничения ресурсов

Установите постоянное жёсткое ограничение на 50 процессов для пользователя `bob`.

<details>
<summary>Ответ</summary>

Добавьте строку в `/etc/security/limits.conf`:

```
bob  hard  nproc  50
```

</details>

### Практическое упражнение 4 — Настройка sudoers

Разрешите группе `developers` перезапускать службу `nginx` без ввода пароля.

<details>
<summary>Ответ</summary>

Отредактируйте `/etc/sudoers` через `visudo`:

```
%developers ALL=(root) NOPASSWD: /usr/bin/systemctl restart nginx
```

</details>
