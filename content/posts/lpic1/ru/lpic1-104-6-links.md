## Что нужно знать

- Создавать жёсткие и символьные ссылки командой `ln`.
- Отличать их по выводу `ls -l` и `ls -i`.
- Понимать разницу между копированием и связыванием.
- Знать ограничения жёстких ссылок: только в пределах одной ФС, нельзя на каталог.

---

## Понятие ссылки и инода

### Что такое инод

Инод (inode) — структура данных ФС, хранящая атрибуты файла: права, владелец, размер, временные метки, список блоков на диске. **Имя файла в иноде не хранится.** Имя существует как запись в каталоге, связывающая строку (имя) с номером инода.

Всё взаимодействие ядра с файлом идёт через инод.

### Два типа ссылок

**Жёсткая ссылка (hard link):** дополнительная запись в каталоге, указывающая на тот же инод, что и исходное имя. Ссылка и оригинал полностью равноправны — нельзя определить, какое из имён «оригинал».

**Символьная ссылка (symlink/soft link):** отдельный файл со своим инодом, в содержимом которого лежит путь к цели. При обращении ядро читает этот путь и перенаправляет к цели.

## Жёсткие ссылки

### Создание

```
ln ЦЕЛЬ ИМЯ_ССЫЛКИ
```

```bash
ln target.txt /home/carol/Documents/hardlink
```

Если опустить имя ссылки — она появится в текущем каталоге с тем же именем, что и цель.

### Счётчик ссылок и инод

```bash
ls -li
# 3806696 -r--r--r-- 2 carol carol 111702 Jun 7 10:13 hardlink
# 3806696 -r--r--r-- 2 carol carol 111702 Jun 7 10:13 target.txt
```

- Первый столбец — номер инода. Одинаковый у обоих имён → одни данные.
- Число `2` после прав — счётчик жёстких ссылок (link count).
- По умолчанию у обычного файла счётчик = 1, у каталога = 2.
- Каждая новая жёсткая ссылка прибавляет 1.

Изменения через одно имя видны через все остальные — данные на диске одни.

### Перемещение и удаление

Жёсткая ссылка ведёт себя как обычный файл: удаляется через `rm`, перемещается через `mv`. Так как она указывает на инод, а не на путь, её можно двигать без риска сломать связь.

`rm` на одном из имён только уменьшает счётчик. Данные живы, пока счётчик > 0.

### Ограничения

- **Нельзя** создать жёсткую ссылку на **каталог** (защита от циклов в дереве).
- **Нельзя** создать жёсткую ссылку **через границу файловой системы/раздела**.

```bash
ln /media/user/FlashA/file.txt ~/link
# ln: failed to create hard link ... : Invalid cross-device link
```

## Символьные ссылки

### Создание

```
ln -s ЦЕЛЬ ИМЯ_ССЫЛКИ
```

```bash
ln -s target.txt /home/carol/Documents/softlink
```

В отличие от жёстких — можно на каталог и можно пересекать границы ФС.

### Идентификация в выводе ls

```bash
ls -lh
# lrwxrwxrwx 1 carol carol 12 Jun 7 10:14 softlink -> target.txt
```

Признаки: первый символ `l`, после имени стрелка `->` и путь к цели.

### Права доступа

Символьная ссылка всегда показывает `lrwxrwxrwx`. Реальный доступ через ссылку определяется **правами цели**.

### Относительные и абсолютные пути

Если цель указана без полного пути — путь интерпретируется **относительно расположения ссылки**. При перемещении ссылки в другой каталог относительная ссылка сломается.

```bash
# Ссылка создана в ~/Documents/
ln -s original.txt softlink      # относительная — опасно!

# После mv softlink ../ ссылка сломается
# Решение: абсолютный путь
ln -s /home/carol/Documents/original.txt softlink
```

### Битые ссылки

При удалении цели ссылка остаётся, но указывает в никуда → **битая (broken) ссылка**. При создании файла с тем же именем по тому же пути ссылка снова заработает.

### Удаление и перемещение

Как обычный файл: `rm softlink`, `mv softlink /новый/путь`.

## Копирование против связывания

| Способ | Инод | Место на диске | Связь с оригиналом |
|---|---|---|---|
| `cp` (копия) | новый | новое | независима |
| `ln` (жёсткая) | общий с целью | общее | полная, данные одни |
| `ln -s` (символьная) | свой, новый | несколько байт (длина пути) | через путь к цели |

---

## Сравнительная таблица

| Свойство | Жёсткая ссылка | Символьная ссылка |
|---|---|---|
| Команда | `ln T L` | `ln -s T L` |
| Тип в `ls -l` | `-` (как файл) | `l` |
| Свой инод | нет, общий с целью | да |
| Увеличивает link count | да | нет |
| Через границы ФС | нет | да |
| На каталог | нет | да |
| При удалении цели | данные живы (счётчик ≥1) | ссылка ломается |
| Размер в `ls -l` | равен размеру цели | длина строки пути |

## Применение в системном администрировании

- **Версионирование ПО:** `/usr/bin/python` → `python3.11`. Перенацелить ссылку — переключить версию без правки скриптов.
- **nginx `sites-enabled`:** файлы — символьные ссылки на `sites-available/`.
- **Инкрементальные бэкапы** (`rsync --link-dest`): неизменённые файлы — жёсткие ссылки на предыдущий снимок, экономия места при полном виде каждого снимка.
- **Удобный доступ:** ссылка в `~` на `/var/log/myapp/`.

## Шпаргалка

```bash
# Жёсткая ссылка
ln target.txt hardlink

# Символьная ссылка
ln -s target.txt softlink

# Символьная ссылка с абсолютным путём (безопасно)
ln -s /home/user/file.txt /home/user/links/file

# Перезаписать существующую ссылку
ln -sf newtarget.txt softlink

# Показать инод и счётчик ссылок
ls -li

# Показать только инод
ls -i

# Удалить ссылку
rm link

# Переместить ссылку
mv link /новый/путь
```

## Типичные вопросы экзамена

1. Что произойдёт при удалении цели жёсткой ссылки? → Данные останутся доступными через ссылку: счётчик упадёт до 1, но не до 0.
2. Что произойдёт при удалении цели символьной ссылки? → Ссылка станет битой: ошибка `No such file or directory`.
3. Можно ли создать жёсткую ссылку на каталог? → **Нет**, только символьную.
4. Можно ли создать жёсткую ссылку через границу ФС? → **Нет**: ошибка `Invalid cross-device link`. Только символьную.
5. Какой ключ `ls` показывает номер инода? → **`-i`**.
6. Какие права показывает символьная ссылка в `ls -l`? → Всегда **`lrwxrwxrwx`**. Реальные права — от цели.
7. Чем копия отличается от жёсткой ссылки? → Копия занимает отдельный инод и место на диске; жёсткая ссылка разделяет инод и данные с целью.
8. Как создать `c.txt` с тем же инодом, что и `a.txt`? → `ln a.txt c.txt`.
9. Увеличивает ли символьная ссылка счётчик ссылок цели? → **Нет**.
10. Почему относительная символьная ссылка ломается при перемещении? → Путь к цели интерпретируется относительно расположения ссылки; в новом месте такого файла нет.
