## Временные зоны

Временные зоны представляют собой регионы Земли с одинаковым местным временем, определяемым как смещение от **UTC** (Coordinated Universal Time). Аббревиатура GMT (Greenwich Mean Time) используется как синоним UTC в именах с указанием смещения.

- `GMT-5` означает, что регион на 5 часов отстаёт от UTC.
- `GMT+3` означает, что UTC на 3 часа позади, то есть регион на 3 часа впереди UTC.

Серверы и облачные сервисы обычно настраивают аппаратные часы на UTC, оставляя коррекцию временных зон отдельным пользователям.

### Проверка текущей временной зоны

```bash
$ date
Mon Oct 21 10:45:21 -03 2019
```

Смещение `-03` показывает, что система на 3 часа отстаёт от UTC (зона GMT-3).

```bash
$ timedatectl
               Local time: Sat 2019-10-19 17:53:18 -03
           Universal time: Sat 2019-10-19 20:53:18 UTC
                 RTC time: Sat 2019-10-19 20:53:18
                Time zone: America/Sao_Paulo (-03, -0300)
System clock synchronized: yes
```

`timedatectl` доступен на системах с systemd.

### Файлы временных зон

| Путь | Назначение |
|---|---|
| `/etc/timezone` | Содержит имя временной зоны, например `America/Sao_Paulo` или `Etc/GMT+3` |
| `/etc/localtime` | Символическая ссылка на файл данных активной зоны в `/usr/share/zoneinfo/` |
| `/usr/share/zoneinfo/` | Каталог с файлами данных всех доступных зон |

Имена с указанием смещения UTC должны включать `Etc`, например `Etc/GMT+3` (не просто `GMT+3`).

```bash
$ cat /etc/timezone
America/Sao_Paulo
```

`/etc/localtime` указывает на `/usr/share/zoneinfo/America/Sao_Paulo`.

### tzselect — интерактивный подбор зоны

`tzselect` интерактивно проводит через меню с выбором континента, страны и региона, чтобы найти правильное имя зоны. Системные настройки не изменяет, только выводит значение TZ:

```bash
$ tzselect
# ... интерактивное меню ...
# Результат:
TZ='America/Sao_Paulo'; export TZ
```

Добавьте эту строку в `~/.profile`, чтобы изменение применялось в будущих сессиях.

### Переменная TZ

`TZ` переопределяет системную временную зону только для текущей сессии, не затрагивая системные файлы:

```bash
$ env TZ='Africa/Cairo' date
Mon Oct 21 15:45:21 EET 2019
```

## Язык и кодировка символов

### Переменная LANG

`LANG` является основной переменной локали. Формат: `язык_РЕГИОН.КОДИРОВКА`, например `pt_BR.UTF-8` (бразильский португальский, UTF-8) или `en_US.UTF-8`.

```bash
$ echo $LANG
pt_BR.UTF-8
```

Системное значение по умолчанию задаётся в `/etc/locale.conf`:

```bash
$ cat /etc/locale.conf
LANG=pt_BR.UTF-8
```

Изменить на системах с systemd:

```bash
localectl set-locale LANG=en_US.UTF-8
```

Для переопределения только в текущей сессии используйте `export LANG=...` или `export LC_ALL=...`.

### Переменные LC_*

Управляют отдельными аспектами локали и имеют приоритет над `LANG` для своей категории:

| Переменная | Управляет |
|---|---|
| `LC_COLLATE` | Алфавитный порядок (сортировка файлов и строк) |
| `LC_CTYPE` | Классификация символов (прописные, строчные, цифры) |
| `LC_MESSAGES` | Язык сообщений программ (программы GNU) |
| `LC_MONETARY` | Формат денежных единиц |
| `LC_NUMERIC` | Разделители целой и дробной части числа |
| `LC_TIME` | Формат даты и времени |
| `LC_PAPER` | Стандартный формат бумаги |
| `LC_ALL` | Переопределяет все остальные `LC_*` и `LANG` |

```bash
$ locale
LANG=pt_BR.UTF-8
LC_CTYPE="pt_BR.UTF-8"
LC_NUMERIC=pt_BR.UTF-8
...
LC_ALL=
```

`LC_ALL` по умолчанию пустая. При установке в `en_US.UTF-8` принудительно задаёт английскую локаль для всех аспектов:

```bash
$ env LC_ALL=en_US.UTF-8 date
Mon Oct 21 10:45:21 -03 2019
```

В скриптах рекомендуется использовать `LANG=C`, чтобы избежать зависящего от локали поведения при сортировке и сравнении символов.

### Кодировки символов

| Кодировка | Описание |
|---|---|
| ASCII | 7-битная, 128 символов; только латинские буквы, цифры, знаки препинания |
| ISO-8859 | Семейство 8-битных кодировок (например ISO-8859-1 для Latin-1); расширяет ASCII для западноевропейских языков |
| Unicode | Универсальный стандарт, присваивающий уникальный номер каждому символу |
| UTF-8 | Кодировка Unicode переменной ширины; обратно совместима с ASCII; стандарт на современном Linux |

### iconv — конвертация кодировок

`iconv` преобразует файл из одной кодировки в другую:

```bash
iconv -f ISO-8859-1 -t UTF-8 original.txt > converted.txt
```

| Параметр | Описание |
|---|---|
| `-f КОДИРОВКА` | Исходная кодировка (`--from-code`) |
| `-t КОДИРОВКА` | Целевая кодировка (`--to-code`) |
| `-o файл` | Выходной файл вместо перенаправления stdout |
| `-l` / `--list` | Список всех поддерживаемых кодировок |

Добавьте `//TRANSLIT` к целевой кодировке, чтобы транслитерировать символы, которые не существуют в целевом наборе:

```bash
iconv -f UTF-8 -t ASCII//TRANSLIT -o ascii.txt readme.txt
```

## Полезные команды для экзамена

```
Файлы временных зон:
  /etc/timezone          имя зоны (напр., America/Sao_Paulo, Etc/GMT+3)
  /etc/localtime         ссылка на /usr/share/zoneinfo/<зона>
  /usr/share/zoneinfo/   файлы данных временных зон

  LANG=язык_РЕГИОН.КОДИРОВКА   основная локаль (напр., en_US.UTF-8)
  LC_ALL                  переопределяет все LC_* и LANG
  LC_COLLATE              порядок сортировки
  LC_CTYPE                типы символов (прописные/строчные)

  /etc/locale.conf        системные параметры локали
```

## Типичные вопросы экзамена

1. В каком файле хранится имя системной временной зоны? → `/etc/timezone`
2. Что такое `/etc/localtime`? → Символическая ссылка на активный файл данных зоны в `/usr/share/zoneinfo/`.
3. Где хранятся файлы данных всех временных зон? → `/usr/share/zoneinfo/`
4. Что делает `tzselect`? → Интерактивно помогает найти временную зону и выводит значение `TZ`; системные настройки не изменяет.
5. Какая команда показывает временную зону с подробностями от systemd? → `timedatectl`
6. Как временно запустить команду в другой временной зоне? → `env TZ='Зона/Имя' команда`
7. Каков формат переменной `LANG`? → `язык_РЕГИОН.КОДИРОВКА`, например `en_US.UTF-8`
8. Что делает `LC_ALL`? → Переопределяет все остальные переменные `LC_*` и `LANG`.
9. Какая переменная `LC_*` управляет разделителями числа? → `LC_NUMERIC`
10. Какая переменная `LC_*` управляет алфавитной сортировкой? → `LC_COLLATE`
11. В каком файле задаются системные параметры локали? → `/etc/locale.conf`
12. Какая команда меняет системную локаль на системах с systemd? → `localectl set-locale LANG=en_US.UTF-8`
13. Что такое UTF-8? → Кодировка Unicode переменной ширины, обратно совместимая с ASCII, стандарт на современном Linux.
14. Какая команда конвертирует файл из ISO-8859-1 в UTF-8? → `iconv -f ISO-8859-1 -t UTF-8 входной.txt > выходной.txt`
15. Что делает `//TRANSLIT` в `iconv`? → Заменяет символы, не представимые в целевой кодировке, похожими символами вместо знаков вопроса.
16. Как получить список всех кодировок, поддерживаемых `iconv`? → `iconv -l` или `iconv --list`
17. Что означает временная зона `Etc/GMT+3`? → Смещение от UTC на +3 часа (имена с указанием смещения UTC должны использовать префикс `Etc/`).
18. Почему в скриптах рекомендуется `LANG=C`? → Локаль C обеспечивает предсказуемое поведение при сортировке и сравнении символов, независимо от настроек системной локали.

## Упражнения

### Практическое упражнение 1 — Чтение временной зоны по выводу date

Команда `date` выводит: `Mon Oct 21 18:45:21 +05 2019`. Какова временная зона в нотации `Etc/`?

<details>
<summary>Ответ</summary>

`Etc/GMT+5` — смещение `+05` означает, что регион на 5 часов впереди UTC. В нотации `Etc/` это `Etc/GMT+5`.

</details>

---

### Практическое упражнение 2 — Настройка /etc/localtime

Чтобы сделать `Europe/Brussels` временной зоной по умолчанию, на что должен указывать `/etc/localtime`?

<details>
<summary>Ответ</summary>

`/etc/localtime` должен быть символической ссылкой на `/usr/share/zoneinfo/Europe/Brussels`.

</details>

---

### Практическое упражнение 3 — Конвертация через iconv

Преобразуйте файл `old.txt` в кодировке WINDOWS-1252 в UTF-8 и сохраните как `new.txt`.

<details>
<summary>Ответ</summary>

```bash
iconv -f WINDOWS-1252 -t UTF-8 -o new.txt old.txt
```

</details>

---

### Исследовательское упражнение 1 — Временная зона для сессии

Сделайте `Pacific/Auckland` временной зоной по умолчанию только для текущей сессии.

<details>
<summary>Ответ</summary>

```bash
export TZ=Pacific/Auckland
```

Временная зона действует только в текущей сессии. Для постоянного применения добавьте строку в `~/.profile` или `~/.bash_profile`.

</details>

---

### Исследовательское упражнение 2 — Транслитерация через iconv

Преобразуйте файл `readme.txt` в кодировке UTF-8 в файл `ascii.txt` в кодировке ASCII, сохранив символы читаемыми.

<details>
<summary>Ответ</summary>

```bash
iconv -f UTF-8 -t ASCII//TRANSLIT -o ascii.txt readme.txt
```

`//TRANSLIT` заменяет символы, не существующие в ASCII, похожими символами ASCII вместо знаков вопроса.

</details>
