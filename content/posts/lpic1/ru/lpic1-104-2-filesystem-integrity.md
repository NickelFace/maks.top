> **Вес на экзамене: 2** — LPIC-1 v5, Экзамен 101

## Что нужно знать

- Проверять целостность файловых систем.
- Мониторить свободное место и inodes.
- Чинить простые проблемы файловых систем.

Утилиты из objectives: `du`, `df`, `fsck`, `e2fsck`, `mke2fs`, `tune2fs`, `xfs_repair`, `xfs_fsr`, `xfs_db`.

---

## Введение

Современные файловые системы Linux журналируемые. Каждая операция сначала пишется во внутренний журнал, потом исполняется. Если её прерывает паника ядра или отключение питания, состояние восстанавливается по журналу. Журналирование сильно сократило потребность в ручных проверках, но они всё ещё бывают нужны.

## Проверка использования диска: du

Команда `du` (disk usage) показывает занятое место и работает рекурсивно.

### Базовый запуск

Без параметров выводит число килобайтных блоков для текущего каталога и всех подкаталогов:

```bash
$ du
4816 .
```

С `-h` — в удобочитаемом формате:

```bash
$ du -h
4.8M .
```

### Параметры du

| Параметр | Что делает |
|---|---|
| `-h` | удобочитаемые единицы (K, M, G) |
| `-a` | показать каждый файл, не только каталоги |
| `-s` | только итоговая строка (без подкаталогов) |
| `-S` | разделить учёт текущего каталога и подкаталогов |
| `-c` | добавить строку `total` в конце |
| `-d N` | глубина рекурсии N уровней |
| `--exclude="ШАБЛОН"` | исключить файлы по шаблону |

**`-a`** — показывает размер каждого файла:

```bash
$ du -ah
432K ./geminoid.jpg
508K ./Linear_B_Hero.jpg
60K  ./Stranger3_Titulo.png
4.8M .
```

**`-S` vs итог по умолчанию:** по умолчанию du включает подкаталоги в итог родителя. С `-S` (заглавная) итог для каждого каталога считается отдельно — видно, сколько весят файлы именно в нём:

```bash
$ du -h           $ du -Sh
4.8M ./Temp       4.8M ./Temp
6.0M .            1.3M .
```

С `-Sc` добавляется строка total:

```bash
$ du -Shc
4.8M ./Temp
1.3M .
6.0M total
```

**Регистр важен:** малое `-s` и заглавное `-S` — разные параметры.

**`-d N`** — ограничить глубину рекурсии:

```bash
$ du -h -d1
224K ./somedir
232K .
```

Скрытые уровни учитываются в итоге, просто не выводятся отдельно.

**`--exclude`** — исключить файлы по шаблону:

```bash
$ du -ah --exclude="*.bin"
```

## Проверка свободного места: df

Команда `df` работает на уровне файловой системы. Перечисляет все смонтированные ФС: общий размер, занятое, свободное место, процент использования, точку монтирования.

### Базовый запуск

```bash
$ df -h
Filesystem Size Used Avail Use% Mounted on
udev       2.9G 0    2.9G  0%   /dev
/dev/sda1  106G 25G  76G   25%  /
```

### Параметры df

| Параметр | Что делает |
|---|---|
| `-h` | удобочитаемые единицы |
| `-i` | показать inodes вместо блоков |
| `-T` | добавить колонку с типом ФС |
| `-t TYPE` | фильтр — показать только ФС данного типа |
| `-x TYPE` | фильтр — исключить ФС данного типа |
| `--output=FIELDS` | задать список и порядок колонок |

**`-i`** — переключиться на inodes:

```bash
$ df -i
Filesystem Inodes  IUsed  IFree   IUse% Mounted on
/dev/sda6  6766592 307153 6459439 5%    /
```

**`-T`** — добавить тип ФС в вывод:

```bash
$ df -hT
Filesystem Type     Size Used Avail Use% Mounted on
/dev/sda1  ext4     106G 25G  76G   25%  /
```

**`-t` vs `-T` — важное различие для экзамена:**

- `-T` (заглавная) добавляет колонку Type в вывод.
- `-t TYPE` (строчная) работает как **фильтр** — оставляет только ФС указанного типа.

```bash
df -ht ext4      # только ext4
df -hx tmpfs     # без tmpfs
```

### Кастомизация через `--output`

Задаёт список полей и их порядок:

```bash
$ df -h --output=target,source,fstype,pcent
Mounted on Filesystem Type     Use%
/dev/       udev       devtmpfs 0%
/           /dev/sda1  ext4     25%
```

Поля для блоков: `source`, `fstype`, `size`, `used`, `avail`, `pcent`, `target`.

Поля для inodes: `itotal`, `iused`, `iavail`, `ipcent`.

```bash
$ df --output=source,fstype,itotal,iused,ipcent
Filesystem Type  Inodes  IUsed  IUse%
/dev/sda1  ext4  7069696 318651 5%
```

## Обслуживание ext2, ext3, ext4

### fsck

`fsck` проверяет ФС на ошибки и чинит их. **Никогда не запускать на смонтированной ФС — приводит к потере данных.**

```bash
fsck /dev/sdb1
```

`fsck` сама не проверяет: она вызывает нужную утилиту по типу ФС. Без указания типа предполагается ext2/3/4 и вызывается `e2fsck`.

Указать тип явно:

```bash
fsck -t vfat /dev/sdc
```

Подсказка: наберите `fsck` и дважды нажмите Tab — увидите все доступные варианты.

| Параметр | Что делает |
|---|---|
| `-A` | проверить все ФС из `/etc/fstab` |
| `-C` | показать прогресс-бар (только ext2/3/4) |
| `-N` | вывести, что было бы сделано, без реальной проверки |
| `-R` | вместе с `-A` — пропустить корневую ФС |
| `-V` | подробный вывод |

### e2fsck

Специализированная утилита для ext2/3/4. Имена `fsck.ext2`, `fsck.ext3`, `fsck.ext4` — ссылки на `e2fsck`.

По умолчанию работает интерактивно. При ошибке спрашивает: `y` (исправить), `n` (пропустить), `a` (исправить эту и все последующие).

Неинтерактивные режимы:

| Параметр | Что делает |
|---|---|
| `-p` | автоматически исправлять ошибки; при серьёзной ошибке описать и выйти |
| `-y` | отвечать `y` на все вопросы |
| `-n` | отвечать `n`, ФС только для чтения — никаких изменений |
| `-f` | принудительно проверить, даже если ФС помечена как `clean` |

### tune2fs

Просматривает и изменяет параметры ext2/3/4.

Просмотр текущих параметров:

```bash
tune2fs -l /dev/sda1
```

Вывод включает: метку, UUID, размер блока, число inodes, mount count, maximum mount count, дату последней проверки.

| Параметр | Что делает |
|---|---|
| `-l` | показать все параметры ФС |
| `-c N` | максимум монтирований без проверки; `-1` отключает |
| `-C N` | установить текущее значение счётчика монтирований |
| `-i N[d|m|y]` | интервал между проверками: `d` дни, `m` месяцы, `y` годы; `0` отключает |
| `-L LABEL` | метка ФС (до 16 символов) |
| `-U UUID` | UUID ФС |
| `-j` | добавить журнал (конвертирует ext2 → ext3) |
| `-J ОПЦИИ` | параметры журнала |
| `-e BEHAVIOUR` | поведение при ошибке: `continue`, `remount-ro`, `panic` |
| `-f` | режим грубой силы — выполнить даже при ошибках |

**Регистр важен:** `-c` (строчная) задаёт максимум монтирований, `-C` (заглавная) устанавливает текущее значение счётчика.

**Mount count:** счётчик увеличивается на 1 при каждом монтировании. Когда достигает maximum mount count, система запускает `e2fsck` на следующей загрузке.

**Поведение при ошибке (`-e`):**

- `continue` — продолжить нормально (по умолчанию)
- `remount-ro` — перемонтировать только для чтения; останавливает запись, предотвращает дальнейшие ошибки
- `panic` — вызвать панику ядра

### Конвертация ext2 в ext3

ext3 = ext2 + журнал. Добавить журнал:

```bash
tune2fs -j /dev/sda1
```

Параметры журнала через `-J`:

| Параметр | Что делает |
|---|---|
| `-J size=N` | размер журнала в МБ |
| `-J location=POS` | позиция на диске (с суффиксом M, G) |
| `-J device=DEV` | хранить журнал на отдельном устройстве |

Несколько опций комбинируются через запятую:

```bash
tune2fs -J size=10,location=100M,device=/dev/sdb1 /dev/sda1
```

## Обслуживание XFS

### xfs_repair

Аналог `fsck` для XFS. Первый шаг — проверка без правок:

```bash
xfs_repair -n /dev/sdb1
```

`-n` означает no modify: ФС проверяется, ошибки выводятся, ничего не пишется. Если ошибки найдены, повторить без `-n`:

```bash
xfs_repair /dev/sdb1
```

| Параметр | Что делает |
|---|---|
| `-n` | только проверить, не чинить |
| `-l LOGDEV` | устройство с внешним log section |
| `-r RTDEV` | устройство с realtime section |
| `-m N` | ограничить использование памяти N МБ |
| `-d` | dangerous mode — чинить ФС, смонтированные только для чтения |
| `-v` | verbose; повтор `-v -v` увеличивает подробность |
| `-L` | обнулить повреждённый журнал (последнее средство, риск потери данных) |

`xfs_repair` не умеет чинить ФС с грязным журналом без `-L`.

### xfs_db

Просмотр внутренних параметров и элементов XFS:

```bash
xfs_db /dev/sdb1
```

Открывает интерактивный режим. `help` — список команд, `help COMMAND` — подробности.

### xfs_fsr

Дефрагментация XFS. Без аргументов работает 2 часа и пытается дефрагментировать все смонтированные XFS из `/etc/mtab`. В некоторых дистрибутивах требует отдельной установки.

## Шпаргалка

```bash
# Размер только файлов в текущем каталоге, без подкаталогов
du -Shd 0

# du с исключением шаблона
du -ah --exclude="*.tmp"

# Глубина рекурсии 1 уровень
du -h -d 1

# Только ext4 ФС с нужными колонками
df -t ext4 --output=source,target,itotal,iavail,pcent

# Тип ФС в выводе
df -hT

# Inodes
df -i

# Проверка ext с автоисправлением
e2fsck -p /dev/sdc1

# Принудительная проверка чистой ФС
e2fsck -f /dev/sdc1

# Конвертация ext2 в ext3 со сбросом счётчика и меткой
tune2fs -j -C 0 -L UserData /dev/sdb1

# Изменить интервал проверки на 30 дней
tune2fs -i 30d /dev/sda1

# Изменить max mount count
tune2fs -c 200 /dev/sda1

# Просмотр параметров ext
tune2fs -l /dev/sda1

# Журнал на отдельном устройстве + интервал 30 дней
tune2fs -j /dev/sdb1 -J device=/dev/sdc1 -i 30d

# Проверка XFS без правок
xfs_repair -n /dev/sdb1

# XFS с внешним журналом, без правок
xfs_repair -l /dev/sdc1 -n /dev/sda1
```

## Типичные вопросы экзамена

1. Какая команда показывает использование диска для всех смонтированных ФС? → **`df`**.
2. Какой параметр `e2fsck` автоматически исправляет ошибки без вопросов? → **`-p`**.
3. Какой параметр `tune2fs` меняет интервал между проверками в днях? → **`-i`**.
4. Какой параметр `tune2fs` меняет максимум монтирований без проверки? → **`-c`** (строчная).
5. Чем `-T` отличается от `-t` у `df`? → `-T` добавляет колонку Type в вывод; `-t TYPE` фильтрует — показывает только ФС указанного типа.
6. Как добавить журнал к ext2? → `tune2fs -j /dev/XXX` — конвертирует ФС в ext3.
7. Как проверить XFS без внесения изменений? → `xfs_repair -n /dev/XXX`.
8. Аналог `fsck` для XFS? → **`xfs_repair`**.
9. Как разместить журнал ext на отдельном устройстве? → `tune2fs -J device=/dev/XXX`.
10. Какой параметр `du` показывает размер только файлов текущего каталога? → **`-S`** (заглавная), часто с `-d 0`.
11. Какой процент места по умолчанию резервируется для root в новой ext4? → **5%**.
12. Что означает `-n` у `xfs_repair`? → no modify — только проверить, без записи на диск.
13. Чем `-c` отличается от `-C` у `tune2fs`? → `-c` задаёт максимум монтирований, `-C` устанавливает текущее значение счётчика.
14. Что такое `mount count` в `tune2fs -l`? → счётчик, сколько раз ФС была смонтирована; при достижении maximum mount count запускается проверка.
15. Что делает `-e remount-ro` в `tune2fs`? → при ошибке ФС перемонтируется только для чтения, предотвращая дальнейшую запись.

## Упражнения

### Упражнение 1 — Размер только файлов текущего каталога через du

С помощью `du` проверить, сколько места занимают именно файлы в текущем каталоге, без подкаталогов.

<details>
<summary>Ответ</summary>

```bash
du -Shd 0
```

`-S` разделяет учёт текущего каталога и его подкаталогов. `-d 0` ограничивает глубину рекурсии нулём. Без `-S` итог всё равно включал бы вложенные файлы, поэтому оба параметра нужны вместе.

</details>

---

### Упражнение 2 — df с фильтром по ext4 и кастомными колонками

Вывести информацию по каждой ext4-ФС. Колонки в порядке: устройство, точка монтирования, общее число inodes, число доступных inodes, процент использованного места.

<details>
<summary>Ответ</summary>

```bash
df -t ext4 --output=source,target,itotal,iavail,pcent
```

`-t ext4` — фильтр по типу. `--output=...` — задаёт список и порядок колонок.

`pcent` показывает процент использованного места. Отдельного поля «процент свободного» у `df` нет.

</details>

---

### Упражнение 3 — e2fsck в неинтерактивном режиме с автоисправлением

Запустить `e2fsck` для `/dev/sdc1` в неинтерактивном режиме с автоматическим исправлением большинства ошибок.

<details>
<summary>Ответ</summary>

```bash
e2fsck -p /dev/sdc1
```

При `-p` утилита чинит то, что может без участия человека. При серьёзной ошибке — описывает её и завершается. Альтернативы: `-y` отвечает yes на всё, `-n` отвечает no и открывает ФС только для чтения.

</details>

### Упражнение 4 — Конвертация ext2 в ext3 со сбросом счётчика и меткой

`/dev/sdb1` — ext2. Конвертировать в ext3, сбросить счётчик монтирований, поменять метку на `UserData`.

<details>
<summary>Ответ</summary>

```bash
tune2fs -j -C 0 -L UserData /dev/sdb1
```

- `-j` — добавить журнал (конвертация ext2 → ext3)
- `-C 0` (заглавная) — сбросить текущий счётчик монтирований
- `-L UserData` — установить метку

Регистр важен: `-c` задаёт максимум монтирований, `-C` устанавливает текущее значение счётчика.

</details>

---

### Упражнение 5 — Проверка XFS без внесения изменений

Проверить XFS-файловую систему на ошибки без реального ремонта.

<details>
<summary>Ответ</summary>

```bash
xfs_repair -n /dev/sdb1
```

`-n` — no modify: ФС проверяется, ошибки выводятся, на диск ничего не пишется.

</details>

### Упражнение 6 — Эффект tune2fs -c 9 при mount count = 8

ext4-ФС на `/dev/sda1`. По выводу `tune2fs -l`: `Mount count: 8`, `Maximum mount count: -1`. Что произойдёт на следующей загрузке после `tune2fs -c 9 /dev/sda1`?

<details>
<summary>Ответ</summary>

Команда установит максимум монтирований в 9. Поскольку текущее значение — 8, на следующей загрузке счётчик станет 9, достигнет лимита, и система автоматически запустит проверку ФС.

Изначальное значение `-1` означало, что проверка по счётчику была отключена.

</details>

### Упражнение 7 — Разбор вывода du -h и расчёт размера файлов

Вывод команды `du -h`:

```
216K ./somedir/anotherdir
224K ./somedir
232K .
```

Сколько места занимают только файлы в текущем каталоге? Как переписать команду, чтобы это было видно явно?

<details>
<summary>Ответ</summary>

Из суммарных 232 КБ подкаталог `somedir` со всем содержимым весит 224 КБ. В текущем каталоге без подкаталогов: 232 − 224 = **8 КБ**.

Чтобы видеть это явно:

```bash
du -Sh
```

`-S` разделяет каталоги в учёте — строка для текущего каталога покажет только его собственные файлы. Дополнительно `-d 0` уберёт строки про подкаталоги.

</details>

### Упражнение 8 — Эффект tune2fs -j -J device= -i 30d

Что произойдёт с ext2-ФС `/dev/sdb1`, если выполнить:

```bash
tune2fs -j /dev/sdb1 -J device=/dev/sdc1 -i 30d
```

<details>
<summary>Ответ</summary>

К `/dev/sdb1` добавится журнал — ФС конвертируется в ext3. Журнал будет храниться на отдельном устройстве `/dev/sdc1`. Автоматическая проверка ФС будет запускаться раз в 30 дней.

- `-j` — создаёт журнал, поднимает версию до ext3
- `-J device=` — размещает журнал на отдельном диске
- `-i 30d` — задаёт интервал автоматической проверки

</details>

### Упражнение 9 — Проверка XFS с внешним log section

Проверить XFS `/dev/sda1`, у которой log section находится на `/dev/sdc1`, без реального ремонта.

<details>
<summary>Ответ</summary>

```bash
xfs_repair -l /dev/sdc1 -n /dev/sda1
```

`-l` (строчная L) указывает устройство с внешним журналом. Не путать с `-L` (заглавная), которая **обнуляет** повреждённый журнал.

`-n` — всегда безопасен, подходит для первичной диагностики.

</details>

### Упражнение 10 — Разница между -T и -t у df

В чём разница между параметрами `-T` и `-t` у команды `df`?

<details>
<summary>Ответ</summary>

- `-T` (заглавная) — добавляет колонку с типом ФС в вывод. Все ФС остаются в выводе.
- `-t TYPE` (строчная) — работает как **фильтр**: в выводе остаются только ФС указанного типа.

Аналогичная пара: `-x TYPE` исключает ФС указанного типа.

</details>
