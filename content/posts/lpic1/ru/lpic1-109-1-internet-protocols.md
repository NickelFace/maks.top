## Адресация IPv4

IPv4-адреса — 32-битные числа, записанные в виде четырёх десятичных октетов, разделённых точками (например `192.168.1.100`).

### Классы адресов

| Класс | Диапазон первого октета | Маска по умолчанию | Назначение |
|---|---|---|---|
| A | 1–126 | /8 | Крупные сети |
| B | 128–191 | /16 | Средние сети |
| C | 192–223 | /24 | Малые сети |
| D | 224–239 | — | Многоадресная рассылка (multicast) |
| E | 240–255 | — | Зарезервировано |
| — | 127.0.0.0/8 | — | Петлевой интерфейс (127.0.0.1) |

### Приватные диапазоны (RFC 1918)

Эти адреса не маршрутизируются в публичном интернете и для выхода наружу требуют NAT.

| Диапазон | CIDR | Класс |
|---|---|---|
| 10.0.0.0 – 10.255.255.255 | 10.0.0.0/8 | A |
| 172.16.0.0 – 172.31.255.255 | 172.16.0.0/12 | B |
| 192.168.0.0 – 192.168.255.255 | 192.168.0.0/16 | C |

### Маски подсети и CIDR

Маска подсети определяет, какие биты адреса относятся к сети, а какие — к узлу.

- `/24` = `255.255.255.0` — 256 адресов, 254 используемых узла
- `/25` = `255.255.255.128` — 128 адресов, 126 используемых узлов
- Используемых узлов = 2ⁿ − 2, где n — количество битов хоста

**Адрес сети**: все биты хоста равны 0 (первый адрес диапазона).  
**Широковещательный адрес**: все биты хоста равны 1 (последний адрес диапазона).

Пример: `192.168.1.0/24` — сеть `192.168.1.0`, широковещательный `192.168.1.255`, используемые `192.168.1.1–192.168.1.254`.

### Специальные адреса

- **Маршрут по умолчанию**: `0.0.0.0/0` — совпадает с любым назначением, используется как шлюз последней надежды.
- **NAT** (Network Address Translation): позволяет нескольким узлам с приватными адресами совместно использовать один публичный IP.
- **TTL** (Time to Live): уменьшается на каждом маршрутизаторе. Пакет отбрасывается при достижении 0, предотвращая петли маршрутизации.

## Протоколы транспортного уровня

### TCP — Transmission Control Protocol

- Протокол с установлением соединения: перед передачей данных выполняется трёхэтапное рукопожатие (SYN → SYN-ACK → ACK).
- Надёжная доставка: гарантирует получение, порядок и контроль ошибок.
- Используется HTTP, HTTPS, SSH, SMTP, FTP.

### UDP — User Datagram Protocol

- Протокол без установления соединения: отправляет датаграммы без предварительного соединения.
- Ненадёжная доставка: нет гарантий.
- Меньше накладных расходов, используется там, где скорость важнее надёжности (DNS-запросы, NTP, потоковое видео).

### ICMP — Internet Control Message Protocol

- Не транспортный протокол, передаёт управляющие и диагностические сообщения.
- Используется `ping` (эхо-запрос/ответ) и `traceroute`.
- Без портов, работает на сетевом уровне.

## IPv6

IPv6-адреса — 128-битные числа, записанные восемью группами по четыре шестнадцатеричных цифры, разделёнными двоеточиями:

```
2001:0db8:0000:0000:0000:0000:0000:0001
```

### Правила сокращения

1. Убрать ведущие нули в каждой группе: `0db8` → `db8`, `0001` → `1`.
2. Заменить одну последовательность подряд идущих нулевых групп на `::` (только один раз в адресе).

```
2001:db8::1
fe80::1
::1       (петлевой)
```

### Типы адресов

| Тип | Описание |
|---|---|
| Глобальный одноадресный | 2000::/3 — публично маршрутизируемый |
| Локально-канальный одноадресный | fe80::/10 — не маршрутизируется за пределы канала |
| Петлевой | ::1 |
| Многоадресный | ff00::/8 |
| Anycast | Один адрес на нескольких интерфейсах; отвечает ближайший |

### Отличия от IPv4

| Параметр | IPv4 | IPv6 |
|---|---|---|
| Размер адреса | 32 бита | 128 бит |
| Широковещание | Есть | Нет (вместо него — multicast) |
| Настройка адресов | DHCP или статически | SLAAC, DHCPv6 или статически |
| Предел переходов | Поле TTL | Поле Hop Limit |
| ARP | Есть | Заменён NDP (Neighbor Discovery Protocol) |
| Запись в URL | `http://192.168.1.1/` | `http://[::1]/` (обязательны скобки) |

## Типичные вопросы экзамена

1. Какова маска подсети по умолчанию для адреса класса B? → `/16` (255.255.0.0)
2. Какой приватный диапазон относится к классу A? → `10.0.0.0/8`
3. Какой широковещательный адрес у сети `192.168.5.0/24`? → `192.168.5.255`
4. Сколько используемых узлов в подсети `/26`? → 62 (2⁶ − 2)
5. Что означает маршрут по умолчанию `0.0.0.0/0`? → Маршрут последней надежды, совпадает с любым назначением.
6. Какой порт использует HTTPS? → 443
7. Какой порт использует SMTP? → 25
8. Какой порт использует DNS? → 53 (TCP и UDP)
9. Какой порт использует SNMP Trap? → 162/UDP
10. В каком файле хранится соответствие имён служб и портов? → `/etc/services`
11. Чем TCP отличается от UDP? → TCP — с установлением соединения и надёжный, UDP — без соединения и ненадёжный.
12. Какой протокол использует `ping`? → ICMP
13. Какой петлевой адрес в IPv6? → `::1`
14. Какой префикс используется для локально-канальных адресов IPv6? → `fe80::/10`
15. Что заменяет ARP в IPv6? → NDP (Neighbor Discovery Protocol)
16. Как сокращаются IPv6-адреса? → Убирают ведущие нули в группе, одну серию подряд идущих нулевых групп заменяют на `::`.
17. Как записывается IPv6-адрес в URL? → В скобках: `http://[2001:db8::1]/`
18. Что такое SLAAC? → Stateless Address Autoconfiguration — механизм автоматической настройки IPv6-адреса без DHCP.

## Упражнения

### Практическое упражнение 1 — Подсетевая адресация

Администратор использует сеть `172.16.10.0/28`. Какой широковещательный адрес и сколько используемых узлов?

<details>
<summary>Ответ</summary>

`/28` — 28 сетевых битов, 4 бита хоста. 2⁴ = 16 адресов.

Сеть: `172.16.10.0`  
Широковещательный: `172.16.10.15`  
Используемые узлы: 14 (адреса `172.16.10.1–172.16.10.14`)

</details>

---

### Практическое упражнение 2 — Приватные адреса

Определите, приватный или публичный каждый адрес: `10.0.0.1`, `172.15.0.1`, `172.16.0.1`, `192.168.100.1`, `8.8.8.8`.

<details>
<summary>Ответ</summary>

- `10.0.0.1` — приватный (10.0.0.0/8)
- `172.15.0.1` — публичный (172.15.x.x выходит за 172.16.0.0/12)
- `172.16.0.1` — приватный (172.16.0.0/12)
- `192.168.100.1` — приватный (192.168.0.0/16)
- `8.8.8.8` — публичный

</details>

---

### Практическое упражнение 3 — Сокращение IPv6

Сократите адрес `2001:0db8:0000:0000:0001:0000:0000:0001`.

<details>
<summary>Ответ</summary>

1. Убрать ведущие нули: `2001:db8:0:0:1:0:0:1`
2. Заменить самую длинную серию нулевых групп на `::`: `2001:db8::1:0:0:1`

</details>

### Исследовательское упражнение 1 — Выбор протокола

Вы создаёте приложение для потоковой передачи видео в реальном времени, где допустимы единичные потери пакетов, но важна минимальная задержка. Какой транспортный протокол выбрать?

<details>
<summary>Ответ</summary>

UDP — без установления соединения, меньше накладных расходов. Потерянные пакеты не повторяются, что допустимо для потокового видео, где задержка важнее абсолютной надёжности.

</details>

### Исследовательское упражнение 2 — Поиск порта

Какой командой найти номер порта для службы `imaps`?

<details>
<summary>Ответ</summary>

```bash
grep imaps /etc/services
```

Вывод покажет `imaps 993/tcp`.

</details>
