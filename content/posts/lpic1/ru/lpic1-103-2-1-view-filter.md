## Текстовые потоки и фильтры

В Linux практически всё строится вокруг текста. Конфиги, логи, вывод команд, таблицы данных, скрипты — всё это текст. Фильтры читают текст из стандартного ввода или файла, что-то с ним делают и отдают результат в стандартный вывод. Дальше этот вывод можно передать в следующий фильтр через конвейер `|`.

Конвейер выглядит так:

```bash
cat /etc/passwd | cut -d: -f1 | sort | uniq
```

Каждая команда получает на вход результат предыдущей. Это и есть философия Unix: маленькие инструменты, каждый делает одно, но делает хорошо.

## Просмотр и вывод файлов

### cat

`cat` (concatenate) выводит содержимое одного или нескольких файлов. Если передать несколько имён, выведет их подряд.

```bash
cat file.txt                 # вывести файл
cat file1.txt file2.txt      # склеить два файла
cat -n file.txt              # с нумерацией строк (все строки)
cat -b file.txt              # нумеровать только непустые строки
cat -A file.txt              # показать спецсимволы: $ в конце строк, ^I для табов
cat > newfile.txt            # записать ввод с клавиатуры в файл (Ctrl+D для завершения)
```

### head и tail

`head` показывает начало файла, `tail` — конец. По умолчанию 10 строк.

```bash
head file.txt                # первые 10 строк
head -n 20 file.txt          # первые 20 строк
head -20 file.txt            # то же самое, короткий синтаксис
head -c 100 file.txt         # первые 100 байт

tail file.txt                # последние 10 строк
tail -n 5 file.txt           # последние 5 строк
tail -f /var/log/syslog      # следить за файлом в реальном времени (follow)
tail -n +5 file.txt          # вывести начиная с 5-й строки до конца
```

`tail -f` особенно полезен при отладке: открыл в одном терминале лог, в другом запускаешь команды и сразу видишь что происходит.

### less

`less` — постраничный просмотр файла. В отличие от `more`, можно листать назад.

```bash
less file.txt
less +G file.txt             # открыть сразу в конце файла
```

Навигация внутри `less`:
- `Space` / `f` — следующая страница
- `b` — предыдущая страница
- `g` — начало файла
- `G` — конец файла
- `/шаблон` — поиск вперёд
- `?шаблон` — поиск назад
- `n` / `N` — следующий / предыдущий результат поиска
- `q` — выход

### nl

`nl` нумерует строки файла. Гибче, чем `cat -n`.

```bash
nl file.txt                  # нумеровать только непустые строки
nl -b a file.txt             # нумеровать все строки (включая пустые)
nl -b t file.txt             # нумеровать только непустые (по умолчанию)
nl -v 10 file.txt            # начать нумерацию с 10
nl -i 5 file.txt             # шаг нумерации 5 (10, 15, 20...)
nl -w 3 -s ': ' file.txt     # ширина поля 3, разделитель ': '
```

### od

`od` (octal dump) выводит файл в восьмеричном, шестнадцатеричном или другом формате. Полезен для просмотра бинарных файлов и скрытых символов.

```bash
od file.txt                  # восьмеричный вывод (по умолчанию)
od -c file.txt               # символьный вывод с escape-последовательностями
od -x file.txt               # шестнадцатеричный (hex) вывод
od -t x1 file.txt            # hex побайтово
od -An -c file.txt           # без адресов, только символы
```

Пример: `echo "AB" | od -c` покажет `A`, `B`, `\n` — виден символ перевода строки.

## Работа с колонками и полями

### cut

`cut` вырезает части строк по символу-разделителю или по позиции символа.

```bash
# Вырезать по разделителю (-d) и номеру поля (-f)
cut -d: -f1 /etc/passwd          # первый столбец (имена пользователей)
cut -d: -f1,3 /etc/passwd        # первый и третий столбцы
cut -d: -f1-3 /etc/passwd        # столбцы с 1 по 3
cut -d' ' -f2 file.txt           # второе слово в каждой строке

# Вырезать по позиции символа (-c)
cut -c1-10 file.txt              # символы с 1 по 10
cut -c5 file.txt                 # только 5-й символ каждой строки
cut -c1,5,10 file.txt            # символы 1, 5 и 10

# Вырезать по байту (-b)
cut -b1-5 file.txt               # байты с 1 по 5
```

Важно: `cut` работает только с одним разделителем. Для нескольких пробелов подряд лучше использовать `awk`.

### paste

`paste` склеивает строки из нескольких файлов горизонтально, добавляя разделитель между ними.

```bash
paste file1.txt file2.txt        # склеить столбцами, разделитель таб
paste -d, file1.txt file2.txt    # использовать запятую как разделитель
paste -d: file1.txt file2.txt    # использовать двоеточие
paste -s file.txt                # транспонировать: строки файла в одну строку
```

Пример: если `names.txt` содержит имена, а `ages.txt` — возрасты, `paste -d, names.txt ages.txt` даст CSV.

## Сортировка и уникальность

### sort

`sort` сортирует строки текстового файла или потока.

```bash
sort file.txt                    # алфавитная сортировка
sort -r file.txt                 # обратный порядок
sort -n file.txt                 # числовая сортировка
sort -rn file.txt                # числовая в обратном порядке
sort -k2 file.txt                # сортировка по второму полю
sort -k2 -t: file.txt            # по второму полю, разделитель ':'
sort -k2,2n file.txt             # по второму полю числово
sort -u file.txt                 # удалить дубликаты при сортировке
sort -f file.txt                 # без учёта регистра
sort -h file.txt                 # сортировка по «человеческим» размерам (1K, 2M)
```

Разница между `sort -n` и `sort`: без `-n` строка "10" идёт перед "9" (лексикографически 1 < 9), с `-n` правильно 9 < 10.

### uniq

`uniq` удаляет (или выводит) повторяющиеся строки. Работает только с уже отсортированным вводом, поэтому почти всегда идёт после `sort`.

```bash
sort file.txt | uniq             # удалить дубликаты
sort file.txt | uniq -c          # показать количество повторений
sort file.txt | uniq -d          # показать только дублированные строки
sort file.txt | uniq -u          # показать только уникальные строки
sort file.txt | uniq -i          # без учёта регистра
```

Пример для подсчёта частоты слов в тексте:

```bash
tr ' ' '\n' < text.txt | sort | uniq -c | sort -rn | head
```

## Полезные команды для экзамена

| Команда | Что делает |
|---------|------------|
| `cat -n` | нумеровать все строки |
| `cat -b` | нумеровать только непустые строки |
| `head -n 5` | первые 5 строк |
| `tail -n 5` | последние 5 строк |
| `tail -f` | следить за файлом в реальном времени |

| `sort -n` | числовая сортировка |
| `sort -r` | обратная сортировка |

| `sort -u` | сортировка с удалением дубликатов |
| `uniq -c` | строки с количеством повторений |
| `uniq -d` | только дублированные строки |
| `uniq -u` | только уникальные строки |
| `cut -d: -f1` | первое поле, разделитель ':' |
| `cut -c1-5` | символы 1–5 |
| `paste -d,` | склеить файлы через запятую |

## Типичные вопросы экзамена

**Какая команда выводит последние 20 строк файла?**
`tail -n 20 file.txt` или `tail -20 file.txt`

**Как отследить обновления лога в реальном времени?**
`tail -f /var/log/syslog`

**Как вывести только первое поле из /etc/passwd (разделитель ':')?**
`cut -d: -f1 /etc/passwd`

**Как посчитать количество строк в файле?**
`wc -l file.txt`

**Как удалить дубликаты из отсортированного файла?**
`sort file.txt | uniq`
