## Что такое раздел и таблица разделов

Прежде чем диск можно использовать, его нужно разбить на разделы. Раздел — это логический кусок физического диска, описанный записью в таблице разделов. Запись содержит первый и последний сектора раздела, его тип и дополнительные сведения.

Каждый раздел операционная система видит как отдельный диск, даже если все они на одном физическом носителе. В Windows разделам выдают буквы C:, D:. В Linux они становятся файлами устройств: `/dev/sda1`, `/dev/sda2`.

Все операции с разметкой требуют прав root.

## Таблицы разделов: MBR и GPT

### MBR

MBR (Master Boot Record) пришёл из MS-DOS, конкретно из PC-DOS 2.0 в 1983 году. Таблица разделов лежит в первом секторе диска вместе с минимальным кодом загрузчика. На Linux туда обычно ставят GRUB.

Ограничения MBR:

- Максимальный размер диска: 2 ТБ.
- Не более 4 первичных разделов на диске.

### GPT

GPT (GUID Partition Table) обходит ограничения MBR. Размер диска практически не ограничен. Число разделов ограничено только ОС — типичная реализация поддерживает до 128 записей. GPT встречается на современных машинах с UEFI.

Каждому диску GPT присваивает 128-битный идентификатор GUID. GPT хранит резервную копию заголовка и таблицы в конце диска, что упрощает восстановление при повреждении основного заголовка.

### Сравнение

| Параметр | MBR | GPT |
|---|---|---|
| Год появления | 1983 | конец 1990-х |
| Максимальный размер диска | 2 ТБ (часто пишут 2.2 ТБ) | до 9.4 ЗБ |
| Максимум разделов | 4 первичных | до 128 по умолчанию |
| Прошивка | BIOS | UEFI |
| Резервная копия таблицы | нет | есть в конце диска |
| Идентификатор диска | 32-битный | 128-битный GUID |

## Управление MBR через fdisk

Стандартная утилита для MBR — `fdisk`. Интерактивная. Запуск:

```bash
fdisk /dev/sda
```

Указывайте само устройство (`/dev/sda`), не раздел (`/dev/sda1`). Изменения хранятся только в памяти и записываются на диск командой `w`. До неё можно безопасно тренироваться. Выход без сохранения — команда `q`.

### Команды fdisk

| Команда | Что делает |
|---|---|
| `m` или `?` | помощь |
| `p` | показать текущую таблицу разделов |
| `n` | создать новый раздел |
| `d` | удалить раздел |
| `t` | сменить тип раздела |
| `l` | список известных типов разделов |
| `F` | показать свободное место |
| `w` | записать изменения и выйти |
| `q` | выйти без сохранения |

### Печать таблицы

```
Disk /dev/sda: 111.8 GiB, 120034123776 bytes, 234441648 sectors
Disk model: CT120BX500SSD1
Disklabel type: dos
Disk identifier: 0x97f8fef5

Device     Boot     Start       End   Sectors   Size Id Type
/dev/sda1           4096 226048942 226044847 107.8G 83 Linux
/dev/sda2      226048944 234437550   8388607     4G 82 Linux swap / Solaris
```

Колонки: `Device` — файл устройства, `Boot` — флаг загрузочного раздела, `Start`/`End` — первый и последний сектор, `Sectors` — число секторов, `Size` — размер, `Id` — числовой код типа, `Type` — текстовое описание.

### Первичные и расширенные разделы

На MBR-диске два сорта разделов: первичные (primary) и расширенные (extended). Первичных не более 4. Если нужно больше разделов, создают один расширенный — контейнер для логических разделов внутри.

Если диск должен быть загрузочным, первый раздел обязан быть первичным.

### Создание раздела

Команда `n`. Размер задаётся через `+1G`, `+500M`. Суффиксы: K, M, G, T, P.

```
Command (m for help): n
Partition type
   p   primary (0 primary, 0 extended, 4 free)
   e   extended (container for logical partitions)
Select (default p): p
Partition number (1-4, default 1): 1
First sector (2048-3903577, default 2048): 2048
Last sector, +/-sectors or +/-size{K,M,G,T,P} (2048-3903577, default 3903577): +1G
```

### Свободное место и его непрерывность

Команда `F` покажет, где лежат куски свободного места и каких они размеров.

**Важная тонкость:** при создании нового раздела максимум ограничен наибольшим непрерывным куском, не суммой. Если свободны два куска по 512 МБ, новый раздел будет максимум 512 МБ.

### Смена типа раздела

Команда `t`, затем номер раздела, затем шестнадцатеричный код. Полный список — команда `l`.

Самые частые коды:

- `83`: Linux
- `82`: Linux swap

## Управление GPT через gdisk

`gdisk` — аналог fdisk для GPT. Интерфейс очень похожий.

### Печать таблицы

```
Command (? for help): p
Disk /dev/sdb: 3903578 sectors, 1.9 GiB
Sector size (logical/physical): 512/512 bytes
Disk identifier (GUID): AB41B5AA-A217-4D1E-8200-E062C54285BE
Partition table holds up to 128 entries
First usable sector is 34, last usable sector is 3903544
Total free space is 1282071 sectors (626.0 MiB)

Number  Start (sector)    End (sector)  Size       Code  Name
   1            2048         2099199   1024.0 MiB  8300  Linux filesystem
   2         2623488         3147775    256.0 MiB  8300  Linux filesystem
```

В шапке вывода уже есть GUID диска и общий объём свободного места.

### Создание и удаление раздела

Создание: команда `n` — помимо номера и размера, gdisk просит тип. Удаление: команда `d`.

Коды типов в gdisk четырёхзначные:

- `8300`: Linux filesystem
- `8200`: Linux swap

### Сортировка номеров

После удаления раздела в нумерации остаётся дыра. Команда `s` пересчитает номера.

### Восстановление через recovery menu

Меню восстановления: команда `r`. Подкоманды:

- `b` / `c`: пересобрать основной заголовок / таблицу GPT
- `d` / `e`: построить резервную копию из основной
- `f`: преобразовать MBR в GPT
- `g`: преобразовать GPT в MBR

### Свободное место в GPT

В отличие от MBR, на GPT новый раздел может занять любое свободное место — ограничения по непрерывности нет.

## Создание файловых систем

После разбиения нужно создать на разделе файловую систему. Если запустить `mkfs` без указания типа, по умолчанию создаётся **ext2**.

### ext2, ext3, ext4

Утилиты `mkfs.ext2`, `mkfs.ext3`, `mkfs.ext4` — символические ссылки на `mke2fs`.

```bash
mkfs.ext4 /dev/sdb1
# или
mke2fs -t ext4 /dev/sdb1
```

Опции `mke2fs`:

| Опция | Что делает |
|---|---|
| `-b SIZE` | размер блока (1024, 2048 или 4096) |
| `-c` | проверка bad blocks |
| `-c -c` | тщательная проверка bad blocks (медленно) |
| `-d DIR` | скопировать содержимое каталога в корень новой ФС |
| `-F` | форсировать создание |
| `-L LABEL` | метка тома (до 16 символов) |
| `-n` | пробный запуск, ничего не создаётся |
| `-q` | тихий режим |
| `-U ID` | UUID: литерал, `clear`, `random` или `time` |
| `-V` | подробный вывод |

### XFS

XFS — высокопроизводительная журналируемая ФС от Silicon Graphics (1993). Стоит по умолчанию в RHEL 7+. Утилиты — пакет `xfsprogs`.

```bash
mkfs.xfs /dev/sda1
```

Опции `mkfs.xfs`:

| Опция | Что делает |
|---|---|
| `-b size=N` | размер блока (512..65536, по умолчанию 4096) |
| `-m crc=0/1` | CRC32c для метаданных (по умолчанию включено) |
| `-m uuid=UUID` | задать UUID |
| `-f` | форсировать (есть ФС — перезаписать) |
| `-l logdev=DEV` | вынести журнал на отдельное устройство |
| `-l size=N` | размер журнала |
| `-L LABEL` | метка (до 12 символов) |
| `-N` | пробный запуск |
| `-q` | тихий режим |

### VFAT и FAT

FAT родом из MS-DOS. VFAT — расширение FAT16 с поддержкой длинных имён (до 255 символов). Утилита — `mkfs.fat` (алиас `mkfs.vfat`).

Ограничения:

- FAT16: том до 4 ГБ, файл до 2 ГБ
- FAT32: том до 2 ПБ, файл до 4 ГБ

```bash
mkfs.fat /dev/sdc1
```

Опции:

| Опция | Что делает |
|---|---|
| `-c` | проверка bad blocks |
| `-F SIZE` | размер таблицы FAT: 12, 16 или 32 |
| `-n NAME` | метка (до 11 символов) |
| `-v` | подробный вывод |
| `-C FILENAME BLOCK_COUNT` | создать FAT-образ в файле |

### exFAT

exFAT появился у Microsoft в 2006 году. Максимальный размер файла — 16 ЭБ, тома — 128 ПБ. Это стандартная ФС для SDXC-карт > 32 ГБ.

Утилита `mkfs.exfat` (ссылка на `mkexfatfs`).

```bash
mkfs.exfat /dev/sdb2
```

Опции: `-n NAME` (метка до 15 символов), `-i VOL_ID` (32-битный Volume ID), `-s SECTORS` (секторов на кластер, степень двойки).

### Btrfs

Btrfs (B-Tree Filesystem) разрабатывается с 2007 года. По умолчанию в SUSE.

Возможности:

- Несколько устройств в одной ФС (аналоги RAID 0, 1, 5, 6, 10)
- Прозрачное сжатие (ZLIB, LZO, ZSTD)
- Снапшоты и инкрементальные бэкапы
- Подтома с квотами
- Дедупликация и онлайн-дефрагментация
- Copy-on-write: новые данные пишутся в свободное место, старые освобождаются только после успешной записи

```bash
mkfs.btrfs /dev/sdb1
mkfs.btrfs /dev/sdb1 -L "New Disk"
```

**Несколько устройств:**

```bash
# Конкатенация без избыточности
mkfs.btrfs -d single -m single /dev/sdb /dev/sdc

# Зеркало данных и метаданных (RAID1)
mkfs.btrfs -d raid1 -m raid1 /dev/sdb /dev/sdc
```

Флаги `-d` (данные) и `-m` (метаданные): `raid0`, `raid1`, `raid5`, `raid6`, `raid10`, `single`, `dup`.

**Подтома:**

```bash
# Создать подтом
btrfs subvolume create /mnt/disk/BKP

# Информация о подтоме
btrfs subvolume show /mnt/disk/BKP/

# Смонтировать подтом отдельно
mount -t btrfs -o subvol=BKP /dev/sdb1 /mnt/bkp
```

Подтом не равен разделу: он делит свободное место с родительской ФС и растёт по мере необходимости, а не резервирует место заранее.

**Снапшоты:**

```bash
# Снапшот (read-write)
btrfs subvolume snapshot /mnt/disk /mnt/disk/snap

# Снапшот только для чтения
btrfs subvolume snapshot -r /mnt/disk /mnt/disk/snap
```

**Прозрачное сжатие** включается при монтировании:

```bash
mount -o compress /dev/sdb1 /mnt/disk
```

## Управление разделами через GNU Parted

GNU Parted работает и с MBR, и с GPT.

**Важно: parted применяет команды немедленно, команды `w` здесь нет.** Тренируйтесь только на пустом или запасном устройстве.

```bash
parted /dev/sdb
```

### Команды parted

| Команда | Что делает |
|---|---|
| `select DEVICE` | переключиться на другое устройство |
| `print` | показать текущую таблицу |
| `print devices` | список всех блочных устройств |
| `print free` | свободное место по разделам |
| `mklabel TYPE` | создать таблицу разделов |
| `mkpart` | создать раздел |
| `rm N` | удалить раздел N |
| `rescue START END` | восстановить случайно удалённый раздел |
| `resizepart N END` | сдвинуть конец раздела N |
| `quit` | выйти |

### Создание таблицы разделов

```
(parted) mklabel msdos   # MBR
(parted) mklabel gpt     # GPT
```

### Создание раздела через mkpart

Синтаксис: `mkpart PARTTYPE FSTYPE START END`

- `PARTTYPE`: `primary`, `logical` или `extended` (для MBR)
- `FSTYPE`: подсказка ОС о типе — parted **не создаёт** ФС, только ставит флаг
- `START`/`END`: позиция в байтах, МБ, % диска и т.д.

```
(parted) mkpart primary ext4 1m 100m
(parted) mkpart primary linux-swap 301m 800m
```

После mkpart файловую систему создают отдельно: `mkfs.ext4 /dev/sdXN`.

### Удаление и восстановление

Удалить: `rm N`.

Восстановить случайно удалённый раздел: `rescue START END` — parted сканирует диск и предлагает восстановить найденные разделы. Можно дать запас вперёд и назад. Находит только разделы с файловой системой.

```
(parted) rescue 90m 210m
```

### Изменение размера

Уменьшение (порядок важен — при перепутанном порядке теряются данные):

1. Уменьшить ФС: `resize2fs /dev/sdb3 88m`
2. Сдвинуть конец раздела: `resizepart 3 300m`

Увеличение:

1. Сдвинуть конец раздела: `resizepart N END`
2. Расширить ФС: `resize2fs /dev/sdbN`

Опция `-M` у `resize2fs` подгоняет ФС под минимально нужный размер по фактическим данным:

```bash
resize2fs -M /dev/sdc1
```

## Раздел и файл подкачки

### Подготовка раздела под swap

Смена типа раздела:

- В fdisk: `t` → код `82`
- В gdisk: `t` → код `8200`
- В parted: указываем `linux-swap` как FSTYPE при создании

### mkswap, swapon, swapoff

После создания раздела записываем заголовок swap:

```bash
mkswap /dev/sda2
```

Без `mkswap` команда `swapon` выдаст `read swap header failed`.

```bash
swapon /dev/sda2     # включить
swapoff /dev/sda2    # выключить
swapon -s            # список активных swap
cat /proc/swaps      # то же из псевдо-ФС
```

### Файл подкачки

Создание swap-файла на 1 ГБ:

```bash
dd if=/dev/zero of=myswap bs=1M count=1024
mkswap myswap
swapon myswap
```

Параметры `dd`: `if=` — источник (`/dev/zero` отдаёт нули), `of=` — куда писать, `bs=` — размер блока, `count=` — число блоков.

Без записи в `/etc/fstab` swap-файл проживёт только до перезагрузки. Рекомендуемые права — `0600`, владелец root.

## Шпаргалка: утилиты и команды

### Разметка дисков

|---|---|
| `fdisk DEV` | редактор MBR-таблицы |

| Команда | Что делает |
|---|---|
| `mkswap DEV/FILE` | подготовить swap |
| `swapon DEV/FILE` | включить swap |
| `swapoff DEV/FILE` | выключить swap |
| `swapon -s` | список активных swap |

### Коды типов разделов

| Тип | fdisk (MBR) | gdisk (GPT) |
|---|---|---|
| Linux filesystem | `83` | `8300` |
| Linux swap | `82` | `8200` |

## Типичные вопросы экзамена

1. Какие основные ограничения у MBR по сравнению с GPT? → MBR: до 4 первичных, диск до 2.2 ТБ. GPT: до 128 разделов, диск практически без ограничений.
2. Сколько разделов поддерживает GPT по умолчанию? → **128**.
3. Какой код типа у Linux swap в fdisk? → **82**.
4. Какой код типа у Linux swap в gdisk? → **8200**.
5. Какая команда меняет тип раздела в fdisk? → **`t`**.
6. Какая команда в parted создаёт MBR-таблицу? → **`mklabel msdos`**.
7. Какую ФС создаёт `mkfs` без параметров? → **ext2**.
8. Создаёт ли parted сам файловую систему через mkpart? → **Нет**, только ставит флаг типа.
9. Какая утилита создаёт XFS? → **`mkfs.xfs`** из пакета `xfsprogs`.
10. Чем отличается подтом Btrfs от обычного раздела? → Подтом делит свободное место с родителем, раздел резервирует заранее.
11. Какие алгоритмы сжатия поддерживает Btrfs? → **ZLIB, LZO, ZSTD**.
12. Какая опция mke2fs проверяет bad blocks? → **`-c`**.
13. Чем отличаются `mkfs.fat` и `mkfs.vfat`? → **Одна утилита**, vfat — алиас.
14. Какой максимальный размер файла на FAT32? → **4 ГБ**.
15. Что такое copy-on-write в Btrfs? → Новые данные пишутся в свободное место, метаданные переключаются, только потом старые данные освобождаются.
16. Как определить UUID и метку ФС? → **`lsblk -f`**.
17. Чем отличается `mkfs.ext4` от `mke2fs -t ext4`? → **Ничем**, первая — символическая ссылка.
18. Что нужно сделать перед `swapon`? → **`mkswap`** на разделе или файле.
19. Как при уменьшении раздела ext4 не потерять данные? → Сначала `resize2fs`, потом `resizepart` (не наоборот).
20. Зачем нужна команда `rescue` в parted? → Восстановить случайно удалённый раздел по диапазону позиций.

## Упражнения

### Упражнение 1 — Таблица разделов для диска 3 ТБ

Диск 3 ТБ, нужно создать три раздела по 1 ГБ. Какой тип таблицы разделов выбрать и почему?

<details>
<summary>Ответ</summary>

GPT. MBR ограничен дисками до 2 ТБ, поэтому 3-терабайтный диск в MBR-схеме не поместится целиком.

### Упражнение 2 — Свободное место в gdisk

Как в gdisk узнать, сколько свободного места осталось на диске?

<details>
<summary>Ответ</summary>

Команда `p` (print) выводит таблицу разделов. В шапке вывода указан общий объём свободного места:

```
Total free space is 1282071 sectors (626.0 MiB)
```

</details>

### Упражнение 3 — Создать ext3 с проверкой bad blocks, меткой и UUID

Создать ext3 на `/dev/sdc1` с проверкой bad blocks, меткой `MyDisk` и случайным UUID.

<details>
<summary>Ответ</summary>

```bash
mkfs.ext3 -c -L MyDisk -U random /dev/sdc1
```

Альтернатива через `mke2fs`:

```bash
mke2fs -t ext3 -c -L MyDisk -U random /dev/sdc1
```

</details>

### Упражнение 4 — Создать раздел в parted

В parted создать ext4-раздел на 300 МБ начиная с отметки 500 МБ.

<details>
<summary>Ответ</summary>

```
(parted) mkpart primary ext4 500m 800m
```

Важно: parted ставит только флаг типа — файловую систему не создаёт. После выхода из parted нужно отдельно запустить `mkfs.ext4 /dev/sdXN`.

</details>

### Упражнение 5 — Btrfs RAID1 из двух разделов по 20 ГБ

Два раздела по 20 ГБ. Как объединить в одну Btrfs с зеркалированием? Какой будет размер?

<details>
<summary>Ответ</summary>

```bash
mkfs.btrfs -d raid1 -m raid1 /dev/sda1 /dev/sdb1
```

Размер — **20 ГБ**: один раздел является зеркалом другого.

</details>

### Упражнение 6 — Раздел 600 МБ на MBR-диске с дырой

MBR-диск 1.9 ГБ. На нём два раздела: `/dev/sdb1` (512 МБ) и `/dev/sdb3` (512 МБ). Между ними и после третьего — свободное место. Можно ли создать раздел на 600 МБ?

<details>
<summary>Ответ</summary>

Нельзя. Суммарно свободно около 1 ГБ, но наибольший **непрерывный** кусок — только 512 МБ. Раздел не может «перепрыгнуть» через занятый — fdisk ответит `Value out of range`.

Команда `F` покажет точную картину свободного места.

</details>

### Упражнение 7 — Ужать раздел до минимального размера

Первый раздел `/dev/sdc1` — 1 ГБ с ext4, содержит около 256 МБ файлов. Ужать до минимально необходимого размера.

<details>
<summary>Ответ</summary>

Операция в два шага — сначала ФС, потом раздел.

Шаг 1 — ужать файловую систему под фактический объём данных:

```bash
resize2fs -M /dev/sdc1
```

Шаг 2 — сдвинуть конец раздела в parted:

```
(parted) resizepart 1 241M
```

Порядок критичен: при уменьшении сначала ФС, потом граница раздела. В обратном порядке — потеря данных.

</details>

### Упражнение 8 — swapon выдаёт "read swap header failed"

Создан раздел `mkpart primary linux-swap 0 1024M`, затем `swapon /dev/sdb1` ругается ошибкой `read swap header failed`. Что не так?

<details>
<summary>Ответ</summary>

`mkpart` создаёт раздел, но **не записывает swap-структуру**. Перед `swapon` нужно запустить `mkswap`:

```bash
mkswap /dev/sdb1
swapon /dev/sdb1
```

</details>

### Упражнение 9 — Восстановить случайно удалённый раздел

На диске была EFI-партиция 250 МБ, swap 4 ГБ и третий раздел 10 ГБ. Третий случайно удалили. Как восстановить?

<details>
<summary>Ответ</summary>

Команда `rescue` с примерными границами.

Считаем позиции: 250 МБ (EFI) + 4×1024 МБ (swap) = 4346 МБ — начало. Конец: 4346 + 10×1024 = 14 586 МБ.

```
(parted) rescue 4346m 14586m
```

Можно дать небольшой запас — геометрия диска может слегка сдвигать границы. `rescue` находит только разделы, на которых была ФС.

</details>

### Упражнение 10 — Превратить /dev/sda3 в активный swap через fdisk

Неиспользуемый раздел `/dev/sda3` 4 ГБ. Активировать как swap.

<details>
<summary>Ответ</summary>

Шаг 1 — в fdisk сменить тип раздела:

```
Command (m for help): t
Partition number (1-3, default 3): 3
Hex code (type L to list all codes): 82

Changed type of partition 'Linux' to 'Linux swap / Solaris'.

Command (m for help): w
```

Шаг 2 — подготовить swap-структуру:

```bash
mkswap /dev/sda3
```

Шаг 3 — включить swap:

```bash
swapon /dev/sda3
```

</details>
