Принципы эффективного изучения
1Сначала сделай, потом читай
Любая тема — с попытки настроить сервис руками. Получил ошибку → читай документацию прицельно под проблему. В десять раз эффективнее пассивного чтения.
2Пиши пока учишься
Блог — проверка понимания. Если не можешь объяснить постом на 800 слов, значит не понял. Плюс контент работает на тебя круглосуточно.
3Не гонись за коллекцией сертов
Максимум один серт в квартал с реальной практикой между ними. Пять сертификатов без проекта в GitHub — красный флаг. Один серт с публичным портфолио — читается как инженер.
4Пропорция 70/30
70% времени — терминал, 30% — видео и книги. Если пропорция перекашивается в сторону пассивного контента, знания не закрепляются.
5Откликайся рано и итерируй
Первые собеседования не про оффер, а про то, чего не хватает. После LPIC-2 уже можно подавать. Отказы — обратная связь.
6Находи сообщество
В Сиднее активное DevOps-комьюнити. Личные контакты решают больше чем CV в ATS. Через сообщество находятся менторы, которых нет онлайн.
7Документируй homelab как production
README, runbooks, DR-план. Формирует правильные привычки и выглядит сильно на интервью, когда рассказываешь про свой setup.
